### 버킷 비교 로직 상세
*(코드: time-off/service/.../AnnualTimeOffPromotionApplicationServiceImpl.kt:227-260)*

문서 목록 표시 시, 현재 정책 기준으로 버킷을 동적 계산 후 비교:
```kotlin
// 현재 버킷 목록 생성 (line 227-232)
val targetBuckets = buckets.map {
    AnnualTimeOffPromotionBucketIdentity(it.dissipateType, it.legalExpirationDate)
}.distinct()

// 숨김 조건 (line 248-260)
if (targetBuckets.none {
    it.expirationDate == boostHistory.dissipatedAt &&
    it.dissipateType === boostHistory.boostType.dissipateType
}) {
    // 정책 변경 혹은 입사일 변경 등으로 소멸일이 바뀐 경우 숨김
    return@mapNotNull null
}
```

MONTHLY 타입의 `legalExpirationDate`는 `joinDate.plusYears(1)`로 계산됨 *(코드: time-off/service/.../TimeOffBucketWithDissipateType.kt:87-93)*.
MONTHLY 버킷 생성 조건: `annualTimeOffAssign.timeRule.time === AnnualTimeOffAssignTime.MONTHLY` *(코드: time-off/service/.../TimeOffBucketWithDissipateType.kt:77-96)*

## 관련 법령
- **근로기준법 제61조 (연차 유급휴가의 사용 촉진)**: 사용자가 연차를 사용하도록 촉진 절차를 밟은 경우, 미사용 연차에 대한 보상 의무가 면제됨
- 연차촉진은 법적 절차이므로, 촉진 알림이 정상 발송되지 않거나 stale 상태로 방치되면 사용자(회사)의 연차촉진 절차 이행 여부에 영향을 줄 수 있음
- 이 이슈는 **불필요한 알림이 발송되는 방향** → 근로자에게 불리하지 않음 ✅ (오히려 알림 과다로 관리자 혼란만 유발)
