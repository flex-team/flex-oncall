## 원인 분석

운영 요청 건. 스폰서십으로 등록된 계정의 관리자 이메일이 퇴사자로 되어 있어 변경이 필요.

### 팀 논의 결과

> 💡 **판단 근거**: 퇴사자 계정이므로 단순 이메일 변경으로는 부족할 수 있음[^9]
> → 이메일 변경 + 비밀번호 찾기 방식[^10] 또는 최고관리자 강제 추가 오퍼레이션[^11] 두 가지 방안 논의
> → **@윤성복 최종 답변: 라쿤(Flex 내부 운영 관리 도구) 최고관리자 메뉴에서 처리 가능**[^12]

**논의된 처리 방안:**

| # | 방안 | 설명 | 비고 |
|---|------|------|------|
| 1 | 이메일 변경 → 비밀번호 찾기 | 라쿤에서 이메일 변경 후, 변경된 이메일로 비밀번호 찾기[^10] | 비번 등록 이력 없어도 가능 여부 확인 필요 |
| 2 | **최고관리자 강제 추가 오퍼레이션** | 새 이메일로 최고관리자를 직접 추가[^11] | @김가영 선호[^13], @윤성복 추천[^12] |

## 해결안

### Operation API를 통한 이메일 변경

`flex-core-backend`에 이메일 변경 Operation API가 존재한다[^3].

**엔드포인트**: `PATCH /action/v2/operation/core/customers/{customerId}/users/{userId}/emails/change`

**Request Body**:
```json
{ "newEmail": "변경할 이메일" }
```

### 대상 계정 조회 결과

> 두 고객사 모두 해당 이메일의 활성 계정이 존재하며, 다법인 사용자가 아닌 독립 계정이다[^6].

| customer_id | userId | primary_user_id | deleted_date | 비고 |
|-------------|--------|-----------------|--------------|------|
| 141809 | **557917** | NULL | NULL | 독립 계정, 활성 |
| 143157 | **560538** | NULL | NULL | 독립 계정, 활성 |

- `primary_user_id`가 둘 다 NULL → 다법인(concurrent) 사용자가 아님[^5]
- 다법인 제약 없이 각 회사별로 독립 변경 가능

<details>
<summary>📋 조회 쿼리</summary>

```sql
SELECT id, customer_id, email, primary_user_id, deleted_date
FROM user
WHERE customer_id IN (141809, 143157)
  AND email LIKE '%ryuseongrak%'
```
- 출처: `flex-core-backend` > `workspace/repository/.../user/User.kt` — `user` 테이블[^4]

</details>

```
PATCH /action/v2/operation/core/customers/143157/users/560538/emails/change

```

### ⚠️ 주의사항

| # | 항목 | 설명 |
|---|------|------|
| 1 | 검증된 이메일 | 이미 검증(verified)된 이메일은 관리자가 변경 불가[^5] |
| 2 | 다법인 사용자 | primary 회사에서만 변경 가능. 두 회사가 같은 계정이면 확인 필요[^5] |
| 3 | 인증 정보 연동 | 이메일 변경 시 인증(authentication) 식별자도 자동 업데이트됨[^3] |
