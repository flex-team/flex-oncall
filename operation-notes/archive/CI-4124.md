## 해결안

### Bulk Email Change Operation API

`flex-core-backend`에 일괄 이메일 변경 Operation API가 존재한다[^5].

**엔드포인트**: `PATCH /action/v2/operation/core/customers/{customerId}/emails/change/bulk`

**Request Body**:
```json
{
  "items": [
    { "oldEmail": "기존이메일", "newEmail": "새이메일" },
    ...
  ]
}
```

**동작 방식**:
- `oldEmail`로 사용자 조회 → 각 사용자별 이메일 변경 + 인증 정보 업데이트 + Kafka 이벤트 발행[^5]
- ⚠️ 존재하지 않는 이메일이 포함되면 `IllegalArgumentException`으로 **전체 실패**[^5]

### 실행 절차

1. 엑셀(`Flex.xlsx`) 내용 확인 — oldEmail/newEmail 매핑 추출[^3]
2. DB로 대상 구성원 존재 확인

<details>
<summary>📋 확인 쿼리</summary>

```sql
SELECT id, email
FROM user
WHERE customer_id = 215051
  AND deleted_date IS NULL
```
- 출처: `flex-core-backend` > `workspace/repository/.../user/User.kt` — `user` 테이블[^6]

</details>

3. Bulk API 호출:

```
PATCH /action/v2/operation/core/customers/215051/emails/change/bulk
Body: { "items": [ { "oldEmail": "...", "newEmail": "..." }, ... ] }
```

4. 변경 후 DB 재조회로 결과 확인

### ⚠️ 주의사항

| # | 항목 | 설명 |
|---|------|------|
| 1 | 전체 실패 | 존재하지 않는 oldEmail이 하나라도 포함되면 전체 롤백됨[^5] |
| 2 | 다법인 사용자 | primary 회사에서만 변경 가능. 대상 중 다법인 사용자 여부 사전 확인 필요[^7] |
| 3 | 인증 정보 연동 | 이메일 변경 시 인증(authentication) 식별자도 자동 업데이트됨[^5] |
