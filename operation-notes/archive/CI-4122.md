## 증상

인사 발령(겸직발령 — 팀/직책 추가)을 진행했는데 임금계약 정보 변경 알림이 함께 발송됨[^1]. 메타베이스[^2]에서 Core 알림의 `title_meta_map`이 `[]`로 표시되어 알림 내용 확인이 불가[^3].

- 확인 대상: `notification_deliver.id = 178286727`

## 원인 분석

### 원인

1. **`title_meta_map`이 비어있는 것은 정상 스펙.** Core 인사정보 변경 알림(`FLEX_USER_DATA_CHANGE` 토픽)의 제목은 "내 정보 변경"이라는 고정 문자열이므로, `titleMetaMap = emptyMap()`으로 설계됨[^4]. 메타베이스 쿼리(#3551)가 `notification_topic.title_meta_map`만 조회하여 내용이 안 보이는 구조[^5].
2. **"임금계약 정보 변경" 알림은 존재하지 않음.** DB 조회 결과[^9][^10], 해당 유저에게 발송된 알림은 "인사정보" 변경 1건 + 조직개편 공지 1건 뿐. 고객이 알림을 확장하면서 하단 알림을 읽고 오인한 것으로 추정[^11].

### 조사 과정

> 💡 **조사 경로**: TopicContext 코드 확인[^4] → 토픽 제목 템플릿 확인[^6] → 알림 본문은 별도 테이블에 저장되는 구조 확인[^7][^8] → DB에서 실제 알림 내용 확인[^9] → 전체 알림 조회로 임금계약 알림 부재 확인[^10]

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | Core 알림의 `title_meta_map`은 원래 비어있는 것이 스펙 | `FlexUserDataChangeTopicContext` 코드 확인 | ✅ 확정 |
| 2 | 알림 발송 시 메타데이터 저장 버그 | 위 가설 확정으로 불필요 | ❌ 소거 |
| 3 | 메타베이스 쿼리가 잘못된 테이블/컬럼을 참조 | 쿼리 분석 + 알림 시스템 구조 확인 | ❌ 소거 (쿼리 자체는 정상, 다만 `notification.message_data_map`을 조회하지 않음) |

<details>
<summary>📋 알림 시스템 데이터 구조 상세</summary>

**알림 데이터 흐름:**

```
발송 요청 (flex-core-backend)
  ├─ MessageContext: messageDataMap (알림 본문 변수)
  │   → notification 테이블에 저장
  │   → 예: {targetUserId: 123, changedDataName: "인사정보", ...}
  │
  └─ TopicContext: titleMetaMap (토픽 제목 변수)
      → notification_topic 테이블에 저장
      → FLEX_USER_DATA_CHANGE의 경우: emptyMap() (제목이 고정이므로)
```

**테이블 관계:**
- `notification_deliver` — 수신자별 배달 기록 (id, notification_id, notification_topic_id)
- `notification` — 알림 본체 (notification_type, **message_data_map**)
- `notification_topic` — 토픽 그룹 (topic_type, **title_meta_map**)

</details>

## 해결

- 메타베이스 쿼리(#3551)에 `notification` 테이블 JOIN + `notification_type`, `message_data_map` 컬럼을 추가하여 Core 알림도 내용 확인 가능하도록 수정 완료[^12]
- 알림타입별 메시지 참고: [flex 알림 메시지 취합](https://www.notion.so/flexnotion/a52c7d620b8a40c698b5a37a59fdea43?v=452623d7196b4d37b4c8f89fdfecba0e&source=copy_link)[^12]

## 다음에 같은 문의가 오면

1. **메타베이스에서 `title_meta_map`이 `[]`인 알림** → Core 알림은 토픽 제목이 고정이므로 정상. 메타베이스 #3551에서 `notification_type`과 `message_data_map` 컬럼으로 확인 가능
2. **실제 알림 내용 확인** → `message_data_map.changedDataName` 값으로 어떤 정보가 변경되었는지 특정
3. **알림타입별 메시지 유추 불가 시** → [flex 알림 메시지 취합](https://www.notion.so/flexnotion/a52c7d620b8a40c698b5a37a59fdea43?v=452623d7196b4d37b4c8f89fdfecba0e&source=copy_link) Notion 문서에서 확인
