## 문의 내용

### 질문 1: 출근시각/퇴근시각 없이 시작시간/종료시간만 있는 기록의 출처

근무 승인을 통해 수정 가능한 설정에서, 출근시각/퇴근시각은 없고 시작시간과 종료시간만 있는 기록 중 별도의 승인 내역이 없는 경우, 아래 중 하나인지 확인 요청:

- 근무수정권한을 가진 자가 직접 수정
- 근무 수정 승인정책이 누구나 가능하던 시점의 기록

### 질문 2: '당일 승인상태'의 '당일' 기준

근태기록리포트에서 '당일 승인상태' 중 '당일'은 엑셀을 내려받은 시점을 의미하는 것인지 확인 요청[^1].

## 원인 분석

### 질문 1 답변: 승인 내역 없이 시작/종료시간만 있는 기록 — **근무수정권한자 직접 수정이 가장 유력**

리포트 컬럼의 "출근시각/퇴근시각"과 "시작시간/종료시간"은 서로 다른 데이터 소스를 참조한다[^2]:

| 리포트 컬럼 | 모델 필드 | 데이터 소스 |
|------------|----------|------------|
| 출근시각 / 퇴근시각 | `realCheckInTime` / `realCheckOutTime` | 출퇴근 기록 (Work Clock) |
| 시작시간 / 종료시간 | `startTime` / `endTime` | 근무기록 이벤트 블록의 `timeFrom` / `timeTo` |

**승인 내역 없이 시작/종료시간만 존재하는 경우의 생성 경로:**

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | 근무수정권한자(`ADMIN`)가 직접 수정 | event의 `actorType` 확인 | ✅ 확정 |
| 2 | 과거 승인정책 변경 이전 데이터 | 승인 엔티티 코멘트 확인 | ❌ 소거 — 과거에도 승인 레코드는 생성됨[^4] |
| 3 | Operation API를 통한 수정 | Operation API 코드 확인 | ❌ 소거 — 조회/무효화만 제공, 직접 수정 불가[^5] |

**확정 근거**: `ForceUserWorkRecordRegisterServiceImpl`[^3]이 호출되면 `actorType = ADMIN`으로 이벤트가 저장되며, **승인(Approval) 레코드를 생성하지 않는다**. 이것이 "승인 내역 없이" 기록이 존재하는 유일한 경로다.

### 질문 2 답변: '당일 승인상태'의 '당일' — **해당 근무일 기준, 엑셀 다운로드 시점에 조회**

"당일"은 **해당 근무일(work date)**을 의미한다[^6]. 단, 승인상태 값은 **엑셀 다운로드 시점에 DB에서 조회한 현재 상태**이다[^7].

즉, 3월 1일의 근무기록에 대해:
- 3월 5일에 다운로드하면 → 3월 5일 시점의 3월 1일 승인상태
- 3월 10일에 다운로드하면 → 3월 10일 시점의 3월 1일 승인상태 (그사이 승인 처리되었으면 값이 달라짐)

**승인상태 표시 우선순위** (복수 건 존재 시)[^8]:
1. 승인대기 (WAITING) — 최우선 표시
2. 승인완료 (APPROVED)
3. 승인반려 (DECLINED)
4. 승인취소 (CANCELED)

**엑셀 컬럼 정의** (`UserDailyWorkScheduleResultExcelWriter.kt:36`):

**승인상태 문자열 변환** (`AbstractDailyFixedUserWorkScheduleExportDateNightType3Converter.kt:38-46`):
```kotlin
protected fun Collection<TimeTrackingApprovalResultModel>.getApprovalStatusString(): String? =
    when {
        this.any { it.status == TimeTrackingApprovalStatus.WAITING } -> "승인대기"
        this.any { it.status == TimeTrackingApprovalStatus.APPROVED } -> "승인완료"
        this.any { it.status == TimeTrackingApprovalStatus.DECLINED } -> "승인반려"
        this.any { it.status == TimeTrackingApprovalStatus.CANCELED } -> "승인취소"
        else -> null
    }
```

**승인 매핑** (`UserWorkScheduleApprovalMappingService.kt:270-278`):


## 해결안

**스펙 문의**로 판별. 버그가 아니며 코드 수정 불필요.

문의자에게 아래 내용 안내:

> **질문 1**: 출근시각/퇴근시각 없이 시작시간/종료시간만 있고 승인 내역이 없는 기록은, **근무수정권한을 가진 관리자가 직접 수정한 기록**입니다. 관리자가 직접 수정하는 경우 별도의 승인 절차를 거치지 않으므로 승인 내역이 남지 않습니다.
>
> **질문 2**: '당일 승인상태'의 '당일'은 **해당 근무일**을 의미합니다. 다만 표시되는 승인상태 값은 엑셀을 다운로드하는 시점의 최신 상태이므로, 같은 날짜의 기록이라도 다운로드 시점에 따라 값이 달라질 수 있습니다.
