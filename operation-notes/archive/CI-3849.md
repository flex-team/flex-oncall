# CI-3849: 세콤연동 활성화 풀림 및 수동 전송 미반영 확인 요청

- **문의자**: lee0ju@simmons.co.kr
- **대상 구성원**: champpjk@simmons.co.kr, jaewon.ryu@simmons.co.kr

  2. 세콤 수동 동기화 후 플렉스 근무에 반영이 안 됨

## 원인 분석

### 1. 세콤연동 활성화가 풀리는 현상
- **조사 방법**: PUT `/api/v2/time-tracking/customers/{customerIdHash}/external-providers/{externalProvider}` API 로그 확인
- **결과**: 로그에서 중간에 `false`로 끈 이력이 확인됨
- **결론**: 시스템이 자동으로 비활성화하는 경우는 **구독 해지**뿐이며, 해당 건은 유저가 직접 비활성화한 것으로 판단

### 2. 수동 동기화 후 미반영
- 세콤 → 플렉스 데이터 전송은 대부분 **실시간 반영** (최대시간 없음)
- 늦어지는 경우는 세콤 측에서 늦게 보내는 경우가 대다수
- **핵심**: 연동이 `false`(비활성화)인 상태에서 세콤이 데이터를 발송한 경우, 이후 `true`로 변경해도 이전 데이터는 소급 반영되지 않음
- 이미 플렉스에 전달된 적 있는 데이터는 수동 전송해도 재반영되지 않음

## 해결
- 버그/장애가 아닌 **사용자 조작 이슈**로 확인
- 고객에게 로그 기반으로 직접 비활성화한 이력을 안내
- 비활성화 기간 중 전송된 데이터는 소급 반영 불가함을 안내

## 비고

### 재발 방지 / 향후 참고
- 세콤연동 활성화/비활성화가 "자동으로 풀린다"는 문의 시 → 구독 해지 외에는 자동 변경 없음. API 로그를 통해 변경 주체 확인 가능
- **로그 확인 방법**: log-dashboard에서 아래 조건으로 조회
  - `json.ipath.keyword`: `/api/v2/time-tracking/customers/{customerIdHash}/external-providers/{externalProvider}`
  - `json.authentication.customerId`: 해당 회사 ID
  - `json.authentication.email`: 변경한 사용자 이메일
- 수동 동기화 미반영 문의 시 → 비활성화 상태에서 전송된 데이터는 소급 불가. 활성화 시점 이후 데이터만 반영됨을 안내
