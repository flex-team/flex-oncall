# CI-4121: sk 케미칼 근무 기록 다운로드 실패 (SocketTimeoutException)

> **상태**: 진행 중 — 2026-03-16

## 증상
- **회사**: sk 케미칼 (Customer ID: 55747)
- **문의자**: suyeon@skdiscovery.com
- **대상 구성원**: 여러 부서에서 발생 (최소 6명 확인)
- 문의 내용:
  1. 여러 부서에서 근무 기록 다운로드가 안 된다고 연락이 옴[^1]
  2. 이전에도 동일 증상으로 문의한 이력이 있으며, 당시 타임아웃이 원인이었음[^2]

## 현재까지 파악된 내용

### 조사 과정

> 💡 **판단 근거**: access log에서 async POST 요청은 모두 200 OK로 정상 수신됨[^5]
> → consumer의 traceId 추적을 통해 실제 처리 흐름 확인[^6]
> → `TrackingFileDownloadMappingService`에서 ERROR 로그 발견[^7]
> → 스택트레이스에서 `SocketTimeoutException` 확인[^8]
> → consumer가 core-api로의 OkHttp 호출 중 소켓 타임아웃 발생

### 에러 상세

| 항목 | 내용 |
|------|------|
| 에러 클래스 | `java.net.SocketTimeoutException: timeout` |
| 발생 위치 | `TrackingFileDownloadMappingService` (time-tracking-consumer) |
| 발생 구간 | OkHttp → `Http1ExchangeCodec.readResponseHeaders()` |
| 실패 원인 | consumer → 내부 API 호출 시 응답 대기 중 소켓 타임아웃 |
| **타임아웃 설정** | **3초** (`WorkScheduleInternalApiClientProperties.timeoutMillis = 3_000`)[^11] |
| 설정 키 | `flex.v2.internal-api.time-tracking.timeout-millis` |
| 첫 번째 호출 | `trackingAuthorizationHelper.getAuthorizedTargetUsers()` — 권한 확인 API[^12] |
| 대상 파일 유형 | `전체_일별_근태기록리포트` (근무 기록 엑셀) |
| 실패 건수 | **32건** (10:00~15:00 KST 전수 확인)[^7] |

### 요청 패턴

엑셀 다운로드는 비동기 처리 구조:

1. 사용자가 UI에서 다운로드 요청 → `POST /api/v2/time-tracking/customers/{id}/work-schedules/excel/async` → 200 OK (즉시 응답)[^5]
2. Kafka 메시지 → `time-tracking-consumer`의 `WithCompensatoryTimeOffExcelReportDownloadConsumer`가 수신[^6]
3. consumer가 내부 API를 순차 호출하며 엑셀 데이터 수집[^12]:
   - `[1]` 휴가 사용 조회 권한 유저 조회 (`trackingAuthorizationHelper.getAuthorizedTargetUsers`)
   - `[2]` 대상 유저 조회 (`workScheduleExportUserModelResolver.getUsersBy`)
   - `[3]` 교대 근무 컨텍스트 조회 (`trackingShiftDailyWorkScheduleExportService`)
   - `[4]` 교대 근무 유저 조회 권한 필터
4. **3번의 첫 단계(`[1]` 권한 확인)에서 내부 API 응답이 3초를 초과하여 `SocketTimeoutException` 발생**[^8][^13]
5. 실패 알림 `FLEX_FILE_STORAGE_DOWNLOAD_PREPARATION_FAILED`가 사용자에게 전송됨[^9]
