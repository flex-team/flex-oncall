# TT-16783: 외부 연동(SECOM) 등록 시 다법인 workspace에서 customerKey 충돌 오류

> **상태**: 조사 중 — 2026-03-17

## 증상
- **회사**: workspace 39602 (다법인 구조)[^1]
- Sentry에서 prod 오류 감지[^2]:
  1. SECOM 외부 연동 등록 시 `java.lang.IllegalStateException` 발생
  2. 에러 메시지: `하나의 계열사 안에 서로 다른 외부 연동 정보가 존재합니다.`
  3. 상세: `동일한 workspace(39602)내에 여러 외부 연동처가 존재합니다. (customerKeys=[secom_jy0r76pz1k, secom_k1ebayvkzy])`

## 현재까지 파악된 내용

### 에러 발생 위치

에러는 `registerNewProvider` 메서드에서 발생한다[^3].

```kotlin
// CustomerExternalProviderUpdateServiceImpl.kt:209-212
// customerKeys가 2개 이상이면 else 분기에서 에러 throw
```

동일 workspace 내 동일 providerType(SECOM)에 대해 `customerKeys`[^4]가 2개 이상 존재할 때, 어느 key를 사용할지 결정할 수 없어 예외를 발생시킨다.

### customerKey 생성 로직

`generateCustomerKey`는 customerId[^5]를 해시하여 key를 생성한다[^3]. customer마다 고유한 key가 만들어지므로, 다법인 구조에서 각 customer가 별도로 등록되면 서로 다른 key가 생성된다.

### 정상 플로우 vs 현재 상태

정상적인 등록 흐름:
1. 첫 번째 customer가 **case 0** (신규 생성)으로 진입 → customerKey 생성
2. 이후 같은 workspace의 다른 customer가 **case 1** (기존 key 재사용)으로 진입 → 첫 번째에서 생성된 key를 공유

현재 workspace 39602의 상태:
- SECOM providerType에 대해 **서로 다른 accountId(customerKey)가 2개** 존재[^1]
  - `secom_jy0r76pz1k`
  - `secom_k1ebayvkzy`

### 추정 원인 시나리오

> 💡 **판단 근거**: customerKey가 2개 존재하려면 두 customer 모두 case 0(신규 생성)에 진입해야 한다[^3]
> → 이는 두 가지 시나리오로 설명 가능:

| # | 시나리오 | 설명 |
|---|---------|------|
| 1 | **동시 등록** | 두 customer가 거의 동시에 등록을 시도하여 양쪽 모두 case 0(신규 생성)에 진입 |
| 2 | **workspace 병합** | 원래 별도 workspace였던 회사가 나중에 하나의 workspace로 합쳐지면서 각각의 customerKey가 공존 |

어느 시나리오인지 확정하려면 OpenSearch 로그에서 생성 이력을 확인해야 한다.
