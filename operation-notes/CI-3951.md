## 원인 분석

> 조사를 통해 도출한 분석 결과. 코드 레벨 조사는 미실시 (승인자 교체 핵심 로직이 다른 레포에 위치).

**승인 요청 레코드(3건)가 존재하지만, 대응하는 실제 휴가 사용 건이 확인되지 않는 데이터 불일치 상태.**

| # | 가설 | 확인 방법 | 상태 |

| 3 | 승인 요청이 만들어졌으나 휴가 사용 건이 최종 저장 실패 (트랜잭션 불일치) | 승인 요청 생성 시점과 time_off_use 테이블의 생성 이력 비교 | 대기 |

**버그 가능성 높음** — 실제 존재하지 않는 휴가에 대한 승인 요청이 잔존하여 "교체 필요" 카운트에 잡히는 것은 의도된 동작이 아님.

**데이터 확인이 우선 필요함.** 코드 조사 전에 아래 데이터를 확인해야 근본 원인을 특정할 수 있음:

1. target_uid 3건에 해당하는 승인 요청 레코드의 상세 정보 (요청자, 생성일, 상태 등)

3. 해당 유저(905694)의 휴가 사용 건 중 취소/삭제된 이력
