# Operation Notes

CI Linear 팀 티켓을 분석/해결하면서 생성되는 운영 업무 기록을 이 디렉토리에 모은다.

## 디렉토리 구조

```
operation-notes/
├── CLAUDE.md          # 운영 업무 가이드 (이 파일)
└── {ticket-id}.md     # 티켓별 분석/해결 기록
```

## 파일 네이밍 규칙

- 파일명: `{ticket-id}.md` (예: `CI-1234.md`)
- 하나의 티켓에 대한 분석과 해결 과정을 하나의 파일에 기록

## 파일 템플릿

```markdown
# {ticket-id}: {제목}

## 증상
- 리포트된 현상 요약

## 원인 분석
- 조사 과정과 근본 원인

## 해결
- 적용한 수정 사항
- 관련 PR/커밋 링크

## 비고
- 재발 방지를 위한 메모
- 관련된 다른 티켓이나 이슈
```

## 운영 업무 진행 절차

1. **티켓 확인**: Linear에서 CI 팀 티켓 내용 파악
2. **분석 기록**: 해당 티켓 ID로 md 파일 생성, 증상 기록
3. **원인 조사**: 코드/로그 분석 후 원인 기록
4. **해결**: 수정 적용 후 해결 내용 기록
5. **회고**: 비고란에 재발 방지 메모 추가
