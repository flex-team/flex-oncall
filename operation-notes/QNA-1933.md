# QNA-1933: 급여정산 해지 시 진행한 정산 관련 문의

> **상태**: 해결 완료 — 2026-03-18

## 증상
- **문제 정의**: 급여정산 구독 해지 시 만료일 이후에 명세서 공개일이 도래하면, 구성원에게 명세서가 공개되고 알림이 발송되는지 여부 확인
- **회사**: 특정 회사 없음 (일반 동작 문의)
- **요청자**: 김한나 (CS팀)[^1], 이윤선이 전달[^2]
- **대상자**: 특정 대상 없음 (일반 동작 문의)
- **영향 범위**: 급여정산을 해지하는 모든 고객
- **문제 시점**: 해지 만료일(예: 24일) vs 명세서 공개일(예: 25일) 시점 차이
- **조사 담당**: 김주원[^3]
- 문의 내용:
  1. 급여정산 구독 중 완료한 정산의 명세서 공개일이 해지 만료일 이후일 때, 해당 명세서가 구성원 급여 탭에서 공개되는지[^1]
  2. 공개 시 구성원에게 알림이 발송되는지[^1]

---

## 원인 분석

**결론: 해지 만료일 이후에도 이미 설정된 명세서 알림은 정상 발송된다 (스펙).**

구독 해지가 기존 명세서 알림 스케줄에 영향을 주지 않는 이유는, payroll → pavement 전체 알림 발송 경로에서 구독 상태를 차단 조건으로 사용하는 코드가 없기 때문이다.

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | 알림 스케줄러가 구독 상태를 체크하여 미발송 | 스케줄러 코드 확인 | ❌ 소거 — 구독 상태 체크 없음[^4] |
| 2 | 구독 해지 시 ACTIVATED 알림을 DEACTIVATED로 변경 | 해지 이벤트 핸들러 확인 | ❌ 소거 — 해지 이벤트 처리 로직 없음[^5] |
| 3 | pavement에서 발송 시 구독 상태를 체크하여 차단 | pavement 알림 발송 로직 확인 | ❌ 소거 — feature 체크는 경고 로그만[^6] |
| 4 | 구독 해지와 무관하게 알림이 발송됨 | 위 3개 가설 소거 결과 | ✅ 확정 |

### 조사 과정

> 💡 **판단 근거**: payroll 스케줄러[^4] → payroll send 로직[^7] → pavement feature 체크[^6] 순서로 전체 알림 발송 경로를 추적한 결과, 어느 단계에서도 구독 상태를 차단 조건으로 사용하지 않음을 확인

**알림 발송 흐름 (3단계)**:
1. **스케줄러** (1분 주기): `ACTIVATED` 상태 + `willNotifyAt`이 지난 알림 추출 → Kafka 메시지 발행[^4]
2. **payroll send**: Kafka 수신 → `ACTIVATED` 상태/시간만 검증 → 대상자(COMPLETED 정산) 추출 → `SendNotificationCommandV1` Kafka 발행[^7]
3. **pavement**: Kafka 수신 → 사용자 상태(퇴직/휴직 등) 필터 → topic 구독 여부 확인 → 발송[^8]

### 스펙 vs 버그 판별

**스펙으로 판단한다.**

- 이미 완료된 정산 건의 명세서 공개는 구독 해지와 무관하게 동작하는 것이 합리적이다
- 급여는 이미 지급이 완료된 상태이므로, 구성원이 명세서를 받는 것은 근로자의 권리에 해당한다
- `deactivate()` 메서드가 존재하지만 호출되지 않는 것은, 해지 시에도 기 설정된 알림을 유지하려는 의도로 해석할 수 있다 (추정)
- 다만, 이것이 명시적으로 설계된 스펙인지 단순 누락인지는 담당자 확인이 필요하다

## 해결안 / 조사 방향

### CS 답변 (합의됨)

이윤선님이 정리한 답변 방향[^13]:
1. 해지 이후에도 완료한 정산의 급여명세서 공개 일정이 되면, 웹/모바일/이메일로 **알림은 발송된다**
2. 그러나 구독 해지 시 **프로필을 통한 급여 조회는 차단**되기 때문에, 알림을 클릭해도 구성원이 실제 급여를 확인하는 데 문제가 발생할 수 있다
3. 급여 오류 발견 시에도 수정할 방법이 없으므로, **급여 공개 이후까지 1달 사용 연장을 권장**한다[^14]

### 제품 정책 판단

- 이윤선님: 공개 기능을 사용하려면 연장하도록 안내하는 것이 맞다[^15]
- 급여 명세 공개 후 오류 수정이 필요한 경우에도 정산이 열려있어야 가능하다[^16]

## 다음에 같은 문의가 오면

1. **확인**: 알림은 발송되는가? → 예. `WorkIncomePayStubNotification` 테이블에서 `status = ACTIVATED`인 알림은 구독 해지와 무관하게 발송됨
2. **주의**: 알림은 가지만, 구독 해지 후 **급여 탭 접근이 차단**되어 구성원이 실제 명세서를 열람할 수 없을 수 있음
3. **권장 안내**: "급여 명세서 공개 이후까지 1달 사용 연장을 권장합니다. 공개 후 오류 수정이 필요한 경우에도 구독이 유지되어야 합니다."

    만약 공개된 명세서에 오류가 발견된다
    그러면 정산 수정을 위해 구독이 재활성화되어야 한다
