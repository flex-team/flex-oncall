## 현재까지 파악된 내용

- 근무유형의 **휴일 1일 기준 시간이 2시간 48분**으로 설정되어 있음을 확인[^7]
- 김영준(Enhance)이 일소정근로시간이 2시간 48분으로 되어 있는 것을 발견하고, 근무유형 상세 설정 공유를 요청[^8]
- 안희종이 "저 값이 휴가 1일 부여량 및 주휴시간 외에 일 안 하는 날의 소정근로시간으로 쓰여지는 게 스펙인가요?"라는 질문을 제기[^9]
- 유지원이 "휴일 1일 기준 시간이 2시간 48분으로 설정되어 있는 것을 확인했다"고 답변[^6]

> 💡 **핵심 쟁점**: 휴일 1일 기준 시간(2시간 48분)이 근무하지 않는 날의 소정근로시간으로 사용되어 연장근무 계산에 영향을 미치는 것이 의도된 스펙인지 확인 필요[^9]

## 원인 분석

**일 연장 계산에서 `baseAgreedDayWorkingMinutes`(168분)가 휴무일의 법적 소정근로시간으로 사용되어, 12h - 2h48m = 9h12m만 일 연장으로 계산됨.**

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | `baseAgreedDayWorkingMinutes`가 휴무일(WEEKLY_UNPAID_HOLIDAY)의 법적 소정근로시간으로 사용됨 | `RegularWorkBaseWorkingMinutes.kt` 라인 67 확인 | ✅ 확정 |
| 2 | 2/22의 168분은 주기 연장(period overtime) 분배에 의한 것 | 근무유형 설정 데이터 + 주기 연장 계산 시뮬레이션 필요 | ⏳ 조사 중 |

### 확정된 원인: 일 연장 계산 (2/17)

<details>
<summary>📋 코드 분석 상세</summary>

**`RegularWorkBaseWorkingMinutes.kt:62-68`** — 단시간 근로자의 일별 법적 기준시간:
```kotlin
when (workMinutesDayCategory) {
    WorkMinutesDayCategory.WORKING_DAY -> this.getBaseWorkingMinutesPerDay(date)
    // 휴일 근무의 연장은 단시간 여부와 관계없이 절대시간 8시간으로 계산된다.
    WorkMinutesDayCategory.WEEKLY_PAID_HOLIDAY -> MAX_LEGAL_BASE_WORK_MINUTES_IN_DAY  // 480분(8시간)
    WorkMinutesDayCategory.WEEKLY_UNPAID_HOLIDAY -> workingHourRule.baseAgreedDayWorkingMinutes  // 168분
}
```

**비대칭 설계**:
- 주휴일(WEEKLY_PAID_HOLIDAY): 항상 **480분(8시간)** — 주석: "절대시간 8시간으로 계산"
- 휴무일(WEEKLY_UNPAID_HOLIDAY): **`baseAgreedDayWorkingMinutes`** — 초단시간은 168분(2h48m)

일반 근로자는 `baseAgreedDayWorkingMinutes = 480분`이므로 차이가 없지만, 초단시간 근로자는 이 비대칭이 극대화됨.

</details>

### 추가 발견: `baseAgreedDayWorkingMinutes` 산출 로직

<details>
<summary>📋 값 계산 과정</summary>

**`CustomerWorkRuleRegisterDto.kt:280-288`**:
```kotlin
baseAgreedDayWorkingMinutes = weekWorkingHourRules.let {
    val workingDays = it.count { it.dayWorkingType === WORKING_DAY }
    val totalUsualWorkingMinutes = it.sumOf { it.usualWorkingMinutes }
    minOf(MAX_LEGAL_BASE_WORK_MINUTES_IN_DAY, (totalUsualWorkingMinutes / workingDays))
}
```

이 고객의 경우:
- workingDays = 근무유형에서 WORKING_DAY로 설정된 요일 수
- totalUsualWorkingMinutes = 전체 소정근로시간 합
- 결과: 168분(2h48m) — 실제 근무유형 설정 데이터로 검증 필요

</details>

## 역사적 맥락 (구현 타임라인)

안희종·김영준의 Linear 코멘트 및 PR #7185 분석을 통해 파악한 구현 이력:

### 타임라인

1. **초기 구현** — 휴무일 연장 기준을 `(주 소정근로시간 / 5)`로 구현. 당시 노무 가이드라인이 없었음[^12]
2. **PR #7185 (2024-04-01)** — `baseAgreedDayWorkingMinutes` 속성을 `CustomerWorkRule`에 추가. 3가지 용도로 설계됨:
   - 휴무일의 연장 판단 기준 시간
   - 휴가 지급 1일 기준 시간
   - 주휴일 기준 시간 (주휴수당 가치)
   - 이 PR에서는 속성 추가 + plumbing만 수행, 실제 연장 계산 반영은 `// FIXME : TT-8768`로 남겨놓고 후속 작업으로 분리
3. **후속 작업** — `WEEKLY_UNPAID_HOLIDAY`에 `baseAgreedDayWorkingMinutes`를 연장 판단 기준으로 연결
4. **노무 가이드 변경** — "휴무일 연장도 8시간 기준으로 하면 된다"라는 가이드가 나왔으나, 코드에 미반영[^12]

### 결론

> **의도된 스펙이 아닌, 가이드 변경 미반영으로 인한 버그로 추정됨.**
> 주휴일(WEEKLY_PAID_HOLIDAY)은 이미 480분(8시간)으로 하드코딩되어 있으나, 휴무일(WEEKLY_UNPAID_HOLIDAY)은 `baseAgreedDayWorkingMinutes`를 그대로 사용 중.

안희종이 @jaeeonjin에게 "저 때 해리님이 주신 가이드가 제가 이해하기로는 휴무일 연장 판단도 8시간 기준으로 해라인 것 같은데 혹시 요거 구현이 안 된 게 맞는지" 확인 요청 중[^13].

### 법적 기준 (안희종 정리)

단시간 근로자의 연장근로시간 판단[^14]:
1. **소정근로일**: 소정근로시간 초과 시 연장
2. **휴무일**: **8시간 초과 시 연장**
3. **주 소정근로시간 초과분**: 모두 연장

→ 현재 코드는 2번이 `baseAgreedDayWorkingMinutes`(168분)를 사용하여 8시간이 아닌 2시간 48분 초과 시 연장으로 계산함
