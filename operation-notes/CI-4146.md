# CI-4146: 무료체험 구성원 등록 시 사용할 수 없는 이메일 표시

> **상태**: 고객 응답 대기 중 — 2026-03-18

## 증상
- **회사**: 주식회사캐리랩 (Customer ID: 224552[^1])
- **문의자**: goatclinic.official@gmail.com[^2]
- **대상 구성원**: 없음
- **라벨**: 구성원/문서/증명서[^1]
- **우선순위**: Medium[^1]
- **온도감**: 중 (D+1~2)[^3]
- 문의 내용:
  1. 무료체험 및 구성원 등록을 시도하면 "사용할 수 없는 이메일"이라는 메시지가 표시됨[^2]
  2. Raccoon[^4]에서 활성 구성원을 검색했으나 등록된 유저가 아닌 것으로 확인됨[^2]

## 현재까지 파악된 내용
- CS 담당 @고정윤이 Raccoon에서 해당 고객사의 활성 구성원을 검색했으나, 등록된 유저가 발견되지 않음[^3]
- @김주원이 "볼게요"로 확인 의사를 표명한 상태[^5]
- 고객에게 에러 화면 스크린샷 + 재현 시도를 요청함 (2026-03-18). 고객 응답 대기 중[^11]

## 원인 분석

"사용할 수 없는 이메일이에요" 메시지는 에러코드 `OBUS_400_004` = `DUPLICATED_EMAIL`로, **이메일 글로벌 중복 체크 실패** 시 발생한다.[^6] 이메일 형식 오류(`OBUS_400_005`)와는 별개.

핵심 증거:
- 구성원 등록 시 `hasDuplicateEmails(email)` → `workspaceUserLookUpService.getUserByUserEmail(email)` 호출[^7]
- 이 메서드는 **customer_id 필터 없이** `user.email`로만 조회하여 **전체 시스템에서 글로벌 중복 체크**[^8]
- 즉, 다른 회사에서 동일 이메일로 활성 사용자가 존재하면 등록 차단됨

| 3 | FE 단 validation 또는 무료체험 가입 플로우 자체에서 에러 | 고객에게 정확한 에러 화면/단계 확인 필요 | 🔍 추가 확인 필요 |

<details>
<summary>상세 코드 분석</summary>

**검증 플로우** (`UserOnboardingRegistrationService`):

1. **형식 검증**: `Rfc5322EmailAddressValidator.check(email)` — 실패 시 `OBUS_400_005`
2. **중복 검증**: `hasDuplicateEmails(email)` — 실패 시 `OBUS_400_004` ("사용할 수 없는 이메일이에요")

**중복 조회 쿼리** (`MemberUserMappingRepository.findByUserEmail`):
```sql
SELECT mapping FROM MemberUserMapping mapping
JOIN FETCH mapping.member
JOIN FETCH mapping.user
WHERE mapping.user.email = :userEmail
AND   mapping.user.deletedAt IS NULL
```
→ `deleted_date IS NULL` 조건으로 활성 사용자만 대상. soft-delete된 사용자는 통과.

**단건 등록** (`registerSingleUser`, line 109-125): 이메일 존재 시 무조건 에러
**겸직 등록** (`registerSingleUserOrConcurrentUser`, line 43-100): 같은 workspace 내 겸직이면 허용, 아니면 에러

</details>
