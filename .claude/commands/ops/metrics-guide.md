# 메트릭스 기록 가이드

> 이 문서는 ops 스킬이 참조하는 공통 가이드다. 스킬 실행 완료 시 아래 규칙에 따라 메트릭스를 기록한다.
> PostToolUse 훅이 ops 스킬 실행 시 자동으로 리마인드하므로, 이 가이드의 절차를 빠짐없이 수행한다.

## 1. 노트 활동 로그

대상 operation-note 최하단(각주 뒤)에 `## Claude 활동 로그` 섹션을 유지한다.
섹션이 없으면 새로 추가한다.

### 테이블 형식

```markdown
## Claude 활동 로그

| 시간 (KST) | 스킬 | 모델 | subagent 토큰 | 소요시간 | 쿡북 참조 | 요약 |
|------------|------|------|-------------|---------|----------|------|
```

### 컬럼별 기록 규칙

| 컬럼 | 규칙 | 예시 |
|------|------|------|
| 시간 (KST) | 스킬 실행 완료 시점, `YYYY-MM-DD HH:mm` | `2026-03-18 15:10` |
| 스킬 | 실행한 스킬명 (`ops:` 접두사 제외) | `investigate-issue` |
| 모델 | Claude 모델 ID 짧은 형태 | `opus-4-6`, `sonnet-4-6` |
| subagent 토큰 | 스킬 실행 중 사용한 **모든 subagent**의 `total_tokens` 합산 (정수, 천 단위 쉼표) | `85,400` |
| 소요시간 | subagent들의 `duration_ms` 합산, 사람이 읽기 쉬운 형태 | `4m 12s`, `45s` |
| 쿡북 참조 | 아래 "쿡북 히트 판정" 참조 | `F3 히트`, `미스`, `—` |
| 요약 | 스킬 실행의 핵심 결과 한 줄 | `원인 분석 완료` |

### 토큰/시간 수집 방법

- subagent(Agent 도구) 결과에 포함된 `total_tokens`와 `duration_ms`를 사용한다
- 스킬 실행 중 여러 subagent를 사용했으면 모두 합산한다
- subagent를 사용하지 않은 스킬은 토큰 `—`, 소요시간은 체감 시간으로 기록한다

## 2. METRICS.md 갱신

`operation-notes/METRICS.md` 의 테이블들을 갱신한다.

### 2-1. 활동 로그 (전체)

노트 활동 로그에 추가한 것과 동일한 행을 **역시간순(최신이 위)** 으로 추가한다.

```
| 시간 (KST) | 이슈 | 스킬 | 모델 | 토큰 | 소요시간 | 쿡북 |
```

### 2-2. 스킬별 사용량

해당 스킬의 행을 찾아 갱신한다:
- **호출 횟수**: +1
- **총 토큰**: 이번 토큰 합산
- **평균 토큰/호출**: 총 토큰 / 호출 횟수 (정수로 반올림)
- 해당 스킬 행이 없으면 새로 추가

### 2-3. 월별 요약

현재 월(`YYYY-MM`)의 행을 찾아 갱신한다:
- **이슈 수**: 이번 달에 활동 로그에 등장하는 고유 이슈 ID 수
- **스킬 호출**: 이번 달 활동 로그 행 수
- **총 토큰**: 이번 달 토큰 합산
- **쿡북 히트**: `investigate-issue` 에서 히트한 횟수 / 전체 `investigate-issue` 횟수
- **히트율**: 쿡북 히트 비율 (%)
- 해당 월 행이 없으면 새로 추가

## 3. 쿡북 히트 판정 (investigate-issue 전용)

`investigate-issue` 실행 시 Agent B(쿡북 참조)의 결과를 기반으로 판정한다.

### 판정 기준

| 판정 | 조건 | 활동 로그 표기 | METRICS.md 동작 |
|------|------|-------------|----------------|
| **히트(Hit)** | 쿡북 플로우의 단계를 따라가서 가설 확정 또는 원인 발견에 기여 | `F{N} 히트` | 플로우 히트 이력 갱신 + COOKBOOK.md `히트: N` +1 |
| **참조(Ref)** | 플로우를 읽었지만 이번 이슈와 무관하여 활용 안 함 | `F{N} 참조` | 기록만 (히트 증가 안 함) |
| **미스(Miss)** | 해당 도메인에 플로우가 없어서 처음부터 조사 | `미스` | 쿡북 미스 로그에 행 추가 |
| **해당 없음** | investigate-issue가 아닌 스킬, 또는 쿡북 참조가 불필요한 경우 | `—` | 동작 없음 |

### 히트 시 추가 동작

1. **COOKBOOK.md 갱신**: 해당 플로우의 `히트: N` 을 +1 하고, 출처 이슈 ID를 추가
2. **METRICS.md 플로우 히트 이력 갱신**:
   - 해당 플로우 행의 "총 히트" +1, "최근 히트" 날짜 갱신, "이슈 목록"에 ID 추가
   - 해당 플로우 행이 없으면 새로 추가

### 미스 시 추가 동작

**METRICS.md 쿡북 미스 로그**에 행 추가:

| 이슈 | 도메인 | 증상 요약 | 쿡북 추가 여부 |
|------|--------|----------|-------------|

- "쿡북 추가 여부"는 이후 `update-artifacts`에서 플로우가 추가되면 갱신한다

## 4. 스킬별 기록 시점

| 스킬 | 기록 시점 | 비고 |
|------|----------|------|
| `note-issue` | Step 6 (결과 보고) 후 | 쿡북 참조 = `—` |
| `investigate-issue` | Step 9 (결과 보고) 후 | 쿡북 히트/미스 판정 포함 |
| `close-note` | Phase 3 (최종 보고) 후 | 쿡북 참조 = `—` |
| `fix-issue` | Phase 6 (마무리) 후 | 쿡북 참조 = `—` |
| `maintain-notes` | Step 6 (최종 리포트) 후 | 이슈 = `일괄`, 월별 요약 행 갱신 |
| `update-artifacts` | Phase 3 (결과 보고) 후 | 쿡북 참조 = `—` |
