# 디폴트 노트 템플릿

> 모든 노트의 뼈대. 도메인별 추가 섹션은 "영향 범위"와 "해결" 사이에 삽입된다.

```markdown
# {ticket-id}: {제목}

## 증상

- 사용자 보고 내용, 에러 로그, 재현 조건 등

## 원인 분석

- 조사 과정과 발견 사항

## 영향 범위

- 이 이슈를 겪을 수 있는 사용자/회사 범위
- 금전적·법적 영향이 있는가 (있다면 도메인별 추가 섹션의 상세 체크포인트 참조)

<!-- 도메인별 추가 섹션(d:tpl)이 여기에 삽입됨 -->

## 해결

- 수행한 조치 (코드 수정, 데이터 패치, 설정 변경 등)
- PR 링크 (있으면)

## 비고

- 관련 이슈 링크
- 향후 주의사항

## 각주

- 참고한 슬랙 스레드, 문서 링크 등
```

## "영향 범위" 섹션 사용 가이드

이슈 타입에 따라 이 섹션의 깊이가 달라진다.

| 이슈 타입 | 영향 범위 작성 수준 | 도메인별 추가 섹션 |
|----------|-------------------|------------------|
| **Error** (500, 에러) | 같은 오류를 겪는 사용자/기능 범위 | 대부분 "해당 없음" |
| **Perf** (느려요) | 영향받는 사용자/기능 범위 | 대부분 "해당 없음" |
| **Auth** (403, 권한) | 접근 차단된 사용자 범위 | 대부분 "해당 없음" |
| **Data** (값이 이상) | 영향 범위 + **금전적·법적 영향 상세** | **필요** — 상세 작성 |
| **Spec** (이게 맞나요) | 스펙 확인 → 버그면 타입 전환 | 전환 후 결정 |
| **Render** (화면 이상) | FE 이슈면 영향도 낮음 | 대부분 "해당 없음" |

**게이트 원칙**: "금전적·법적 영향이 있는가?"에 대한 답이 "없음"이면 도메인별 추가 섹션은 간단히 처리. "있음"이면 상세 체크포인트로 진입.
