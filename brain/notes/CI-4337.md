# CI-4337: 휴가 취소 후 휴가 사용 내역 상태가 "승인완료"로 유지됨

> **상태**: 조사 중 — 2026-04-07

## 증상
- **문제 정의**: 휴가 취소 처리 완료 후에도 "휴가 사용 내역" 탭과 엑셀 다운로드에서 상태가 "승인완료"로 표시되고 "휴가 취소"로 갱신되지 않음
- **회사**: 주식회사 포티파이 (Customer ID: 179342)
- **요청자**: 이주화 (CS팀)
- **대상자**: yul_kim@40fycorp.com (user_id: 738824)
- **영향 범위**: 1명
- **문제 시점**: 2025-06-24 전일 휴가 등록 → 2025-06-26 09:09 취소
- 문의 내용:
  1. 6월 26일 오전 9시 9분에 휴가 취소했으나, 휴가 사용 내역 탭에서 상태가 여전히 "승인완료"로 표시됨
  2. 엑셀 다운로드 시에도 "승인완료"로 확인됨

## 현재까지 파악된 내용

### DB 조사 결과

`v2_user_time_off_event` 이벤트 현황[^1]:

| id | type | date_from | date_to | reference_id | db_created_at |
|----|------|-----------|---------|--------------|---------------|
| 7243527 | REGISTER | 2025-06-24 | 2025-06-24 | — | 2025-06-23 15:57:45 |
| 7266756 | CANCEL | 2025-06-24 | 2025-06-24 | 7243527 | 2025-06-26 09:09:40 |

- CANCEL 이벤트(id=7266756)가 2025-06-26 09:09:40에 정상 저장되어 있음. 고객 취소 시각과 일치.
- `reference_time_off_event_id = 7243527`으로 REGISTER 이벤트와 정상 연결됨.
- `v2_user_time_off_use`: user_id=738824 레코드 0건 (취소 시 물리 삭제됨)[^2].

**결론**: DB 데이터 누락 아님. 취소 자체는 정상 처리됨.

### 시스템 아키텍처 파악

- "휴가 사용 내역" 탭 API: `/api/v2/time-off/users/{userIdHash}/time-off-uses/by-use-date-range/{from}..{to}`[^3]
- 상태 표시는 Event Sourcing 기반 — `v2_user_time_off_event`의 REGISTER 이벤트를 읽고, 동일 `reference_time_off_event_id`를 가진 CANCEL/INVALIDATE 이벤트 존재 여부로 상태 결정
- 관련 클래스: `UserTimeOffUseLookUpMappingService`, `CalculatedUserTimeOffRegisterEventLookUpService`, `UserTimeOffEventLatestCancelAmongSameReferenceIdFilter`

### 현재 가설

| 가설 | 근거 | 상태 |
|------|------|------|
| H1: 상태 계산 로직이 CANCEL 이벤트를 누락 | CANCEL 이벤트 DB 존재, but UI "승인완료" | 미검증 |
| H2: 결재(approval) 레코드가 "승인완료" 상태로 남아있고 UI가 approval 기반으로 상태를 표시 | TimeOffUseStatus에 APPROVAL_COMPLETED, CANCEL_APPROVAL_COMPLETED 두 상태 존재 | 미검증 |
| H3: 직접 취소(non-approval 경로)이므로 approval 레코드 미갱신 | cancelRegisteredTimeOff() → cancelApproval() 별도 트랜잭션 | 미검증 |
