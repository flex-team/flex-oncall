# CI-4355: 구성원 일괄 업로드 후 일부 인사정보 누락

> **상태**: 원인 확정 — 2026-04-07

## 증상

- **문제 정의**: 구성원 여러 명 일괄 업로드 완료 후 사번·이메일·입사일만 저장되고, 조직·직5정보[^4]·주민등록번호·계좌정보가 누락됨[^1]
- **회사**: 케이유엠 유한책임회사 (Customer ID: 217253)[^1]
- **요청자**: 오시은 (CS팀 담당자)[^1]
- **대상자**: 일괄 등록된 구성원 560명 (오후 요청), 23명 (오전 요청)[^2]
- **영향 범위**: 해당 업로드 건에 한정 (전사 미영향)
- **문제 시점**: 2026-04-07 오전 10:49 (23명), 오후 15:58 (560명)[^2]
- 문의 내용:
  1. 구성원 일괄 업로드 완료 알림 수신 후 조직·직5정보·주민등록번호·계좌정보가 구성원 프로필에 반영되지 않음
  2. 인사정보관리의 인사발령 이력 없음
  3. 정보 일괄 변경 이력도 없음
  4. 업로드 오류 발생 후 웹뷰에서 조직 수기 입력 후 등록 완료한 케이스[^1]

## 원인 분석

### 조사 과정

FE 교차 조사(payload 정상 포함, taskId=14869 확인)로 BE 문제를 좁힌 후, DB에서 task_details 상태와 cron ERROR 로그로 원인을 확정했다.[^5]

1. DB 조회: task 14869의 task_details 중 `PERSONNEL_APPOINTMENT`(FAILED, 07:01:35~07:01:38 UTC), `BASIC`(FAILED, 07:01:38~07:02:36 UTC) 확인[^5]
2. cron ERROR 로그 조회 (be-cron 인덱스, pod `flex-prod-prod-core-cron-blue-6d7f768fcf-sl8j6`)[^6]:
   - `07:01:37 UTC` — `JpaObjectRetrievalFailureException: Unable to find JobGroup with id -9223372036854775807` → PERSONNEL_APPOINTMENT 실패
   - `07:02:36 UTC` — `UBAN_400_005:INVALID_ACCOUNT_NUMBER_FORMAT` (11명 계좌번호 오류) → BASIC 실패
3. `failedUsers` 카운트 구조 코드 확인: `sendNotification()`이 USER 타입 task_details 결과만 집계 → PERSONNEL_APPOINTMENT/BASIC 실패가 "완료"로 가려짐[^5]

### 원인

**원인 1 — PERSONNEL_APPOINTMENT 실패**: 엑셀의 직군(jobGroup) 값이 DB에 없는 ID(`-9223372036854775807`, Long.MIN_VALUE 계열)로 파싱됨. `JpaObjectRetrievalFailureException`으로 `createSetAndApplyToWaiting()` 전체 실패 → 조직·직5정보 미저장.[^6]

**원인 2 — BASIC 실패**: 560명 중 최소 11명의 계좌번호에 하이픈 등 비숫자 문자 포함(`UBAN_400_005`). `BulkChangeUnitServiceImpl` 유효성 검사 실패 → 주민번호·계좌 전원 미저장.[^6]

**구조적 맹점**: `UserOnboardingTaskCronService.sendNotification()`의 `createdUsers`/`failedUsers`가 USER 타입만 집계. PERSONNEL_APPOINTMENT·BASIC 실패는 카운트에 포함되지 않아 "완료" 알림 발송 + `failedUsers: 0`으로 표시됨.[^5]

## 영향 범위

- task 14869로 등록된 구성원 560명 전원 — 조직, 직5정보 미저장
- 계좌번호 오류 사용자(최소 11명) 포함 560명 전원 — 주민번호, 계좌 미저장
- 인사발령 이력 없음 → 이력 관리 공백[^1]
- 근태·급여 처리 영향 가능성[^2]

## 전파 범위

- `personnelAppointment` 저장 실패 → `EVENT_FLEX_USER_DATA_CHANGE` 미발행 → change-history + search 도메인 미반영
- 조직 미설정 구성원 → 권한 체크(`isPrimary` 기준) 이상 가능성

## 겸직/복수 소속

- 이 건에서 겸직 구성원 관련 여부 미확인

## 해결 방안

**즉시 데이터 복구 (CS 조치)**:
1. 직군(JobGroup) ID 오류 수정: 엑셀 재업로드 전에 직군 이름이 DB에 존재하는 값인지 확인 필요
2. 계좌번호 형식 수정: 하이픈 제거 후 숫자만 포함하도록 엑셀 수정 후 재업로드
3. 또는 등록된 구성원의 조직/직5/주민번호/계좌를 수동으로 직접 입력

**코드 개선 (별도 티켓 권고)**:
- `UserOnboardingTaskCronService.sendNotification()`: PERSONNEL_APPOINTMENT/BASIC/WORK 실패도 `failedUsers`에 포함하도록 수정 — 현재는 사용자에게 실패가 숨겨짐
- 업로드 실패 시 구체적인 에러 메시지를 알림에 포함
