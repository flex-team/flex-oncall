# CI-4117: 평가 등급 체계 — 등급 15개 생성 후 "다음" 클릭 시 validation 에러

> **상태**: 진행 중 — 2026-03-16

## 증상
- **문의자**: 나유라 (yura+103@flex.team)
- **환경**: PROD
- 문의 내용:
  1. 평가 생성 > 등급 체계 단계에서 등급을 최대 15개까지 만든 후 "다음" 버튼을 클릭하면 "등급은 최대 15개까지 추가할 수 있어요" 알럿이 뜸[^1]

## 현재까지 파악된 내용
- FE에서는 등급 15개 이상이면 추가 버튼을 숨기는 클라이언트 validation이 존재[^2]
- 빠르게 입력하면 임시 저장 API가 16개 이상 저장되는 race condition 발생[^3]
- 유라님 데이터에 임시 저장이 16개가 되어 있음[^4]
- BE 등급 임시 저장 API에는 15개 제한 validation이 없음[^5]

## 원인 분석

`DraftGradeItemCommandService.create()` 에서 등급 아이템 생성 시 개수 제한 검증이 없어, 15개를 초과하여 저장 허용됨. 검증은 "다음" 클릭 시 호출되는 `DraftEvaluationStageGradingSystemValidator.validateGradeBasics()` 에서만 수행됨.[^6][^7]

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | BE 등급 아이템 생성 API에 개수 validation 없음 | `create()` 메서드 코드 확인 | ✅ 확정 — `addDraftGradeItem()` 에 개수 체크 없음[^6] |

### 조사 과정

> 💡 **판단 근거**: Linear 코멘트에서 "임시 저장 api에 15개 validation 없음" 언급[^5]
> → `DraftGradeItemCommandService.create()` 확인[^6] — 개수 검증 로직 부재
> → `DraftEvaluationStageGradingSystemValidator.validateGradeBasics()` 에서만 `MAX_GRADE_COUNT = 15` 체크[^7]
> → BE 생성 API에 validation 누락 확정

### 스펙 vs 버그

**버그**. FE에서만 검증하고 BE에서 검증하지 않아, race condition으로 제한을 우회할 수 있음.

### 수정 시 사이드이펙트 포인트

| # | 영향 지점 | 위험도 | 설명 |
|---|----------|--------|------|
| 1 | `createAll()` 메서드 | 🟡 중간 | 동일하게 개수 검증 누락. 함께 수정 필요[^8] |
| 2 | 기존 16개 이상 저장된 데이터 | 🟢 낮음 | 추가 생성만 차단, 수정/삭제는 가능 |
| 3 | FE 에러 핸들링 | 🟢 낮음 | 에러 코드가 다를 수 있으나 동일한 400 응답 |

## 해결안 / 조사 방향

`DraftGradeItemCommandService` 의 `addDraftGradeItem()` 에서 `DraftLinearGrade` 인 경우 `items.size >= 15` 이면 예외를 발생시킨다. grade 모듈 자체 예외(`DraftGradeError.GRADE_COUNT_EXCEEDED`)를 추가하여 evaluation 모듈 의존 없이 처리.
