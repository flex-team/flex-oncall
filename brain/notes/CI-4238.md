<details>
<summary>📋 코드 분석 — 근본 원인 체인</summary>

**1단계: 대상자 생성 시 competencyGroupMappings 설정** (`EvaluationRevieweeCommandService.kt:138-158`)[^7]:
```kotlin
val competencyGroupMappings = evaluationStepAndFactor
    .filter { it.factorType == EvaluationFactorType.COMPETENCY }
    .takeIf { evaluation.useCompetencyItem }  // useCompetencyItem=false → null
    ?.map { ... }
    ?: emptyList()  // → 빈 배열
```
`useCompetencyItem=false`이면 `.takeIf`가 null을 반환하여 `competencyGroupMappings = []`로 설정된다. 이것은 **의도된 동작**이다.

**2단계: RevieweeEvaluationItem 생성 시 필터** (`RevieweeEvaluationItemCommandService.kt:1177-1197`)[^8]:
```kotlin
val competencyFactorEnabled = reviewee.evaluateStepAndFactors
    .any { it.factorType == EvaluationFactorType.COMPETENCY }  // → true (TOP_DOWN/COMP)

val hasCompetencyGroup = reviewee.competencyGroupMappings
    .any { v -> v.evaluationCompetencyGroupIdentity != null }  // → false (빈 배열)

val canCreateForCompetency = !competencyFactorEnabled || hasCompetencyGroup
// = !true || false = false

(performanceFactorEnabled || competencyFactorEnabled) && canCreateForCompetency
// = true && false = false → 대상자 필터링됨
```
`canCreateForCompetency`가 `evaluation.useCompetencyItem` 플래그를 참조하지 않는다. **이것이 버그다.**

</details>

### 5 Whys

1. 왜 할일이 발송되지 않았나? → `reviewee_evaluation_item`이 0건이라 발송 대상이 없음[^9]
2. 왜 `reviewee_evaluation_item`이 생성되지 않았나? → `createAll` 필터에서 대상자가 제외됨[^8]
3. 왜 필터에서 제외됐나? → `canCreateForCompetency=false` (COMPETENCY factor 존재 + competencyGroup 없음)[^8]
4. 왜 competencyGroup이 없나? → `useCompetencyItem=false`일 때 의도적으로 빈 배열 설정[^7]
5. **왜 필터가 이를 고려하지 않나?** → `createAll` 필터가 `evaluation.useCompetencyItem` 플래그를 참조하지 않는 누락[^8]

### 스펙 vs 버그 판별: **버그**

- `EvaluationRevieweeCommandService`는 `useCompetencyItem=false`일 때 `competencyGroupMappings`를 의도적으로 비운다[^7]
- `RevieweeEvaluationItemCommandService.createAll`은 이 케이스를 고려하지 않고, COMPETENCY factor가 있으면 competencyGroup이 필수라고 판단한다[^8]
- **두 서비스 간 계약 불일치**가 근본 원인이다

## 다음에 같은 문의가 오면
1. `reviewee_evaluation_item` 테이블에서 해당 evaluation_id로 조회하여 대상자 존재 여부 확인
2. 0건이면 → `evaluation_reviewee`의 `evaluate_step_and_factors`에 COMPETENCY factor가 있는지, `competency_group_mappings`가 비어있는지 확인
3. 위 조건 해당 시 → `evaluation.use_competency_item` 값 확인
4. `use_competency_item=0` + COMPETENCY factor + competencyGroupMappings=[] → **이 버그에 해당** (PR#5199 배포 후 해결됨)
5. 배포 전 환경이면: 대상자 제외 후 재추가 + 항목 요청 재실행
