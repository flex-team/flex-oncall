# CI-3998: 교대근무 엑셀 스케줄 업로드 시 연차 미반영

> **상태**: Pending — 2026-02-27 (KST)
> **verdict**: bug-fixed — `toDraftRegisterModel()`에서 `filterIsInstance<TimeOff>()` 결과가 빈 리스트일 때 null과 혼동되어 DraftTimeOff 미생성

## 증상

- **문제 정의**: 교대근무에서 엑셀로 스케줄 업로드 시 다른 코드(근무 유형 등)는 정상 반영되나 연차만 반영되지 않음
- **회사**: 테크타카로지스틱스 (Customer ID: 70145)
- **요청자**: jude@techtaka.com
- **라벨**: 근태/근무/휴가
- **담당자**: 김영준(Enhance)
- **Linear**: [CI-3998](https://linear.app/flexteam/issue/CI-3998/교대근무-엑셀로-스케줄-업로드시-연차가-반영되지-않음)

## 원인 분석

- 김영준 님이 원인 파악:
  - `toDraftRegisterModel()`에서 `filterIsInstance<TimeOff>()`가 빈 리스트를 반환하는 경우 null과 혼동됨
  - 게시된 근무 스케줄이 있는 날짜에 엑셀로 연차를 추가하면 `DraftTimeOff`가 생성되지 않는 버그
- 이지선 님이 실제 재현 확인: "엇 영준님 진짜로 연차 업로드가 안되는데요...????"

## 해결

- **수정 PR**: [flex-timetracking-backend#11874](https://github.com/flex-team/flex-timetracking-backend/pull/11874)
  - `!= null` 조건을 `!isNullOrEmpty()`로 변경하여 빈 리스트와 null을 구분
- **cherry-pick PR (RB 포함)**: [flex-timetracking-backend#11878](https://github.com/flex-team/flex-timetracking-backend/pull/11878)
- 로컬 확인 후 dev 머지 → 이번 주 수요일 배포에 포함 (2026-03-05 배포)
- 같은 고객사 다른 관리자분에게도 동일 문의 인입 → 배포 전까지 수동 업로드 안내
