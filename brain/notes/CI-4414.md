# CI-4414: 개인 목표 생성 후 내목표 탭에서 조회 안 됨

> **상태**: 진행 중 — 2026-04-13

## 증상

- **문제 정의**: 개인 목표를 생성했으나 목표 > 내목표 탭에서 해당 목표가 보이지 않음
- **회사**: INNIT ENTERTAINMENT (Customer ID: 195210)
- **요청자**: 이주화 (CS 담당), intercom 인입
- **대상자**: psm@innitent.com (user_id: 753130, 삭제 계정 아님)
- **영향 범위**: 최초 1명 확인이나, `relation_action` 토픽 밀림으로 인해 해당 시간대 목표 생성 사용자 전반 영향
- **문제 시점**: 2026-04-13 15:04 최초 문의
- 문의 내용:
  1. 목표 > 목표 생성 > 개인목표 추가 → 목표명 "개인목표설정테스트중" 생성
  2. 생성 후 목표 > 내목표 탭에서 해당 목표가 조회되지 않음

## 원인 분석

### 조사 과정

> 처음 FE invalidate 문제로 추정(이지훈)[^1]되었으나, 권병준이 prod에서 재현 후 BE/Matrix 문제로 정정[^2].
> 상세 조회(DB 직접)는 정상이나 목록 조회(matrix)에서만 미표시 → matrix 인덱싱 문제로 특정[^3].
> 양태일이 인가 컨슈머 에러를 확인, `relation_action` Kafka 토픽 밀림으로 최종 특정[^4].

### 원인

- **`relation_action` Kafka 토픽 밀림** → matrix authorization 인덱싱 처리 지연[^4]
- 상세 조회: DB 직접 조회 → 목표 생성 직후 정상 반영
- 목록 조회(내목표 탭): matrix authorization 인덱스 기반 → 토픽 밀림으로 갱신 지연

### 해결

- 양태일: "밀림 현상은 해소되었다고 합니다" (2026-04-13 15:32)[^4]
- 윤성복: matrix authorization batch-index operation API 실행[^5]
  ```
  POST /api/operation/v2/matrix/authorization/resolved-auth/from/{fromCustomerId}/to/{toCustomerId}/batch-index
  ```

## 영향 범위

- `relation_action` 토픽 밀림 구간 동안 목표를 생성한 사용자 전반 (내목표 탭 미표시)
- 밀림 해소 후 자연 반영 또는 batch-index로 강제 동기화 가능

## 발견한 스펙/제약

- 내목표 탭 목록 조회는 DB가 아닌 matrix authorization 인덱스 기반[^3]
- 상세 조회(개별 목표 클릭)는 DB 직접 조회라 즉시 반영[^3]
- `relation_action` 토픽 밀림 시 matrix authorization 인덱싱 전체가 지연됨[^4]
