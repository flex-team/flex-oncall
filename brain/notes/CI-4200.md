# CI-4200: 재직자 메일주소 일괄 변경 작업 가능 여부 검토

## 증상
- **문제 정의**: 회사 도메인 변경으로 재직자(최대 400명) 메일주소를 일괄 변경할 수 있는지, 소요 기간은 얼마인지 문의
- **회사**: 주식회사 이도 (Customer ID: 212445)
- **요청자**: 안상연 (@sangyeon, CS팀)[^1]
- **대상자**: 재직자 최대 400명 (미확정)[^2]
- **영향 범위**: 전사 (재직자 전원)
- **문제 시점**: 2026-03-24
- 문의 내용:
  1. 등록된 구성원(재직자만) 메일주소 일괄 변경이 가능한지
  2. 작업 소요 기간은 얼마나 되는지

## 원인 분석

### 조사 과정
> 💡 문의성 티켓으로 별도 원인 분석 불필요. 기존 Operation API 기능으로 대응 가능한 건.
> → CI-4124에서 동일 유형(일괄 이메일 변경) 작업을 Operation API로 처리한 선례 확인[^3]
> → 해당 API가 prod에 배포되어 있는지 확인 필요 (김영준 확인 예정)[^4]

### 원인
- 기능 문의 건으로 시스템 이슈 없음. Operation API를 통한 일괄 이메일 변경 기능이 이미 존재[^5]

## 해결
- Operation API 일괄 이메일 변경 기능으로 처리 가능함을 안내[^5]
- 엑셀 템플릿(Flex.xlsx)을 첨부하여 데이터 준비 방법 가이드[^6]
- 작업 소요 기간: 영업일 1일로 안내[^7]
- 실제 변경 작업이 필요할 때 새 티켓으로 요청하도록 안내 후 종료[^8]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 구성원 이메일 일괄 변경 요청인지 확인. 단건이면 CI-4118 참고
2. **원인 판별**: 문의성 티켓 — Operation API(`PATCH /action/v2/operation/core/customers/{customerId}/emails/change/bulk`)로 처리 가능[^3]
3. **조치**:
   - 엑셀 템플릿 전달 (변경 전/후 이메일 목록 준비 요청)
   - 실제 작업 시 새 티켓 생성 요청
   - 소요 기간: 영업일 1일

- 이메일 중복 체크 로직(`hasDuplicateEmails`)으로 인해 다른 고객사에 동일 이메일이 존재하면 실패 가능 — 사전 확인 필요
