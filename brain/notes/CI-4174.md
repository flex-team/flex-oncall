# CI-4174: 건강보험 제외 대상인데 중도정산 시 2025년 사회보험 연말정산금액에 건강보험, 고용보험 금액이 들어가 있습니다

## 증상

- **문제 정의**: 건강보험 제외 대상자의 중도정산에서 2025년 사회보험 연말정산 금액에 건강보험/고용보험 금액이 잘못 표시됨
- **회사**: 엔엑스씨 (Customer ID: 209287)
- **요청자**: sjyang@nxc.com (CS: 이주화)
- **대상자**: blatun@naver.com
- **영향 범위**: 최소 1명 (같은 조건의 다른 구성원 미확인)
- **문제 시점**: 2026-03-20 중도정산 진행 중
- **긴급도**: High — 오늘 정산 마감 필요
- 문의 내용:
  1. 대상자의 건강보험 공제 이력이 전혀 없음[^1]
  2. 건강보험 가입으로 상태 변경 시 금액이 나왔으나, 출처 불명[^2]
  3. 현재 진행 중인 정산에서도 건강/장기요양 모두 0원[^2]
  4. 2025년 1~5월 급여자료 이관 파일에도 건강보험 연말정산/장기요양보험 연말정산 금액 없음[^5]
  5. 2025년 연말정산 시에도 건강보험료 공제금액이 0원이었음[^8]

### 문제 3가지 (CS 정리)

| # | 문제 | 설명 |
|---|------|------|
| 1 | 공제 없이 납부 금액 표기 | 공제한 금액이 없는데 총 납부한 금액에 금액이 표기됨[^4] |
| 2 | 제외 전환 시 환급 발생 | 건강보험을 제외로 바꾸면 금액이 그대로 환급으로 돌아감[^4] |
| 3 | 결정세액 영향 | 건강보험 금액이 결정세액에 영향을 미침[^4] |

### 용어 정리

- **중도정산**: 구성원이 퇴직 등의 사유로 연도 중간에 급여·세금을 정산하는 절차
- **확정/확정해제**: 중도정산 단계에서 정산 결과를 확정(잠금)하거나 확정을 해제(잠금 해제)하는 조작
- **전년도 사회보험 연말정산**: 전년도에 납부한 사회보험료(건강보험·고용보험 등)의 과부족을 정산하는 절차. 당해연도 급여에 정산금이 반영됨

## 원인 분석

**중도정산 단계 확정/확정해제 시 전년도 사회보험 연말정산 금액이 리버트되지 않음 + 2월 말 픽스로 해당 금액이 납부한 보험료에 합산됨**[^17]

### 재현 경로

1. 전년도 사회보험 연말정산금이 발생한 상태에서 중도정산 단계 확정 → 정산 결과에 금액 저장
2. 확정 해제 시 저장된 금액이 리버트되지 않음 (기존 스펙, 지금까지는 문제없었음)
3. 2월 말 픽스 이후, 최신 정보 반영 시 당해연도 정산 결과 연말정산 항목의 사회보험 금액도 납부한 보험료에 합산하게 됨 → 리버트 안 된 (1)의 금액이 합산되어 건강보험/고용보험 금액이 표시됨

### 스펙 vs 버그: **버그**

확정 해제 시 정산 결과 금액이 리버트되지 않는 것이 기존 스펙이었으나, 2월 말 픽스로 합산 로직이 추가되면서 기존 스펙의 허점이 드러난 것이다[^17].

## 해결

### 즉시 대응 (상담 종료)

임시 워크어라운드를 고객에게 안내하여 정산 마감 완료[^18]:
1. 건강보험을 **제외**로 변경
2. 중도정산 단계 **확정**
3. 다시 **확정 해제** 후 **포함**으로 변경 → 전년도 사회보험 납부한 보험료가 0으로 리셋됨[^17]

### 근본 해결 — 코드 수정 (예정)

확정 해제(2-a 상황) 시 저장된 정산 결과 금액을 리버트하는 처리 추가[^19]. 이한결님-김주원 대면 논의 완료, 코드 대응 예정[^19].

## 다음에 같은 문의가 오면

1. **먼저 확인**: 중도정산 대상자가 "확정 → 확정해제 → 최신 정보 반영"을 거쳤는지 확인. 전년도 사회보험 연말정산금이 발생한 상태에서 이 경로를 거치면 동일 증상이 재현됨.
2. **원인 판별**: 건강보험 공제 이력이 없는데 납부 금액이 표시되면 이 버그에 해당.
3. **조치**: 워크어라운드 적용 — 건강보험을 제외로 변경 → 확정 → 확정해제 → 포함으로 변경. 코드 픽스가 배포되었으면 최신 정보 반영만으로 해결됨.
