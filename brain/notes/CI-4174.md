# CI-4174: 건강보험 제외 대상인데 중도정산 시 사회보험 연말정산금액에 건강보험/고용보험 포함

> **상태**: In Progress — 2026-03-20 (KST)
> **verdict**: bug-fix — 중도정산 확정 해제 시 연말정산 항목 리버트 누락 버그, 핫픽스 배포 완료

## 증상

- **문제 정의**: 건강보험 제외 대상 구성원의 중도정산 시 2025년 사회보험 연말정산금액에 건강보험, 고용보험 금액이 잘못 포함됨. 해당 구성원은 건강보험 공제 이력이 전혀 없고, 2025년 연말정산에서도 건강보험료 공제금액이 0원이었음. 당일 정산 마감이 필요한 긴급 건.

## 원인 분석

이한결이 dev에서 재현 후 원인을 파악:

**재현 경로:**
1. 전년도 사회보험 연말정산금이 발생한 상태에서 중도정산 단계 확정
2. 다시 확정 해제
3. 해당 구성원 중도정산 최신 정보 반영 → 제보된 상황 발생

**근본 원인:**
1. 중도정산 단계 확정 시, 정산 결과 - 연말정산 항목에 사회보험 연말정산 금액이 저장됨 (1-a)
2. 확정 해제 시, 1-a에서 저장된 금액이 **리버트되지 않음** (기존 스펙상 리버트 안 해도 문제없었음)
3. 2월 말 핫픽스 이후, 전년도 사회보험 연말정산 납부 보험료에 당해연도 정산 결과 연말정산 항목의 사회보험 금액도 합산하도록 변경 (의도된 기능)
4. 이 합산 과정에서 2-a에서 리버트되지 않은 1-a 금액이 함께 합산됨

**문제점:**
- 공제한 금액이 없는데 총 납부한 금액에 금액이 표기됨
- 제외로 바꾸면 해당 금액이 그대로 환급으로 전환됨
- 원천징수영수증에 표기되는 상황이라 단순 0원 처리 불가

## 해결

- **임시 해결 (제품 내 수동 조치):**
  1. 건강보험을 "제외"로 변경
  2. 중도정산 단계 확정
  3. 다시 확정 풀고 1번 옵션을 "포함"으로 변경 → 전년도 사회보험 납부 보험료 0으로 리셋됨

- **코드 수정:** 이한결이 김주원과 논의 후, **확정 해제 시 연말정산 항목 리버트** 방식으로 코드 대응
- **핫픽스 배포 완료** (이한결, 2026-03-20 18:41 KST)
- 다행히 표준세액공제 적용 대상이라 소득세가 발생하지만 세금에 실질적 영향은 없었음
