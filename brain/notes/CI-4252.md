# CI-4252: 쉬는 날 기본유형 중 근로자의 날(5월 1일) 삭제 요청

> **상태**: 처리 완료, 고객 확인 대기 — 2026-03-30

## 증상
- **문제 정의**: 해외 법인 적용을 위해 기본 휴일 정책에서 근로자의 날(LABOR_DAY, 5월 1일) 삭제 요청
- **회사**: glluga JAPAN (Customer ID: 205274)[^1]
- **요청자**: 고은아 (euna.go@ohora.com)[^1] — CS 담당자 김현욱이 접수
- **대상자**: euna.go@ohora.com[^1]
- **영향 범위**: glluga JAPAN 전사 (기본 휴일 정책 적용 대상)
- **문제 시점**: 2026-03-30 접수, 당일 처리 (2026년부터 적용)
- 문의 내용:
  1. 설정 > 쉬는 날 에서 기본 유형의 근로자의 날(5월 1일) 삭제 요청
  2. 해외(일본) 법인 적용이 목적이므로 기본 유형에서만 수정 요청[^2]

## 현재까지 파악된 내용

### 처리 과정

1. CS팀 김현욱이 이슈 접수 후 온콜에 전달[^3]
2. 해당 회사에 '기본', '일본법인' 두 개의 휴일 정책이 있음을 확인[^4]
3. 일본법인 정책은 테스트용으로 생성된 것으로 확인 → **기본 정책**에서 삭제하는 것이 맞음[^5]
4. 2026년부터 삭제 적용 확정[^6]
5. 김영준(Enhance)이 Operation API를 통해 처리 완료[^7]

### 처리 방법

**Step 1: 휴일 그룹 ID 조회**[^7]
```sql
select *
from v2_customer_holiday_group
where customer_id = 205274;
-- 결과: targetCustomerHolidayGroupId = 194223 (기본 정책)
```

**Step 2: Operation API로 근로자의 날 삭제**[^7]
```
POST /action/operation/v2/holiday/customers/customer-holiday-groups/delete
{
  "actorCustomerId": 205274,
  "actorUserId": 865518,
  "targetCustomerHolidayGroupId": 194223,
  "publicHolidayName": "LABOR_DAY",
  "selectedYear": "2026",
  "appliedEveryHoliday": true,
  "rootHoliday": "--05-01"
}
```

> ⚠️ `appliedEveryHoliday: true` — 2026년 이후 매년 적용됨을 의미. 특정 연도만 제거하려면 `false`로 설정할 것[^7]

## 관련 법령
- **근로자의 날 제정에 관한 법률 제1조**: 5월 1일을 근로자의 날로 정하며, 이 날은 「근로기준법」에 따른 유급휴일로 한다
- 근로자의 날은 **법정유급휴일**이므로 국내 법인에서 삭제 시 법적 리스크 발생
- 이번 케이스는 해외(일본) 법인 적용이 목적이므로 법적 문제 없음[^2]
- **주의**: 국내 법인 기본 정책에서 근로자의 날 삭제 요청이 오면 반드시 법령 위반 여부 검토 필요

## 다음에 같은 문의가 오면

1. **먼저 확인**: 삭제할 휴일 정책이 어떤 그룹인지 특정 → `v2_customer_holiday_group WHERE customer_id = {id}` 조회
2. **원인 판별**: 여러 정책이 있으면 어떤 정책에서 삭제할지 고객에게 재확인
3. **조치**: Operation API `POST /action/operation/v2/holiday/customers/customer-holiday-groups/delete` 호출
   - `appliedEveryHoliday: true` → 해당 연도부터 매년 삭제
   - `appliedEveryHoliday: false` → 특정 연도만 삭제
4. **주의**: 국내 법인의 법정유급휴일(근로자의 날, 공휴일) 삭제는 법적 리스크 검토 후 진행
