# CI-4309: 감사로그 다운로드 요청 (스트라드비젼)

## 증상
- **문제 정의**: 엔터프라이즈 플랜이 아닌 고객이 장기간 감사로그 데이터 다운로드를 요청
- **회사**: (주)스트라드비젼 (Customer ID: 25546)
- **요청자**: 유지원 (CS팀)
- **대상자**: gyusun.lee@stradvision.com (이규선)[^1]
- **영향 범위**: 1명 (문의자)
- **문제 시점**: 2025-09-22 ~ 2026-01-30
- 문의 내용:
  1. 감사로그 데이터 다운로드 요청 — 법적 대응 목적[^2]
  2. 이벤트: 수정, 삭제, 생성
  3. 정보 유형: 근무정보, 휴가정보, 기타

## 원인 분석

### 조사 과정
> 💡 제품 내 감사로그 다운로드 기능은 **7일 제한**이 있으며, 장기간 조회는 엔터프라이즈 플랜 전용[^3]
> → 고객이 법적 대응 목적으로 약 4개월치 데이터가 필요하여 별도 요청 발생
> → raccoon audit operation API를 통해 기간 제한 없이 다운로드 가능[^4]

### 원인
- 제품 기능의 기간 제한(7일)으로 인해 고객이 직접 다운로드할 수 없음[^3]

## 해결
- raccoon audit operation API로 감사로그 다운로드 실행[^5]
- API 호출 시 문의자 메일로 파일이 직접 발송됨
- 4건 중복 발송됨 (김영준 2회 + 김주원 2회 클릭)[^6]

### Operation API 정보

```
POST https://flex-raccoon.grapeisfruit.com/proxy/audit/action/operation/v2/audit/{customerId}/download?userId={userId}
```

| 파라미터 | 값 | 설명 |
|---------|---|------|
| `customerId` (path) | 25546 | 고객사 ID |
| `userId` (query) | 857604 | 문의자 user ID |

```json
{
  "personalInformationType": [
    "근태_근무정보", "근태_휴가정보", "기타"
  ],
  "eventCategory": [
    "CREATE", "UPDATE", "DELETE"
  ],
  "from": "2025-09-22T00:00:00+09:00",
  "to": "2026-02-01T00:00:00+09:00"
}
```

**응답**: 200 OK (body 없음) — 파일은 userId에 해당하는 사용자의 이메일로 발송됨[^7]

## 발견한 스펙/제약
- 감사로그 다운로드 API는 **200 응답만 반환**하고, 실제 파일은 이메일로 비동기 발송[^7]
- 제품 내 감사로그 다운로드는 **7일 기간 제한**[^3]
- 장기간 감사로그 요청은 법적 요건으로 협조 필요 — 엔터프라이즈 유도 목적의 제한이지만 법적 대응 시 거부 불가[^8]

## 다음에 같은 문의가 오면

1. **먼저 확인**: [감사로그데이터 전달 대응 관련 문서](https://www.notion.so/flexnotion/2450592a4a928024b02eebe84fd722b6) 참조[^9]
2. **raccoon Swagger 접속**: `https://flex-raccoon.grapeisfruit.com/swagger/audit` → "audit operation" 섹션
3. **API 실행**: `POST /action/operation/v2/audit/{customerId}/download?userId={userId}`
   - `personalInformationType`: 고객 요청에 맞게 선택 (근태_근무정보, 근태_휴가정보, 기타 등)
   - `eventCategory`: CREATE, UPDATE, DELETE 중 선택
   - `from`/`to`: KST 기준 시작/종료 시간 (ISO 8601)
4. **주의**: 200 응답만 오고 파일은 **메일로 발송**됨 — 중복 클릭 주의
