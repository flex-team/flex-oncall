# CI-4410: 기본정보 일괄 변경 템플릿 필드 오류

> **상태**: 조사 중 (버그 1 재검토 필요) — 2026-04-13

## 증상

- **문제 정의**: 기본정보 일괄 변경 템플릿에서 급여계좌 금융기관 셀에 SC제일은행이 없고, 국적 셀에 국적 영문 항목이 빠져 있어 업로드 시 오류 발생
- **회사**: 이도에코제주 (Customer ID: 224479)[^1]
- **요청자**: jemoon12@yido.com[^1]
- **대상자**: 기본정보 일괄 변경 기능을 사용하는 전체 고객사
- **영향 범위**: 전체 고객사 공통 — SC제일은행 급여계좌 보유 구성원 또는 국적 영문 항목 입력이 필요한 구성원 포함 시 일괄 변경 불가
- **문제 시점**: 2026-04-13 문의 접수[^1]
- 문의 내용:
  1. 급여계좌 금융기관 작성 셀에 SC제일은행이 없음[^1]
  2. 국적 작성 셀에 국적 영문 항목이 빠져 있으며, 현재 상태 그대로 업로드 시 오류 발생[^1]

## 문제 평가
> assess 완료 — 2026-04-13

- **도메인**: `:account` (계정/구성원) — repo: `flex-core-backend`
- **이슈 타입**: Data (템플릿 필드 누락) + Error (업로드 시 오류)

### 성격
정적 xlsx 파일(`기본정보_변경_템플릿_v1.0.9.xlsx`)의 드롭다운 옵션이 백엔드 코드(`BankCode`, `UserPersonalCountry` enum)와 동기화되지 않은 상태.
- `BankCode.kt`에 `SC제일은행(CODE_023)`이 존재하지만 xlsx 파일에 누락됨[^5]
- 국적 드롭다운 값 형식이 한글명("모로코")이나 백엔드가 기대하는 형식은 enum name("MA") — 형식 불일치로 업로드 오류 발생[^8]
- 템플릿 다운로드는 `ClassPathResource`로 정적 파일을 그대로 내려줌 — 동적 드롭다운 생성 없음[^7]

### 영향 범위
전체 고객사 공통 — `기본정보_변경_템플릿_v1.0.9.xlsx`를 사용하는 모든 회사에 동일하게 영향.
실제로 문제를 겪는 케이스:
- SC제일은행 급여계좌를 가진 구성원을 포함한 회사 (기본정보 일괄 변경 시 드롭다운에서 선택 불가)
- 국적 컬럼을 포함한 기본정보 일괄 변경을 시도하는 모든 회사 (국적 값 형식 불일치로 업로드 오류 발생)

진행형 여부: 지금도 동일 버전 템플릿이 배포 중이므로, 해당 케이스를 시도하는 모든 회사가 영향받음.

### 긴급도
**일반** — 데이터 오염 없음 (업로드 자체가 오류로 차단됨). 특정 조건(SC제일은행 or 국적 영문)의 사용자만 차단.

### 조사 방향
COOKBOOK 해당 없음. 기술 조사 완료:
- **verdict**: `bug` — 정적 xlsx 파일이 enum 변경에 동기화되지 않음
- **해결**: xlsx 파일에 SC제일은행 드롭다운 추가 + 국적 영문 항목 추가 후 버전업 필요

### 확정 원인
두 가지 독립된 버그로 조사 시작. **버그 1은 재검토 중**.

1. **SC제일은행 누락** [재검토 중]: 초기 조사에서 서버 배포 v1.0.9에 없다고 판단했으나, `flex-core-backend` main 브랜치 확인 결과 v1.0.5~v1.0.9 전 버전에 `[023] SC제일은행` 포함되어 있음[^10] — 초기 조사에 사용한 파일이 실제 서버 배포 파일이 아닐 가능성. Production 직접 확인 후 결론.

## 해결안

### 버그 1 (SC제일은행 누락) — 보류 중
Production 직접 확인 후 결정. main 브랜치에 이미 포함되어 있을 경우 코드 변경 불필요.
- 확인 방법: production에서 기본정보 일괄 변경 템플릿 다운로드 → sheet2 D컬럼 확인
- 확인 후 누락 시: `기본정보_변경_템플릿_v1.0.9.xlsx`의 sheet2 D 컬럼에 `[023] SC제일은행` 추가 후 버전업, `BulkChangeExcelTemplate.kt`의 `EMPLOYEE_INFO` 파일명 변경

### 버그 2 (국적 값 형식 불일치)
두 가지 수정 방향:

**A. xlsx 수정 (단기)**: sheet2 C 컬럼 국적 드롭다운 값을 enum name(`MA`, `KR` 등)으로 교체
- 단점: 사용자가 `MA`, `KR` 등 코드를 선택해야 하므로 UX 저하

**B. 백엔드/프론트엔드 변환 로직 추가 (권장)**: `UserPersonalCountry`의 `labelKorean`("모로코") → enum name("MA") 변환 deserializer 또는 프론트엔드 변환 로직 추가
- `BulkChangeUnitData.personalNationality` 필드에 custom deserializer 적용 (`labelKorean` 매핑 지원)
- xlsx는 현행 유지(한글명), 백엔드에서 변환 처리

### 장기 해결 (공통)
동적 드롭다운 생성으로 전환 — BankCode/UserPersonalCountry enum에서 xlsx 생성 시 자동 주입. 이후 enum 추가 시 자동 반영. 다만 xlsx 생성 로직 구현 필요.
