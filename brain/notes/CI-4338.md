### 스펙 vs 오퍼레이션 판별

**스펙 (제품 정책)**: 진행 중인 리뷰의 질문지 수정은 제품 비지원[^7]
- "리뷰를 게시한 이후에는 수정 안됨" — 이윤주 발언[^7]
- 유저 가이드에도 명시됨[^4]

**오퍼레이션 가능 범위**: **질문 제목/설명 텍스트 수정만** 예외 허용[^3]
- 질문 추가/삭제/타입 변경 → 불가능
- 섹션명(SUBTITLE 타입 question의 question 값) 수정 → 가능 (동일 패턴)

## 발견한 스펙/제약
- **구리뷰 섹션명 저장**: `review_question` 테이블의 `question_type = 'SUBTITLE'` 행의 `question` 컬럼[^6]
- **진행 중 리뷰 질문 수정**: 제품 미지원. DML + question_log 동기화로만 가능[^3][^4]
- **question_log 동기화 필수**: `review_question.question` 변경 후 반드시 `question_log.content`도 동기화해야 리뷰어 화면에 반영됨[^5]
- **템플릿 독립**: 리뷰와 템플릿은 서로 독립 개체. 리뷰 생성 시 복사본 템플릿이 연결. 템플릿을 수정해도 진행 중 리뷰에는 영향 없고, 역방향도 동일[^7]

-- 하향평가 리뷰의 template_id 확인 (review_set.id 확인 후)

SELECT rq.id, rq.template_id, rq.question_type, rq.question, rq.description
FROM flex_review.review_question rq
WHERE rq.template_id = {template_id}

### Step 3: UPDATE 실행 (결재 필요)

```sql
-- 섹션명 변경: "종합 self review" → "종합 review"
UPDATE flex_review.review_question
SET question = '종합 review'
WHERE id = {subtitle_question_id}
  AND template_id = {template_id}
  AND question_type = 'SUBTITLE';

-- 질문1 변경: "나의 Impact를 스스로 평가" → "피평가자의 Impact..."
UPDATE flex_review.review_question
SET question = '지난 7주 동안의 피평가자의 Impact를 스스로 평가해 주세요. (5점 만점)'
WHERE id = {question1_id}
  AND template_id = {template_id};

-- 질문2 변경: "내가 일하는 방식" → "피평가자의 일하는 방식"
UPDATE flex_review.review_question
SET question = '지난 7주 동안의 피평가자의 일하는 방식에 얼마나 부합했는지를 스스로 평가해주세요. (5점 만점)'
WHERE id = {question2_id}
  AND template_id = {template_id};
```

### Step 4: question_log 동기화 (결재 필요)

```sql
-- review_question 변경 후 question_log 동기화
UPDATE flex_review.question_log ql
JOIN flex_review.review_question rq ON ql.question_id = rq.id
SET ql.content = rq.question,
    ql.description = rq.description
WHERE rq.template_id = {template_id}
  AND (rq.question != ql.content OR rq.description != ql.description);
```

> ⚠️ **주의 사항**:
> - 변경 전 반드시 백업 SELECT 실행 후 결과 보관
> - UPDATE는 결재 승인 후 실행
> - 진행 중인 하향평가 답변이 이미 있는 경우, 리뷰어 화면에서는 질문 텍스트만 변경되며 기존 답변은 그대로 유지됨
