# CI-4362: 워크플로우 승인 문서 승인 동기화 오류

> **상태**: 해결 완료 — 2026-04-08
> **담당**: 임형태 (Enterprise Product BE)

## 증상

- **문제 정의**: 승인자가 문서를 승인했으나 승인 버튼이 계속 활성화되고, "승인라인에 포함되지 않은 사용자에요" 오류 메시지가 노출된다
- **회사**: 차봇 모빌리티 주식회사 (Customer ID: 2738)
- **요청자**: js.jo@chabot.co.kr[^1]
- **대상자**: js.jo@chabot.co.kr (user_id: 465509, 승인자 본인)[^1][^7]
- **영향 범위**: 해당 문서 1건 (문서번호: 2026-521, 출장기안) + **시스템 전체 78건/일 sync failure, 117건/일 승인 실패**[^11]
- **문제 시점**: 2026-04-07 12:39 이후 (js.jo 승인 처리 이후)[^8]
- 문의 내용:
  1. 승인자로 지정되어 문서 승인을 수행했으나 승인 버튼이 계속 활성화 상태[^1]
  2. "승인라인에 포함되지 않은 사용자에요" 오류 메시지 노출[^1]
  3. 예외 클래스: `ApprovalProcessNotActableUserException`[^2]

## 원인 분석

**승인 처리 중 타임아웃이 발생하여, approval-api에서는 승인이 완료되었지만 impact-api(결의/워크플로우)로의 이벤트 전파가 실패했다.** 이로 인해 impact-api는 해당 사용자가 아직 승인하지 않은 것으로 판단하고 승인 버튼을 계속 노출했다. 사용자가 재시도하면 approval-api에서는 이미 APPROVED 상태이므로 PENDING actor가 없어 예외가 발생했다.

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | js.jo가 이미 승인 완료했으나 UI 미갱신으로 재시도 시 예외 | approval_line_actor에서 js.jo status 확인 | ✅ 확정 — actor status=APPROVED[^8] |
| 2 | resolve 시점에 js.jo가 resolvedTarget에 미포함 | approval_line_actor의 resolve_target_type/value 확인 | ❌ 소거 — type=USER, value=465509로 정상 resolve[^8] |
| 3 | impact-api의 approvalRequired 계산에서 approval-api 상태 동기화 실패 | impact-api accessibleApprovers() 코드 + 에러 로그 분석 | ✅ 확정 — 상태 불일치 확인[^11][^12] |
| 4 | 승인 처리 타임아웃으로 approval → impact 이벤트 전파 실패 | access log 확인 | ✅ 확정 (임형태 분석)[^17] |

### 조사 과정

> 💡 **판단 근거**:
> → DB 조회: js.jo(465509)의 actor status = **APPROVED** (acted_at: 2026-04-07 12:39)[^8]
> → 임형태 분석: 승인 처리 타임아웃 발생 access log 확인[^17]
> → **승인은 처리되었으나 결의는 오류로 승인 처리되지 않음**[^17]
> → impact-api의 `accessibleApprovers()`에서 `approvalRequired=true` 유지 → 사용자에게 승인 버튼 노출[^12]

### 원인

- **표면 원인**: js.jo가 이미 승인 완료한 문서에서 승인 버튼이 계속 활성화되어 재시도 → `ApprovalProcessNotActableUserException`[^8][^9]
- **근본 원인**: **승인 처리 중 타임아웃** 발생 → approval-api에서 승인은 완료됐지만 impact-api(결의)로의 이벤트 전파 실패[^17]. impact-api는 해당 actor를 여전히 PENDING으로 인식하여 `approvalRequired=true` 반환[^12]

### 스펙 vs 버그

**판정: 타임아웃에 의한 시스템 간 상태 불일치 버그**
- approval-api: js.jo의 승인이 올바르게 반영됨[^8]
- impact-api: 타임아웃으로 이벤트 수신 실패 → 승인 상태 미갱신[^17]
- **시스템 전체 차원**: 78건/일 sync failure, 117건/일 승인 실패[^11]

## 해결

### 이 문서(2026-521) — 해결 완료

1. 임형태가 승인 롤백 방안 검토: 승인 상태 복구용 operation API가 없어 DB 직접 수정으로 진행[^18]
2. 권민철이 js.jo(465509)의 actor를 PENDING으로 롤백[^19]:
   ```sql
   UPDATE flex_approval.approval_line_actor t
   SET t.acted_user_id = null,
       t.acted_at      = '9090-09-09 18:09:09.000000',
       t.status        = 'PENDING',
       t.updated_at    = '2026-04-07 03:00:20.685639'
   WHERE t.id = '01KNJY4HKQP6HFH5CZNY40CV03';
   ```
3. 사용자(js.jo)가 재승인 수행 → 이번에는 이벤트 전파 정상 완료[^20]
4. 1차 시도에서 `acted_at`이 `18:09`로 잘못 들어간 것을 발견, 원복 후 재수행하여 완료[^21]

## 다음에 같은 문의가 오면

1. **먼저 확인**: approval-api access log에서 타임아웃 발생 여부 확인 + `approval_line_actor`에서 해당 사용자의 actor status 확인
   ```sql
   SELECT ala.status, ala.acted_user_id, ala.acted_at
   FROM flex_approval.approval_line_actor ala
   JOIN flex_approval.approval_line al ON ala.approval_line_id = al.id
   JOIN flex_approval.approval_process ap ON al.approval_process_id = ap.id
   WHERE ap.customer_id = {customer_id}
     AND ap.target_uid = (SELECT task_key FROM flex.workflow_task
                          WHERE customer_id = {customer_id} AND code = '{문서번호}')
     AND ala.resolve_target_value = '{user_id}'
   ```
2. **원인 판별**:
   - actor status = APPROVED + impact-api에서 승인 버튼 노출 → 타임아웃에 의한 이벤트 전파 실패. DB 롤백 후 재승인 필요
   - actor status = PENDING인데 예외 발생 → `approval_line_actor_resolved_user`에서 실제 resolve된 user_id 확인
   - 모든 actor APPROVED인데 line이 PENDING → `syncApprovalProcessStepPromotion` 실행
3. **조치** (EPBE-386 operation API 완성 전):
   - actor를 PENDING으로 DB 롤백 → 사용자 재승인 → 이벤트 정상 전파 확인
   - 롤백 SQL: `UPDATE flex_approval.approval_line_actor SET acted_user_id=null, acted_at='9090-09-09 18:09:09', status='PENDING' WHERE id='{actor_id}'`
   - ⚠️ `acted_at` 값 주의: `18:09:09`가 아닌 정확한 default 값을 사용해야 함[^21]

## 발견한 스펙/제약

- `approval_process.deleted_at`의 soft delete 마커: `9090-09-09 09:09:09` (일반적인 `9999-12-31`이 아님)[^8]
- `approval_line_actor.acted_at`의 미사용 마커: `9090-09-09 18:09:09`[^19]
- 다단계 승인 시 동일 step의 모든 actor가 승인해야 해당 step이 APPROVED로 전이[^9]
- `ApprovalProcessNotActableUserException`은 "이미 승인 완료"와 "승인라인 미포함" 두 경우 모두에서 동일하게 발생 — 구분 불가[^9]
- impact-api의 `reviewWithoutStep`은 `step=-1`(DELEGATE_STEP_ENTRY)로 approval-api에 위임[^13]
- 승인 상태 복구용 operation API 미존재 — DB 직접 수정만 가능. EPBE-386에서 작업 예정[^18][^22]
- `user_workflow_task_box` 테이블에 `is_follower`, `is_reviewer` 컬럼으로 사용자별 문서 역할 관리[^23]
