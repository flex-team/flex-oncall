# CI-4186: 주휴일 휴일대체 지정 가능 기한 변경 (한국로날드맥도날드하우스)

> **상태**: 해결 완료 — config PR #39 머지 (2026-03-23)

## 증상

- **문제 정의**: 주휴일 휴일대체 지정 가능 기간을 기본값(±6일)에서 **±30일**로 변경 요청
- **회사**: 재단법인한국로날드맥도날드하우스 (Customer ID: `223522`)
- **요청자**: ms.cho@rmhc.or.kr (고객사), CS팀 김한나[^1]
- **영향 범위**: 해당 회사 전체
- **문제 시점**: 2026-03-23 인입
- 문의 내용:
  1. 휴일대체 근무 시 대체일을 1주일 안에 지정하도록 되어 있으나, 업무 특성상 기간이 부족
  2. 전후 30일로 변경 요청[^2]

---

## 원인 분석

**스펙 이슈** — 현재 주휴일 휴일대체 gap 기본값은 ±6일이며, 회사별 커스텀 설정으로 변경 가능하다.

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | config 변경으로 해결 가능 (CI-3897 동일 패턴) | 쿡북 체크리스트 2번 + CI-3897 선례 확인 | ✅ 확정 |

> 💡 **판단 근거**: CI-3897[^3]에서 동일 요청(유모스원, ±14일 변경)을 `experimental.json`의 `customGapOfWeeklyHoliday` 설정 변경으로 해결한 선례가 있음 → CI-4186도 동일 방식 적용 가능

### 코드 위치

| 역할 | 위치 |
|-----|------|
| 동적 설정 로드 | `flex-timetracking-backend` > `applications/application-config/.../TrackingExperimentalDynamicConfig.kt` |
| 상수 정의 | `flex-timetracking-backend` > `time-tracking/service/.../alternativeholiday/commons/CommonConstants.kt` — `MAX_GAP_OF_WEEKLY_HOLIDAY_IN_DATES = 6L` |

<summary>📋 설정 로드 우선순위 (CI-3897에서 확인)</summary>

2. 정적 프로퍼티 (`application.yml` > `flex.v2.tracking.alternative-holiday.customGapOfWeeklyHoliday`)
3. 기본값 `6`

</details>

### 스펙 vs 버그

**스펙** — 주휴일 gap은 회사별 커스텀 설정이 설계 시점부터 지원됨. 기본값(±6일)은 근로기준법 제55조에 근거[^4].

---

## 해결

`flex-timetracking-config` > `experimental.json` > `customGapOfWeeklyHoliday`에 `"223522": 30` 추가[^6].

- [config PR #39](https://github.com/flex-team/flex-timetracking-config/pull/39) — 머지 완료, Central Dogma 1분 내 자동 반영[^7]
- 고객 요청이 "전후 30일"임을 Linear 이슈 설명에서 확인[^2]

| 항목 | 내용 |
|-----|------|
| 작업량 | config 1줄 추가, 코드 변경 없음 |
| 변경 효과 | ±6일(기본) → **±30일** (총 61일 범위) |
| 배포 | 불필요 (Central Dogma 동적 반영) |
| 근로자 영향 | 선택 범위 확대 — **유리** ✅ |
| 법적 리스크 | 없음 — 근로자에게 유리한 방향[^5] |

## 다음에 같은 문의가 오면

1. **쿡북 체크리스트 2번** 확인 — "휴일대체 기간 문의 → `TrackingExperimentalDynamicConfig` gap 설정"
2. **CI-3897 선례** 참조 — 동일 패턴으로 config PR 생성
3. `flex-timetracking-config` > `experimental.json` > `customGapOfWeeklyHoliday`에 `"{customer_id}": {일수}` 추가
4. 고객에게 "전후 대칭만 가능"한 시스템 제약 안내 (비대칭 요구 시)
