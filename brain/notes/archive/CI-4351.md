# CI-4351: 근무 스케줄 삭제 로그 요청

## 증상

- **문제 정의**: 시몬스 고객사가 '갤러리F' 교대근무 스케줄 삭제 이력(삭제 시점 + 삭제자 이메일)을 요청
- **회사**: 시몬스 (Customer ID: 77952)
- **요청자**: lee0ju@simmons.co.kr[^1]
- **대상자**: lee0ju@simmons.co.kr[^1]
- **영향 범위**: 1개 회사(시몬스) — 로그 조회 요청, 데이터 변경 없음
- **문제 시점**: 2026-04-04 이후 스케줄 미노출[^1]
- 문의 내용:
  1. 교대 근무 설정 > 스케줄 관리에서 '갤러리F' 스케줄이 삭제되었으며, 삭제 발생 시점과 수행자 이메일 확인 요청[^1]

## 원인 분석

### 조사 과정

> 💡 "삭제 이력" 문의 → 감사로그(raccoon audit API) 방향 검토 → 실제 DB 조회로 전환
> → `v2_customer_work_plan_template WHERE customer_id = 77952 AND name = '갤러리F'` 조회[^2]
> → 스케줄 존재 확인. 삭제 이력 없음. 2026/4/7 13:13에 생성 + 2회 변경 이력만 있음[^2]
> → 4월 4~6일 사이 삭제된 게 아니라, 고객이 오늘 재생성한 것 확인[^2]

### 원인

- 갤러리F 스케줄은 **삭제된 게 아님** — `v2_customer_work_plan_template` 에 존재[^2]
- 4월 4~6일 사이 스코프 변경으로 안 보였던 것으로 추정[^3]
- 고객이 안 보인다고 판단하여 4월 7일 재생성[^2]

## 영향 범위

- 1개 회사(시몬스) — 데이터 변경 없음. 로그 조회 요청이므로 운영 대응 완료

## 해결

- DB 직접 조회로 스케줄 삭제 이력 없음 확인[^2]
- 스코프 변경으로 안 보인 것으로 추정 — 고객에게 스코프 확인 안내[^3]
- 고객 확인 완료 ("확인 감사합니다 영준님! 👍")[^4]

## 발견한 스펙/제약

- 교대근무 스케줄(`v2_customer_work_plan_template`)의 **별도 삭제 이력 저장 없음**[^2]
  - raccoon audit API(감사로그)는 계정/구성원 정보 전용 — 교대근무 스케줄 삭제는 기록 안 됨
  - 삭제 여부는 테이블 직접 조회로만 확인 가능
- "스케줄이 안 보인다" 문의 시 스코프 변경 가능성을 먼저 확인해야 함

## 다음에 같은 문의가 오면

1. **먼저 확인**: `v2_customer_work_plan_template WHERE name = ? AND customer_id = ?` 직접 조회
   - 존재하면 → 삭제 안됨. 스코프 변경으로 안 보인 것 가능성. 고객에게 스코프 확인 안내
   - 없으면 → 실제 삭제됨. 별도 이력 없으므로 access log에서 DELETE 호출 확인
2. **raccoon audit API(감사로그)는 이 케이스 미적용** — 교대근무 스케줄은 audit 대상 아님
