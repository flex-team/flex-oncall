# CI-4313: 엑셀 발령 기능 오픈 요청

## 증상
- **문제 정의**: 엑셀 인사발령 기능 오픈 요청 (feature flag 활성화)
- **회사**: 주식회사 테크타카 (Customer ID: 3718)
- **요청자**: jin@techtaka.com[^1]
- **대상자**: 미기재 (전사 기능 오픈)
- **영향 범위**: 전사 — 엑셀 인사발령 기능을 사용할 수 있는 관리자 전체
- **문제 시점**: 2026-04-03
- 문의 내용:
  1. 엑셀 인사발령 기능 오픈 요청[^1]
  2. 유의사항 안내 완료, 고객이 직접 테스트 후 진행하겠다고 확인[^2]

## 원인 분석

운영성 요청 — 원인 분석 대상 아님. 엑셀 인사발령은 **Flagsmith** feature flag로 고객사별 제어된다[^4]. 과거 동일 패턴 10건+ 처리됨[^3].

## 해결

### 오픈 절차

1. **prod Flagsmith 접속**
2. **Segments > `인사발령_엑셀업로드_우선_오픈_고객사` 클릭**
3. **Description에 회사명 추가, `customer_id` 값에 `3718` 추가**[^4]
   - customer_id 앞에 **공백이 들어가지 않도록 주의**
4. **[Update Segment] 버튼 클릭**

> ⚠️ **사전 확인 필요**: CI-4214 (엑셀 발령 후 "정보 변경하러가기" 버튼 미동작 FE 버그)가 수정 진행 중이다. 기능 오픈 시 고객에게 이 제한사항을 안내할 것.

### 고객 안내 주의사항

- 임시로 제공되는 기능이라는 점[^4]
- 엑셀에 기입된 정보로 **덮어쓰기**되므로, 엑셀에 기재되지 않은 정보는 소실된다[^4][^5]
- 관리자가 덮어쓰기 특성을 모르고 사용하면 과거 발령 히스토리가 날아갈 수 있음[^5]


## 다음에 같은 문의가 오면

1. **먼저 확인**: CC팀이 유의사항(임시 기능, 덮어쓰기 특성)을 고객에게 안내했는지 확인
2. **조치**: prod Flagsmith > Segments > `인사발령_엑셀업로드_우선_오픈_고객사` > customer_id에 해당 회사 ID 추가 > [Update Segment]
3. **사후 확인**: CI-4214 (FE 버그) 수정 여부를 확인하고, 미수정 시 제한사항 안내
