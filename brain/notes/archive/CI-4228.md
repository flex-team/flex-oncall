# CI-4228: 승인 라인에 있는 삭제한 구성원 정보 대체 요청

## 증상
- **문제 정의**: 승인 라인에 포함된 구성원 계정(jwc@chvolvo.co.kr)을 삭제하여 진행 중인 승인건 처리 불가 — 대체 계정으로 교체 요청
- **회사**: (주)천하자동차 (Customer ID[^13]: 223459)[^1]
- **요청자**: CS팀 김현욱 (hrm@chvolvo.co.kr 고객사 HR 담당자 대리 문의)[^1]
- **대상자**: 삭제된 계정 jwc@chvolvo.co.kr (userId: 953972)[^2]
- **영향 범위**: WORK_RECORD 1건 (targetUid: `01kmkz7aemse0cs8sq755qy79j`)[^3]
- **문제 시점**: 2026-03-26
- 문의 내용:
  1. 2차 조직장 계정을 삭제하여 승인 라인이 깨짐[^4]
  2. 삭제된 구성원을 다른 계정(hrm@chvolvo.co.kr / 958764)으로 대체하여 승인 진행 요청[^1]

## 원인 분석

삭제된 구성원이 승인 라인에 남아 승인 진행이 불가한 상태. 제품의 "퇴직자 승인자 교체" 기능은 삭제된 구성원에 적용되지 않음[^7].

### 조사 과정

> 💡 **판단 근거**: 퇴직자 교체 Operation API(`replacement-targets/migrate`) 시도[^5]
> → 삭제된 사용자는 퇴사 이벤트가 발생하지 않아 `approval_replacement_target`[^14]에 등록되지 않음[^6]
> → 퇴직자 교체 불가 확정 → 강제 승인으로 전환

### 원인
- **표면 원인**: 고객사 관리자가 결재 라인 수정 의도로 2차 조직장 **계정 자체를 삭제**함[^2]
- **근본 원인**: 구성원 삭제 시 퇴사 이벤트가 발행되지 않아, 승인 서비스의 `approval_replacement_target`에 교체 대상으로 등록되지 않음[^6]. 제품에서는 퇴직자만 승인자 교체를 지원하므로 삭제된 구성원은 교체 대상에서 누락됨[^7].

## 해결
임형태가 approval Operation API로 강제 승인 처리 완료[^8]:

<details>
<summary>📋 처리 절차 상세</summary>

**1차 시도 — 퇴직자 교체 (실패)**:
```
POST /proxy/approval/api/operation/v2/approval/replacement-targets/migrate
Body: { "customerIds": [223459], "resignationDateRange": { "start": "2026-03-25", "endInclusive": "2026-03-25" } }
```
→ 삭제된 사용자는 퇴사 조회에 나오지 않아 교체 대상 미등록[^6]

**2차 시도 — 강제 승인 (성공)**:
```
POST /proxy/approval/api/operation/v2/approval/process/customers/223459/users/953972/bulk-approve-for-user
Body: { "categories": ["TIME_OFF", "WORK_RECORD"] }
```

**결과**:
```json
{
  "succeededProcesses": [
    { "targetCategory": "WORK_RECORD", "targetUid": "01kmkz7aemse0cs8sq755qy79j" }
  ],
  "failedProcesses": []
}
```

</details>

## 발견한 스펙/제약
- **삭제 vs 퇴직 차이**: 구성원 삭제 시 퇴사 이벤트가 발행되지 않아, 승인 서비스의 퇴직자 교체 기능(`replacement-targets/migrate`)이 동작하지 않음[^6]
- **`bulk-approve-for-user` API**: 삭제된 구성원의 미처리 승인건을 카테고리별로 일괄 강제 승인 가능[^8]
  - 경로: `POST /api/operation/v2/approval/process/customers/{customerId}/users/{userId}/bulk-approve-for-user`
  - Swagger: `https://flex-raccoon.grapeisfruit.com/swagger/approval`

## 다음에 같은 문의가 오면

1. **먼저 확인**: 삭제된 구성원의 userId 확인 + [Metabase 퇴사자 승인 대시보드](https://metabase.dp.grapeisfruit.com/dashboard/245)에서 미처리 승인건 조회
2. **원인 판별**:
   - 퇴직자(resigned) → 제품의 "퇴직자 승인자 교체" 기능 사용 가능
   - 삭제된 구성원(deleted) → 제품 기능 불가, Operation API 필요
3. **조치 (삭제된 구성원)**:
   - 고객에게 "강제 승인 처리"해도 되는지 확인
   - `bulk-approve-for-user` API 호출:
     ```
     POST /proxy/approval/api/operation/v2/approval/process/customers/{customerId}/users/{userId}/bulk-approve-for-user
     Body: { "categories": ["TIME_OFF", "WORK_RECORD"] }
     ```
   - 카테고리: `WORK_RECORD`(근무), `TIME_OFF`(휴가) 등 대상에 맞게 설정
