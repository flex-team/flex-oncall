# CI-4174: 건강보험 제외 대상인데 중도정산 시 사회보험 연말정산금액에 건강보험/고용보험 포함

## 증상

- **문제 정의**: 건강보험 제외 대상 구성원의 중도정산[^11] 시 2025년 사회보험 연말정산금액에 건강보험, 고용보험 금액이 잘못 포함됨
- **회사**: 엔엑스씨 (Customer ID: 209287)[^2]
- **요청자**: sjyang@nxc.com[^2]
- **대상자**: blatun@naver.com[^2]
- **영향 범위**: 중도정산 확정/해제를 수행한 구성원 중 전년도 사회보험 연말정산금이 있는 경우
- **문제 시점**: 2026-03-20[^2]
- **라벨**: 급여/원천세/전표
- **담당자**: 김주원
- 문의 내용:
  1. 건강보험 공제 이력이 전혀 없고 2025년 연말정산에서도 건강보험료 공제금액 0원인 구성원[^1]
  2. 중도정산 시 사회보험 연말정산금액에 건강보험/고용보험 금액이 표기됨[^2]
  3. 당일 정산 마감이 필요한 긴급 건[^2]

## 원인 분석

### 조사 과정

> 💡 이한결이 dev 환경에서 재현 후 원인을 파악[^3]
> → 전년도 사회보험 연말정산금이 있는 상태에서 중도정산 단계 확정 → 확정 해제 → 최신 정보 반영 순서로 재현 성공[^4]
> → 확정 해제 시 정산 결과 연말정산 항목이 리버트되지 않는 것이 근본 원인[^5]
> → 2월 말 핫픽스로 추가된 합산 로직과 결합하여 문제 발생[^5]

### 원인

1. 중도정산 단계 확정 시, 정산 결과의 연말정산 항목에 사회보험 연말정산 금액이 저장됨[^5]
2. 확정 해제 시, 저장된 금액이 **리버트되지 않음** (정상: 리버트되어 0원이어야 함)[^5] [^12]
3. 2월 말 핫픽스 이후, 전년도 사회보험 연말정산 납부 보험료에 당해연도 정산 결과 연말정산 항목의 사회보험 금액도 합산하도록 변경됨[^5]
4. 이 합산 과정에서 리버트되지 않은 금액이 함께 합산됨[^5]

**문제점:**
- 공제한 금액이 없는데 총 납부한 금액에 금액이 표기됨[^6]
- 제외로 바꾸면 해당 금액이 그대로 환급으로 전환됨[^6]
- 원천징수영수증에 표기되는 상황이라 단순 0원 처리 불가[^7]

## 해결

- **임시 해결 (제품 내 수동 조치)**[^4]:
  1. 건강보험을 "제외"로 변경
  2. 중도정산 단계 확정
  3. 다시 확정 풀고 "포함"으로 변경 → 전년도 사회보험 납부 보험료 0으로 리셋

- **코드 수정**: 이한결이 김주원과 논의 후, **확정 해제 시 연말정산 항목 리버트** 방식으로 코드 대응[^8]
- **핫픽스 배포 완료** (2026-03-20 18:41 KST)[^9]
- 표준세액공제 적용 대상이라 소득세가 발생하지만 세금에 실질적 영향은 없었음[^7]

## 발견한 스펙/제약

- 중도정산 확정 해제 시 정산 결과의 연말정산 항목을 리버트해야 한다 (핫픽스로 반영됨)[^5]
- 2월 말 핫픽스로 전년도 사회보험 연말정산 납부 보험료 합산 로직이 추가되었으며, 이 로직은 당해연도 정산 결과의 연말정산 항목 값도 참조한다[^5]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 해당 구성원의 중도정산 이력에서 확정/확정해제를 반복한 이력이 있는지 확인
2. **원인 판별**: 전년도 사회보험 연말정산금이 있는 상태에서 확정 후 해제한 경우 → 리버트 누락 버그 (2026-03-20 핫픽스로 수정 완료)
3. **조치**: 핫픽스 배포 이전 데이터라면 임시 해결법(제외→확정→해제→포함) 적용. 핫픽스 이후에는 정상 동작.

    만약 중도정산 단계를 확정 해제한다
    그러면 정산 결과의 연말정산 항목에 저장된 사회보험 연말정산 금액이 리버트된다
    그리고 최신 정보 반영 시 사회보험 연말정산금액에 건강보험/고용보험 금액이 포함되지 않는다
