## 원인 분석

### 배경
- 평가 리스트 API: `POST /action/v3/evaluation/list-view`[^13] — `evaluation` + `draft_evaluation` 두 테이블을 조회
- `draft_evaluation` 테이블은 soft-delete[^18] 방식 (`deleted_at IS NULL` 필터)[^14]
- `DraftEvaluationJdbcEntity.title`은 `String?` (nullable) → title이 NULL인 DRAFT도 정상 조회됨[^15]

**이 평가들은 삭제된 적 없는 DRAFT 상태의 임시 저장 평가이며, 3/18 핫픽스로 정상 노출되기 시작한 것이다 (스펙).**

> 💡 **판단 근거**: access log에서 isDeleted=true인 평가 없음[^7] → 이해나 확인: title 없는 DRAFT 평가가 안 보이던 버그를 3/18 핫픽스로 수정[^8] → 고객이 "이전에 안 보이던 것이 보임"을 "삭제한 것이 복원됨"으로 오해

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | DB `deleted` 플래그가 `false`로 롤백됨 | access log + DB 확인 | ❌ 소거 — isDeleted=true인 평가 없음[^7] |
| 2 | 최근 배포로 BE 조회 로직 변경 | git log 확인 | ❌ 소거 — review_set 관련 BE 코드 변경 없음[^6] |
| 3 | 3/18 FE 핫픽스로 기존에 숨겨졌던 DRAFT 평가가 정상 노출 | Linear 코멘트 + Slack 스레드 | ✅ 확정[^8] |

### 조사 과정

1. API 응답 분석: 25건의 제목 없는 DRAFT 평가 (evaluationTitle 없음, revieweeUserCount=0) + 12건의 정상 평가 확인[^9]
2. access log 확인 (이지선): isDeleted=true인 평가가 응답에 포함되지 않음[^7]
3. 이해나 확인: 3/18 화요일에 "임시저장된 제목 없는 평가가 목록에서 안 보이는 버그" 핫픽스 배포[^8]
4. 결론: 고객이 삭제했다고 인식한 평가들은 실제 삭제되지 않은 DRAFT 평가

<details>
<summary>📋 API 응답 상세 (DRAFT 25건 + 정상 12건)</summary>

응답에 포함된 DRAFT 평가 25건은 모두 아래 특징을 공유한다:
- `status: "DRAFT"`, `evaluationTitle` 필드 없음 (제목 미설정)
- `evaluationDateRange: {}` (기간 미설정)
- `revieweeUserCount: 0` (피평가자 없음)
- 생성 기간: 2025-11-11 ~ 2026-03-16

정상 평가 12건은 제목 있음 (예: "2025 성과평가 XX팀"), `RESULT_ADJUSTMENT`/`CLOSED` 상태, revieweeUserCount > 0.

</details>

<details>
<summary>📋 소거된 가설 상세</summary>

**가설 1 (DB deleted 플래그 롤백)**: 이지선이 오늘 아침 access log에서 isDeleted=true인 평가가 없음을 확인[^7]. 또한 김보라가 "삭제 안된 임시 저장되어 있는 평가 중에 title이 없는 (제목 없음) 경우의 평가가 25개"라고 확인[^10].

**가설 2 (BE 코드 변경)**: git log 확인 결과, 2026-03-17~19 사이 v4.26.13/v4.26.14 배포는 form/report 관련 수정만 포함. review_set 도메인 코드 변경 없음[^6]. 김보라도 "어제 기준으로는 FE 배포만 있었다"고 확인[^11].

</details>

### 스펙 vs 버그 판별: **스펙**

이해나: "원래 보여야하는게 안보였던것이 이슈였는데, 제대로 보이는 것을 버그라고 느끼신것이 아닐지"[^12]

- 이전에 title=null인 DRAFT 평가가 목록에서 숨겨지는 **FE 버그**가 있었음
- 3/18 핫픽스로 해당 버그 수정 → DRAFT 평가가 정상 노출[^8]
- 고객은 이전에 안 보이던 것이 보이기 시작한 것을 "삭제한 평가 복원"으로 오해

- 고객에게 "삭제되지 않은 임시 저장 평가이며, 불필요하면 직접 삭제 가능"하다고 안내[^19]

## 시나리오

```gherkin
# language: ko
기능: 평가 리스트 조회 시 DRAFT 평가 노출

  시나리오: 제목이 없는 DRAFT 평가가 평가 리스트에 정상 노출된다
    주어진 관리자가 평가를 생성하고 제목을 입력하지 않은 채 임시 저장했다
    그리고 해당 평가는 삭제하지 않았다
    만약 관리자가 평가 리스트를 조회한다
    그러면 제목이 "(제목 없음)"으로 표시된 DRAFT 평가가 리스트에 노출된다

  시나리오: 삭제한 DRAFT 평가는 평가 리스트에 노출되지 않는다
    주어진 관리자가 DRAFT 평가를 삭제했다
    만약 관리자가 평가 리스트를 조회한다
    그러면 삭제한 평가는 리스트에 노출되지 않는다
```

## 다음에 같은 문의가 오면
1. "삭제한 평가가 다시 보인다" → 먼저 API 응답에서 해당 평가의 `isDeleted` 값 확인
2. `isDeleted: false`이면 → 삭제된 적 없는 DRAFT 평가. 고객에게 삭제 방법 안내
3. `isDeleted: true`이면 → 실제 버그. `draft_evaluation` 테이블에서 해당 레코드의 `deleted_at` 컬럼 확인
