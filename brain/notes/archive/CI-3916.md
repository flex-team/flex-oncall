# CI-3916: 연말정산에서 소기업소상공인공제부금이 제외되지 않음

> **상태**: Done — 2026-02-19 (KST)
> **verdict**: bug-fix — 소기업소상공인공제부금 자동 밸리데이션 대상을 SYSTEM 레코드로 한정하지 않아 제외 처리 무효화

## 증상

- **문제 정의**: 연말정산에서 소기업소상공인공제부금(노란우산공제) 항목을 제외 처리해도 "좀비처럼 살아남" — 제외해도 다시 공제가 반영되는 현상. 실제 법인대표자가 아닌 일반 근로자(타 회사 법인 대표자)에게 해당 공제가 자동 적용되어 제외가 불가능했음. 연말정산 결과 오픈 전날이라 긴급 대응 필요.
- **회사**: Ackerton Technology Services (Customer ID: 167437)
- **요청자**: 정인지 (내부)
- **이슈 대상**: cym1992@ackertonts.com
- **라벨**: 신고/세금, 급여/원천세/전표
- **담당자**: 이한결
- **Linear**: [CI-3916](https://linear.app/flexteam/issue/CI-3916/연말정산에서-소기업소상공인공제부금이-제외되지-않음)

## 원인 분석

- 소기업소상공인공제부금 자동 밸리데이션 로직이 **모든 레코드**를 대상으로 실행 → 사용자가 제외 처리한 USER 레코드까지 다시 공제 처리
- SYSTEM 레코드만 대상으로 밸리데이션해야 하는데 범위가 넓었음
- 이한결이 긴급 대응으로 해당 건 수동 제외 처리 후, 근본 수정 PR 작성

## 해결

- **긴급 대응**: 이한결이 당일 밤 수동으로 해당 대상자 공제 제외 처리 → 정인지 확인 "잘 제외 되었습니다"
- **코드 수정**: [flex-yearend-backend PR #4093](https://github.com/flex-team/flex-yearend-backend/pull/4093) — 소상공인 자동 밸리데이션 대상을 SYSTEM 레코드로 한정
- **후속 대응 필요 대상**: 코스틱폼텍(206674) 이석환 — 로그 확인 결과 공제 제외 처리했으나 실제 계산 시 공제 적용됨. 고객 핸들링 요청: https://flex-cv82520.slack.com/archives/C02L4FZ897U/p1771476369837619
- **세법 참고**: 일반 근로자가 타 회사 법인 대표자인 경우, 총급여 8,000만원 이하이면 노란우산공제 소득공제 적용 가능 (2016.1.1. 이후 가입자 기준)
