# CI-4325: 토큰 생성시 접근오류 확인됨

## 증상
- **문제 정의**: OpenAPI 토큰 생성 시 '접근할 수 없습니다' 오류 발생으로 토큰 생성 불가
- **회사**: GS에너지 (Customer ID: 180694)
- **요청자**: 유지원 (seruya@gsenergy.co.kr)[^1]
- **대상자**: 미기재
- **영향 범위**: 해당 회사 — 토큰 생성 차단으로 계정 조회 등 API 활용 불가
- **문제 시점**: 2026-04-06 접수[^2]
- 문의 내용:
  1. 토큰 생성 시도 시 '접근할 수 없습니다' 오류 메시지 확인[^1]
  2. 급하게 계정 조회할 일이 있어 빠른 조치 요청[^3]
  3. 조치 지연 시 임시로 사용할 refresh token 또는 access token 발급 가능 여부 문의[^3]

## 원인 분석

설정 조회 시 불필요한 추가 권한이 요구되는 버그[^7]

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | `allowedCustomerIdList` 화이트리스트 미등록 | prod 설정 확인 | ❌ 소거 — 담당자가 권한 이슈로 확인[^7] |
| 2 | 설정 조회에 불필요한 추가 권한 요구 | 담당자(윤성복) 확인 | ✅ 확정[^7] |

### 조사 과정

> 💡 코드 분석에서 화이트리스트 접근 제어를 발견하여 가설 1을 수립했으나[^8][^9], 담당자(윤성복)가 실제 원인을 "설정 조회 시 불필요한 추가 권한 요구"로 확인[^7]. 초기 가설은 소거됨.

<details>
<summary>📋 초기 코드 분석 (소거된 가설 1 관련)</summary>

토큰 생성 API에 화이트리스트 기반 접근 제어가 있으나, 이번 이슈의 직접 원인은 아니었음.

```kotlin
// ClientCredentialsApiController.kt:66-67
if (!checkAllowedCustomerId(customerIdentity.customerId)) {
    throw OpenApiForbiddenException()
}
```

- `specificCustomerOnly=true`(기본값)이면 `allowedCustomerIdList`에 없는 Customer ID는 차단
- local-dev: `specificCustomerOnly=false`[^10]
- prod: AWS Secrets Manager로 관리[^11]

</details>

## 영향 범위
- GS에너지 — OpenAPI 토큰 생성 기능 사용 불가
- 금전적/법적 영향: 해당 없음 (인증/접근 이슈)

## 해결
- 윤성복님이 설정 조회 시 불필요한 권한 요구를 제거하는 hotfix 배포[^7]
- 약 10분 후 해결 완료[^12]
- 고객(유지원) 확인 완료[^13]

## 다음에 같은 문의가 오면

1. **먼저 확인**: OpenAPI 토큰 생성 시 403/접근 오류인지 확인
2. **원인 판별**:
   - 에러 코드가 `OPENAPI_403_001` → 권한/인증 관련
   - 특정 회사만 발생 → `allowedCustomerIdList` 설정 확인
   - 전체적으로 발생 → 코드 변경 영향도 확인 (최근 배포 여부)
3. **조치**: OpenAPI 담당자(윤성복)에게 에스컬레이션
