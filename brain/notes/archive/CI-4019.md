# CI-4019: 승인 완료 문서가 진행중 문서함에 표시됨

> **상태**: Done — 2026-03-05 (KST)
> **verdict**: bug — 워크플로우-승인 간 동기화 오류로 승인완료 문서가 진행중 상태로 표시

## 증상

- **문제 정의**: 문서번호 2025-2560이 승인 완료되었으나 진행중 문서함에 남아 있음. 승인 시스템에서는 승인완료 상태이나 워크플로우 문서함에는 진행중으로 표시.
- **회사**: 크로노스튜디오 (Customer ID: 185358)
- **요청자**: jyyun@chronostudio.net
- **라벨**: 워크플로우
- **담당자**: 임형태
- **Linear**: [CI-4019](https://linear.app/flexteam/issue/CI-4019)

## 원인 분석

- 승인 시스템에서는 승인완료 상태로 정상 존재하나, 워크플로우 문서 상태와 동기화가 깨진 상태
- 해당 고객사에서만 이런 동기화 오류가 반복 발생하는 패턴 확인됨
- Operation API로 동기화 시도 시 `WorkflowLegacyBusinessException` 발생:
  ```
  c.f.w.e.WorkflowLegacyBusinessException: null
    at t.f.w.d.a.ApprovalDocumentCommandServiceImpl.get(ApprovalDocumentCommandService.kt:412)
    ...
  Wrapped by: t.f.b.v.e.FlexUnknownException
  ```
- 최초 OS 동기화 API로 시도 → 문서함에서는 여전히 진행중 표시 → 승인-워크플로우 간 sync API로 보정 필요

## 해결

1. **1차 시도 — OS 동기화**:
   ```
   /api/operation/v3/impact/approval-documents/customers/{customerId}/approval-documents/sync-os
   ```
   실행 → 동기화 완료했으나 문서함 상태는 여전히 진행중

2. **2차 시도 — 승인-워크플로우 동기화 API**:
   ```
   /api/v2/operation/workflow/customers/{customerId}/users/{userId}/approval-document/{documentKey}/sync-with-approval
   ```
   - userId: `f37ef47cb8f9461192d277d44bc30b2f`
   - 실행 중 `WorkflowLegacyBusinessException` 발생했으나 최종 보정 완료
