- **문제 시점**: 2026-04-09 오전 (전날까지 정상 동작)[^1]

  4. 게시글 작성 버튼 등 주요 기능 사용 불가[^1]

### 원인

- **표면 원인**: 구독 기간이 2026-04-08에 만료되어 4/9부터 `customer_flex_subscription_feature` 레코드가 삭제됨 → `FlexTeam-ActiveFeatures` 헤더가 빈 값 → FE가 모든 피처 UI를 숨김[^8][^9]
- **근본 원인**: 고정윤이 수동 청구 시 구독을 1개월(3/9~4/8)만 설정. 구독 갱신/연장이 이루어지지 않아 만료됨[^10]

## 영향 범위

- 도안회계법인 전사 — 모든 핵심 피처가 비활성화되어 제품 사용 자체가 불가한 상태[^2]
- 금전적/법적 영향: 해당 없음 (구독/피처 활성화 이슈로, 데이터 변조나 금전 오류 아님)

## 해결

- 이해나가 구독을 추가하여 해결[^11]
- 임준규: "그로스에서 확인해주셔야할듯" — 향후 구독 갱신은 그로스팀 확인 필요[^10]

## 발견한 스펙/제약

- 구독 만료 시 `customer_flex_subscription_feature` 레코드가 **삭제**됨 (INACTIVE 전환이 아닌 물리 삭제)[^8]
- `flexteam-activefeatures` 헤더는 `flex-backend-gateway`에서 설정. `expired=false`인 피처만 activeFeatures에 포함[^12]
- 수동 청구의 대상 기간 종료일이 곧 구독 만료일. 자동 갱신이 아닌 경우 종료일에 피처가 즉시 비활성화됨[^9]

## 다음에 같은 문의가 오면

1. **먼저 확인**: access log에서 `flexteam-activefeatures` 헤더 값 확인 — 빈 값이면 구독 만료
2. **원인 판별**:
   - activeFeatures 빈 값 → `customer_flex_subscription_feature` 조회 (0건이면 구독 만료 확정)
   - activeFeatures 있음 → 다른 원인 (권한, API 에러 등)
3. **조치**: raccoon에서 구독 추가 (필수 플랜 포함). 일할 결제 필요 시 수동 청구도 함께 처리.
   - 참고: https://flex-raccoon.grapeisfruit.com/billings/222963/subscription
