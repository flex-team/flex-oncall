# CI-4365: 워크플로우 문서 승인 버튼이 없음

> **상태**: 해결 완료 — 2026-04-09
> **compact**: 원본 249줄 → 정제 119줄 (2026-04-09). git log로 원본 추적 가능.

## 증상
- **문제 정의**: 워크플로우 문서에서 본인의 승인 차례임에도 승인 버튼이 보이지 않는다
- **회사**: 바이엘코리아 (Customer ID: 199890)
- **요청자**: youjung.choi.ext@bayer.com[^1]
- **대상자**: 최유정 (youjung.choi.ext@bayer.com, hashedId: 248NmJoBEe)[^1][^4]
- **영향 범위**: 해당 문서 1건 (문서번호: 2026-832) + 시스템 전체 78건/일 sync failure[^12]
- **문제 시점**: 2026-04-08
- 문의 내용:
  1. 워크플로우 문서에서 본인 승인 차례가 되어 승인하려 하나 승인 버튼이 보이지 않음[^1]
  2. 1단계 승인권자가 승인한 것으로 상태 표시되나 활동 내역에 승인 내역이 보이지 않음[^2]

## 원인 분석

**1단계 승인 처리(act API) 시 타임아웃(~25s)이 발생하여, impact-api에서 이벤트 동기화가 실패 → 최유정의 UserBox 미생성 → `myBoxInfo = null`**

### 조사 과정

> 💡 FE가 myBoxInfo null 확인[^4]
> → `myBoxInfo`는 `userBoxes.find { ... }`로 생성 — UserBox가 없으면 null[^7]
> → `workflow_task.last_modified_date = 2026-04-07 12:35:49` — 생성 이후 미갱신[^10]
> → act API에서 `FlexRemoteUnknownStateException: timeout` 발생[^13]
> → 48시간 내 동일 타임아웃 14건[^14]

### 원인

- **표면 원인**: 최유정의 `user_workflow_task_box` 레코드 미존재 → `myBoxInfo = null`[^7][^10]
- **근본 원인**: approval-api 호출 타임아웃(~25s)으로 impact-api 후속 동기화 실패. approval-api에서는 승인 커밋 완료, impact-api에서는 `patchAllUserBoxes()` 미실행[^13][^9]

### 스펙 vs 버그

**판정: 버그** — approval-api 정상, impact-api 동기화 실패[^10]

## 영향 범위
- CI-4362 조사에서 **78건/일 sync failure** 확인[^12]. 모든 고객사에서 발생 가능

## 해결

- 임형태님이 정합성 복구 Operation API 구현/배포 ([EPBE-388](https://linear.app/flexteam/issue/EPBE-388/), [PR #2236](https://github.com/flex-team/flex-impact-backend/pull/2236))[^17]
- sync API로 해당 문서 보정 처리 완료[^18]
- 고객사 확인 완료[^19]

### Operation API 주요 기능[^17]

- 단건/벌크 sync: `POST .../approval-process/sync`, `.../sync/bulk`
- `dryRun=true` 기본값, 활동로그 자동 추가, 23개 테스트 통과

## 발견한 스펙/제약

- `myBoxInfo`는 nullable. UserBox가 없으면 null[^7]
- UserBox 갱신은 approval 이벤트 수신에 의존하는 비동기 구조 — 이벤트 유실 시 UI에 즉시 영향[^9]

## 다음에 같은 문의가 오면

1. **먼저 확인**: FE에서 `myBoxInfo`가 null인지 확인 (API 응답 `GET /api/v3/approval-document/approval-documents/{documentKey}`)
2. **원인 판별**:
   - `myBoxInfo = null` → `workflow_task.last_modified_date`가 생성 이후 갱신되었는지 DB 확인
   - 미갱신이면 → 승인 이벤트 동기화 실패 (이 이슈와 동일)
   - 갱신되었는데 null → UserBox 생성 로직 자체의 버그 (다른 원인)
3. **조치**: sync Operation API (`POST .../approval-process/sync?actorUserId={}&dryRun=false`)로 보정
