# CI-4335: 구성원 정보 변경 알림을 받음

> **상태**: 해결 완료 (Not a Bug) — 2026-04-07
> **verdict**: not-a-bug — 알림은 권한(CORE_USER_ATTACHMENT_UPDATE) 기반 발송이 스펙. 승인정책관리 > 문서변경 참조 설정과 무관

## 증상

- **문제 정의**: 문서변경 참조[^5] 설정 없이 구성원 문서 변경 알림이 발송됨[^1]
- **회사**: 주식회사 에스엘즈 (Customer ID: 34792)[^1]
- **요청자**: minhee.shin@slz.kr (신민희)[^1]
- **대상자**: kisae.lee@slz.kr (이기재)[^1]
- **영향 범위**: 참조 설정 없이 알림을 수신한 대상자 1명 이상[^1]
- **문제 시점**: 신규입사자(jehyoung.park@slz.kr) flex 가입·문서 등록 시점[^2]
- 문의 내용:
  1. kisae.lee@slz.kr이 jehyoung.park@slz.kr의 문서 변경 알림을 수신함[^1]
  2. 설정 > 승인정책관리 > 구성원정보 > 문서변경 참조에 kisae.lee@slz.kr이 설정되어 있지 않음에도 알림이 발송됨[^1]
  3. Intercom 원문: "신규입사자가 와서 flex 가입하는데, 다른 분들에게 개인정보 등록 알림이 간다고 하네요"[^2]

> ⚠️ Intercom 원문과 Linear 이슈 설명이 표현상 차이가 있으나 동일 현상으로 보임. Intercom은 "개인정보 등록 알림", Linear는 "문서 변경 참조 알림"으로 표현.

## 현재까지 파악된 내용

- 스크린샷 2장이 코멘트에 첨부되어 있으나 내용 미확인[^3]
- 알림 수신자(kisae.lee)가 발신자(jehyoung.park)의 문서변경 참조로 등록되지 않은 상태에서 알림을 받은 것이 핵심[^1]

### 코드 조사 결과 (flex-core-backend)

이성환님 코멘트가 정확하다. 문서/개인정보 변경 알림 모두 "문서변경 참조" 설정이 아닌 **권한(permission) 기반**으로 수신자를 결정한다[^6].

**문서 변경 알림 (`UserDocumentFileServiceImpl.sendNotificationToManager`)**[^6]
- 수신자: `resolveUserAuthorityGrantedUsers(CORE_USER_ATTACHMENT_UPDATE)` — 해당 대상자에 대해 이 권한을 가진 모든 사용자
- 승인정책관리 > 문서변경 참조 설정과는 **무관**

**개인정보 변경 알림 (`UserPersonalBundleServiceImpl.sendNotificationToManager`)**[^8]
- 수신자: `resolveUserAuthorityGrantedUsers(CORE_USER_PERSONAL_UPDATE)` — 동일하게 권한 기반
- 추가: 변경 대상자 본인에게도 1단계 알림 발송 (`UserDataChangeNotificationService`)
- 단, 관리자(권한 보유자)가 직접 수정한 경우엔 2단계 알림 스킵

> 💡 **결론**: 개인정보 변경도 **권한 기반**으로 발송된다. 최고관리자 고정이 아니라 `CORE_USER_PERSONAL_UPDATE` 권한을 보유한 사람 전원에게 발송. 문서/개인정보 모두 동일한 패턴.

| 항목 | 문서 변경 | 개인정보 변경 |
|------|---------|-------------|
| 수신자 기준 | `CORE_USER_ATTACHMENT_UPDATE` 권한 보유자 | `CORE_USER_PERSONAL_UPDATE` 권한 보유자 |
| 대상자 본인 알림 | 없음 | 있음 (항상) |
| 관리자 수정 시 | 알림 발송 | 알림 스킵 (관리자가 수정한 경우) |

### 알림 타입 (flex-pavement-backend)

문서/개인정보 변경 관련 알림 타입[^7]:
- `FLEX_CORE_USER_DATA_CHANGED` — 구성원 정보 변경 기본 알림 (대상자 본인)
- `FLEX_CORE_USER_DATA_CHANGE_REQUEST` — 변경 요청 알림
- `FLEX_CORE_USER_DATA_CHANGE_ONGOING` — 변경 진행 중 알림

## 영향 범위
