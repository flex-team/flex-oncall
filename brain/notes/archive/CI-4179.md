# CI-4179: 국세청 연동 이전 세금계산서 내역 연동 요청

## 증상
- **문제 정의**: 국세청 세금계산서 데이터가 2025년 9월부터만 연동되어 있어, 2025년 7월부터의 데이터를 요청
- **회사**: 자원 (ZAONE) (Customer ID: 214379)
- **요청자**: 이주화 (CS팀)[^1]
- **대상자**: nari@zaone.org (고객사 담당자)[^2]
- **영향 범위**: 1개 회사
- **문제 시점**: 2025년 7~8월 세금계산서 누락
- 문의 내용:
  1. 국세청 세금계산서 2025년 7월~8월 데이터 소급 연동 요청[^1]
  2. 신한카드 2025년 9월 1일부터 데이터 재연동 요청[^3]

## 원인 분석

### 조사 과정
> 💡 CS팀(이주화)이 Notion 스크래핑 대상 문서에서 최근 12개월 수집 가능 확인[^4]
> → 담당자(김성훈)가 연동 가능 범위를 금융사별로 확인[^5]
> → 세금계산서는 범위 내이므로 운영 도구로 즉시 동기화
> → 신한카드는 고객사 측에서 연동 자체가 안 되어 있어 우리 측 조치 불가

### 원인
- 세금계산서: 초기 연동 시 2025년 9월부터만 수집되었고, 7~8월은 수동 동기화가 필요했음[^5]
- 신한카드: 고객사가 신한카드 연동을 수행하지 않은 상태[^6]

## 해결
- **세금계산서 7~8월**: 김성훈님이 운영 도구로 즉시 동기화 완료[^5]
- **신한카드 11~12월**: 고객사 연동 확인 후 수동 동기화로 완료[^9]
- **신한카드 9월**: API별 조회 가능 기간이 상이하여 2025-09-15 이전 데이터는 별도 코드 작업이 필요했음[^10]. 익일(03-24) 신한카드 9월 내역까지 모두 동기화 완료[^11]

## 발견한 스펙/제약
- 국세청 스크래핑 수집 가능 범위: 최근 12개월[^4]
- 연동 가능 범위는 금융사마다 다름[^5]
- 카드사 연동은 고객사가 직접 수행해야 하며, 연동 후에야 운영 도구로 데이터 동기화 가능[^6]
- 카드 데이터는 승인/매입이 서로 다른 API를 사용하며, API별 조회 가능 기간이 상이[^10]
- 특정 기간 이전 데이터는 표준 수동 동기화로 불가, 별도 코드 작업이 필요할 수 있음[^10]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 요청 대상이 무엇인지 파악 (카드사 / 국세청 / 홈택스 등) + 희망 기간
2. **원인 판별**:
   - 해당 서비스가 연동되어 있는지 확인 → 미연동이면 고객사에서 직접 연동 필요
   - 연동 완료 상태이면 → Notion 스크래핑 대상 문서에서 수집 가능 범위 확인
3. **조치**:
   - 수집 범위 내 → 어드민쉘 > 파이낸스 > 고객사 통합 조회 > 금융기관 연동 상태 > 대상 회사 선택 > 수동 동기화[^12]
   - 수집 범위 외 또는 특정 기간 이전 → 담당 개발자(김성훈)에게 별도 코드 작업 요청 필요[^10]
   - 동기화 결과 확인: `#squad_finance_notification_critical` 채널[^12]

### 재발 방지 / 향후 참고
