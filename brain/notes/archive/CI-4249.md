# CI-4249: 캡스 기기 변경 후 근태처리옵션 계정 오설정으로 동기화 불량

> **compact**: 2026-03-31. git log로 원본 추적 가능.

## 증상
- **문제 정의**: 캡스 기기 변경 후 캡스 프로그램 근태처리옵션 계정이 잘못 설정되어 타각 데이터가 플렉스 근무와 정상 동기화되지 않음
- **회사**: (주)뉴빌리티 (Customer ID: 29042)
- **요청자**: 유지원[^1] (CS 담당자)
- **대상자**: jaun@neubility.co.kr[^1]
- **영향 범위**: 특정 조건 (기기 변경 후 재설정 미완료)
- **문제 시점**: 2026-03-27 캡스 기기 변경 이후
- 문의 내용:
  1. 3월 27일 캡스 기기 변경 작업 이후 타각 데이터가 플렉스 근무와 정상 동기화되지 않음
  2. draft 기록에는 연동이 정상으로 보이나 근무 기록이 이상하게 쌓임
  3. 근무 시작 → 종료 시 출근 기록 없음 / 시작 → 종료 중첩근무 존재 패턴 반복[^2]

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 
> 1. draft 기록을 확인한 결과, 타각 데이터가 플렉스 근무로 정상 전송됨[^2]
> 2. 그러나 플렉스 데이터를 조회하면 "근무 시작 → 종료 시 출근 기록 없음 / 시작 → 종료 중첩근무" 패턴 반복[^2]
> 3. → 데이터가 수신되고 있으나 잘못된 계정 정보로 인해 매핑 오류 발생 중인 것으로 판단[^3]
> 4. → 캡스 프로그램의 근태처리옵션 계정 재설정을 고객사에 안내[^3]
> 5. 고객사에서 근태처리옵션 계정 수정 후 연동 정상화 확인[^3]

### 원인
- 캡스 기기 변경 후 캡스 프로그램의 근태처리옵션 계정이 잘못 설정됨[^3]
- 플렉스 연동 자체는 살아있었으나, 잘못된 계정 설정으로 데이터가 올바르게 매핑되지 않음

## 해결
- 고객사(뉴빌리티)에서 캡스 프로그램 근태처리옵션 계정을 수정[^3]
- 수정 후 연동 정상 확인[^3]

## 발견한 스펙/제약
- 캡스 기기 변경 시 캡스 프로그램의 근태처리옵션 계정도 재설정이 필요함[^3]
- 원본선택 항목은 캡스에서 제공하는 값이며 플렉스에서 설정하는 영역이 아님[^4]
  - 플렉스에서 설정하는 것은 열이름(컬럼명)
  - 원본선택 항목이 가이드와 다르게 표시되면 캡스 버전 차이일 수 있음 (확인된 버전: ACServer v5.06-T06)[^5]
- 원본선택이 다른 경우 캡스 담당자에게 문의하여 사용 가능한 값 목록을 파악해야 함[^4]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 캡스 기기 교체/변경 이력 여부 확인
2. **원인 판별**:
   - 기기 변경 후 발생 → 캡스 프로그램 근태처리옵션 계정 설정 오류 가능성 높음
   - 원본선택 항목이 가이드와 다르게 보임 → 캡스 버전 차이 (캡스 측 담당자 확인 필요)
3. **조치**:
   - 근태처리옵션 계정 재설정: 캡스 프로그램에서 올바른 계정 정보 입력 후 테이블 선택 재진행
   - 원본선택 불일치: 캡스 담당자에게 버전 확인 및 가용 값 목록 요청 → 백로그 처리
