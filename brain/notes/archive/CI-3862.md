# CI-3862: 주휴일 배치 주기가 일주일 초과 허용인데 근무표 게시 시 제한됨

> **상태**: 진행 중 — 2026-02-10

## 증상
- **회사**: 시몬스 (Customer ID: 77952)
- **문의자**: hy.kim@simmons.co.kr
- **대상 구성원**: jeongmin.jo@simmons.co.kr (657905 / WL8JVvYJ8K), changhee.son@simmons.co.kr (895693 / WL8JK54j0K)
- 문의 내용:
  1. 근무 유형에서 주휴일 배치 주기를 일주일을 넘겨도 게시할 수 있도록 허용해 두었음
  2. 2월 근무표 게시 시 1.26~2.1 주기에 주휴일 미배치로 게시 불가 오류 발생
  3. 주휴일을 배치하여 게시는 완료했으나, 배치하지 않는 것이 의도였기 때문에 확인 요청

## 현재까지 파악된 내용
- dry-run 결과에서 두 구성원 모두 25/12/29, 26/01/05, 26/01/12, 26/01/19 주기에서 "일주일에 주휴일 1회 이상 배치" 오류 발생 확인
- BE dry-run API는 `allowLackOfWeeklyHolidays = true` 설정을 올바르게 반영하여 `WARN` 레벨로 응답 (게시 차단 아님)
- **원인 확정**: FE에서 dry-run 응답의 `WARN` 레벨을 `ERROR`와 동일하게 처리하여 게시 버튼을 비활성화하는 클라이언트 버그
- 김한나님 테스트 계정 확인: UI에서 "경고"가 아닌 "확인 필요"로 표시되며 게시 버튼이 비활성화됨
- **FE 수정 PR**: [flex-frontend-apps-time-tracking/pull/2101](https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2101) — warning 발생 시에도 제출 가능하도록 파서 수정 (담당: 이지선)

### 코드 분석 결과

**검증 로직**: `WeeklyPaidHolidayMissingInWorkingPeriodVerifier.kt`

| 항목 | 내용 |
|------|------|
| 검증 단위 | `makeWorkingPeriodAsWeeksByDates()` — 항상 **7일 단위 주(week)**로 분할 검증 (근무 주기 아님) |
| 설정 필드 | `workRecordRule.allowLackOfWeeklyHolidays` |
| true일 때 | 검증 레벨 = **`WARN`** (경고, 게시 차단 안 됨) |
| false일 때 | 검증 레벨 = **`ERROR`** (게시 차단) |

```kotlin
// WeeklyPaidHolidayMissingInWorkingPeriodVerifier.kt:97-102
workShiftRegisterVerificationLevel = if (it.allowLackOfWeeklyHolidays) {
    WorkShiftRegisterVerificationLevel.WARN   // 허용 → 경고만
} else {
    WorkShiftRegisterVerificationLevel.ERROR  // 비허용 → 차단
}
```

**v3 체크룰** (`UserWorkShiftScheduleWeeklyHolidayCheckRuleExtractor.kt`)에서는 더 강하게 처리:
```kotlin
.filterNot { it.allowLackOfWeeklyHolidays } // 유연한 주휴일 대응 → 아예 검증 스킵
```

**결론**: BE는 정상 동작. `allowLackOfWeeklyHolidays = true`이면 `WARN` 반환하여 게시 차단하지 않음.
실제 차단은 FE에서 `WARN`을 `ERROR`와 동일하게 처리한 클라이언트 버그.
