# CI-4110: PC 근무 시작 시 '근무 시작 시간을 확인해 주세요' 오류

> **상태**: Done — 2026-03-13 (KST)
> **verdict**: client-bug — 프론트엔드 PR 사이드이펙트로 클라이언트-서버 시간 오차 발생, 웹 핫픽스 배포

## 증상

- **문제 정의**: 2026-03-13 10:55경 PC(웹)로 근무 시작 버튼 클릭 시 '근무 시작 시간을 확인해 주세요' 오류 노출. 모바일은 정상 동작. 지난 24시간 동안 318명에게 동일 에러 발생 (Sentry 확인). 웹에서만 400 에러 발생.

## 원인 분석

- access log 확인 결과: 요청 시각 `2026-03-13 11:39:07.067`, `targetWorkStartTimeStamp: 2026-03-13 11:38:00` — 약 1분 차이로 서버 validation 실패 (400)
- 원인 PR: [flex-frontend-apps-time-tracking#2185](https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2185#discussion_r2850611782)의 사이드이펙트
  - **AS-IS**: `onTime: false`일 때 서버에 `targetWorkStartTimeStamp`로 `undefined`(null)를 보냄 → 서버가 자체적으로 현재 시간 사용
  - **TO-BE** (변경 후): `onTime: false`일 때 클라이언트에서 `now`를 보냄 → 서버시계와의 diff 보정이 빠져서 클라이언트 시간이 서버 시간보다 과거로 들어가 validation 실패
- 서버와 클라이언트 시간 오차 1분 정도는 일반적으로 발생 가능하며, 기존에는 `undefined` 전송으로 이 문제를 회피하고 있었음

## 해결

- **웹 핫픽스**: [flex-frontend-apps-time-tracking#2312](https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2312) — `onTime: false`일 때 다시 `null`(undefined)을 보내도록 복원
- 관련 서버 핫픽스: [flex-timetracking-backend#11967](https://github.com/flex-team/flex-timetracking-backend/pull/11967) (전날 배포됨, 지금 출근에도 적용되는 validation)
- 이지선 님이 핫픽스 작성 → 출근 테스트 후 점심 전 배포 완료
- 대부분 사용자가 2~3번 실패 후 재시도로 출근 성공. 추가 문의 인입 없어 별도 안내 불필요
