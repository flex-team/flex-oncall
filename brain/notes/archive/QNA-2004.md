# QNA-2004: 실시간 근무 기록 반영 시 승인 문서 이전 기록 미표시 문의

## 증상
- **문제 정의**: 실시간 근무 기록이 인정 근무[^5]를 대체한 후, 초과근무 승인 문서에서 "이전 기록" 비교가 표시되지 않는 현상
- **회사**: 정보 없음
- **요청자**: 이주화 (가이드 작성 중 발견하여 문의)
- **대상자**: 특정되지 않음 (문의에서 "홍길동"은 예시 이름)[^1]
- **영향 범위**: 단일 문의
- **문제 시점**: 2026-04-09
- 문의 내용:
  1. 근무 유형에서 "실시간 근무 기록을 구성원의 근무 시간으로 반영" ON 설정[^1]
  2. 구성원이 전날 근무를 사전 등록(9시~19시30분) → 초과근무(연장 근무) 승인 발생[^1]
  3. 당일 실시간 근무 시작(9시) 후 예정보다 늦게 종료(21시) → 인정 근무[^5]가 실시간 기록으로 대체됨[^1]
  4. 승인 문서에서 "기록 전(9시~19시30분)" 비교가 표시되지 않음 — 이것이 현재 스펙인지 문의[^1]

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 승인 문서에서 이전 기록 비교가 미표시된다는 보고[^1]
> → 해당 승인 문서의 상태 확인 — 승인이 아직 "진행 중"이었는지 질의[^2]
> → 문의자가 승인이 진행 중 상태였음을 확인[^3]
> → 승인이 완료(인정)되지 않은 상태이므로 이전 기록 비교가 미표시되는 것은 정상

### 원인
- 초과근무 승인이 아직 "진행 중" 상태여서 승인 문서의 이전/현재 기록 비교가 표시되지 않음[^2][^3]
- 승인이 인정(완료)되기 전이므로 비교 대상 자체가 확정되지 않은 상태 — 정상 동작[^4]

### 추가 확인 (위젯과 승인의 관계)
- 질문의 상황은 **미래의 근무**이므로 일반적인 케이스와 다름[^7]
- 해당일에 위젯 기록이 있고, 위젯으로부터 근무 등록 + 승인 대기 중인 상황이라면: 승인 문서에 근무 기록은 안 보이지만 **위젯은 보인다**[^7]
- **위젯은 승인에 묶여있지 않다** — 위젯에는 "대기 중" 상태가 없음[^7]

## 영향 범위
- 단일 문의 건. 동일 조건(실시간 근무 ON + 사전 등록 + 승인 진행 중)에서 재현 가능하나 스펙에 의한 정상 동작
- 금전적·법적 영향 없음

## 소급 영향 범위
- 해당 없음 (코드 변경 아님, 스펙 확인 건)

## 배포 영향도
- 해당 없음

## 법적/컴플라이언스 리스크
- 해당 없음

## 해결
- 승인이 아직 진행 중인 상태임을 확인하여 정상 동작으로 종결[^4]

## 발견한 스펙/제약
- 실시간 근무 기록 ON 시, 사전 등록한 인정 근무[^5] 그래프가 실시간 기록으로 대체된다[^1]
- 승인 진행 중 상태에서는 승인 문서의 "이전 기록 비교"가 표시되지 않는다[^2]
- 위젯은 승인 프로세스에 묶여있지 않다 — 승인 대기 중이어도 위젯은 독립적으로 표시됨[^7]

## 다음에 같은 문의가 오면
1. **먼저 확인**: 해당 승인 문서의 상태가 "진행 중"인지 "완료"인지 확인
2. **원인 판별**: 승인 진행 중이면 → 이전 기록 비교 미표시는 정상 동작. 승인 완료 후에도 미표시면 → 추가 조사 필요
3. **조치**: 진행 중이면 승인 완료 후 확인하도록 안내. 완료 후에도 문제면 코드 조사 진행

    만약 구성원이 실시간 근무를 기록하여 인정 근무를 대체한다
    그러면 승인 문서에서 이전 기록과 현재 기록의 비교가 표시된다
