# CI-4197: 급여정산 결과에서 이름·사번 검색이 원활하지 않음

## 증상
- **문제 정의**: 급여정산 결과 화면의 검색 필터에서 한 글자 입력 시 포커스가 빠지면서 타이핑이 되지 않음
- **회사**: 케이비라이프파트너스 (Customer ID: 59115)
- **요청자**: 안성연 (sungyeon@flex.team)[^1]
- **대상자**: PP[^2]
- **영향 범위**: 급여정산 결과 화면을 사용하는 전체 사용자
- **문제 시점**: 2026-03-24 급여정산 업데이트 이후 발생[^3]
- 문의 내용:
  1. 급여정산 결과 화면에서 이름·사번으로 검색이 원활하지 않음
  2. 필터 칸에 타이핑하면 한 글자 쓸 때마다 포커스가 빠짐[^4]
  3. 어제 밤까지는 정상 동작했음[^5]

## 원인 분석

### 조사 과정
> 💡 유애진이 백엔드(테이블 검색)에는 추가 작업이 없었음을 확인[^6]
> → 프론트엔드 이슈로 판단, 이지우에게 전달[^7]
> → 이지우가 당일 플래그 open한 여파로 추정[^8]
> → 테이블에 `onFilterChanged` 로직이 추가되면서 발생한 이슈로 확인[^9]

### 원인
- 급여정산 결과 테이블의 floating filter에 `onFilterChanged` 이벤트 핸들러가 추가되면서, 한글 입력 시 매 글자마다 필터 이벤트가 트리거되어 포커스가 유실됨[^9]

## 해결
- 프론트엔드 핫픽스 배포: `flex-frontend-apps-payroll#1708`[^10]
- 정산 결과 테이블 floating filter에서 한글 입력 시 포커스 유실되는 버그 수정[^10]
- 고객 확인 완료 — "이제 된다"[^11]

## 시나리오

```gherkin
# language: ko
기능: 급여정산 결과 테이블 floating filter 한글 입력

  시나리오: 급여정산 결과 화면에서 이름으로 검색할 때 한글 입력이 정상 동작한다
    주어진 급여정산 결과 화면이 열려 있다
    그리고 테이블 floating filter에 onFilterChanged 이벤트 핸들러가 적용되어 있다
    만약 사용자가 이름 필터에 "홍길동"을 입력한다
    그러면 입력 중 포커스가 유실되지 않는다
    그리고 "홍길동"이 포함된 구성원만 필터링되어 표시된다

  시나리오: 급여정산 결과 화면에서 사번으로 검색할 때 입력이 정상 동작한다
    주어진 급여정산 결과 화면이 열려 있다
    만약 사용자가 사번 필터에 "2024001"을 입력한다
    그러면 입력 중 포커스가 유실되지 않는다
    그리고 해당 사번의 구성원만 필터링되어 표시된다
```
