# CI-4202: 캡스 수동 동기화 시 실패 — 테이블 매핑 설정 오류

## 증상
- **문제 정의**: 캡스(CAPS) 수동 동기화 시 "전송 실패"로 표시되며 출퇴근 데이터가 flex로 반영되지 않음
- **회사**: 클레로보틱스 주식회사 (Customer ID: 70136)
- **요청자**: 이주화 (CS팀)
- **대상자**: hkjee@clerobotics.com
- **영향 범위**: 해당 회사 캡스 연동 사용자 전체 (추정)
- **문제 시점**: 2026-03-24 문의 접수
- 문의 내용:
  1. 캡스 연동은 성공한 것으로 보이나, 수동 동기화 시 "전송 실패"로 표시됨[^1]
  2. 실시간 근무 기록 사용은 활성화 상태[^1]
  3. flex와 캡스의 구성원 사번이 일치한다고 주장[^1]

## 원인 분석

### 조사 과정
> 💡 김영준(Enhance)이 Grafana 캡스 RDB 모니터링 대시보드[^7]를 확인
> → 로그에서 `e_date` 컬럼 관련 오류 발견[^3]
> → `e_date`는 캡스 DB의 원본 컬럼명으로, flex 연동 테이블에는 존재하지 않는 컬럼[^4]
> → 고객이 캡스 테이블 매핑 설정을 잘못 구성하여 발생한 문제로 판정[^3]

### 원인
- **고객이 캡스 테이블 매핑 설정을 잘못 구성함**[^3]
- Grafana 로그에 `e_date` 컬럼이 보이면 테이블 매핑 설정 오류를 의미[^4]
- flex 가이드 문서에 올바른 설정 방법이 있으나, 고객이 따르지 않았음[^6]

## 해결
- 고객에게 올바른 테이블 매핑 설정 방법을 안내 (Linear 코멘트에 올바른 설정 스크린샷 공유)[^5]
- **코드 변경 불필요** — 고객 측 설정 수정으로 해결되는 건[^6]

## 다음에 같은 문의가 오면

1. **먼저 확인**: Grafana 캡스 RDB 모니터링 대시보드[^7]에서 해당 회사의 로그 확인 (`username`으로 필터링)
2. **원인 판별**: 로그에 `e_date` 같은 캡스 원본 컬럼명이 보이면 → 테이블 매핑 설정 오류. 그 외 오류면 → 연결 설정(ODBC, CONNECTION LIMIT 등) 확인
3. **조치**: 고객에게 올바른 테이블 매핑 설정 가이드 안내. 가이드 문서 준수 여부 확인 요청
