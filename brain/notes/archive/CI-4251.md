## 원인 분석

### 원인 (확정)

**초기 세팅 시 각 구성원별 조직설정이 되지 않아, 업로드된 근무기록이 UI에 표시되지 않았다**[^9].

데이터 자체는 DB에 정상 저장되었으나[^6], 조직설정이 없는 상태에서는 UI가 근무정보를 표시하지 않는 구조.

### 스펙 vs 버그 판별

**판정: 설정 이슈 (스펙)** — 벌크 업로드 기능 자체는 정상 동작. 조직설정이 선행 조건이나, 업로드 시 이를 검증하거나 안내하지 않는 점은 UX 개선 여지가 있다.

### 부수 발견

- 3/28: `.xls` 포맷 업로드 시도 → 서버는 `.xlsx`만 지원하여 dry-run 전부 400[^3]
- 3/30 첫 dry-run: row 32 종료시각 미입력, row 44 시작/종료시각 미입력[^4]

## 해결

CS팀(@장민규)이 각 구성원별 조직설정을 완료하여 근무정보가 정상 표기됨을 확인[^9]. 코드 수정 불필요.

## 다음에 같은 문의가 오면

1. **먼저 확인**: 해당 회사가 신규 가입 회사인지, 구성원별 조직설정이 완료되어 있는지 확인
2. **원인 판별**:
   - 조직설정 미완료 → 조직설정 먼저 진행하도록 안내
   - 조직설정 완료 상태 → access log에서 bulk-upload 요청 responseStatus 확인 (F-sched-upload 플로우)
   - dry-run 400 → 엑셀 데이터 검증 실패. `.xls` 형식이면 `.xlsx`로 변환 안내
3. **조치**: 조직설정 완료 후 재업로드 안내
