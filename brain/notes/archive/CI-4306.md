# CI-4306: 그룹 구글캘린더 수동 싱크 요청

## 증상
- **문제 정의**: 그룹 구글캘린더 재연동 후, 미연동 일정의 수동 동기화 요청
- **회사**: (주)이노바이드 (Customer ID: 199157)
- **요청자**: 유지원 (CS팀)[^1]
- **대상자**: halimseo@dentlink.app (구글캘린더 연동 계정)[^2]
- **영향 범위**: 1명 (해당 캘린더 연동 계정)
- **문제 시점**: 2026-01 ~ 2026-04 기간의 일정
- 문의 내용:
  1. 구글캘린더 연동이 원활하지 않아 재연동을 수행함[^2]
  2. 재연동 이전에 등록한 일정이 구글캘린더에 동기화되지 않음[^2]
  3. 2026년 1월 ~ 4월 기간의 미연동 일정 수동 싱크 요청[^2]

## 원인 분석

### 조사 과정
> 💡 Metabase 대시보드 244에서 halimseo@dentlink.app의 캘린더 연동 상태 확인[^6]
> → 그룹캘린더 미연동 이벤트 **349건** 확인[^6]
> → 개인캘린더는 "No results" — 그룹캘린더만 해당[^6]
> → CI-4262 선례와 동일 패턴 — COOKBOOK F1 플로우 적용[^4]

### 원인
- 그룹 구글캘린더 재연동 시 기존 연동이 해제되면서, 재연동 이전에 생성된 flex 캘린더 이벤트가 구글캘린더에 동기화되지 않은 상태로 남음[^3]

## 해결
- Metabase 쿼리 7387로 미연동 eventIds 349건 추출[^7]
- raccoon Operation API `sync-created-member-schedule-group-calendar-by-ids` 호출[^8]
  - batch 20건 + 2초 delay, 총 18회 호출
  - **전체 18/18 성공 (200 OK)**

## 다음에 같은 문의가 오면

1. **먼저 확인**: Metabase 대시보드 `/dashboard/244?email={사용자이메일}`에서 미연동 건 확인
2. **eventIds 추출**: Metabase 쿼리 7387로 미연동 eventIds 추출
3. **조치**: raccoon Operation API로 재동기화
   - 엔드포인트: `/proxy/calendar/api/operation/v2/calendar/customers/{customerId}/google-calendar-events/sync-created-member-schedule-group-calendar-by-ids`
   - 300건 이상 시 Google Calendar API rate limit 주의 → batch 20건 + 2초 delay 권장
   - 50건 이상 시 raccoon audit log URL 길이 초과 가능 → batch 5~20건 권장
