# CI-4226: 기본 정보 변경 엑셀 미리보기에서 로우 중복 생성

> **상태**: 해결 완료 (프론트엔드 수정) — 2026-03-30

## 증상
- **문제 정의**: 기본 정보 변경 엑셀 템플릿 업로드 시 미리보기에서 일부 구성원이 중복 로우로 생성되고, 중복 중 하나는 "알 수 없음"으로 표시됨
- **회사**: 타르틴코리아 (Customer ID: 224947)
- **요청자**: jieon.kim@tartinekorea.com (CS 접수: 오시은)[^2]
- **대상자**: 154명 전원 (엑셀 업로드 대상 전체)
- **영향 범위**: 해당 회사 엑셀 일괄 변경 사용 시
- **문제 시점**: 2026-03-26
- 문의 내용:
  1. 구성원 > 인사 정보 관리 > 정보 일괄 변경 > 정보 변경하기 > 엑셀로 변경 > 기본정보 경로에서 엑셀 템플릿 업로드 시 미리보기 화면에 일부 구성원이 중복 로우로 표시됨[^1]
  2. 실제 엑셀 템플릿에는 해당 데이터가 1건만 존재한다고 전달됨[^1]
  3. 중복 생성된 로우 중 하나는 "알 수 없음"으로 표시됨[^1]

---

## 원인 분석

프론트엔드가 엑셀의 이메일 컬럼을 사번(employeeNumber)으로 잘못 파싱하여, 이메일 기반 검색(0건)과 사번 기반 검색(154건)이 미리보기에 합쳐져 동일 구성원이 중복 표시됨. 이메일 기반 엔트리는 매칭 실패로 "알 수 없음" 표시.[^3][^4]

### 스펙 vs 버그 판별

**버그** — 프론트엔드 엑셀 파싱 로직의 컬럼 매핑 오류.

### 핵심 근거

> 💡 **판단 근거**: access log에서 동일 사용자가 2분 간격으로 두 번 검색한 기록 확인[^3][^4]
> → 1차 검색: employeeNumbers 필터에 **이메일 주소** 154건 전송 → 결과 0건 (매칭 실패)[^3]
> → 2차 검색: employeeNumbers 필터에 **실제 사번** 154건 전송 → 결과 154건 (정상)[^4]
> → 프론트엔드가 1차(이메일 파싱)와 2차(사번 파싱) 결과를 합쳐 미리보기를 구성
> → 이메일 기반 엔트리는 매칭 실패이므로 "알 수 없음"으로 표시

### 가설 테이블

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | 프론트엔드가 이메일 컬럼을 사번으로 잘못 파싱하여 이중 엔트리 생성 | access log에서 employeeNumbers에 이메일이 포함된 요청 확인[^3] | ✅ 확정 |
| 2 | 백엔드 candidate search에서 중복 반환 | 2차 검색 154건 모두 고유 확인[^4] | ❌ 소거 |
| 3 | 백엔드 validate 응답에서 중복 | validate 호출 자체가 0건[^7] | ❌ 소거 |

## 해결안

1. **프론트엔드 수정 (근본 해결)**: 엑셀 파싱 시 사번 컬럼 인덱스를 올바르게 매핑하도록 수정. 이메일 형식(`@` 포함 문자열)이 사번 필드에 들어오면 무시하거나 경고 표시.
2. **엑셀 템플릿 개선 (보조)**: 사번 컬럼에 데이터 유효성 검사(이메일 형식 거부) 추가, 컬럼 헤더에 "사번만 입력" 명시.
