### 세콤 퇴근 자동 조정 로직 (코드 확인)

세콤/캡스 타각 시 정책에 맞는 시각으로 조정하여 자동 퇴근하는 것이 기본 스펙[^10]:

```
세콤 퇴근 타각
  → 1단계: 정시 퇴근 조정 (REAL_TIME이면 스킵)
  → 2단계: dry-run
  → 3단계: OVER_WORK_MINIMUM_INPUT_MINUTES_RULE_VIOLATION 감지 시
    → 퇴근시각을 연장근무 시작점으로 조정 (연장근무 제거)
    → 재 dry-run
  → 4단계: 재 dry-run 통과하면 자동 퇴근 완료
```

- `WORK_SCHEDULE_REQUIRED_MEMO`는 `earlyReturn=false` → dry-run이 **SuccessResultModel** 반환[^11]
- 따라서 3단계 연장근무 조정은 **정상 실행됨**[^11]
- 조정 후 연장근무 0분이면 OVER_WORK 승인 불필요 → REQUIRED_MEMO 해소되어야 함

#### 기준값 불일치 상세

이 유저: FIXED, FULL_TIME, 소정 7시간(420분)[^5]
- `shortTimeWork = false`, `exceedStatutoryWorkingMinutesSettingEnabled = false`[^18]
- → `RegularWorkBaseWorkingMinutes.getLegalBaseWorkingMinutesPerDay()` 에서 `MAX_LEGAL_BASE_WORK_MINUTES_IN_DAY = 480` 반환[^18]

| | 변경 전 (`baseAgreedDayWorkingMinutes`) | 변경 후 (`legalBaseWorkingMinutesPerDay`) |
|---|---|---|
| 기준값 | **420분** (소정근로) | **480분** (법정기준) |
| 연장 시작 시점 | 08:30 + 420분 + 휴게60분 = **16:30** | 08:30 + 480분 + 휴게60분 = **17:30** |
| 16:47 퇴근 시 | 16:47 > 16:30 → candidate 생성 → **16:30으로 조정** | 16:47 < 17:30 → candidate 없음 → **조정 안 됨** |

```
dry-run: 소정근로 420분 기준으로 "17분 연장" 감지 → OVER_WORK_MINIMUM 위반
조정 로직: 법정기준 480분 기준으로 "연장 없음" 판단 → candidate 없음 → 조정 안 함
→ 기준값 불일치로 감지는 되지만 조정은 안 되는 상태 → FAIL_DRY_RUN
```
