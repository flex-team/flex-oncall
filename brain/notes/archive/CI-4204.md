# CI-4204: 평가 등급 배분 완료 탭 클릭 시 오류 발생

## 증상
- **문제 정의**: 평가 전체 설정 중 "7. 등급 배분" 완료 탭을 누르면 "알 수 없는 오류가 발생했습니다" 메시지 표시
- **회사**: 포트로직스 (Customer ID: 181498)
- **요청자**: 이주화 (CS팀)[^1]
- **대상자**: gabean@portlogics.com (이가빈)[^2]
- **영향 범위**: 해당 평가 1건 (evaluation ID: `01kmf4cpshab13r1b6ssws39ke`)
- **문제 시점**: 2026-03-24 14:10경 (KST)[^2]
- 문의 내용:
  1. 평가 전체 설정 > 7. 등급 배분 메뉴에서 완료 탭을 누르면 오류 발생[^2]
  2. 대상 평가명: [수습] 1차 수습 평가_정희진[^2]

## 원인 분석

### 조사 과정
> 💡 홍석민이 Sentry에서 사용자 이메일로 오류를 특정[^3], traceId 확보 후 OpenSearch 로그 확인[^4]
> → 김주원이 코드 라인과 로그를 특정[^5]
> → 김보라가 `evaluation.gradeToCalculation`이 비어있는데 `evaluationStep.setting.enabledGradeFactors[CUSTOM]`이 존재하는 불일치 발견[^8]
> → 셀프 단계에는 등급 미설정, 하향 단계에만 CUSTOM 등급 설정 확인[^9]
> → 등급을 한 단계에서만 낼 경우 "등급 산출" 메뉴가 UI에 나타나지 않아 `grades_to_calculate`가 비어있는 것이 원인[^10]
> → 이 버그는 과거에 수정 배포되었으나, 수정 전에 생성된 평가를 복제하여 만든 평가에서 잔존 데이터로 인해 재발[^11]

- **trace ID**: `01f175e134b6e24ec5c8bcd5b0427770`[^4]
- **오류 위치**: `DraftEvaluationStageValidator.assertToComplete()` (DraftEvaluationStageValidator.kt:342)[^5]
- **예외**: `EvaluationValidationException`[^5]
- **호출 스택**[^5]:
  ```
  DraftEvaluationStageValidator.assertToComplete-FoTxr_Y(DraftEvaluationStageValidator.kt:342)
  DraftEvaluationStageProgressCommandService.complete-FoTxr_Y(DraftEvaluationStageProgressCommandService.kt:211)
  DraftEvaluationFinalizeService.finalize-pVAYERY(DraftEvaluationFinalizeService.kt:72)
  DraftEvaluationActionController.finalizeDraftEvaluation(DraftEvaluationActionController.kt:68)
  ```

### 원인
- 과거 버그: 평가 생성 시 단일 단계에서만 등급을 산출하는 경우 `grades_to_calculate` 필드가 비어있는 채로 저장되는 버그가 있었음[^11]
- 이 버그는 이미 코드 수정 후 배포 완료[^11]
- 그러나 **수정 전에 생성된 평가를 복제(copy)하여 새 평가를 만들면** 잔존 데이터가 그대로 복사되어 동일 오류가 재발[^11]
- `draft_evaluation.grades_to_calculate = '[]'` (비어있음)인데 `evaluationStep.setting.enabledGradeFactors`에 `CUSTOM`이 존재하여 finalize 시 validation 실패[^8]

## 해결
- 고객 확인: 하향평가 단계에서 커스텀 평가 요소에 대한 등급 산출을 원함[^12]
- 김보라가 `draft_evaluation` 테이블에 DML 실행하여 데이터 보정[^13]:

```sql
-- 적용 쿼리
UPDATE flex_review.draft_evaluation SET
  grades_to_calculate = '["FACTOR_GRADE"]',
  factor_grade_calculations = '[{"calculation": "WEIGHTED", "evaluationFactorType": "CUSTOM"}]'
WHERE id = '01kmf4cpshab13r1b6ssws39ke';
```

```sql
-- 백업 쿼리
UPDATE flex_review.draft_evaluation SET
  grades_to_calculate = '[]',
  factor_grade_calculations = null
WHERE id = '01kmf4cpshab13r1b6ssws39ke';
```

- 적용 완료 후 이주화에게 확인 요청[^14]

## 발견한 스펙/제약
- 평가 복제(copy) 시 원본 평가의 `grades_to_calculate`, `factor_grade_calculations` 값이 그대로 복사됨[^11]
- 과거 버그로 인해 잘못된 값을 가진 평가를 원본으로 복제하면 동일 문제가 재발할 수 있음
- 등급을 단일 단계에서만 산출하는 경우 UI에 "등급 산출" 메뉴가 표시되지 않아 사용자가 직접 수정할 수 없음[^10]

## 다음에 같은 문의가 오면

1. **먼저 확인**: Sentry/OpenSearch에서 `EvaluationValidationException` + `DraftEvaluationStageValidator.assertToComplete` 확인. 해당 evaluation ID의 `draft_evaluation.grades_to_calculate` 값 확인
2. **원인 판별**:
   - `grades_to_calculate`가 비어있고(`[]`) `enabledGradeFactors`에 값이 있으면 → 등급 산출 설정 누락 (이 이슈와 동일)
   - 고객에게 해당 평가가 기존 평가의 복제본인지 확인
3. **조치**: 고객에게 원하는 등급 산출 방식 확인 후, `draft_evaluation` 테이블의 `grades_to_calculate`와 `factor_grade_calculations`를 DML로 보정
