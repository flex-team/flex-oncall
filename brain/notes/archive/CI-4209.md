## 원인 분석

**원인 확정**: 교대근무 엑셀에 "일근제"(WorkForm — 일반 근무유형, 예: 일근제/재택)를 입력한 날짜가 shift schedule의 WorkPlanTemplate(교대근무 전용 스케줄 템플릿, 예: 조근/후근) 매칭에서 실패하여 draft가 빈 상태로 저장되고, publish 시 "변경 없음"으로 판정되어 해당 날짜만 게시에서 누락됨.

> 💡 **판단 근거**: 엑셀 분석에서 3명만 교대근무(조근/후근)+일근제 혼합 패턴 확인[^4] → 코드에서 `workPlanTemplatesTable`이 교대근무 전용 WorkPlanTemplate만 포함하는 구조 확인[^5] → DB에서 draft 전체 존재(93행)하나 publish block은 87행만 생성(6개 날짜 누락) 확인[^6]

### 조사 과정

<details>
<summary>5 Whys 체인</summary>

1. **왜 특정 날짜가 게시되지 않았나?** → publish의 `changedDates`(draft와 기존 published를 비교해 변경된 날짜만 추린 목록)에서 제외됨[^7]
2. **왜 "변경 없음"으로 판단됐나?** → 해당 날짜의 draft `work_plan=[]`(빈 배열)이라 기존 published와 동일[^6]
3. **왜 draft `work_plan`이 비어있나?** → 엑셀 업로드 시 "일근제" 셀이 `isInvalidSchedule=true`(셀 값이 WorkPlanTemplate 목록에 없어 유효하지 않은 스케줄로 마킹하는 플래그)로 마킹되어 `toDraftRegisterModel()`에서 `filterNot { it.isInvalidSchedule }` 처리됨[^8]
4. **왜 "일근제"가 invalid인가?** → v3 shift schedule 엑셀 템플릿의 `workPlanTemplatesTable`에는 교대근무 전용 WorkPlanTemplate만 포함. "일근제"는 WorkForm(근무유형)이지 WorkPlanTemplate이 아님[^5]
5. **왜 오류 메시지가 없었나?** → `isInvalidSchedule`은 셀 단위로 `validationSuccess=false`를 마킹하지만, 행 단위 `validationSuccess`는 유지됨[^9]. dry-run에서 `UNKNOWN_SCHEDULE` 경고가 표시되지만 진행 가능

</details>

### 스펙 vs 버그 판별

**버그**. 교대근무 엑셀 업로드에서 "일근제" 같은 일반 WorkForm을 사용할 수 없다는 것은 UI 차원에서 명확히 안내되지 않으며, dry-run 경고를 무시하고 진행할 수 있는 구조임. 또한 validation 실패한 셀이 draft에 빈 배열로 저장되어 publish에서 "변경 없음"으로 처리되는 것은 사용자 기대와 다른 동작.

### 사이드이펙트 포인트

| # | 수정 포인트 | 사이드이펙트 위험 | 비고 |
|---|-----------|----------------|------|
| 1 | `ShiftScheduleExcelReader`에서 WorkForm fallback 매칭 추가 | 기존 교대근무 전용 매칭 로직에 영향 가능. WorkForm과 WorkPlanTemplate 이름 충돌 시 우선순위 결정 필요 | 영향 범위: v3 엑셀 업로드 경로 전체 |
| 2 | `isInvalidSchedule` 셀의 dry-run 경고 강화 (진행 차단 또는 명확한 안내) | UI 변경 필요. 기존에 경고 무시하고 진행하던 사용자 플로우가 달라짐 | 프론트엔드 협의 필요 |
| 3 | draft 저장 시 `filterNot { it.isInvalidSchedule }` 로직 변경 | 빈 배열 대신 기존 published 값 유지하도록 변경 시, 사용자가 의도적으로 비운 경우와 구분 불가 | 가장 신중한 검토 필요 |

## 해결안 / 조사 방향

### 즉시 대응 (표면 원인 해결)
- **고객 안내**: 교대근무 엑셀 템플릿에는 교대근무 전용 스케줄(조근/후근 등)만 입력 가능하며, "일근제" 같은 일반 근무유형은 별도로 배치해야 함
- 누락된 날짜는 관리자가 이미 수동 재게시하여 현재는 해결된 상태[^1]

### 근본 해결 (재발 방지)
1. **엑셀 업로드 시 UNKNOWN_SCHEDULE 경고 강화**: dry-run 단계에서 인식 불가한 스케줄이 있을 때 사용자에게 명확히 표시하고, 해당 날짜가 빈 상태로 저장될 것임을 안내
2. **일반 WorkForm 지원 검토**: 교대근무 엑셀에서도 일근제 등 일반 근무유형을 입력할 수 있도록 WorkForm 매칭 fallback 추가

## 발견한 스펙/제약
- v3 교대근무 엑셀 업로드의 `workPlanTemplatesTable`은 엑셀 2번째 시트에 기록된 WorkPlanTemplate만 포함. WorkForm(근무유형)은 별도 체계이며 매칭 대상이 아님.
- `isInvalidSchedule`인 셀은 셀 단위로 `validationSuccess=false`가 되지만, 행 전체의 `validationSuccess`에는 영향을 주지 않음. 행은 저장되되 해당 셀의 스케줄만 누락.
- publish는 `changedDates` 비교 기반으로, draft를 기존 published에 merge한 결과가 동일하면 해당 날짜를 skip함.
