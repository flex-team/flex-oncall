# CI-4045: 직무 대량 삭제 후 저장 시 삭제한 직무가 복구됨

> **상태**: Done — 2026-03-06 (KST)
> **verdict**: bug — 분산락 반납 실패로 인한 직무 삭제 롤백 + 후속 요청 차단

## 증상

- **문제 정의**: 490개 직무 중 400개를 삭제하고 저장 버튼을 눌렀으나 응답이 오래 걸림. 저장된 것 같아 화면을 나갔다가 다시 직무 탭에 진입하면 삭제한 직무가 복구되어 있음. 이후 하나씩 삭제 시도해도 "예상하지 못한 오류" 발생.

## 원인 분석

- **1차 요청**: 대량 삭제 요청이 약 320초 소요. 도중에 타임아웃으로 인한 thread interrupt 발생
  - Kibana 로그: `flex-app.be-api-2026.03.05` (traceId: `cb81b8a514e53389495ee7a166ae5414`)
- **분산락 반납 실패**: interrupt 발생 시 after complete / lock release 처리 과정에서 오류 발생. 분산락이 제대로 반납되지 않음 (TTL 10분 설정)
- **2차 요청**: 사용자가 멈춘 줄 알고 재시도 → Redis 분산락 획득 실패로 500 에러 반환
- **근본 원인**: 발령(appointment) 로직에 분산락이 트랜잭션 안에서 여러 개 중첩 호출되며, exception에 따라 분산락 반납이 제대로 이루어지지 않는 사례 존재
  - thread interrupt + 내부 여러 겹의 분산락 반납 과정에서 `UnexpectedRollbackException`으로 변경됨
  - 409성 exception이 전파 과정에서 변질
- 개별 직무에 대한 발령 히스토리가 매우 많은 오래된 고객사

## 해결

1. **분산락 TTL 만료 대기**: 최초 실패 후 10분 경과하면 분산락 자동 반납
2. **개별 삭제 우회**: "마케터" 직무에 연결된 유저가 모두 퇴직 유저임을 확인 → 발령으로 직무를 먼저 떼고 삭제하는 방식 안내
3. **"디자이너" 직무 삭제 성공**: 연결된 유저가 없는 직무는 정상 삭제됨
4. **"마케터" 직무 보정**: 퇴직 유저에 연결된 직무 → 직무를 떼는 발령 후 삭제 성공
