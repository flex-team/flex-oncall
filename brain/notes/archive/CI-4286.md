# CI-4286: 퇴사자 미승인건 강제 승인 처리

## 증상
- **문제 정의**: 퇴사자가 최종 승인자로 설정된 워크플로우 문서(지출결의서)가 승인 불가 상태
- **회사**: 주식회사 플러그링크 (Customer ID[^1]: 61839)
- **요청자**: soojung.lee@pluglink.kr[^2]
- **퇴직 승인자**: jiyeon.lee@pluglink.kr (user_id: 279805)[^2]
- **문서번호**: 2023-647 (task_key: `294d2b8ed0594b17b94a0424e0d25edc`)[^3]
- **문서 유형**: 지출결의서[^4]
- **문제 시점**: 2023-03-31 작성, 2026-04-01 강제 승인 요청 인입

## 원인 분석

승인라인 3단계 중 step 2(최종)의 승인자가 퇴사하여 승인 진행 불가[^5].

| Step | Status | 승인자 (user_id) | 비고 |
|------|--------|-----------------|------|
| 0 | APPROVED | 260220 | 2023-03-31 승인 |
| 1 | APPROVED | 306689 | 2025-11-05 승인 |
| **2** | **PENDING** | **279805** | **퇴사자 — 미승인** |

개인 승인라인 (USER 타입) — 조직 승인라인 아님[^5]. CI-4228, CI-4266과 동일 패턴[^6].

## 해결

### 1단계: 강제 승인 (bulk-approve-for-user)

```
POST /api/operation/v2/approval/process/customers/61839/users/279805/bulk-approve-for-user
Body: { "categories": ["APPROVAL_DOCUMENT"] }
→ succeededProcesses: 1건, failedProcesses: 0건
```

### 2단계: workflow 상태 동기화 (sync-with-approval)

`bulk-approve-for-user`로 approval_process는 APPROVED가 되었으나 workflow_task는 IN_PROGRESS로 잔류[^7]. sync-with-approval API로 상태 동기화 수행.

```
POST /api/v2/operation/workflow/customers/61839/users/419359/approval-document/{task_key}/sync-with-approval
subject: APPROVAL_DOCUMENT_MODIFIED_BY_OPERATION
```

| 항목 | Before | After |
|------|--------|-------|
| approval_process | ONGOING | **APPROVED** |
| workflow_task | IN_PROGRESS | **DONE** |

## 다음에 같은 문의가 오면

1. **먼저 확인**: `workflow_task`에서 문서번호로 `task_key` 조회, `approval_process`에서 `target_uid`로 ONGOING 건 확인, `approval_line_actor`에서 PENDING 승인자 user_id 확인
2. **원인 판별**: 승인자가 퇴사자/삭제 → `bulk-approve-for-user` 대상. 조직 승인라인이면 `act-approval-process` 문서별 개별 처리 (CI-4244 참조)
3. **조치**:
   - `bulk-approve-for-user` API 호출 (categories에 `APPROVAL_DOCUMENT` 포함)
   - 승인 후 `workflow_task`가 IN_PROGRESS로 잔류하면 `sync-with-approval` API 추가 호출 필수

### 재발 방지 / 향후 참고
- `bulk-approve-for-user` 후 `workflow_task`가 자동 동기화되지 않는 경우 `sync-with-approval` 필수[^7]
- `produce-approval-process-event` API는 이미 APPROVED 상태에서 500 에러 — `sync-with-approval`이 정확한 해결 방법[^8]
- categories에 `APPROVAL_DOCUMENT`도 포함해야 함 (기존 가이드는 TIME_OFF/WORK_RECORD만 언급)[^9]
