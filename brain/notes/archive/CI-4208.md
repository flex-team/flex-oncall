- 라벨: 구성원/문서/증명서[^1]

## 원인 분석

**이슈는 2건으로 분리됨:**[^9]

### (a) 발급번호/관리번호 자동채번 — 미구현 기능 (스펙)

발급번호/관리번호 자동채번은 현재 미구현 기능이다. 데이터 모델에 필드는 정의되어 있으나, 값을 생성하거나 입력받는 경로가 존재하지 않는다[^3][^10][^11].

> 💡 **판단 근거**: API 요청 DTO에 필드 없음[^10] → 서비스 레이어 기본값 `null`[^11] → 렌더러에서 빈 값 출력

- 김주원 확인: "발급번호 속성에 대한 setter가 어디에도 없습니다. 속성만 있을 뿐, 사용자로부터 값을 받거나 시스템에서 자동으로 값을 채워주는 기능은 애초에 없었던 것으로 보입니다."[^12]
- "소득자 본인이 직접 증명서를 받는데 발급번호를 넣는 것은 어색하긴 합니다. 관리자가 받을 때는 번호를 넣어 줄 수도 있을 것 같네요."[^12]
- 2개 오퍼레이션 존재: `downloadWholeWithholdingReceiptClassAPdfByMe` (본인), `downloadWholeWithholdingReceiptClassAPdfByUserId` (관리자)[^12]

### (b) 양식 서식 불일치 — 버그

원본 법정 양식(1995년 최종판)에서 우측 상단은 **"처리기간"**인데, flex에서는 **"관리번호"**로 잘못 표기됨[^2][^6][^7].

> 💡 **판단 근거**: 2022년 FE PR에서 "관리번호"라고 지정[^8] → 법정 서식과 불일치 → 소득세법 시행규칙 90조 자체가 삭제되어 1995년 서식이 최신[^7]
