# CI-4283: 주법인 전자계약 서식 계열사 복제 요청

> **상태**: 해결 완료 — 2026-04-01
> **compact**: 원본 71줄 → 정제 71줄 (2026-04-02). 정제 대상 없음(단순 운영 대응). git log로 원본 추적 가능.

## 증상
- **문제 정의**: 주법인(엘케이홀딩스)의 전자계약 서식을 계열사(엔마트 세류점)로 복제 요청
- **회사**: 엔마트 세류점 (Customer ID[^5]: 227471)
- **요청자**: 박지훈 (Intercom 인입)[^1]
- **대상자**: 엔마트 세류점 관리자 (hyundal1705@naver.com)
- **영향 범위**: 1개 계열사
- **문제 시점**: 2026-04-01
- 문의 내용:
  1. 주법인(엘케이홀딩스, customer_id=208733)의 전자계약 서식[^7] 전체를 계열사(엔마트 세류점, customer_id=227471)에 복사[^2]
  2. 계약서 제목 그대로 복사[^2]
  3. 복사 받은 회사(엔마트 세류점)의 권한자가 계약서 편집 가능하도록 확인[^2]

## 원인 분석

### 조사 과정
> 💡 이한결님이 `duplicateTemplates` 오퍼레이션 존재를 안내[^8]
> → raccoon prod Swagger UI에서 `POST /api/operation/v2/digicon/duplicate-templates` 확인[^9]
> → 코드 분석: 원본 고객사의 placeholder + template을 대상 고객사에 복제하는 로직[^9]

### 원인
- 계열사 간 전자계약 서식 공유 기능이 제품에 없어 Operation API를 통한 수동 복제가 필요[^3]

## 해결
- raccoon prod에서 `duplicateTemplates` 오퍼레이션 실행[^10]
  - `originalCustomerId`: 208733 (엘케이홀딩스)
  - `targetCustomerIds`: [227471] (엔마트 세류점)
  - `postfix`: "" (제목 그대로 복사)
- 응답: `200 OK`, 처리 시간 366ms[^10]

## 다음에 같은 문의가 오면
1. **먼저 확인**: 주법인 customer_id와 복제 대상 계열사 customer_id 목록을 파악
2. **조치**: raccoon prod Swagger → digicon → `POST /api/operation/v2/digicon/duplicate-templates` 실행
   ```json
   {
     "originalCustomerId": {주법인 customer_id},
     "targetCustomerIds": [{대상 계열사 customer_id 목록}],
     "postfix": ""
   }
   ```
3. **주의**: postfix를 빈 문자열로 설정하면 원본 제목 그대로 복사. 구분이 필요하면 "_복사본" 등 접미사 지정

## 발견한 스펙/제약
- 계열사 간 전자계약 서식 공유/복제 기능이 제품에 없어 Operation API(`duplicateTemplates`)로 수동 처리 필요[^8]
- 이한결: 이 오퍼레이션은 비엔지니어도 가능한 수준(customerId 입력이 전부)이므로 어드민 셸에 메뉴를 뚫어도 됨[^11]
