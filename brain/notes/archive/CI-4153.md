# CI-4153: 구성원 페이지 오류창 지속 노출

> **상태**: 롤백으로 대응 중 — 2026-03-18

## 증상
- **문제 정의**: core v3.93.0 권한세분화 배포 이후, 구성원 페이지(`/people`)에서 오류창이 지속적으로 노출됨
- **회사**: (주)미리디 (Customer ID: 210542)
- **요청자**: CS팀 @권오은
- **대상자**: easyny@miridih.com
- **영향 범위**: 최소 1명 확인. 이성환님에 따르면 기존에도 403이 뜨는 경우가 있었으나 빈도가 특별히 달라지지는 않았다고 함[^6]
- **문제 시점**: 2026-03-18 (core v3.93.0 배포 이후)
- 문의 내용:
  1. 구성원 페이지에서 오류창이 지속적으로 노출됨
  2. 캐시 삭제 후 재부팅하면 순간적으로 정상 노출되었다가 다시 오류페이지 노출[^7]

## 원인 분석

**core v3.93.0 권한세분화 릴리즈가 원인으로 추정된다.**[^8]

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | core v3.93.0 권한세분화 배포로 `/people` 페이지의 권한 체크 로직이 변경되어 기존 사용자가 권한없음으로 처리됨 | 배포 전후 비교, 롤백 후 정상화 여부 확인 | ✅ 확정 (롤백 진행) |

### 조사 과정

> 💡 **판단 근거**: Slack 스레드에서 지무근님이 "이도 권한세분화 배포 하셨었군요"라며 배포 스레드를 연결[^8] → 임준규님(배포 담당)이 "일단 롤백 쳐봅니다"로 즉시 대응[^9]

**경위:**
1. 임준규님이 core v3.93.0(권한세분화 릴리즈) 배포[^10]
2. 배포 직후 easyny@miridih.com 사용자에게 `/people` 페이지 오류 발생
3. Sentry에 `/people` 404 로그 기록됨[^1]
4. 이성환님이 해당 유저에게 "권한없음"이 반복 표시되는 것 확인[^2]
5. 임준규님이 롤백 진행[^9]

- userId를 DB에서 조회하려 했으나 `flex_core.user` 테이블이 허용 목록에 없어 미수행
