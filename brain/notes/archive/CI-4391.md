# CI-4391: 근무 승인 요청했는데 1단계 진행상태가 '대기중'으로 뜨는 원인 확인 요청

## 증상
- **문제 정의**: 근무 승인을 요청했으나 1단계 승인권자에게 전달되지 않고, 승인 상태가 "대기중"으로 유지됨
- **회사**: 주식회사 디지털아이덴티티솔루션코리아 (Customer ID: 198613)[^1]
- **요청자**: sa.oh@disk-global.com (문의자 겸 대상자)[^1]
- **대상자**: sa.oh@disk-global.com (user_id: 745078) — 4월 9일 근무 승인 요청자[^1][^8]
- **1단계 승인권자**: l.jeon@disk-global.com (user_id: 807200)[^9]
- **영향 범위**: 회사 198613에서 WORK_RECORD ONGOING+PENDING 건 **6건 / 4명**[^5]
- **문제 시점**: 2026-04-08 ~ 04-09[^5]
- 문의 내용:
  1. sa.oh 구성원이 4월 9일 근무에 대해 승인을 올림[^1]
  2. 1단계 승인권자(l.jeon)에게 요청이 전달되지 않음[^1]
  3. 승인 상태가 "대기중"으로 표시되며 "1단계 승인 진행중"으로 전환되지 않음[^1]

## 원인 분석

### 조사 과정

> 💡 **판단 근거**: approval_process → approval_line → approval_line_actor_resolved_user → cloud_event_entity → workflow_task 순으로 데이터 대조
> → approval 시스템(프로세스·승인라인·resolve) 모두 정상[^5][^9]
> → cloud event 정상 발행됨 (sa.oh 건)[^6]
> → WORK_RECORD는 workflow_task를 사용하지 않음[^10]
> → 이벤트 소비자(time-tracking 측) 처리 실패로 판단

### 원인

**sa.oh(745078) 건**: approval 시스템 정상 — 이벤트 발행까지 성공했으나, **이벤트 소비자(time-tracking)가 처리에 실패**하여 UI에 "대기중" 표시[^5][^6][^9][^10]

## 영향 범위

- 회사 198613 내 WORK_RECORD 6건 / 4명[^5]
- 금전적·법적 영향: 근무 승인 지연으로 근태 처리에 영향 가능

## 해결

- `produce-event-to-operation-topic` Operation API로 6건 일괄 보정[^7][^11]
  - `POST /action/operation/v2/approval/process/produce-event-to-operation-topic`
  - Body: `customerIdRange=198613~198613, category=WORK_RECORD, chunkSize=100`
  - 응답: 200 OK
- ONGOING 건 전체에 `APPROVAL_PROCESS_SYNC` 이벤트 재발행 완료[^11]

> ⚠️ 보정 후 고객이 여전히 "대기중"으로 보인다고 재보고 (유지원, 04-10 13:24)[^12]. 추가 확인 필요.

## 발견한 스펙/제약

- **WORK_RECORD 카테고리는 `flex.workflow_task`를 사용하지 않음** — 근무 승인 상태 표시는 time-tracking 이벤트 소비자가 직접 관리[^10]
- `produce-event-to-operation-topic`은 별도 operation 토픽(`event.flex.approval.process.operation.v1`)으로 발행 — 원본 created 이벤트와 토픽이 다름[^7]

## 다음에 같은 문의가 오면

1. **먼저 확인**: `approval_process` + `approval_line` 상태 확인 (ONGOING + step 0 PENDING이면 승인권자 미행동)
2. **이벤트 확인**: `cloud_event_entity`에서 해당 건의 created 이벤트 존재 여부 확인 (extensions.flexcustomerid + time 범위)
3. **원인 판별**:
   - 이벤트 없음 → 2PC Phase 2 실패. `produce-event-to-operation-topic` API로 재발행
   - 이벤트 있음 + UI "대기중" → 소비자 처리 실패. 동일 API로 sync 이벤트 재발행
4. **보정**: `POST /action/operation/v2/approval/process/produce-event-to-operation-topic` — customerIdRange, category, chunkSize 지정
