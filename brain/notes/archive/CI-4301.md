### 코드 분석

`evaluationReviewers is empty` 로그는 `EvaluationReviewerCommandService.kt`의 `resetAll()` 메서드에서 발생[^5]:

```kotlin
val evaluationReviewers = evaluationReviewerRepository
    .findAllByCustomerIdentityAndEvaluationIdentityAndRevieweeIn(...)
    .filterNot { it.writingStatus(now) == EvaluationReviewerWritingStatus.BEFORE_REQUESTED }

if (evaluationReviewers.isEmpty()) {
    logger.info { "evaluationReviewers is empty..." }
    return
}
```

비어있게 되는 조건[^5]:
1. DB에서 조회되는 레코드가 없는 경우
2. 모든 레코드의 `writingStatus(now)`가 `BEFORE_REQUESTED` — 즉 `reviewPeriod`이 NULL인 상태

## 발견한 스펙/제약
- 일괄 설정(finalize[^7]) 이후 추가된 reviewer는 UserForm[^8] 초기화를 메시지 큐에 의존[^6]
- 큐 처리 실패 시 lazy initialization(reviewer 직접 접근)이 fallback이지만, admin 화면 즉시 정상화에는 Operation API 필요[^6]
