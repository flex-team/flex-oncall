# CI-4291: 구독 플랜 추가 후에도 무료체험 종료 화면 표시

> **상태**: 해결 완료 — 2026-04-01 | **판정**: 스펙

## 증상
- **문제 정의**: 구독 플랜을 추가했으나 로그인 시 무료체험 종료 화면이 표시됨

- **요청자**: CS팀 hanna.kim[^1]
- **대상자**: tes@braincommerce.com (문의자 이메일)[^1]
- **영향 범위**: 해당 회사 전체
- **문제 시점**: 2026-04-01
- 문의 내용:
  1. 계열사로 계정을 생성했으나 카드 등록 없이 무료체험 기간(3/31)이 종료됨[^1]
  2. CS에서 별도로 구독 플랜을 추가해줌[^1]
  3. 라쿤[^2]에서는 구독이 반영되어 있으나, 로그인 시 여전히 무료체험 종료 화면이 표시됨[^1]

## 원인 분석

무료체험 종료일(3/31)이 지나고 카드가 미등록인 상태에서, 구독 플랜만 추가해도 접속 차단은 해제되지 않는다 — **스펙**[^3].

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | 카드 미등록으로 billing 차단 유지, 플랜 추가만으로는 차단 해제 불가 | 코드 분석 + 권민철님 확인 | ✅ 확정 |

### 조사 과정

> 💡 **판단 근거**: `isShutdownActivated()` 메서드의 조건 순서 확인[^3]
> → 카드 null 체크(라인 151)가 활성 피처 체크(라인 155)보다 **먼저** 실행됨
> → 카드가 없으면 구독 플랜 유무와 무관하게 `return true` (차단)
> → 권민철님이 무료체험 종료일이 3/31로 되어 있음을 확인[^4]

<details>
<summary>isShutdownActivated() 판단 로직 상세</summary>

```kotlin
// CustomerBillingStatusLookUpServiceImpl.kt:117-170
private fun isShutdownActivated(...): Boolean {
    if (currentBillingRule == null) return true           // 1. billingRule 없음
    if (isBlocked) return true                            // 2. 차단 상태
    if (!shutdownOnTrialExpiration) return false           // 3. 차단 비활성화 (force-open)
    if (baseDate <= trialExpireDate) return false          // 4. 무료체험 기간 내
    if (currentDefaultCreditCard == null) return true      // 5. 카드 없음 → 즉시 차단
    return !hasAnyActiveFeature                            // 6. 활성 피처 확인
}
```

이 이슈의 경우: 무료체험 종료(조건 4 통과) → 카드 없음(조건 5에서 `return true`) → 피처 체크(조건 6)에 도달하지 않음.

`force-open`은 `shutdownOnTrialExpiration = false`로 설정하여 조건 3에서 조기 반환(`return false`)시킨다.

</details>

## 해결

권민철님 안내에 따라 CS팀(hanna.kim)이 직접 조치[^5]:
1. 라쿤에서 무료체험 종료일을 오늘(4/1)로 변경하여 접속 허용
2. 고객이 로그인 → 카드 등록 → 결제 생성

## 다음에 같은 문의가 오면

1. **라쿤 확인**: 빠른 회사 검색 > 상세 정보 > 무료체험 종료일 + 카드 등록 여부 확인[^4]
2. **원인 판별**:
   - 무료체험 종료일이 지났는가?
   - 카드가 등록되어 있는가?
3. **조치 분기**:
   - 무료체험 종료 + 카드 미등록 → 무료체험 종료일을 오늘 이후로 변경 → 고객에게 카드 등록 안내
   - 카드 등록됨 + 접속 불가 → `force-open`[^6]으로 임시 접근 허용 → 카드 재등록/결제 확인 → `close-forced-open`
   - 구독 플랜만 추가하면 접속이 될 것이라는 기대는 잘못됨 — **카드 등록 또는 무료체험 종료일 연장이 필수**
