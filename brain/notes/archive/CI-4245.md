# CI-4245: 구성원 직책 2개 설정 불가

> **상태**: 해결 완료 (스펙 확인) — 2026-03-30

## 증상
- **문제 정의**: 같은 조직에 대한 겸직[^12](직책 2개) 등록이 엑셀 일괄등록과 프로필 UI 모두에서 불가
- **회사**: 주식회사 너바나나 (Customer ID: 226325)
- **요청자**: jun@nirvanana.net[^1]
- **대상자**: 기획그룹 소속 특정 구성원 (이름 미기재)[^1]
- **영향 범위**: 해당 구성원 1명
- **문제 시점**: 2026-03-27[^1]
- 문의 내용[^1]:
  1. 특정 구성원이 기획그룹에 속해 있으며 직책이 2개인 상황
  2. 엑셀 일괄등록으로 겸조직+겸직책 설정 시 "겸조직이 동일 조직" 오류로 업로드 불가[^4]
  3. 프로필 UI에서 같은 조직에 겸직책을 반영하면 인사정보에 겸조직/겸직책 모두 노출 안됨[^4]
  4. 조직 없이 겸직책만 반영하려 해도 조직 없음으로 인해 겸직책 반영 불가[^6]

## 원인 분석

### 조사 과정
> 💡 김주원이 dev 환경에서 재현[^3] → 같은 조직 겸직 등록 시 재조회에서 두 번째 조직/직책 미노출 확인
> → 김선미가 엑셀/프로필 두 경로의 증상 정리[^4]
> → 코어 담당자(@junyoung, @yoonsun)에게 에스컬레이션[^5]
> → **이윤선 확인: 같은 조직 내 직책 겸직 발령 불가는 현재 스펙**[^11]

### 원인
- 직책은 조직과 결합된 정보로 취급되므로, **같은 조직에 대한 겸직 발령은 의도적으로 불가**하도록 설계됨[^11]
- 엑셀 일괄등록: 겸조직이 주조직과 동일하면 서버 측 validation에서 차단[^4]
- 프로필 UI: 저장은 되지만 조회 시 같은 조직의 겸직 정보가 필터링됨[^3][^4]

### 두 가지 등록 경로의 증상 정리[^4]

| 경로 | 증상 | 비고 |
|------|------|------|
| 엑셀 일괄등록 | 겸조직이 동일 조직이면 validation 오류 발생 → 업로드 차단 | 서버 측 검증 — 스펙 |
| 프로필 UI 직접 입력 | 저장은 되지만 재조회 시 겸조직/겸직책 미노출 | 스펙이나, 저장 허용 후 미노출은 UX 개선 여지 있음 |

## 해결
- **스펙 확인**: 같은 조직 내 직책 겸직 발령 불가는 현재 스펙[^11]
- 고객에게 같은 조직 내 직책 2개 설정은 지원하지 않음을 안내
- 다른 조직 간 겸직은 정상 지원됨

## 발견한 스펙/제약
- 직책은 조직과 결합된 정보이므로, 겸직은 반드시 **다른 조직** 간에만 가능[^11]
- 프로필 UI에서는 같은 조직 겸직을 저장 자체는 허용하나 조회에서 필터링됨 — 엑셀과 동작 불일치[^4]

## 다음에 같은 문의가 오면
1. **먼저 확인**: 겸직 등록하려는 조직이 주조직과 **동일한지** 확인
2. **동일 조직이면**: 같은 조직 내 직책 겸직은 스펙상 불가 → 고객에게 안내[^11]
3. **다른 조직이면**: 겸직 정상 동작해야 함 → 별도 조사 필요

## 시나리오

```gherkin
# language: ko
기능: 구성원 겸직 발령

  시나리오: 같은 조직에 겸직 발령 시도 — 엑셀 일괄등록
    주어진 구성원 A가 기획그룹에 주직으로 소속되어 있다
    만약 관리자가 엑셀 일괄등록으로 기획그룹에 겸직책을 추가하려 한다
    그러면 "겸조직이 동일 조직" validation 오류가 발생한다
    그리고 업로드가 차단된다

  시나리오: 같은 조직에 겸직 발령 시도 — 프로필 UI
    주어진 구성원 A가 기획그룹에 주직으로 소속되어 있다
    만약 관리자가 프로필 UI에서 기획그룹에 겸직책을 추가하여 저장한다
    그러면 저장은 성공하지만 인사정보 재조회 시 겸조직과 겸직책이 노출되지 않는다

  시나리오: 다른 조직 간 겸직 발령
    주어진 구성원 A가 기획그룹에 주직으로 소속되어 있다
    만약 관리자가 디자인그룹에 겸직 발령을 등록한다
    그러면 기획그룹 주직과 디자인그룹 겸직이 모두 정상 표시된다
```
