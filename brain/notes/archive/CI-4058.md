# CI-4058: 영문 버전 품의서 결재란 이름 검색 시 문서 전체 삭제됨

> **상태**: Done — 2026-03-09 (KST)
> **verdict**: not-a-bug — 크롬 내장 번역기(구글 번역)가 DOM을 변경하여 UI가 깨지는 외부 요인

## 증상

- **문제 정의**: flex 영문 버전에서 품의서 작성 중 결재란에 이름을 검색하면 해당 문서가 임시저장 없이 완전 삭제됨. 시크릿 모드, 다른 브라우저에서도 동일 재현된다고 보고됨.

- **담당자**: 김종현

## 원인 분석

- 처음에 문의자(hcko)와 실제 문제를 겪은 사용자(graysun)가 다른 점이 확인됨 — 로그에서 문의자의 에러를 찾을 수 없었던 이유
- **근본 원인**: 크롬 내장 구글 번역 기능이 켜져 있는 상태에서 구성원 검색을 하면, 구글 번역이 DOM을 변경하면서 에러를 발생시켜 UI가 깨지는 현상
- 구글 번역을 끄면 문제 재현 안 됨
- 고객이 구글 번역을 사용한 이유: flex 영문 버전을 쓰고 있지만, 워크플로우 문서 내용 요약이 한글로 되어 있어 번역기 추가 사용

## 해결

- **수정 불가**: 크롬 구글 번역 측의 문제로 flex 측에서 수정 어려움
- 고객에게 상황에 따라 구글 번역 on/off 하여 사용하도록 안내 완료
- flex 자체 영문 번역 기능에는 문제 없음 확인
