# CI-4008: 출장 근무 승인이 발생하지 않은 원인 확인

> **상태**: Done — 2026-03-04 (KST)
> **verdict**: not-a-bug — 출장 근무 등록 자체가 되지 않았던 것으로, 문의자의 오해

## 증상

- **문제 정의**: 출장 근무 승인 설정이 되어 있으나, 12월 8일과 2월 11일 출장근무 승인이 발생하지 않았다고 원인 확인 요청. 관리자가 등록하지 않았고 변경허용 옵션도 꺼져 있다고 함.
- **회사**: 주식회사 에스엘즈 (Customer ID: 34792)
- **요청자**: nari.jeon@slz.kr
- **이슈 대상자**: kisu.lee@slz.kr
- **라벨**: 근태/근무/휴가
- **담당자**: 미배정
- **Linear**: https://linear.app/flexteam/issue/CI-4008

## 원인 분석

- 진재언이 2월 11일 근무기록을 확인한 결과, **위젯으로 등록된 기록만 존재**하고 출장근무로 등록된 기록은 없었음
- 스크린샷과 실제 데이터가 서로 달랐음 (문의자가 제공한 스크린샷의 근무기록이 해당 이메일 계정과 불일치)
- 12월 8일 건은 약 3개월 전이라 데이터 확인 어려움
- 문의자가 관리자임에도 상황을 명확히 파악하지 못한 채 문의한 것으로 확인됨

## 해결

- 문의가 Intercom에서 종결 처리됨
- 문의자(유지원) 측에서 "관리자도 명확하지 않은 채로 문의하신 것이라서 확인 안 해주셔도 된다"고 연락

## 비고

- Slack: https://flex-cv82520.slack.com/archives/CRU35U9FC/p1772516281863619
- 출장 근무와 위젯 출퇴근 등록은 별개 경로이므로, 출장근무 승인이 발생하려면 출장근무로 등록해야 한다는 점이 핵심
