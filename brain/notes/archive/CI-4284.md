# CI-4284: 목표 엑셀 업로드 시 담당 주체 조직명 매칭 실패

> **상태**: 해결 완료 (Done) — 2026-04-02
> **verdict**: spec — 조직은 시계열 데이터, 목표 시작일 시점에 존재하는 조직만 매칭

## 증상
- **문제 정의**: 목표 엑셀 템플릿으로 업로드 시 담당 주체 조직명이 "잘못 입력했다"는 오류로 거부됨
- **회사**: 디스페이스코리아 (Customer ID: 178246)
- **요청자**: 이주화 (CS팀)[^1]
- **대상자**: hyoon@dspace.kr (디스페이스코리아 관리자)[^1]
- **영향 범위**: 해당 회사의 목표 시작일이 조직 생성일보다 이전인 모든 목표 엑셀 업로드
- **문제 시점**: 2026-04-01
- 문의 내용:
  1. 엑셀 템플릿 다운로드 후 양식에 맞춰 업로드 시 E8~E13행의 담당 주체 조직명이 "잘못 입력했어요" 오류로 거부됨[^1]
  2. 이미 존재하는 조직이고, 조직명을 정확히 입력했으며, 제품 내 조직명을 복사·붙여넣기해도 동일 오류 발생[^1]
  3. 조직장의 사번도 정상 확인됨[^2]

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 김보라가 엑셀 업로드 로직에서 인사기준일 기반 조직 조회 제약을 발견[^3]
> → 엑셀 업로드 시 담당 주체를 `name`(조직명)을 key로 조회하는데, 내부적으로 목표 시작일을 인사기준일로 사용하여 해당 시점의 조직을 조회[^3]
> → 디스페이스코리아의 해당 조직들은 2026-02-01에 생성되었으나, 목표 시작일이 그보다 이전으로 설정됨[^4]
> → 목표 시작일 시점에 해당 조직이 존재하지 않으므로 매칭 실패 → "조직명을 잘못 입력했다" 오류 표시

### 원인
- 엑셀 업로드 시 부서명(`name`)을 key로 조직을 조회하는데, 조직은 시계열 데이터[^5]이므로 목표 시작일 기준으로 해당 시점에 존재하는 조직만 매칭됨[^6]
- 목표 시작일이 조직 생성일보다 이전이면 해당 시점에 조직이 존재하지 않아 매칭 실패[^3]
- 오류 메시지가 "조직명을 잘못 입력했다"로만 표시되어, 고객 입장에서 원인을 알 수 없음[^7]

## 해결
- **임시 해결**: 고객사에 목표 시작일을 조직 생성일(2026-02-01)과 동일하게 설정 후 업로드하도록 안내. 엑셀 업로드 후에는 목표 시작일 변경 가능[^3]
- 이주화가 실제로 시작일 조정 후 업로드 성공 확인[^4]
- **근본 개선 티켓 생성**:
  - [EPBE-317](https://linear.app/flexteam/issue/EPBE-317/): 엑셀 업로드 시 부서 name 매칭 개선[^8]
    - a) 일치하는 부서 1개 → 해당 부서 사용
    - b) 일치하는 것 없음 → 현재 validation 유지
    - c) 일치하는 것 2개 이상 → 인사정보 기준일을 목표 시작일로 조회
  - [EPBE-318](https://linear.app/flexteam/issue/EPBE-318/): 엑셀 업로드 시 validation message 정의[^8]
  - 두 티켓 모두 `quick-pick` 라벨, backlog 상태

## 발견한 스펙/제약
- 조직은 시계열 데이터이므로, 엑셀 업로드 시 목표 시작일 기준으로 존재하는 조직만 매칭 가능 — 임준규가 의도된 스펙으로 확인[^6]
- 과거 조직으로는 목표를 만들 수 없는 제약이 존재[^9]
- 임준규는 조직 코드를 제품 내에서 사용할 수 있도록 개선하는 것이 더 가성비 좋다고 판단[^9]
- 이윤주(PM)는 이 제약이 의도한 스펙이 아니라고 인식 — 담당 주체는 언제든 바꿀 수 있어야 한다고 생각했음[^10]

## 다음에 같은 문의가 오면
1. **먼저 확인**: 오류가 발생하는 조직의 **생성일**과 엑셀에 설정된 **목표 시작일**을 비교
2. **원인 판별**: 목표 시작일이 조직 생성일보다 이전이면 → 시계열 매칭 실패가 원인
3. **조치**: 목표 시작일을 조직 생성일과 동일하거나 이후로 설정 후 업로드 안내. 업로드 완료 후 목표 시작일은 자유롭게 변경 가능
