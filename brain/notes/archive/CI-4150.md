# CI-4150: 권한 부여 시점 문의

## 증상
- **문제 정의**: 특정 구성원의 최고관리자 권한 그룹 포함 시점과 수행자 확인 요청
- **회사**: 주식회사 디지털아이덴티티솔루션코리아 (Customer ID: 198613)
- **요청자**: l.jeong@disk-global.com (user 802588)
- **대상자**: ys.kim@disk-global.com (user 742691)
- **영향 범위**: 1명
- **문제 시점**: 2025년 12월 중 — 최고관리자 권한 부여/회수 이력 확인 필요[^2]
- 문의 내용:
  1. ys.kim 구성원이 언제부터 최고관리자 권한 그룹에 포함되어 있었는지[^1]
  2. 누가 해당 구성원을 최고관리자 권한 그룹에 포함시켰는지[^1]
  3. 감사로그에서 확인이 안 되는 사항이라 별도 확인 요청[^1]

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: `flex_grant_subject` 조회[^5] → 최고관리자 grant[^4]에 ys.kim 미포함, l.jeong만 존재
> → Slack에서 윤성복님이 ys.kim이 **회사 최초 유저**임을 확인[^7]
> → 회사 생성 시 최초 유저에게 최고관리자가 자동 부여되는 구조[^7]
> → l.jeong이 2026-01-08에 ys.kim의 최고관리자를 인지하고 회수한 것으로 판단[^8]

### 원인
- **ys.kim은 회사의 최초 유저**였기 때문에 회사 생성 시점부터 최고관리자 권한이 자동 부여되어 있었다[^7]
- 감사로그에 없는 이유: 회사 초기 세팅 시 자동 부여된 것이므로 별도 변경 이력이 기록되지 않음[^3]
- l.jeong이 2026-01-08에 이를 인지하고 ys.kim의 최고관리자를 회수, 동시에 본인을 최고관리자로 추가[^5]

## 해결
- **스펙 확인**: 최초 유저에게 최고관리자가 자동 부여되는 것은 정상 동작[^7]
- **스펙 확인**: 감사로그가 권한 변경을 기록하지 않는 것도 현재 설계상 의도된 동작[^3]
- 고객에게 사실관계를 안내하고 티켓 종료[^9]

## 다음에 같은 문의가 오면
1. **먼저 확인**: `flex_authorization.flex_grant_subject`에서 해당 회사의 최고관리자 grant 멤버 조회
   ```sql
   SELECT gs.subject_id, gs.created_by, gs.created_at
   FROM flex_grant_subject gs
   JOIN flex_grant g ON g.id = gs.grant_id
   WHERE gs.customer_id = ? AND g.title_key = 'authority.administrator_title';
   ```
2. **원인 판별**:
   - 대상 사용자가 grant에 있으면 → `created_at`이 포함 시점, `created_by`가 수행자
   - 대상 사용자가 grant에 없으면 → 이미 회수됨 (물리 삭제). 회사 최초 유저인지 확인 → 최초 유저라면 회사 생성 시 자동 부여된 것
3. **감사로그 관련**: 권한 변경은 감사로그(Envers) 기록 대상이 아님을 안내
