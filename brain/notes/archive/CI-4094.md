# CI-4094: 구성원 실제 근무 시작 시간과 기록 시작 시간 상이

> **상태**: 해결 완료 — 2026-03-13 (KST)
> **verdict**: bug — 클라이언트 위젯 시각 갱신 문제 + 서버 validation 미비, 핫픽스 배포 완료

## 증상

- **문제 정의**: 구성원이 플렉스 웹에서 10시 17분경 근무 시작했으나 9시 35분으로 출근 기록됨. 정시출퇴근 인정범위 설정은 '정시만 인정'.
- **회사**: 패스트레인 (Customer ID: 190921)
- **요청자**: boram.kim@fastlane.kr
- **이슈 대상자**: lucy.kim@fastlane.kr
- **라벨**: 근태/근무/휴가
- **담당자**: 장우현
- **Linear**: [CI-4094](https://linear.app/flexteam/issue/CI-4094/확인요청-구성원이-실제-근무-시작한-시간과-기록-시작된-시간-상이)

## 원인 분석

- Access log 확인 결과: **클라이언트에서 09:35 시각으로 요청을 전송**한 것이 확인됨
- 당시 설정 확인:
  - onTimeRecord: enabled=true, bufferMinutes=0, recordType=PLAN
  - 시차(flextime): 07:00 ~ 10:30
  - 기존 깔려있는 근무기록 없었음
- 이른출근허용 설정은 맞았으나, **서버 validation이 일부 케이스에서 동작하지 않는 버그** 발견
- 프론트엔드 위젯(`actorNow`)의 현재 시각이 어떤 이유로 갱신되지 않았을 가능성이 유일한 원인
  - 슬립모드/탭 전환으로 테스트해도 재현 안 됨
  - 유저 컴퓨터 시간 조작 가능성도 있으나, 출근 실패 후 재시도 시 성공한 점에서 깔끔하게 설명되지 않음

## 해결

- **서버 측**: validation 버그 핫픽스 배포 완료 (진재언, 2026-03-12)
  - 일부 케이스에서 서버가 이른 시각을 validation으로 튕겨내지 못하던 버그 수정
- **프론트엔드 측**: Sentry warning 로깅 추가 PR 생성
  - PR: https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2299
  - 위젯의 `actorNow` stale 감지를 위한 로깅으로, 향후 동일 이슈 발생 시 원인 추적 가능
- **근본 해결**: 관련 이슈([CI-4096 추정](https://flex-cv82520.slack.com/archives/CRU35U9FC/p1773369856394639)) 해결 시 출근 당시 현재 시각을 백엔드에 판단 위임하게 되어 재발 방지
