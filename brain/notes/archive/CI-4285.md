# CI-4285: 구캘 미연동 일정 재연동 요청

## 증상
- **문제 정의**: 구글 캘린더 연동 사용 중 최근부터 휴가 일정이 구글 캘린더에 동기화되지 않음
- **회사**: 주식회사 알피 (Customer ID: 189332)
- **요청자**: 이주화 (CS팀)[^1]
- **대상자**: ella80@arpi.ai[^2]
- **영향 범위**: 1명
- **문제 시점**: 최근 (구체적 시점 미확인)
- 문의 내용:
  1. 구글 캘린더 연결하여 사용 중인데 최근부터 휴가 연동이 안 되고 있음[^3]
  2. 과거 신청건은 연동이 되어 있으나, 최근 건이 미연동[^3]
  3. 미연동 일정의 재연동 요청[^1]

## 원인 분석

### 조사 과정
> 💡 Metabase 대시보드 244에서 ella80@arpi.ai의 캘린더 연동 상태 확인[^6]
> → 미연동 eventIds 41건 추출[^6]
> → CI-4262 동일 패턴 확인 — Operation API `sync-created-member-schedule-group-calendar-by-ids`로 재동기화[^7]

### 원인
- 구글 캘린더 연동 후 일부 일정이 동기화되지 않은 상태로 남음[^3]. 연동 해제/재연동 또는 동기화 타이밍 이슈로 추정.

## 해결
- Metabase 쿼리로 미연동 eventIds 41건 추출[^6]
- raccoon Operation API `sync-created-member-schedule-group-calendar-by-ids` 호출 (prod)[^8]
  - customerId: 189332, eventIds: 41건, chunkSize: 100, delayInMilli: 1000
  - 응답: **200 OK** (약 9초 소요)
  - 1회 호출로 전체 처리 완료

## 다음에 같은 문의가 오면

1. **먼저 확인**: Metabase 대시보드 `/dashboard/244?email={사용자이메일}`에서 미연동 건 확인
2. **eventIds 추출**: Metabase 쿼리 7387로 미연동 eventIds 추출
3. **조치**: raccoon Operation API로 재동기화
   - 엔드포인트: `/proxy/calendar/api/operation/v2/calendar/customers/{customerId}/google-calendar-events/sync-created-member-schedule-group-calendar-by-ids`
   - 300건 이상 시 Google Calendar API rate limit 주의 → batch 20건 + 2초 delay 권장
   - 50건 이상 시 raccoon audit log URL 길이 초과 가능 → batch 5~20건 권장
