# QNA-1922: 선택적 근무 — 실시간 휴게 기록 시 추천 휴게 자동 입력 여부

## 증상
- 문의 내용[^1]:
  1. 선택적 근무(소정근로시간 내 시작/종료를 자율 선택하는 근무 유형) 에서 "휴게 시간 입력 허용" 설정 후, 실시간 근무 기록 중 휴게 입력/종료를 하면 추천 휴게 시간(근무 유형에 미리 설정된 권장 휴게 시간대)이 자동으로 들어가지 않는다
  2. CS 담당자가 직접 테스트했을 때는 추천 휴게가 정상 입력되었으나, 실제 구성원의 기록에서는 입력되지 않았다

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 이주화가 과거 동일 맥락의 Slack 스레드를 공유[^2]
> → 해당 스레드에서 추천 휴게 시간의 동작 방식이 이미 정리되어 있었음[^3]
> → CS 담당자(김현욱)가 테스트 시 정상으로 보인 이유는 **근무 확정 시점에 추천 휴게가 자동 등록**되었기 때문
> → 실시간 기록 화면에서는 추천 휴게가 표시되지 않는 것이 정상

### 원인
- 추천 휴게 시간은 **실시간 근무 기록 시점이 아닌, 근무 확정 시점**에 판단되어 입력된다[^3]
- 실시간 기록 중에는 추천 휴게가 반영되지 않으며, 근무 확정 시 별도 등록한 휴게와 추천 휴게가 겹치지 않으면 자동 등록된다[^4]

### 스펙/버그 판별
**스펙** — 실시간 휴게 기록과 추천 휴게는 독립적으로 동작하며, 추천 휴게는 근무 확정 시점에 판단/입력되는 것이 설계 의도이다.

## 해결
- 스펙 확인으로 종료[^4]
- CS 담당자에게 동작 방식 안내 완료

## 다음에 같은 문의가 오면

1. **먼저 확인**: 문의 내용이 "실시간 기록 중 추천 휴게가 안 보인다"인지, "근무 확정 후에도 추천 휴게가 안 들어간다"인지 구분
2. **원인 판별**:
   - 실시간 기록 중 미표시 → **스펙**. 추천 휴게는 근무 확정 시점에 판단
   - 근무 확정 후에도 미입력 → 별도 등록한 휴게와 추천 휴게의 시간이 겹치는지 확인. 겹치면 추천 휴게 미등록이 정상
   - 겹치지 않는데도 미입력 → 버그 가능성, 추가 조사 필요
3. **안내**: "실시간 근무 기록 중에는 추천 휴게 시간이 표시되지 않습니다. 근무 확정 시 별도 등록한 휴게와 겹치지 않으면 추천 휴게가 자동으로 입력됩니다."
