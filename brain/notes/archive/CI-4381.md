# CI-4381: 정산결과의 지급액 표기가 0원으로 나옵니다

## 증상
- **문제 정의**: 2023년 11월 급여 정산결과의 지급액이 0원으로 표시되나, 지급 항목별 확인 시에는 정상 조회됨
- **회사**: 주식회사 포스타입 (Customer ID: 326)
- **요청자**: 김한나 (CS팀)[^1]
- **대상자**: park.yuncheol@postype.com (문의자)[^2]
- **영향 범위**: 고객사 단위 — 2025년 3월 마이그레이션 시 누락된 전체 고객사가 영향권[^3]
- **문제 시점**: 2023년 11월 귀속[^5] 급여 정산결과
- 문의 내용:
  1. [2023년 11월 포스타입 급여]의 지급액이 0원으로 보여짐[^2]
  2. 지급에서 항목별 확인 시에는 정상 조회됨[^2]

## 원인 분석

`work_income_payment_settlement_payee_result_item` 테이블에 해당 정산결과(id=37750)의 지급 항목 데이터가 존재하지 않았음[^6]. 2025-03-19 backfill 마이그레이션[^4] 누락이 원인.

### 조사 과정
> 💡 김한나가 이전 Slack 스레드(CI-4265, 녹십자수의약품)[^7]와 동일 이슈로 접수
> → COOKBOOK #16 체크리스트 매칭[^8]
> → `work_income_payment_settlement_payee_result_item` 0건 확인[^6]
> → CI-4227용 Operation API로 dryRun 검증 → wetRun 실행

### 원인
- 정산결과 목록 API는 `WorkIncomeSettlementResult.payment.totalAmount`를 참조하며, 이 값은 `paymentItems`에서 합산[^9]
- `paymentItems`가 마이그레이션되지 않아 0건 → totalAmount = 0원
- 항목별 조회 API는 `WorkIncomeSettlementPayeeResult`에서 개별 payee 데이터를 직접 읽으므로 정상[^10]

## 정산 상태 확인

- 정산 기간: 2023년 11월 귀속, 지급일 2023-12-08[^6]
- 정산 상태: COMPLETED[^6]
- 정산 유형: REGULAR (정기 급여)
- settlement_result_id: 37750, payees_count: 29[^6]

## 금전적 영향

- 표시 오류만 해당. 실제 급여 지급에는 영향 없음[^2]

## 해결
- CI-4227용 Operation API로 마이그레이션 실행[^11]
  ```
  POST /api/operation/v2/payroll/settlement-results/payment/item/migrate-ci4227/326?dryRun=false
  ```
- 결과: migratedPayeeCount=29, skippedPayeeCount=2160[^11]
- DB 검증: payee 29명 전원 item 존재 확인 (총 33건), 누락 0건[^12]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 해당 고객사의 정산결과에서 `work_income_payment_settlement_payee_result_item`이 존재하는지 확인
   ```sql
   SELECT sr.id, sr.customer_id, sr.belonged_date, sr.name,
     (SELECT COUNT(*) FROM flex_payroll.work_income_payment_settlement_payee_result_item item
      JOIN flex_payroll.work_income_settlement_payee_result spr ON item.settlement_payee_result_id = spr.id
      WHERE spr.settlement_result_id = sr.id) AS item_count
   FROM flex_payroll.work_income_settlement_result sr
   WHERE sr.customer_id = {customer_id} AND sr.belonged_date = '{문제 귀속월}'
   ```
2. **원인 판별**: item_count = 0이면 backfill 마이그레이션 누락
3. **조치**: Operation API 실행
   - dryRun: `POST /api/operation/v2/payroll/settlement-results/payment/item/migrate-ci4227/{customerId}?dryRun=true`
   - 결과 확인 후 wetRun: `dryRun=false`
