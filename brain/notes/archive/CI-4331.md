# CI-4331: 평가 → 구리뷰 전환 가능 여부 확인 및 리뷰 작성기간 연장

## 증상

- **문제 정의**: 구리뷰에서 평가로 업그레이드된 고객사에서 구리뷰 원복 요청 + 진행 중 리뷰의 작성기간 수정 불가 상황
- **회사**: 골든플래닛 (Customer ID: 175870)[^1]
- **요청자**: CS juhwa님 (Intercom 인입)[^1]
- **대상자**: shkim@goldenplanet.co.kr[^1]
- **영향 범위**: 175870 고객사의 2025년 인사평가 리뷰 9건
- **문제 시점**: 2026-04-06 (구리뷰 → 평가 업그레이드 이후)
- 문의 내용:
  1. 구리뷰에서 평가로 업그레이드됨. 업그레이드 시 구리뷰에서 생성한 리뷰의 작성기간 수정이 가능할 것으로 안내받았으나 실제로는 불가
  2. 갱신 협의 중인 고객사로, 구리뷰 원복이 어려우면 해지 의사 표명
  3. 구리뷰 원복 가능 여부 + 진행 중인 리뷰 작성기간 4/10까지 연장 요청

## 원인 분석

### 조사 과정

> 💡 **판단 근거**: 김보라님이 구리뷰 원복 가능 여부를 확인한 결과, flag 설정으로 구리뷰 메뉴를 다시 노출시킬 수 있음이 확인됨[^2]
> → dev에서 테스트: 마감된 리뷰의 작성기간 수정은 `progress_status=IN_PROGRESS` 상태에서만 가능[^3]
> → 단, 날짜를 직접 데이터 보정으로 변경하면 담당자에게 할일이 발송되지 않으므로, 상태만 `IN_PROGRESS`로 변경하고 기간 수정은 리뷰관리자가 직접 처리해야 함[^4]

### 원인

- 구리뷰 → 평가 업그레이드 후에는 기본적으로 구리뷰 메뉴가 숨겨짐. flag 설정으로 재노출 가능[^2]
- 마감(`ANSWER_NOT_OPEN`) 상태인 리뷰는 관리자가 기간 수정 불가. `IN_PROGRESS`로 상태 변경 필요[^3]

## 영향 범위

- 175870 고객사 단독
- 대상 리뷰: 9건 (2025년 인사평가)[^5]

## 해결

1. 구리뷰 flag 설정으로 구리뷰 메뉴 재노출[^2]
2. 대상 리뷰 9건 `progress_status`를 `IN_PROGRESS`로 변경 (기안 승인 후 데이터 보정)[^6]
3. 고객(리뷰관리자)에게 직접 작성기간 수정 안내

**적용 SQL**[^6]:
```sql
-- 적용 쿼리
UPDATE flex_review.review_set SET progress_status = 'IN_PROGRESS'
WHERE customer_id = 175870 and id in (52519,52515,52514,52511,52510,52509,52505,52504,52513);

-- 롤백 쿼리
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52504;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52505;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52509;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52510;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52511;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52513;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52514;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52515;
UPDATE flex_review.review_set SET progress_status = 'ANSWER_NOT_OPEN' WHERE id = 52519;
```

## 발견한 스펙/제약

- **구리뷰 → 평가 업그레이드 후 원복**: flag 설정으로 구리뷰 메뉴 재노출 가능. 단, 구리뷰 전환 시점에 진행 중이었던 리뷰 및 캘리브레이션 세션은 데이터가 남아있지 않을 수 있음[^7]
- **마감된 리뷰 기간 수정**: `progress_status=IN_PROGRESS`로 변경해야 관리자 메뉴(리뷰관리 > 진행 중인 리뷰 > `...` 메뉴)에서 수정 가능[^3]
- **기간 수정은 반드시 리뷰관리자가 직접**: 데이터 보정으로 날짜를 직접 바꾸면 담당자 할일이 발송되지 않음[^4]
- **구리뷰 잔존 고객사 관리**: Notion 잔존 고객사 리스트에 등록 필요[^8]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 구리뷰 → 평가 업그레이드 고객사인지 확인. 해당 고객사의 flag 설정 여부 확인
2. **원인 판별**:
   - 구리뷰 메뉴가 안 보인다 → flag 설정으로 재노출
   - 리뷰 작성기간 수정 불가 → `flex_review.review_set.progress_status` 확인. `ANSWER_NOT_OPEN`이면 `IN_PROGRESS`로 변경 필요
3. **조치**:
   - flag 설정 (구리뷰 메뉴 재노출)
   - 기안 승인 후 SQL로 `progress_status` 변경
   - 고객(리뷰관리자)에게 관리 메뉴에서 기간 직접 수정하도록 안내
   - Notion 구리뷰 잔존 고객사 리스트에 추가
