# CI-4182: 2단계 승인 완료되었음에도 시스템상에서 '진행중'으로 표시

> **상태**: Done — 2026-03-23 (KST)

<details>
<summary>이벤트 재발행 시도 (1차 — 실패)</summary>

`produce-approval-process-event` Operation API[^6]로 승인 이벤트를 재발행했으나, `workflow_task` 상태는 변경되지 않음.

```
POST /action/operation/v2/approval/customers/155827/produce-approval-process-event
```

- 이 API는 `ApprovalProcessUpdatedEventV1`을 Kafka로 재발행하는 역할[^6]
- 이벤트 재발행은 성공했으나 워크플로우 컨슈머가 이를 소비하지 못한 것으로 추정
- 워크플로우 컨슈머의 이벤트 소비 실패 근본 원인은 미확정

</details>

### 5 Whys

1. 왜 문서가 진행중으로 표시되나? → `workflow_task.status`가 IN_PROGRESS[^4]
2. 왜 workflow_task가 갱신되지 않았나? → 승인 완료 이벤트(`ApprovalProcessUpdatedEventV1`)가 워크플로우에 반영되지 않음
3. 왜 이벤트가 반영되지 않았나? → 이벤트 재발행(`produce-approval-process-event`)을 해도 워크플로우 상태가 변경되지 않음[^7]
4. 왜 재발행된 이벤트도 무시되나? → (근본 원인 미확정) 워크플로우 컨슈머의 이벤트 처리 로직에 문제가 있거나, 이벤트 구조/조건 불일치 가능성

### 스펙 vs 버그 판별

**버그** — 승인이 완료된 문서는 워크플로우에서도 DONE으로 표시되어야 한다. `approval_process.status = APPROVED`인데 `workflow_task.status = IN_PROGRESS`인 것은 동기화 오류이며, CI-4019에서도 동일 패턴이 bug로 확정됨[^2].

   - 이벤트는 발행되었으나 `workflow_task` 상태 변경 없음

## 다음에 같은 문의가 오면

1. **증상 확인**: 승인 완료된 문서가 진행중 문서함에 표시
2. **DB 확인**: `approval_process` 테이블에서 해당 문서의 `status`가 `APPROVED`인지 확인
3. **불일치 확인**: `workflow_task` 테이블에서 동일 문서의 `status`가 `IN_PROGRESS`인지 확인
4. **보정**: `sync-with-approval` Operation API 호출 (Raccoon Swagger UI)
   - 경로: `POST /api/v2/operation/workflow/customers/{customerId}/users/{userId}/approval-document/{documentKey}/sync-with-approval`
   - userId: `view_user` 테이블에서 기안자의 `id` 조회
   - documentKey: `approval_process` 테이블의 `target_uid`
5. **검증**: `workflow_task` 상태가 DONE으로 변경되었는지 확인
