# CI-4309: 감사로그 다운로드 요청 — (주)스트라드비젼

## 증상
- **문제 정의**: 고객이 제품 감사로그 UI의 90일 조회 제한을 넘는 기간의 감사로그 다운로드를 요청
- **회사**: (주)스트라드비젼 (Customer ID: 25546)
- **요청자**: 유지원 (CS팀)[^1]
- **대상자**: gyusun.lee@stradvision.com (userId: 857604)[^2]
- **영향 범위**: 1명 (요청자)
- **문제 시점**: 2025-09-22 ~ 2026-01-30 (약 4개월, 90일 초과)
- 문의 내용:
  1. 감사로그 데이터 다운로드 요청 (이벤트: 수정/삭제/생성, 정보 유형: 근무정보/휴가정보/기타)[^1]
  2. 고객은 현재 법적 대응 중으로 감사로그 필요[^3]

## 원인 분석

### 조사 과정
> 제품 감사로그 UI는 최대 90일(기본 7일) 조회 제한이 있어, 고객이 직접 다운로드할 수 없는 기간의 데이터를 요청한 건이다.
> 과거 CI-2095에서 동일 유형 문의가 있었고, 그때 구축된 raccoon operation API로 처리 가능한 상태였다[^4].

### 원인
- 제품 감사로그 UI의 기간 제한 (90일)[^5]. 엔터프라이즈 플랜 과금 목적으로 설정되었으나 현재 해당 플랜은 없음[^6]
- 법적 요건상 접속기록 2년 보관 의무가 있으나, 제공 의무는 flex에 없음. 다만 CS팀 선행 결정에 따라 제공 가능[^7]

## 해결
- 김영준님이 raccoon audit operation API 호출하여 요청자 이메일로 감사로그 발송[^8]

### raccoon 호출 정보
```
POST https://flex-raccoon.grapeisfruit.com/proxy/audit/action/operation/v2/audit/25546/download?userId=857604
{
  "personalInformationType": [
    "근태_근무정보", "근태_휴가정보", "기타"
  ],
  "eventCategory": [
    "CREATE", "UPDATE", "DELETE"
  ],
  "from": "2025-09-22T00:00:00+09:00",
  "to": "2026-02-01T00:00:00+09:00"
}
```

> **200 응답만 오고 UI 피드백이 없어서 영준님 2회 + 주원님 2회 = 총 4건 중복 발송됨**[^9]. Amazon SES 발송 이력에서 4건 확인[^10].

### 메일 발송 확인 방법
- Amazon SES > Virtual Deliverability Manager > Dashboard > 메시지 탭에서 수신자로 필터링[^11]
- admin-shell: https://admin-shell.flexis.team/pavement/mail/ses-feedback [^12]
  - 단, ses-feedback-log 뷰는 CS팀에만 권한 부여, 개발자는 접근 불가[^13]

## 발견한 스펙/제약
- 감사로그 조회 기간: 제품 UI 최대 90일, operation API 최대 2년[^5]
- operation API 호출 시 200 응답만 반환되고 별도 UI 피드백 없음 → 중복 클릭 주의[^9]
- 메일 발송은 flex 고객사는 Mailgun, 그 외는 Amazon SES 사용[^14]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 감사로그 다운로드 요청인지, 요청 기간이 90일을 초과하는지 확인
2. **참고 문서**: [감사로그데이터 전달 대응 관련 문서 (Notion)](https://www.notion.so/flexnotion/2450592a4a928024b02eebe84fd722b6)[^15]
3. **조치**:
   - raccoon > audit operation API에서 아래 파라미터로 호출
   - URL: `POST https://flex-raccoon.grapeisfruit.com/proxy/audit/action/operation/v2/audit/{customerId}/download?userId={userId}`
   - Body: `personalInformationType`, `eventCategory`, `from`, `to` 설정
   - **200 응답 오면 완료 — 버튼 한 번만 클릭할 것** (중복 발송 방지)
4. **발송 확인**: admin-shell SES feedback 로그 또는 AWS SES 콘솔에서 확인
5. **CS팀에 고객 수신 확인 요청**
