# CI-4309: 감사로그 다운로드 요청 — (주)스트라드비젼

## 증상
- **문제 정의**: 고객이 제품 감사로그 UI의 기간 제한(기본 7일, 최대 90일)을 넘는 기간의 감사로그 다운로드를 요청
- **회사**: (주)스트라드비젼 (Customer ID: 25546)
- **요청자**: 유지원 (CS팀)[^1]
- **대상자**: gyusun.lee@stradvision.com (이규선, userId: 857604)[^2]
- **영향 범위**: 1명 (문의자)
- **문제 시점**: 2025-09-22 ~ 2026-01-30 (약 4개월)
- 문의 내용:
  1. 감사로그 데이터 다운로드 요청 — 법적 대응 목적[^3]
  2. 이벤트: 수정, 삭제, 생성
  3. 정보 유형: 근무정보, 휴가정보, 기타

## 원인 분석

### 조사 과정
> 💡 제품 내 감사로그 다운로드 기능은 **기본 7일, 최대 90일 제한**이 있으며, 장기간 조회는 엔터프라이즈 플랜 전용이었으나 현재 해당 플랜은 없음[^4]
> → 고객이 법적 대응 목적으로 약 4개월치 데이터가 필요하여 별도 요청 발생
> → 과거 CI-2095에서 구축된 raccoon audit operation API로 처리 가능한 상태였다[^5]

### 원인
- 제품 감사로그 UI의 기간 제한으로 인해 고객이 직접 다운로드할 수 없음[^4]
- 법적 요건상 접속기록 2년 보관 의무가 있으나, 제공 의무는 flex에 없음. 다만 CS팀 선행 결정에 따라 제공 가능[^6]

## 해결
- 김영준님이 raccoon audit operation API 호출하여 요청자 이메일로 감사로그 발송[^7]
- API 호출 시 문의자 메일로 파일이 직접 발송됨
- 4건 중복 발송됨 (김영준 2회 + 김주원 2회 클릭)[^8]

### Operation API 정보

```
POST https://flex-raccoon.grapeisfruit.com/proxy/audit/action/operation/v2/audit/{customerId}/download?userId={userId}
```

| 파라미터 | 값 | 설명 |
|---------|---|------|
| `customerId` (path) | 25546 | 고객사 ID |
| `userId` (query) | 857604 | 문의자 user ID |

```json
{
  "personalInformationType": [
    "근태_근무정보", "근태_휴가정보", "기타"
  ],
  "eventCategory": [
    "CREATE", "UPDATE", "DELETE"
  ],
  "from": "2025-09-22T00:00:00+09:00",
  "to": "2026-02-01T00:00:00+09:00"
}
```

**응답**: 200 OK (body 없음) — 파일은 userId에 해당하는 사용자의 이메일로 발송됨[^9]

> ⚠️ **200 응답만 오고 UI 피드백이 없어서 영준님 2회 + 주원님 2회 = 총 4건 중복 발송됨**[^8]. Amazon SES 발송 이력에서 4건 확인[^10].

### 메일 발송 확인 방법
- 메일 서비스: flex 고객사는 Mailgun, 그 외는 Amazon SES 사용[^11]
- Amazon SES 확인: SES > Virtual Deliverability Manager > Dashboard > 메시지 탭에서 수신자로 필터링[^12]
- admin-shell: https://admin-shell.flexis.team/pavement/mail/ses-feedback [^13]
  - 단, ses-feedback-log 뷰는 CS팀에만 권한 부여, 개발자는 접근 불가[^14]

## 발견한 스펙/제약
- 감사로그 조회 기간: 제품 UI 기본 7일/최대 90일, operation API 최대 2년[^4]
- operation API 호출 시 200 응답만 반환되고 별도 UI 피드백 없음 → **중복 클릭 주의**[^8]
- 장기간 감사로그 요청은 법적 요건으로 협조 필요 — 엔터프라이즈 유도 목적의 제한이지만 법적 대응 시 거부 불가[^15]

## 다음에 같은 문의가 오면

1. **먼저 확인**: [감사로그데이터 전달 대응 관련 문서 (Notion)](https://www.notion.so/flexnotion/2450592a4a928024b02eebe84fd722b6)[^16]
2. **raccoon Swagger 접속**: `https://flex-raccoon.grapeisfruit.com/swagger/audit` → "audit operation" 섹션
3. **API 실행**: `POST /action/operation/v2/audit/{customerId}/download?userId={userId}`
   - `personalInformationType`: 고객 요청에 맞게 선택 (아래 enum 참조)
   - `eventCategory`: CREATE, UPDATE, DELETE 중 선택
   - `from`/`to`: KST 기준 시작/종료 시간 (ISO 8601)
4. **주의**: 200 응답만 오고 파일은 **메일로 발송**됨 — **버튼 한 번만 클릭할 것** (중복 발송 방지)
5. **발송 확인**: admin-shell SES feedback 로그 또는 AWS SES 콘솔에서 확인
6. **CS팀에 고객 수신 확인 요청**
