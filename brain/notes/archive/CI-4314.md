# CI-4314: 평가지 질문 필수→선택 변경 후에도 미작성 시 제출 불가

> **상태**: Done — 2026-04-03. 데이터 보정으로 즉시 해결[^15], 근본 수정은 EPBE-336에서 후속 진행[^16]
> **compact**: 원본 255줄 → 정제 119줄 (2026-04-03). git log로 원본 추적 가능.

## 증상
- **문제 정의**: 평가지 질문의 필수 여부를 "필수→선택"으로 변경했으나, 구성원이 해당 항목을 비워두고 제출하면 "제출에 실패했어요" 오류 발생
- **회사**: 주식회사 메디테라피 (Customer ID: 17953)
- **요청자**: 김한나 (CS팀)[^1]
- **대상자**: 복수 구성원[^2]
- **문제 시점**: 2026-04-03 (당일 데이터 보정으로 해결)

## 원인 분석

**`form_user_answer.required`가 `false`로 갱신되지 않는 버그** — `RequiredChangeInfo.detectRequiredChanges()`가 선택→필수(turned ON)만 감지하고, 필수→선택(turned OFF)은 감지하지 않음[^10]. finalize 시 `hasChanges()=false` → 조기 리턴 → `form_user_answer.required`가 `true`로 유지.

**버그 (Bug)** — 선택→필수 변경은 정상 동작하므로, 반대 방향은 구현 누락으로 판단.

### 발현 조건

| 조건 | 셀프평가 form (정상) | 직책자 역량 form (버그) |
|------|---------------------|----------------------|
| UserAnswer 생성 시점 | step IN_PROGRESS 전환 시 새로 생성 | 기존 UserAnswer 유지 |
| required 전파 | 생성 시 draft 값 반영 | `RequiredChangeInfo`가 turned OFF 미추적 → 미갱신 |

## 영향 범위
- **이 회사**: 셀프평가 65건(보정 완료) + 직책자 역량 셀프리뷰 19건(미보정)[^13]
- **같은 조건의 모든 회사**: 평가 진행 중 질문을 필수→선택으로 변경한 모든 평가에서, 기존 UserAnswer가 유지되는 form에 동일 버그 발생 가능
- 금전적·법적 영향: 해당 없음

## 발견한 스펙/제약
- `form_user_answer.required`는 `form_question.required`의 비정규화 복사본이며, 별도 동기화가 필요하다[^8]

## 해결

### 즉시 대응 (데이터 보정) — 완료
김보라가 셀프평가 form 65건 UPDATE[^15]. 김한나가 제출 정상 동작 확인[^17].

**추가 발견**: 직책자 역량 셀프리뷰 form 19건 미보정[^13]:
```sql
UPDATE flex_review.form_user_answer
SET required = 0, updated_at = NOW(6)
WHERE customer_id = 17953
  AND question_id = '01kn3xyj79ttbe6pnjzwd6vscw'
  AND required = 1 AND deleted_at IS NULL;
```

### 근본 해결 (코드 수정) — EPBE-336
후속 이슈 EPBE-336에서 진행[^16]:
1. `RequiredChangeInfo`에 `requiredTurnedOffBlockIdentities` 필드 추가
2. `detectRequiredChanges()`에서 필수→선택 감지 추가
3. 전체 reviewer의 UserAnswer를 갱신하도록 확장

## 다음에 같은 문의가 오면

1. **먼저 확인**: 관리자가 최근에 해당 질문의 필수/선택 설정을 변경했는지 확인
2. **원인 판별**: `form_user_answer` 테이블에서 해당 질문의 `required` 값 조회 — `true`이면 이 버그에 해당
3. **조치**: DML로 `form_user_answer.required = false` 업데이트 (결재 필요)

## 시나리오

```gherkin
# language: ko
기능: 평가지 질문 필수/선택 설정 변경 시 UserAnswer 동기화

  시나리오: 관리자가 평가 진행 중 질문을 필수에서 선택으로 변경하면 기존 UserAnswer의 required가 갱신된다
    주어진 평가가 진행 중(IN_PROGRESS)이다
    그리고 "가중목 달성 근거 첨부" 질문이 필수(required=true)로 설정되어 있다
    그리고 구성원의 form_user_answer에 required=true로 기록되어 있다
    만약 관리자가 해당 질문을 선택(required=false)으로 변경하고 finalize한다
    그러면 기존 form_user_answer의 required가 false로 갱신된다
    그리고 구성원이 해당 항목을 비워두고 제출하면 정상 제출된다

  시나리오: 필수→선택 변경은 evaluation_step_form_mapping에 없는 form에도 적용된다
    주어진 평가에 "직책자 역량 셀프리뷰" form이 별도 경로로 생성되어 있다
    그리고 해당 form의 질문이 필수에서 선택으로 변경된다
    만약 관리자가 finalize한다
    그러면 해당 form의 form_user_answer.required도 false로 갱신된다
```
