# CI-4230: 워크플로우 참조문서 증빙 제출 건이 엑셀에서 삭제된 문서로 표시

## 증상
- **문제 정의**: 증빙 제출 시 참조문서로 워크플로우를 참조하여 제출한 건이, 관리자 엑셀 다운로드 시 "삭제된 문서"로 표시됨 (실제 해당 워크플로우 문서는 삭제되지 않은 상태)
- **회사**: 주식회사 유니크플랜 (Customer ID: 211781)
- **요청자**: 나유라 (CS팀)[^1]
- **대상자**: dahee@uniqueplan.kr[^2]
- **영향 범위**: 워크플로우 참조문서가 있는 모든 증빙에서 발생[^11]
- **문제 시점**: Bullseye 인덱싱 코드에 이중 접두사 버그가 포함된 시점부터
- 문의 내용:
  1. 증빙 제출 시 참조문서 내 워크플로우를 참조하여 제출
  2. 관리자가 엑셀 다운로드 시 해당 건이 "삭제된 문서"로 표시됨
  3. 실제로 해당 워크플로우 문서는 삭제되지 않은 상태

## 원인 분석

`SpendingBullseyePublishService`의 Bullseye 인덱싱 코드에서 `"${it.additionalType}:${it.approvalDocumentKey}"` 형태로 키를 저장하면서, 이미 `workflow:` 접두사가 포함된 `approvalDocumentKey`에 다시 `workflow:` 접두사가 붙어 **이중 접두사** `"workflow:workflow:80caec..."`가 발생했다[^11]. 엑셀 다운로드 시 이 키를 파싱하면 실제 문서 ID 대신 `"workflow"`라는 문자열이 API에 전달되어 빈 응답 → `deleted = true` → "삭제된 문서" 표시.

### 가설 목록

|---|------|------|
| 1 | approval document API에 잘못된 키 전달 | ✅ 확정 — Bullseye 이중 접두사[^9] |

### 조사 과정

> 💡 **판단 근거**: access log에서 API 요청/응답 확인[^9]
> → DB 저장 키 `"workflow:80caec..."` 정상 확인[^10]
> → Bullseye에 `"workflow:workflow:80caec..."` 이중 접두사로 저장됨[^11]
> → `groupedType()` 파싱 결과 `"workflow"` 문자열만 API에 전달
> → API 빈 응답 → `deleted = true`

referencedApprovalDocumentKeys = evidence

**`groupedType()` 파싱 시 잘리는 과정:**
1. 입력: `"workflow:workflow:80caec..."`
2. `split(":")` → `["workflow", "workflow", "80caec..."]`
3. `parts.size > 1` → key 그대로 유지
4. 다시 `split(":")` → `["workflow", "workflow", "80caec..."]`
5. `groupBy(keySelector={it[0]}, valueTransform={it[1]})` → `{"workflow": ["workflow"]}`
6. `extractWorkflow` 호출: `approvalDocumentKeys = ["workflow"]`
