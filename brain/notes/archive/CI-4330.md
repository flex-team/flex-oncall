## 원인 분석

**스펙 확정**: 연동 해제는 "앞으로의 동기화를 차단"하는 것이며, 기존에 구글캘린더에 생성된 이벤트를 삭제하는 로직이 존재하지 않는다.[^5]

핵심 근거:
- `GoogleCalendarConnectionUpdateService.deleteCalendarConnection()`[^5]은 DB에서 연결 정보만 삭제하고 구글 Calendar API를 호출하지 않음
- 연동 해제 시 `google_calendar_event_delete_sync` 레코드가 생성되지 않음[^6]
- 이는 모든 회사에서 동일하게 발생하는 의도된 설계

### 조사 과정

> 💡 **판단 근거**:
> → `flex-calendar-backend` > `GoogleCalendarConnectionUpdateService.deleteCalendarConnection()` 확인[^5]
> → `externalCalendarConnectionRepository.delete()` + 히스토리 기록만 수행, **구글 API 호출 없음**
> → 연동 해제 시 `google_calendar_event_delete_sync` 레코드 미생성[^6]
> → force-sync-deleted/sync-deleted API는 이 레코드를 조회하므로 연동 해제 케이스에 **적용 불가**
> → `cleansing` API는 이벤트 ID를 직접 지정하므로 **적용 가능**

## 다음에 같은 문의가 오면

1. **현황 파악**: "연동 해제 시 기존 이벤트 자동 삭제 없음" — 이것이 현재 스펙임을 고객에게 안내
2. **수동 삭제 요청 시**: cleansing API 사용
   - `flex.v2_external_calendar_connection_history`에서 `customer_id` + `connection_state='DISCONNECTED'`로 `calendarConnectionHistoryId` 조회
   - `flex.v2_oauth_user_token`에서 `customer_id` + 그룹 캘린더 소유 계정 이메일로 `oauthTokenId` 조회
   - `flex_calendar.google_calendar_event_sync`에서 `query_key LIKE '{customerId}%'` + `google_calendar_id = '{그룹캘린더ID}'`로 이벤트 ID 목록 조회 (Metabase 사용 권장 — MCP 화이트리스트 미허용)
   - raccoon 브라우저 세션 (`https://flex-raccoon.grapeisfruit.com`) 에서 `X-CSRF-TOKEN` 헤더 포함하여 `POST /proxy/calendar/api/operation/v2/calendar/customers/{customerId}/events/cleansing` 호출
   - 300건 이상 시 20건씩 배치, 배치 간 2초 딜레이 (Google Calendar API rate limit)

## 발견한 스펙/제약

- **연동 해제 ≠ 이벤트 삭제**: 그룹 구글캘린더 연동 해제는 DB 연결 정보만 삭제하며, 구글캘린더에 이미 생성된 이벤트를 삭제하지 않음
- **force-sync-deleted / sync-deleted API 적용 범위**: 이 API들은 `google_calendar_event_delete_sync` 레코드가 있을 때만 동작. 연동 해제 케이스에는 해당 레코드가 없으므로 적용 불가
- **cleansing API**: `calendarConnectionHistoryId`를 통해 이미 삭제된 연동에 대해서도 처리 가능

```gherkin
    Then flex DB에서 연결 정보(external_calendar_connection)가 삭제된다
    And 구글캘린더에 기존 생성된 이벤트는 삭제되지 않고 잔존한다
    # 이것이 현재 스펙: 연동 해제 = 앞으로의 동기화 차단, 기존 이벤트 삭제 아님

  Scenario: 수동 삭제 요청 시 cleansing API로 처리한다
    Given 그룹 구글캘린더 연동이 이미 해제된 회사가 있다
    And 구글캘린더에 잔존 이벤트가 있다
    When 온콜 담당자가 calendarConnectionHistoryId와 oauthTokenId를 확인한다
    And google_calendar_event_sync 테이블에서 잔존 이벤트 ID 목록을 추출한다
    And POST /api/operation/v2/calendar/customers/{customerId}/events/cleansing 를 호출한다
    Then 구글캘린더에서 해당 이벤트들이 삭제된다
```
