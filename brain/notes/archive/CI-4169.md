# CI-4169: 결제 취소 후 라쿤에서 체험 종료일/청구일 반영 불가

## 증상
- **문제 정의**: 결제 취소로 인해 고객사 로그인 차단 상태에서, 라쿤 billing operation으로 체험 종료일/청구일 변경 시도 시 기존 결제 이력으로 반영 불가
- **회사**: 라인앤컴퍼니주식회사 (Customer ID: 224686)[^1]
- **요청자**: @yeogyung (CS팀)[^1]
- **대상자**: 라인앤컴퍼니주식회사 관리자[^1]
- **영향 범위**: 1사 (전사 로그인 차단)
- **문제 시점**: 2026-03-03 결제 취소 이후[^1]
- 문의 내용:
  1. 3/3 결제 취소 후 로그인 불가 상태[^1]
  2. 라쿤에서 체험 종료일을 3/29로, 청구일을 30일로 반영하려 하지만 기존 결제 이력으로 불가[^1]
  3. 청구목록 기록 삭제 또는 종료일/청구일 직접 반영 가능 여부[^1]

## 원인 분석

### 조사 과정
> 💡 결제 취소 시 고객사 접근이 차단되는 것은 정상 스펙[^2]
> → 무료체험 종료일은 결제 발생 이력이 있는 고객사에 대해 변경 불가[^4]
> → 청구서는 원칙적으로 삭제 불가[^2]
> → raccoon billing operation의 `force-open`으로 임시 접근 허용 후, 카드 재등록으로 해결하는 경로가 표준 대응[^3]

### 원인
- 결제 이력이 있는 고객사는 무료체험 종료일 직접 변경이 불가하도록 설계되어 있음[^4]
- 결제 취소 시 고객사 접근이 차단되므로, 카드 재등록을 위해 임시 접근(force-open)이 필요[^2]

## 해결
- raccoon > billing operation의 `force-open`으로 강제 진입 처리[^2]
- 고객사가 로그인하여 결제카드 재등록 완료[^5]
- `close-forced-open`으로 강제 진입 플래그 원복[^6]

## 발견한 스펙/제약
- **결제 이력 고객사의 체험 종료일 변경 불가**: 무료체험 종료일은 결제 발생 고객사에 대해 변경할 수 없음. 0원 구독 또는 청구 시작일로 조정해야 함[^4]
- **청구서 삭제 불가**: 청구서는 원칙적으로 삭제할 수 없음[^2]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 결제 취소 여부 및 고객사 로그인 가능 상태 확인
2. **원인 판별**: 결제 취소로 인한 접근 차단이면 아래 조치 진행
3. **조치**:
   - raccoon > billing operation > `force-open` 실행하여 임시 접근 허용[^3]
   - 고객에게 로그인 → 카드 등록 안내
   - 카드 등록 완료 확인 후 `close-forced-open`으로 원복[^6]
   - 체험 종료일 직접 변경은 불가 — 0원 구독 또는 청구 시작일 조정으로 대응[^4]
