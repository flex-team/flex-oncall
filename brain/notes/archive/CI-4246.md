# CI-4246: 세콤 수동전송 시 출근 이벤트 중복 처리 버그

> **compact**: 2026-03-31. git log로 원본 추적 가능.

## 증상
- **문제 정의**: 세콤 출근 타각 3건을 수동전송하자 잘못된 START 이벤트 2개가 추가 생성되어 PC·모바일 표시가 불일치함
- **회사**: 퍼시스홀딩스 (Customer ID: 224004)
- **요청자**: 안상연[^1] (CS 담당자)
- **대상자**: haneung_lim@fursys.com[^1]
- **영향 범위**: 1명 (테스트 중 발생)
- **문제 시점**: 2026-03-27 07:55, 10:37, 11:42 출근 태깅
- 문의 내용:
  1. 세콤 연동 테스트 중 동일 사용자가 3회 출근 태깅 후 수동전송[^3] (타사 출근 시스템에서 수작업으로 이벤트를 Flex 플랫폼으로 일괄 전송하는 기능)
  2. PC 웹: 마지막 타각 시점부터 근무 시작으로 표시
  3. 모바일 (iOS 26.1 / flex app 2.121.0): 근무가 3개 구간으로 나뉘어 표시됨

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 출근 이벤트 3개가 뒤늦게 한번에 도착한 패턴 확인[^2]
> → 수동전송[^3]으로 이벤트가 일괄 전달됨
> → 기존 버그[^4]로 인해 근무로 전환된 위젯 이벤트에 맞춰 잘못된 START 이벤트 2개가 추가 생성됨

### 원인
- 세콤 수동전송 시 이벤트가 일괄 처리되는 과정에서 알려진 버그가 발동[^4]
- 정상 근무 전환 1건(START 이벤트) 외에 잘못된 START 이벤트[^6] 2건이 추가 생성됨[^2]
- PC 웹과 모바일이 서로 다른 이벤트 기준으로 렌더링하여 표시 불일치 발생

## 해결
- 잘못 생성된 `v2_user_work_clock_event`[^7] (출근 이벤트를 저장하는 DB 테이블) 2건을 CANCEL 처리[^2] (이벤트를 취소 상태로 표시하는 작업)
  - ID: 127831183
  - ID: 127831227
- 핫픽스 배포 완료[^5]

## 발견한 스펙/제약
- 세콤 이벤트 수동전송 시 이벤트가 일괄 도착하면 버그가 재현될 수 있음[^4]
- 출근 가능 시간(08:30) 이전 타각과 이후 타각이 섞인 경우 근무 구간 처리가 복잡해짐[^1]

## 다음에 같은 문의가 오면

1. **먼저 확인**: PC 웹과 모바일 표시가 다른지, 세콤 수동전송 여부 확인
2. **원인 판별**:
   - 수동전송 + 이벤트 중복 → `v2_user_work_clock_event` 에서 해당 유저/날짜 CANCEL 대상 이벤트 탐색
   - 자동전송인데 불일치 → 다른 원인 탐색
3. **조치**:
   - 잘못된 START 이벤트 → `v2_user_work_clock_event`에 CANCEL 이벤트 삽입
   - 버그 재발 시 핫픽스 이력 확인
