# CI-4107: 특정 필드의 세로 가운데 정렬 미동작

> **상태**: In Progress — 2026-03-16
>
> **한 줄 요약**: 전자계약 양식에서 임금수당 리스트의 세로 가운데 정렬이 미리보기에서는 정상이나, 실제 발송 후에는 상단 정렬로 표시되는 문제. 플레이스홀더 뒤 불필요한 `<br>` 태그가 원인.

## 증상
- **회사**: athometrip (Customer ID: 202640)
- **문의자**: jiwonwoo@athometrip.com
- **라벨**: 전자계약
- **우선순위**: Low
- **담당자**: 지무근
- 문의 내용:
  1. 양식 "01. [앳홈트립] 근로계약서(정규직)"에서 임금수당 리스트가 다른 항목들과 달리 세로 가운데 정렬이 적용되지 않음[^1]
  2. 표에서 입력 후 저장 완료, 미리보기까지는 정상이나 실제 계약서 발송 후에는 상단 정렬로 표시됨[^1]
  3. 해당 양식에서만 문제 발생 — prd 테스트 계정에서 재현 확인됨[^2]

## 원인 분석

정렬 자체는 동작하지만, 임금수당 리스트 플레이스홀더[^8] 뒤에 불필요한 줄바꿈(`<br>`)이 삽입되어 상단으로 올라가 보이는 상태[^3].

> 💡 **판단 근거**: 미리보기에서는 정상 → 발송 후에만 상단 정렬 → 렌더러[^9] 변환 과정에서 차이 발생 의심 → 플레이스홀더 뒤 `<br>` 태그가 세로 정렬을 깨뜨리는 것으로 확인[^3]

## 해결안

### 수정 내용

두 개의 PR로 수정 진행 중:

1. **렌더러 수정**: 블록 플레이스홀더 세로 가운데 정렬 수정 (V1_13)[^4]
   - digicon renderer 버전을 `V1_12` → `V1_13`으로 업그레이드 필요[^7]
2. **프론트엔드 수정**: 템플릿 저장 시 플레이스홀더 뒤 불필요한 `<br>` 제거[^5]

### 배포 상태

- renderer 배포 완료[^6]
- digicon-backend에서 renderer 버전 반영 필요 — 온콜 담당에게 요청됨[^6]

## 해결

> 미완료 — renderer 배포 완료, digicon-backend에서 V1_13 반영 대기 중.
