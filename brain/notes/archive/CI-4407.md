## 원인 분석

**3건 (2026-10, 2026-66, 2026-67)**: `approval_process.status = APPROVED`인데 `workflow_task.status = IN_PROGRESS` → 이벤트 동기화 실패 버그. 이전 광범위 오류(CI-4385 참조)의 후속 여파.

**4건 (2026-23, 2026-43, 2026-46, 2026-59)**: `approval_process.status = ONGOING` → 이전 일괄 강제 승인 처리(2026-04-07경) 시 offboarded 사용자(userId 930246, songdm@safedoc.io)가 승인자였던 건들이 처리되지 않고 잔존.

## 해결

### 3건 보정 (2026-04-13 11:57~11:58)

`approval-process/sync` v3 API 사용.

> **주의**: v3 API의 actorUserId는 기안자(writer_id)가 아닌 승인라인에 실제 등재된 단독 actor여야 한다.
> step에 actor가 2개 이상(동시 승인 구조)이면 "multiple actors found" 에러 → 단독 actor인 step의 userId를 선택.

| 문서 | actorUserId | 보정 결과 |
|------|-------------|---------|
| 2026-10 | 911597 (step 1) | IN_PROGRESS → DONE |
| 2026-66 | 941623 (step 1) | IN_PROGRESS → DONE |
| 2026-67 | 941623 (step 1) | IN_PROGRESS → DONE |

### 4건 강제 승인 (2026-04-13 06:41, by 임형태)

offboarded 사용자 930246 (songdm@safedoc.io)에 대해 `bulk-approve-for-user` API로 강제 승인 처리 완료.

```
POST /api/operation/v2/approval/process/customers/161808/users/930246/bulk-approve-for-user
Body: { "categories": ["APPROVAL_DOCUMENT"] }
```

## 다음에 같은 문의가 오면

1. `workflow_task` 상태 확인 → `approval_process` 상태 확인
2. `approval_process.status = APPROVED` + `workflow_task = IN_PROGRESS` → **sync-with-approval v3 API**
   - actorUserId: `approval_line_actor`에서 단독 actor 조회 (step에 2명 이상이면 다른 step 선택)
3. `approval_process.status = ONGOING` → **`approval_line_actor`에서 PENDING 승인자 확인**
   - offboarded 사용자가 있으면 → **F1 플로우 (bulk-approve-for-user)**
   - `view_user.status` 확인 후 `bulk-approve-for-user` API 호출
