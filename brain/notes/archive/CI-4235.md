# CI-4235: 구글 캘린더 미연동 일정 재연동 요청

> **compact**: 원본 142줄 → 정제 72줄 (2026-03-27). git log로 원본 추적 가능.

## 증상
- **문제 정의**: 구글 캘린더 연동 설정 후 11건의 일정이 동기화되지 않음
- **회사**: 마인이스 (Customer ID: 53244)
- **대상자**: kate@mineis.io[^1]
- **영향 범위**: 1명
- 문의 내용: 구글 캘린더를 연동해놨으나 미연동된 일정 11건이 존재, 재연동 요청[^1]

## 원인 분석

간헐적 Kafka/캘린더 API 동기화 누락 — SP팀 운영 가이드에서 "듬성듬성 연동 누락"으로 분류되는 known issue[^2]

### 조사 과정

> 💡 **판단 근거**: 코드 조사에서 캘린더 기능 활성 확인[^3] → 운영 가이드에서 "듬성듬성 연동 누락" 패턴과 대응 절차 확인[^2] → 에러 시 로깅만, 재시도 없는 구조[^4] → Operation API 재동기화로 대응

**스펙에 가까움** — 동기화 파이프라인에 재시도 로직이 없는 것은 의도된 구조이며, SP팀에서 Operation API를 통한 수동 재동기화 절차를 운영 중[^2]

## 해결
- raccoon Operation API[^5]로 미연동 11건 재동기화 완료[^6]
- 티켓 Done 처리 완료

## 다음에 같은 문의가 오면

1. **확인**: Metabase 대시보드(`/dashboard/244?email=`)에서 미연동 건 수와 calendar event ID 확인
2. **원인 판별**: 미연동 건 존재 → "듬성듬성 연동 누락" (known issue). 토큰 오류(`insufficientPermissions`) → 재연동 필요
3. **조치**:
   - 미연동 건 → Metabase 쿼리 7387로 eventIds 추출 → raccoon Operation API로 재동기화 (batch 5건, 2초 delay)
   - 토큰 오류 → 사용자에게 설정 > 연결 설정 > 구글 캘린더 재연동 가이드

## 발견한 스펙/제약

| 항목 | 내용 |
|------|------|
| **동기화 재시도 없음** | 에러 시 catch + 로깅만, 재시도 로직 없음[^4] |
| Rate limit | batch 5건 + 2초 delay 권장. 24년 10월 이전 데이터는 중복 생성 우려로 비권장[^2] |
