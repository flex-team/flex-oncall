# CI-3938: 중간정산 이력 시 퇴직사유가 중간정산으로 잘못 신고됨

> **상태**: Done — 2026-02-25 (KST)
> **verdict**: bug-fix — Payee 모델 최신화 시 중간정산 사유 미초기화

## 증상

- **문제 정의**: 퇴직소득 지급명세서 신고 시 중간정산 이력이 있는 경우, 실제 퇴직사유(퇴직)가 아닌 '중간정산'으로 신고됨. 원천징수영수증에서도 동일하게 '중간정산'으로 표기. 신고 오류는 발생하지 않아 그동안 발견되지 못했으나, 실제 사유와 다르게 신고되어 수정 필요. 2025년 귀속 전체 대상자 보정 필요.

## 원인 분석

- 이원국이 코드 분석으로 원인 단서 발견:
  ```kotlin
  fun of(payee: SeveranceSettlementPayeeModel): 소득자_퇴직사유 {
      if (payee.interimSettlementReason != null) return 중간정산  // ← 여기
      return when (payee.severanceSettlementReason!!) {
          SeveranceSettlementReasonType.RETIREMENT -> 정년퇴직
          ...
      }
  }
  ```
- `payee.interimSettlementReason`이 null이 아니면(이전 중간정산 사유 정보가 있으면) 무조건 퇴직사유를 '중간정산'으로 리턴
- 재현 흐름:
  1. 처음에 중간정산 대상자로 시작
  2. 중간정산 사유 입력
  3. 대상자 최신화 등으로 구분이 퇴직으로 변경됨
  4. 중간정산 사유는 남아 있는 채로 이후 정산 진행 → 퇴직사유가 '중간정산'으로 잘못 설정
- 이한결이 prod 데이터 확인: 화면에 드러나지 않지만(FE에서 구분=퇴직이면 중간정산 사유 열을 안 보여줌) 데이터상 최종 퇴직금 정산에 중간정산 사유가 남아 있음
- dev에서 바로 재현 성공
