# CI-4361: SAML 로그인 시 사용할 수 없는 계정 문구 노출

## 증상
- **문제 정의**: SAML 2.0[^7] SSO 로그인 시 "사용할 수 없는 계정이에요" 에러 노출
- **회사**: (주)랜덤워크 (Customer ID: 229748)
- **요청자**: 장민규 (CS팀)
- **대상자**: admin@randomwalk.co.kr
- **영향 범위**: 해당 계정 1명 (무료체험 중)
- **문제 시점**: 2026-04-08 오전
- 문의 내용:
  1. SAML 2.0 연동 환경에서 로그인 시 "사용할 수 없는 계정" 문구 노출
  2. 사내 PC, 개인 PC 동일 증상. 캐시 삭제, 시크릿 창에서도 동일[^1]

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 윤성복님이 SAML 로그인 세션을 확인하여 원인 특정[^2]
> → flex에 등록된 이메일: `admin@randomwalk.co.kr`
> → 실제 SSO(Azure AD)로 로그인한 이메일: `admin@rdwalk.com`[^2]
> → SAML assertion[^7]에 포함된 이메일이 flex에 등록된 이메일과 불일치
> → `Saml2VerificationProcessor`에서 이메일로 계정 조회 실패 → `AUTH_404_300`[^4]

### 확인 방법
`/api-public/v2/auth/verification/identifier`로 해당 이메일의 로그인 세션을 찾아 session id를 추출 후 검색[^3]

### 원인
- flex 등록 이메일(`admin@randomwalk.co.kr`)과 Azure AD SSO 이메일(`admin@rdwalk.com`)이 서로 다른 도메인[^2]
- SAML 인증 후 assertion의 이메일로 flex 계정을 조회하는데, `admin@rdwalk.com`에 매칭되는 계정이 없어 `AccountNotFoundSaml2Exception` (AUTH_404_300) 발생[^4]

## 영향 범위
- 해당 계정 1명에 국한
- SSO IdP[^8](Azure AD)에 등록된 이메일과 flex에 등록된 이메일이 다른 모든 사용자에게 동일 증상 발생 가능[^4]

## 전파 범위

- 해당 없음 — 인증 단계에서 차단되므로 하위 전파 없음

## 겸직/복수 소속

- 해당 없음

## 캐시/동기화

- 해당 없음

## 해결
- 고객사에 원인 안내: flex 등록 이메일과 SSO 로그인 이메일이 일치해야 함[^5]
- 고객사 측에서 둘 중 하나를 맞추면 해결:
  1. flex 등록 이메일을 `admin@rdwalk.com`으로 변경, 또는
  2. Azure AD에서 로그인 이메일을 `admin@randomwalk.co.kr`로 변경

## 다음에 같은 문의가 오면

1. **먼저 확인**: SAML 로그인 에러 코드가 `AUTH_404_300`인지 access log에서 확인
2. **원인 판별**: `/api-public/v2/auth/verification/identifier`로 로그인 세션을 찾아 session id 추출 → SAML assertion의 이메일과 flex 등록 이메일 비교
3. **조치**: 이메일 불일치면 고객사에 flex 이메일 또는 IdP 이메일 중 하나를 맞추도록 안내
