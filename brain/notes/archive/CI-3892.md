# CI-3892: 교대근무 스케줄에서 여러날 휴가가 표시 안 되고 배치 불가

> **상태**: BE 수정 PR 생성 완료 — 2026-02-12
> **PR**: https://github.com/flex-team/flex-timetracking-backend/pull/11811

- **회사**: (주)아이넷캅 (Customer ID: 214734)
- **문의자**: iseul.jang@inetcop.org
- **대상 구성원**: kyoungseop.kim@inetcop.org
- 문의 내용:
  1. kyoungseop.kim 구성원의 승인된 연차가 교대근무 스케줄표에 표시되지 않음
  2. 연차가 나타나지 않아 배치하려고 해도 배치가 안 됨

## 현재까지 파악된 내용

### 조사 과정

> 💡 **1차 확인 — BE 응답 정상 여부**
> → 김영준이 교대근무 스케줄 access log를 시각화한 결과, BE 응답에는 연차 데이터가 포함되어 있음을 확인 *(Linear 코멘트 @김영준(Enhance), 2026-02-12)*
> → 즉 BE는 정상이고, FE에서 뭔가의 이유로 연차를 숨기고 있는 상황

> 💡 **2차 확인 — draft 데이터 비어있음**
> → 이지선이 해당 구성원의 draft data가 비어있는 것을 확인 *(Linear 코멘트 @이지선, 2026-02-12)*
> → 김영준이 유지원 자리에서 직접 확인: 임시저장 상태를 복구하니 연차가 보였다가 다시 사라짐 *(Linear 코멘트 @김영준(Enhance), 2026-02-12)*

> 💡 **3차 확인 — 임시저장 API 응답 분석**
> → 이지선이 임시저장 로직을 분석:
>   - 임시저장 시 클라이언트가 빠른 UI 렌더링을 위해 연차를 직접 넣어줌
>   - 임시저장 API 응답이 오면 그 응답으로 교체함
>   - 그런데 **임시저장 API 응답의 draft에 연차가 포함되지 않음** (published에만 담겨옴)
> *(Linear 코멘트 @이지선, 2026-02-12)*

> 💡 **근본 원인 특정**
> → 이지선이 여러날에 걸친 휴가 등록 시 재현에 성공 (화면 녹화 첨부) *(Linear 코멘트 @이지선, 2026-02-12)*
> → **여러날 휴가가 등록된 날에 스케줄을 편집하면, 등록했던 여러날 휴가가 전부 날아가는 것이 근본 원인** *(Linear 코멘트 @이지선, 2026-02-12)*

### 관련 배경

- 김영준: 여러날 휴가 삭제를 못하게 하는 dry-run validation은 이미 들어가 있음 ([TT-15807](https://linear.app/flexteam/issue/TT-15807/교대-근무-drywet-run에서-여러날-휴가-검증-추가)) *(Linear 코멘트 @김영준(Enhance), 2026-02-12)*
- 안희종: 신 교대근무 draft에 여러날 휴가에 대한 처리가 안 되어 있을 것으로 판단, 여러날 휴가는 제거도 불가능한 걸로 알고 있었으나 draft는 아니었음 *(Linear 코멘트 @안희종, 2026-02-12)*
- 이지선: 구 교대근무 쪽도 이 처리가 안 되어 있었음. 이전에 구 교대근무에서 처리를 넣었다가 롤백한 적이 있음 *(Linear 코멘트 @이지선, 2026-02-12)*
- 이지선: 클라이언트에서 여러날 연차 삭제를 못하게 하려면 대공사가 필요한 상황 — x버튼 삭제 막기, delete key 삭제 막기, 스케줄 셀렉터 disabled, 툴바 처리, 복사&붙여넣기 처리 등 *(Linear 코멘트 @이지선, 2026-02-12)*

### FE의 여러날 휴가 판단 한계 및 BE 대응 필요

> 💡 **FE 여러날 휴가 판별 방식의 구조적 한계**
> → 현재 FE는 조회기간 내에 같은 `timeoffEventId`로 등록된 휴가가 있는지로 여러날 휴가인지를 판단함
> → **다른 조회기간에 걸친 휴가는 판단할 수 없는 구조적 한계가 있음**
> *(Slack 스레드 @이지선, 2026-02-12)*

- **BE 대응 필요**: 교대근무 조회 API 응답의 published에 등록된 휴가일 경우, **실제 휴가일(전체 날짜 범위)을 응답에 추가**해야 함
  - FE가 조회기간에 국한되지 않고 여러날 휴가 여부를 정확히 판단할 수 있도록 하기 위함

### 당면 조치 및 해결 방향

- **고객 임시 조치**: 여러날 연차 등록했던 날을 **전부 다** 선택 → 초기화하면 근무표에 연차가 다시 보임 (이지선이 안내, 화면 녹화 첨부) *(Linear 코멘트 @이지선, 2026-02-12)*
- **영구 수정 방향 (FE)**: "여러날 휴가가 등록된 날은 아예 편집을 못하게 막는 처리"를 해서 prod에 내보낼 예정 *(Linear 코멘트 @이지선, 2026-02-12)*
- **영구 수정 방향 (BE)**: 교대근무 조회 API 응답에 published 휴가의 실제 휴가일 정보를 추가 *(Slack 스레드, 2026-02-12)*
