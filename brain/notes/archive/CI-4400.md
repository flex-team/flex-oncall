# CI-4400: 과거 요청한 일 내역 클릭 시 알 수 없는 오류 발생

## 증상

- **문제 정의**: 요청한일(todo/sent)에서 과거 외근 승인 요청 클릭 시 "알 수 없는 오류" 발생
- **회사**: (주)하이컨시 (Customer ID: 216339)[^1]
- **요청자**: hjjeon@hiconsy.com[^1]
- **대상자**: 미특정 (문의자 본인으로 추정)[^1]
- **영향 범위**: 삭제된 근무정책(workForm)에 연결된 승인 건을 가진 모든 사용자[^7]
- **문제 시점**: 2025년 12월 요청한 외근 승인 건[^1]
- 문의 내용:
  1. 요청한일(진행 중) 목록에서 과거 할일을 클릭하면 "알 수 없는 오류" 발생[^1]
  2. 해당 건은 12월에 요청한 외근 승인 요청[^1]
  3. 외근 정책을 중간에 삭제한 적이 있음[^1]

## 원인 분석

### 조사 과정

> 💡 Sentry에서 해당 사용자의 에러가 캡처된 것을 확인[^6]
> → 이해나가 Sentry 로그 분석 — FE에서 실패한 것으로 판단[^7]
> → 삭제된 근무정책(workForm)에 매핑되는 항목이 없어서 error throw[^7]
> → 이지선이 이미 develop에 수정 배포해둔 상태 확인[^9]

### 원인

- **프론트엔드 버그**: 근무 미리보기 위젯에서 삭제된 근무정책(workForm)을 매핑하려 할 때, 매핑 대상이 없으면 error를 throw하는 구조[^7]
- API는 정상 응답하지만, FE에서 workForm 매핑 실패 시 적절한 empty view를 보여주지 않고 에러를 던짐[^8]

## 영향 범위

- 삭제된 근무정책에 연결된 과거 승인 건을 조회하는 모든 사용자에게 동일 증상 발생 가능[^7]
- 금전적/법적 영향 없음 — 조회 오류일 뿐 데이터 변경 없음

## 해결

- 이지선이 develop에 수정 배포 완료[^9]
- 이해나가 체리픽 핫픽스 PR 생성[^10]: [flex-frontend-apps-time-tracking#2470](https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2470)
- 원본 커밋: [PR #2466의 커밋 afc83463](https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2466/changes/afc83463c6d9458dc11f5cb446dbd5f0b1f7be60)[^11]

## 발견한 스펙/제약

- 근무정책(workForm) 삭제 시 해당 정책에 연결된 기존 승인 건의 FE 렌더링이 고려되지 않았음[^7]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 문의자가 클릭하는 할일의 근무 유형(외근/재택 등)에 연결된 근무정책이 삭제되었는지 확인
2. **원인 판별**: Sentry에서 해당 사용자 이메일로 FE 에러 검색. workForm 매핑 실패 에러면 이 건과 동일
3. **조치**: 핫픽스(PR #2470) 배포 후에는 삭제된 정책도 empty view로 표시되어 오류 없음. 배포 전이면 핫픽스 배포 독촉
