# CI-4327: 진행중인 평가 등급 배분율 설정 수정 요청 (라라스윗)

- **문제 정의**: 평가 단계가 시작된 후 등급 배분율 설정을 변경할 수 없어, 인원 초과 시 제출 제한 비활성화 불가

- **요청자**: hwanghj@lalasweet.kr[^1]
- **대상자**: 라라스윗 전체 평가 담당자
- **영향 범위**: 라라스윗 — [라라스윗] 2026 상반기 평가 대상자
- **문제 시점**: 2026-04-06 (평가 진행 중)
- 문의 내용:
  1. 등급 배분율 설정에서 배분율 초과 시 제출 불가 옵션이 활성화되어 있음
  2. 평가할 팀원이 2명인데 둘 다 A 등급을 부여하고 싶어도 제출이 안 됨
  3. 인원 초과 시 제출 제한 비활성화 요청[^1]

## 원인 분석

### 조사 과정
> 💡 **판단 근거**: 평가 단계가 시작된 이후에는 평가 설정 UI 수정이 불가능한 스펙[^2]
> → `flex_review.evaluation_grade_distribution.allow_submission_if_exceed = 0` 설정이 문제[^3]
> → UI로 변경 불가하므로 DML 보정 필요
> → [라라스윗] 2026 상반기 평가_계약직은 배분율 설정 자체가 없어 대상 제외[^2]

### 원인
- `evaluation_grade_distribution.allow_submission_if_exceed = 0` (제출 제한 활성화 상태)[^3]
- 평가 단계 시작 후에는 UI에서 설정 변경이 불가능한 시스템 스펙[^2]

## 영향 범위
- 라라스윗 단일 고객사의 특정 평가 1건
- 금전적·법적 영향 없음 (평가 등급 배분 설정 수정)

## 해결
- `evaluation_grade_distribution.allow_submission_if_exceed = 1` 로 DML 업데이트[^3]
- 적용 대상: `evaluation_id = '01kmczjvyqd5yyvqp9vqkdb8mj'`, `id = '01kmcztk7syakpac7y5sege9rp'`
- [라라스윗] 2026 상반기 평가_계약직은 배분율 설정 없음 → 제외[^2]

## 발견한 스펙/제약
- **평가 단계 시작 후 설정 변경 불가**: 평가 진행 중에는 등급 배분율 설정 UI 편집이 잠긴다. 변경이 필요하면 DML 보정으로만 가능하다.[^2]
- `allow_submission_if_exceed`[^3]: 0 = 배분율 초과 시 제출 차단, 1 = 초과해도 제출 허용

## 다음에 같은 문의가 오면

1. **먼저 확인**: `flex_review.evaluation_grade_distribution`에서 해당 evaluation의 `allow_submission_if_exceed` 값 확인
   ```sql
   SELECT id, evaluation_id, grade_factor_id, allow_submission_if_exceed
   FROM flex_review.evaluation_grade_distribution
   WHERE customer_id = ? AND evaluation_id = ?;
   ```
2. **원인 판별**: `allow_submission_if_exceed = 0`이면 배분율 초과 시 제출 차단 설정. 평가가 이미 시작되어 UI 변경 불가라면 DML 보정 필요
3. **조치**: 결재 후 DML 실행
   ```sql
   -- 백업
   UPDATE flex_review.evaluation_grade_distribution
   SET allow_submission_if_exceed = 0 WHERE id = ?;
   -- 적용
   UPDATE flex_review.evaluation_grade_distribution
   SET allow_submission_if_exceed = 1
   WHERE customer_id = ? AND evaluation_id = ? AND id = ?;
   ```
