# CI-4399: 정산결과 지급액 표기 0원 데이터 보정 요청

## 증상

- 회사: 주식회사 매드엔진 (customer_id=2879)
- 23.12 [정기급여] 정산결과 지급액이 0원으로 표시됨
- 퇴사자 급여명세서 보관을 위해 다운받다가 발견
- CI-4227, CI-4265, CI-4381과 동일 패턴

## 문제 평가

> backfill 누락으로 인한 지급액 0원 표시 · customer_id=2879 단일 고객사 · 낮음 (데이터 조회 이상, 금전 영향 없음)

- **성격**: work_income_payment_settlement_payee_result_item backfill 누락 — CI-4227에서 확인된 반복 패턴
- **범위**: customer_id=2879 단일 고객사, settlement_result_id=44842 (23.12 정기급여, 173명 전원 미마이그레이션)
- **긴급도**: 낮음 — 표시 문제이며 실제 급여 지급에는 영향 없음
- **조사 방향**: migrate-ci4227 Operation API로 해결 / COOKBOOK: F-PAY-002

## 원인 분석

- **verdict**: ops
- CI-4227에서 확인된 근본 원인과 동일: 2025-03-19 backfill 시 일부 settlement result의 payee result item이 신 테이블로 마이그레이션되지 않음
- settlement_result_id=44842 (2023-12-01): 173명 전원 미마이그레이션 상태
- 그 외 44개 settlement result에도 소수의 미마이그레이션 payee가 산재

## 해결

- migrate-ci4227 Operation API 실행으로 해결
- dryRun 확인 후 wetRun 수행
  - `POST /proxy/payroll/api/operation/v2/payroll/settlement-results/payment/item/migrate-ci4227/2879?dryRun=false`
  - traceId: `01b6154538ae2449914ff38476d6e0a7`
  - 결과: settlementResultIds=[44842], migratedPayeeCount=198, skippedPayeeCount=13958
- wetRun 후 검증 쿼리로 마이그레이션 완료 확인
  - settlement_result_item: 954건
  - settlement_payee_result_item: 53823건
- 고객사 확인 완료 (김한나: "확인하셨다고 합니다")
