# CI-3842: home 화면 접속 시 yearend settlements API 400 에러

> **상태**: Done — 2026-02-09 (KST)
> **verdict**: bug-fix — 연말정산 대상자가 아닌 유저 세션에서 연말정산 API 호출 시 예외 발생

## 증상

- **문제 정의**: DEV 환경에서 home 화면 접속 시 `api/v2/yearend/settlements/2025/users/{userIdHash}` 에서 400 에러가 반복 발생. 새로고침해도 지속. QA 환경에서도 동일 현상 확인.
- **회사**: 내부 (DEV/QA 환경)
- **요청자**: 이지선 (내부)
- **라벨**: Payroll Squad, bug, 연말정산
- **담당자**: 이원국
- **Linear**: [CI-3842](https://linear.app/flexteam/issue/CI-3842/home-화면-접속시-apiv2yearendsettlements2025usersuseridhash-에서-400-에러가-발생해서)

## 원인 분석

- 이지우가 원인 추정: [flex-frontend-apps-payroll PR #1484](https://github.com/flex-team/flex-frontend-apps-payroll/pull/1484) 의 여파 — develop 향 배너 로직 작업
- 이원국이 확인: expose된 microApp(연말정산 배너 - `ForcedYearEndToDoItem`)에서 **해당 연도 연말정산 대상자로 등록되지 않은 유저 세션**으로 `getYearEndSettlementUserByUserId`를 호출 → throw Exception 발생
- 사전 조건(연말정산 대상자 여부) 확인 없이 API를 호출한 것이 원인

## 해결

- **코드 수정**: 사전 조건이 충족되지 않은 경우 API를 호출하지 않도록 수정
- 하위 티켓 [PAY-15363](https://linear.app/flexteam/issue/PAY-15363) 생성하여 이원국이 작업
- QA로 픽스 머지 후 DEV로 백머지하는 방식으로 배포
- 이원국이 DEV, QA 모두 배포 완료 후 재현 계정에서 픽스 확인
