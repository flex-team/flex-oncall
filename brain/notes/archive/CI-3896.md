# CI-3896: 근무지 스케줄 잘못 추가 후 삭제 불가

> **상태**: Done — 2026-02-12 (KST)
> **verdict**: bug-fix — 스케줄 편집 권한이 근무지 내 조직만 바라보던 버그 수정

## 증상

- **문제 정의**: 호텔 근무지에서 스케줄을 추가한 후 근무지의 사용 조직을 변경했더니, 기존 스케줄(변경 전 조직에 소속)의 편집/삭제가 불가능해짐. 최고관리자여도 편집 불가 상태 발생.
- **AS-IS**: [호텔] 근무지에 하위 조직 포함으로 생성 → b레스토랑 오픈 스케줄 추가
- **TO-BE**: 근무지명을 "호텔-식음료부_A레스토랑"으로 변경, 사용 조직을 식음료부-A레스토랑으로 변경 → b레스토랑 스케줄 편집/삭제 불가
- **라벨**: Tracking Squad, bug, 근태/근무/휴가
- **담당자**: 이지선
- **Linear**: [CI-3896](https://linear.app/flexteam/issue/CI-3896/근무지-스케줄을-잘-못-추가했는데-삭제가-되지-않습니다)

## 원인 분석

- 스케줄 편집 권한을 "스케줄의 사용 조직에 대해 근무 편집 권한이 있는가"로 변경하면서 발생한 사이드 이펙트
- **근본 원인**: 스케줄 편집 권한 체크 시 `근무지에 속한 조직 중 편집 가능한 조직`을 기준으로 판단 → 스케줄의 사용 조직이 근무지 조직에 속하지 않으면 편집 불가로 처리됨
- 근무지의 조직을 나중에 변경하면, 기존 스케줄의 사용 조직이 근무지에 속하지 않은 상태가 되어 최고관리자도 편집 불가
- 안희종이 지적한 올바른 로직: (1) 내가 현재 '사용 조직'에 대한 편집 권한이 없거나, (2) 근무지 내의 조직 중 편집 가능한 조직이 하나도 없는 경우에만 편집 불가

## 해결

- **수정**: 스케줄 편집 권한을 근무지 내 조직이 아닌 **회사의 전체 조직**을 바라보도록 변경
- **PR**: https://github.com/flex-team/flex-frontend-apps-time-tracking/pull/2117
- 수정 후 배포 완료, 이주화 확인 완료 ("와아 수정됩니다~")
