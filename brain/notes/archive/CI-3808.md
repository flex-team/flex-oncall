# CI-3808: 연말정산 체크리스트 반영 후 최종 세액에서 제외됨

> **상태**: Done — 2026-02-13 (KST)
> **verdict**: user-error + operation — 체크리스트 요건 미충족으로 자동 제외, 오퍼레이션 API 부재로 수동 대응

## 증상

- **문제 정의**: 아쿠쉬네트코리아 '신가연' 님의 장기주택저당차입금 이자상환액 체크리스트를 포함 체크한 뒤 `신고 포함 > 결과 확인 > 확정하기`로 진행하면, 최종 세액란에서 해당 항목이 제외되어 미반영 상태로 되돌아감. 최종검토기간이 마감되어 관리자가 직접 반영하려 했으나 체크리스트 수정 불가.

## 원인 분석

- 이원국이 로그 조회 결과, 해당 구성원이 **주택취득 당시 기준시가가 공제 대상 요건에 해당하지 않도록 선택**하여 자동 제외된 것으로 확인
- 체크리스트 데이터:
  ```json
  {
    "houseTake": "NOT_DEDUCTED",
    "houseTakeDetail": {
      "allOneHouse": "YES",
      "householder": "YES",
      "whenAcquiredHasAnyMaintenanceCondition": "YES",
      "whenAcquiredHasAnyCostCondition": "NO"  // ← 이 값이 NO여서 공제 대상 미충족
    }
  }
  ```
- `whenAcquiredHasAnyCostCondition: "NO"`로 인해 공제 요건 미충족 → 시스템이 자동 제외 처리한 것
- 버그가 아니라 사용자가 체크리스트를 잘못 작성(혼동)한 케이스

## 해결

- 체크리스트는 관리자가 대리 수행할 수 없는 구조
- 두 가지 방법 제시:
  1. 담당자가 작성기간을 수정해서 접근 가능하도록 열고, 해당 유저가 직접 수정
  2. 체크리스트 데이터를 원하는 상태로 밀어넣는 오퍼레이션 수행 후 관리자가 재검토
- 오퍼레이션 API가 없어 **방법 1(작성기간 수정 후 유저 직접 수정)으로 안내**
- 향후를 위해 **오퍼레이션 API 생성 티켓을 CI-3808 subissue로 생성**함
