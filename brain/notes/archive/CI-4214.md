## 원인 분석

**프론트엔드 폼 검증 버그** — 최근 추가된 대상자 input 길이 검증이 엑셀 업로드 경로를 고려하지 않음.

> 💡 **확정 원인**: 최근 프론트엔드에 폼 검증이 추가되어, 선택된 대상자 input의 길이가 0이면 not-valid로 판정한다. 엑셀 업로드 경로에서는 대상자를 input 필드가 아닌 엑셀 파일로 선택하므로 input 길이가 항상 0 → 폼이 유효하지 않다고 판단하여 다음 단계로 진행 불가.[^8]

## 해결

- 이지우 님(FE)이 폼 검증 로직 수정 진행 중[^8]
- 엑셀 업로드 경로에서는 대상자 input 길이 검증을 스킵하거나, 엑셀 데이터 기반으로 유효성을 판단하도록 변경 예상
- BE 조치 불필요

## 다음에 같은 문의가 오면

1. **먼저 확인**: 인사발령 > 엑셀로 발령 경로에서 버튼이 반응하지 않는지 직접 재현 시도
2. **원인 판별**: 버튼 클릭 후 API 호출이 발생하는지 브라우저 네트워크 탭 확인
   - API 호출 없음 → FE 폼 검증 이슈 (이 건과 동일 유형)
   - API 호출 있지만 에러 → BE 이슈 (validate API 응답 확인)
3. **조치**: FE 이슈이면 프론트엔드 담당자(이지우 님 등)에게 전달
