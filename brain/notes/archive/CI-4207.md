# CI-4207: 세콤 배치 동기화 시 중복 START 등록으로 인한 자동퇴근 실패 알림

> **상태**: 수정 완료 (PR 생성) — 2026-03-25
> **compact**: 원본 186줄 → 정제 112줄 (2026-03-26). git log로 원본 추적 가능.
> **verdict**: bug — `checkWorkClockStatus()` eventTimeStamp 필터 역전
> **PR**: [#12058](https://github.com/flex-team/flex-timetracking-backend/pull/12058) (코드 수정 + 시나리오 테스트)

## 증상

- **문제 정의**: 세콤으로 퇴근했으나 다음 날 새벽 "근무를 종료하지 못했어요" 알림(`WORK_CLOCK_STOP_FAILURE_GO_CONFIRM`) 수신
- **회사**: 조앤강 주식회사 (Customer ID: 218902)
- **영향 범위**: 1명 확인 (동일 패턴의 세콤 배치 동기화 사용 고객사에서 재발 가능)
- **문제 시점**: 2026-03-24 18:55경 퇴근 → 2026-03-25 05:10 알림 수신

## 원인 분석

### 결론 (L1)

세콤 배치 동기화로 출근 이벤트 2건이 거의 동시에 수신될 때, `checkWorkClockStatus()`의 타임스탬프 필터가 기존 START를 누락하여 **중복 START가 등록**되는 버그. 퇴근(STOP)이 두 번째 START에만 연결되어 첫 번째 START가 미종료 상태로 남고, AUTO_WORK_STOP이 이를 감지하여 실패 알림을 발송함.

### 핵심 증거 (L2)

1. 세콤에서 출근 이벤트 2건이 배치로 동시 수신됨 (실제 타각: 09:10:38, 09:11:15 / flex 등록: 09:12:30, 09:12:31)[^5]
2. `checkWorkClockStatus()` 필터에서 `eventTimeStamp`(flex 등록 시각)을 `targetTime`(실제 타각 시각)과 비교[^1]:
   - 첫 번째 START의 `event_time_stamp` truncated = **09:12:00**[^5]
   - 두 번째 이벤트의 `targetWorkStartTimeStamp` = **09:11:15**[^5]
   - 09:12:00 <= 09:11:15 → **FALSE** → 첫 번째 START가 필터에서 제외됨
3. `onGoingEventPack = null` → `WorkClockContinuousRegisterValidator` 통과[^2] → 중복 START 등록

> **판단 근거**: `eventTimeStamp`(flex 등록 시각)과 `targetTime`(실제 타각 시각)의 역전이 핵심.
> 세콤 배치 동기화에서는 등록 시각이 실제 타각 시각보다 늦어지므로, 시간순으로 뒤의 이벤트를 먼저 dry-run할 때 기존 START를 찾지 못함.

### 5 Whys (L3)

<details><summary>5 Whys 전개</summary>

1. **왜 "근무를 종료하지 못했어요" 알림이 발생했나?** → AUTO_WORK_STOP이 미종료 START에 대해 실행 → dry-run 실패[^10]
2. **왜 미종료 START가 있나?** → STOP이 두 번째 START(126672702)에만 연결, 첫 번째(126672691)는 미종료[^5]
3. **왜 START가 2건 등록됐나?** → 두 번째 dry-run에서 `WorkClockContinuousRegisterValidator` 가 통과[^2]
4. **왜 Validator가 통과했나?** → `onGoingEventPack = null`로 판단 (첫 번째 START가 필터에서 제외됨)[^1]
5. **왜 필터에서 제외됐나?** → `eventTimeStamp`(09:12:00 truncated) > `targetTime`(09:11:15)으로 필터 조건 불충족. 세콤 배치 동기화로 등록 시각이 실제 타각 시각보다 늦기 때문[^1][^5]

</details>

## 해결

### 원인 커밋
`0c2275343f` (#11865, 2026-02-27) — ON_TIME 보정 출근 버그 수정 시 필터 기준을 `targetTime` → `eventTimeStamp.truncatedToMinutes()`로 변경. 이 변경이 세콤 배치 동기화에서 역전을 유발[^12].

### 수정
`min(eventTimeStamp, targetTime)` 사용으로 ON_TIME과 세콤 배치 양쪽 모두 해결[^13].

### PR
- [#12058](https://github.com/flex-team/flex-timetracking-backend/pull/12058) — 코드 수정 + 주석 보완 + 시나리오 테스트
- [#12060](https://github.com/flex-team/flex-timetracking-backend/pull/12060) — 조사 과정에서 독립 생성 (12058로 통합 예정)
