# CI-4222: 건강보험 연말정산 분할납부금액이 전년도 납부금액에 합산됨

> **상태**: 해결 완료 — 2026-03-26

## 증상

- **문제 정의**: 건강보험 연말정산 시 2024년도 분할납부금액(74번 코드)[^1]이 2025년도 기납보험료[^12]에 잘못 합산됨
- **회사**: (주)위시켓 (Customer ID: 34686)
- **요청자**: CS팀 @정인지[^2]
- **대상자**: pjw@wishket.com[^2]
- **영향 범위**: 8개 고객사 / 15명 — 2/27 이후 확정된 정산에서 중도정산이 실행된 대상자 중 26년 1~3월 74번 코드 금액이 0이 아닌 구성원[^15]
- **문제 시점**: 2025년 연말정산 처리 시[^2]
- **라벨**: 신고/세금
- **담당자**: 이한결 (현재 휴가 중)[^5]
- 문의 내용:
  1. 2026년에 공제한 2024년도 건강보험 정산 분할납부금액(74번 코드)이 2025년도 납부금액으로 합산되고 있음[^2]
  2. 이로 인해 보험료가 실제보다 덜 공제될 수 있어, 고객사가 개인에게 보험료를 회수해야 하는 상황 발생 가능[^2]

## 원인 분석

**버그 확정** — `PaidSocialInsuranceCalculator.getYearEndTotalAmountByType()`이 74번 코드의 귀속연도를 구분하지 않고 전부 기납보험료에 합산한다[^6].

CI-4174 핫픽스(2월 말)에서 "귀속연도[^13] 다음연도의 연말정산 보험료를 기납보험료에 합산"하는 로직을 추가했는데[^7], 74번 코드(정산분할고지보험료)는 **원래 귀속연도와 무관하게** `YEAR_END_REASON_CODES`에 포함되어 있어 무조건 합산 대상이 된다[^8].

핵심 증거:
1. 74번은 `YEAR_END_REASON_CODES = setOf(10, 20, 57, 67, 71, 74, 75)`에 포함[^8] → `isYearEnd=true`
2. `getYearEndTotalAmountByType()`은 `yearEndSettlementResult.healthInsurance`를 **전부** 합산 — 귀속연도 필터 없음[^6]
3. 2024년도 정산 분할납부(74번)가 2026년에 공제 → 2026년 정산 결과에 포함 → 2025년 기납보험료로 잘못 합산

<details>
<summary>5 Whys 체인</summary>

1. **왜 74번 금액이 2025년 기납보험료에 합산되나?** → `PaidSocialInsuranceCalculator.getPaidInsurance(thisYear, nextYear)`가 2026년 정산 결과의 `yearEndSettlementResult`를 전부 합산[^6]
2. **왜 전부 합산하나?** → `getYearEndTotalAmountByType()`에 귀속연도 필터가 없음[^6]
3. **왜 74번이 yearEnd 경로로 들어가나?** → `YEAR_END_REASON_CODES`에 74번이 포함[^8]
4. **왜 이전에는 문제가 없었나?** → CI-4174 핫픽스(2월 말) 이전에는 "귀속 다음연도 연말정산 보험료 합산" 로직 자체가 없었음[^7]
5. **왜 핫픽스에서 74번을 구분하지 않았나?** → 74번의 귀속연도 차이(2024년 정산 분할이 2026년에 공제)가 핫픽스 시점에 고려되지 않았음(추정)

</details>

### 스펙 vs 버그 판별

**버그**. 74번 코드(2024년도 건강보험 정산 분할납부)는 2024년 귀속이므로 2025년 기납보험료에 포함되면 안 된다. 코드에 의도적 분기 처리가 없으며, Slack에서도 버그로 합의되었다[^3].
