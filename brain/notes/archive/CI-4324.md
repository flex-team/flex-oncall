# CI-4324: 지출결의 영수증 업데이트 시 선택 초기화 및 무한 업데이트 요구

> **compact**: 원본 182줄 → 정제 82줄 (2026-04-06). git log로 원본 추적 가능.

## 증상
- **문제 정의**: 지출결의[^8] 작성 시 "업데이트 하기" 클릭하면 기존 선택 영수증이 전부 해제되고, 이미 업데이트한 영수증을 재선택해도 다시 "업데이트 하기"가 표시됨
- **회사**: 버즈앤비 (Customer ID: 150211)
- **요청자**: 이주화 (CS팀)
- **대상자**: glen@bzznbyd.com[^1]
- **영향 범위**: **서비스 전반 이슈** — 최소 5개 고객사에서 동시 인입, 전체 구성원 발생 확인[^9]
- **문제 시점**: 2026-04-06

## 원인 분석

**Bullseye 매트릭스 색인 누락 → `findAllBy()` 빈 결과 → compare API 빈 배열 → FE가 `search-with-ids`를 빈 배열로 호출 → 선택 영수증 전부 사라짐.**

이해나가 access log에서 compare API 빈 배열 응답을 확인했고[^4], 임형태가 matrixQL로 `fins_spending_entire_v1` 색인 상태를 검증하여 원인을 확정했다[^5].

### 조사 과정

> 💡 **판단 근거**: compare API(속성/버전 비교) 코드 분석에서 `modified`, `needCheck`, `needCheckPop` 모두 false → 이해나가 compare 빈 배열 응답 발견[^4] → 이성환이 매트릭스 색인 이슈 가능성 제기[^6] → 임형태 matrixQL 검증으로 확정[^5]

### 원인
1. **표면 원인**: `fins_spending_entire_v1` Bullseye 색인이 누락되어 compare API가 빈 배열 반환[^5]
2. **근본 원인**: 색인 누락 원인은 미확정 (배포 전후 색인 동기화 실패 가능성)

## 영향 범위
- **서비스 전반** — 최소 5개 고객사(버즈앤비 150211, 무하유 69518, 딜리버드로지스틱스 182346, 코니바이에린 501, 유니브 215569)[^9]
- 금전적/법적 영향: 없음 (지출결의 제출 자체가 차단되는 것, 잘못된 금액 처리 위험 없음)

## 해결
- 버즈앤비 전체 문서 재색인 완료[^2]
- 비용관리 사용 전체 고객사 재동기화 완료 (KST 13:36)[^3]
- 버즈앤비 이슈 해결 확인 (KST 13:46)[^7]
- 전체 고객사 해소 확인 (KST 13:52, 14:36)[^10][^11]
- 근본 원인(왜 색인이 깨졌는지)은 별도 후속 조사 필요

## 다음에 같은 문의가 오면

1. **먼저 확인**: compare API(`/api/v2/electronic-approval/documents/receipts/compare`) access log에서 응답 `list` 필드가 빈 배열인지 확인
2. **원인 판별**:
   - 빈 배열이면 → Bullseye 매트릭스 색인 이슈. 임형태에게 연락, `fins_spending_entire_v1` 재동기화 요청
   - modified/needCheck=true 이면 → 영수증 속성 변경 정상 동작 (고객 안내)
3. **조치**: `POST /api/operation/v2/fins/spending/customers/{customer_id}/synchronize` 또는 전체 재동기화 요청
