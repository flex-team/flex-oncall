# CI-4220: 워크플로우 임시저장된 문서가 사라짐

> **상태**: 해결 완료 — 2026-03-26

## 증상
- **문제 정의**: 워크플로우 기안서 작성 중 화면을 벗어난 후 돌아왔을 때 임시저장된 문서가 사라짐
- **회사**: 퀸라이브 (Customer ID: 209434)
- **요청자**: 이주화 (CS팀)
- **대상자**: sally@queenlive.co.kr (userId: 807961, 닉네임: 셀리)
- **영향 범위**: 1명
- **문제 시점**: 2026-03-25 18:25경[^1]
- 문의 내용:
  1. "기안서(일반) - 제목을 반드시 수정해주세요" 양식으로 문서를 작성 중이었음[^2]
  2. 작성 중간에 작성 화면을 벗어남
  3. 이어서 작성하기 위해 돌아왔으나 이전에 작성한 내용이 초기화되어 있음[^3]

## 원인 분석

**"할일 > 작성하기" 목록에서 작성하기를 재선택하면, 해당 양식의 임시저장 문서가 초기화(재생성)된다.**

### 조사 과정

> 💡 **판단 근거**: 임형태가 access log를 추적하여 원인 특정[^8]
> → 16:09 — 임시저장 최초 생성[^8]
> → 16:14 — "마케팅 크레딧 보상 품의서"로 1차 작성 완료[^8]
> → 18:25:34 — "작성하러가기" 할일 조회[^8]
> → **18:25:42 — 임시저장된 문서 초기화됨**[^8]

### 원인
- **표면 원인**: 사용자가 "할일 > 작성하기"를 다시 선택하여 기존 임시저장 문서가 초기화됨[^9]
- **근본 원인**: 작성하기 재진입 시 기존 draft를 불러오지 않고 새로운 draft로 재생성하는 동작[^9]

### 가설 목록

| # | 가설 | 상태 |
|---|------|------|
| 1 | Draft 저장 API 호출 자체가 실패 | ❌ 소거 (89건 전부 200 OK)[^5] |
| 2 | Draft 데이터가 DB에서 삭제됨 | ❌ 소거 (`workflow_task_draft` 레코드 존재)[^6] |
| 3 | `workflow_task` 레코드 미생성으로 목록에서 누락 | ❌ 소거 (실제 원인은 할일 재선택에 의한 초기화)[^8] |
| 4 | "작성하기" 재진입 시 기존 draft 초기화 | ✅ 확정[^8] |

## 해결
- 임형태가 draft 데이터를 추출하여 CS 담당자(이주화)에게 DM으로 전달[^10]
- 이주화가 종료 가능 확인[^11]

## 발견한 스펙/제약
- 기안서 자동저장은 백엔드 기능: `POST /api/v3/approval-document/approval-documents/draft` → `flex.workflow_task_draft` 테이블[^6]
- **"할일 > 작성하기" 재진입 시 기존 임시저장 문서가 초기화된다** — 사용자가 의도하지 않은 데이터 소실 가능[^9]
- v3 approval-document 모듈은 flex-flow-backend에서 제거되어 별도 서비스로 운영 중[^7]

## 다음에 같은 문의가 오면

1. **먼저 확인**: `flex.workflow_task_draft` 테이블에서 해당 사용자의 draft 존재 여부 확인
   ```sql
   SELECT id, user_id, workflow_task_key, created_date, last_modified_date
   FROM flex.workflow_task_draft
   WHERE user_id = ? AND customer_id = ?
   ORDER BY last_modified_date DESC;
   ```
2. **원인 판별**:
   - draft가 있고 내용이 초기화된 상태 → "작성하기 재진입"에 의한 초기화 (이 건과 동일)
   - draft 자체가 없음 → 다른 원인 (보관 기한 만료 등)
3. **조치**: draft data의 `text` 필드에서 HTML 본문을 추출하여 고객에게 전달. DB 수삽 복구는 위험하므로 지양.
