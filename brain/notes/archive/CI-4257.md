# CI-4257: 계약서 선택 발송 시 미발송 임시저장 계약서 삭제됨

## 증상
- **문제 정의**: 다수의 계약서 임시저장 후 일부 인원만 선택하여 발송하면 나머지 미발송 계약서가 삭제됨
- **회사**: 미확인[^1]
- **요청자**: @장지은 (CS)[^2]
- **대상자**: 미확인[^1]
- **영향 범위**: 20명 가량의 계약서 셋팅 데이터 삭제 → 재작업 필요[^2]
- **문제 시점**: 2026-03-31 접수[^3]
- 문의 내용:
  1. 다수의 계약서 임시저장 후 일부 인원만 선택하여 발송하면 나머지 미발송 계약서가 삭제됨[^2]
  2. 선택 발송 이후에도 다른 임시저장 값은 그대로 남아있어야 함[^2]

## 원인 분석

### 조사 과정

> 💡 `DigiconCandidateSetExecuteServiceImpl.execute()` 코드 분석에서 출발[^4]
> → `targetDigiconCandidateUnitIdentities`가 비어있지 않으면, 전체 CandidateUnit[^14] 중 미선택 건을 필터링하여 `deleteAllById()`로 물리 삭제[^4]
> → `DigiconCandidateSetExecuteRequest.targetDigiconCandidateUnitIdHashes`는 nullable[^5] — 값이 있으면 선택 발송, null이면 전체 발송
> → 현재 설계는 **하나의 CandidateSet[^15] = 한 번의 발송**으로 간주하는 구조
> → 임준규(PE) 확인: "스펙입니다"[^11]
> → 이윤선 확인: VOC-2410에 개선사항으로 이미 다수 등록되어 있으며, 현재 스펙으로 확인[^12]

### 원인
- 선택 발송 시 미선택 CandidateUnit을 물리 삭제하는 것은 **의도된 설계(스펙)**[^11]
- 하나의 CandidateSet이 한 번의 발송 단위로 설계되어 있어, 발송 실행 시 미선택 건을 정리하는 구조[^4]

### 발송 흐름

```
사용자: 20명 임시저장 → 5명만 선택 발송
  ↓
execute() 호출 (targetDigiconCandidateUnitIdHashes = [5명])
  ↓
나머지 15명의 DigiconCandidateUnit → deleteAllById() 로 물리 삭제
  ↓
CandidateSet 상태: IN_REVIEW → QUEUED
  ↓
결과: 15명분 임시저장 데이터 영구 삭제, 복구 불가
```

## 해결
- **스펙 확인**: 현재 동작은 의도된 설계. 수정 불필요[^11]
- **UX 개선 요청**: VOC-2410으로 이미 개선사항 등록됨 — "전자계약 부분 취소 및 임시저장 후 대상자 조정 지원"[^12]
- 이윤선이 UX 개선 방향 검토 중[^13]

## 발견한 스펙/제약
- CandidateSet은 `IN_REVIEW`[^16] 상태에서만 삭제 가능 (`deletable()` = IN_REVIEW only)[^8]
- 선택 발송 시 미선택 Unit은 **물리 삭제**되며 복구 경로 없음 — 이것이 현재 스펙[^4][^11]
- 하나의 CandidateSet = 한 번의 발송 단위. 부분 발송 후 나머지를 유지하는 기능은 미구현[^4]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 이 동작은 현재 스펙임을 안내 — 선택 발송 시 미선택 계약서는 삭제되는 것이 의도된 동작
2. **원인 판별**: 사용자가 "일부만 발송했더니 나머지가 사라졌다"고 하면 이 스펙에 해당
3. **조치**: 삭제된 임시저장 데이터는 복구 불가. 재작업 필요. VOC-2410으로 개선 요청이 등록되어 있음을 안내

## 시나리오

```gherkin
# language: ko
기능: 전자계약 대량 발송 세트의 선택 발송

  시나리오: 임시저장된 대량계약 세트에서 일부 인원만 선택하여 발송
    주어진 20명의 계약서가 임시저장(IN_REVIEW) 상태인 대량계약 세트가 있다
    만약 관리자가 5명만 선택하여 발송을 실행한다
    그러면 선택된 5명의 계약서만 발송 처리된다
    그리고 나머지 15명의 임시저장 계약서(DigiconCandidateUnit)는 물리 삭제된다
    그리고 삭제된 임시저장 데이터는 복구할 수 없다

  시나리오: 임시저장된 대량계약 세트에서 전체 인원을 발송
    주어진 20명의 계약서가 임시저장(IN_REVIEW) 상태인 대량계약 세트가 있다
    만약 관리자가 대상자를 지정하지 않고 발송을 실행한다
    그러면 전체 20명의 계약서가 발송 처리된다
    그리고 어떤 임시저장 계약서도 삭제되지 않는다
```
