## 원인 분석

- 권한/인증 문제가 아닌 **internal API client 타임아웃** 이슈
- 엑셀 다운로드 시 스토리지 internal API 호출이 3.49초 소요되었으나, 클라이언트 타임아웃이 3.15초로 설정되어 있어 타임아웃 오류 발생
- SK케미칼은 휴가조회권한 있는 유저 모두 조회 대상이라 데이터량이 많아 시간 초과
- TT 액셀 다운로드 실패가 최근 잦았으며, **타임아웃 오류가 TT 오류 중 대다수**를 차지

## 해결

- **즉시 대응**: 이슈 보고 시점에 이미 다운로드 가능 상태로 복구됨 (간헐적 타임아웃)
- **장기 대응**: internal API client 타임아웃 증가 작업 진행
  - 스토리지 쪽 제공 internal API client는 전체 타임아웃만 조절 가능, 특정 API만 늘리려면 common 쪽 공통 작업 선행 필요
  - 과거 유사 작업: https://github.com/flex-team/flex-v2-backend-commons/pull/1114
  - TT에서 직접 retrofit client를 새로 만들어 bean으로 등록하는 방식으로 해결 시도
  - Draft PR: https://github.com/flex-team/flex-timetracking-backend/pull/11985
  - 윤성복: `flex.okhttp3` 설정으로 타임아웃 변경 가능하나 모든 API client에 적용되는 문제 있음
