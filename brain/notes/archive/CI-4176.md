# CI-4176: 최고관리자 OTP 설정으로 카드등록 로그인 불가

## 증상

- **문제 정의**: 기존 관리자가 OTP 2차인증을 설정한 채 퇴사하여, 신규 관리자가 로그인 불가 → 카드 등록 불가
- **회사**: 한국앱티브커넥션시스템즈 유한책임회사 (Customer ID: 222855)
- **요청자**: CS팀 @채효진
- **대상자**: so-hee.kim@aptiv.com (신규 관리자)
- **영향 범위**: 최고관리자 1명 (전사 영향 — 유료전환 차단)
- **문제 시점**: 2026-03-20 (신규계약 시작일, 무료체험 3/23 만료)
- **온도감**: 🔴 상 (우선 해결)
- 문의 내용:
  1. 해당 계정의 OTP 설정을 임의로 해제할 수 있는지[^1]
  2. 최고관리자가 로그인할 수 있도록 지원 가능한지[^1]
  3. 불가 시 계정 초기화 후 재세팅 방향 가이드 필요[^1]

## 현재까지 파악된 내용

- Notion "이슈 대응 공통 가이드"에 동일 케이스 가이드 존재[^2]:
  - 보안상 이유로 OTP 해제 operation API 없음
  - **유일한 해결책: 해당 회사의 OTP 2차인증 설정을 완전히 끄는 것**
  - 설정 off 요청의 증적을 확실히 받고 수행해야 함
  - 우회로: SSO 설정이 있으면 SSO 로그인 시 OTP 확인 없이 가능
- workspace_id = 211747[^5]
- customer의 original_workspace_id는 NULL[^5]
- OTP 필수 여부는 `flex_auth.customer_credential_t_otp_setting` 테이블의 `required` 컬럼으로 관리[^7]
- `allowed_authentication_method` 테이블에도 OTP 타입 3건 활성화 상태[^3]
- `customer_authentication_setting`(`flex` DB)에는 해당 고객 데이터 없음[^6]

## 원인 분석

기존 관리자가 2026-01-28에 회사 단위 OTP 2차인증을 활성화한 채 퇴사 → 신규 관리자는 OTP 기기 없이 인증 불가 → 로그인 차단[^7].

### 스펙 vs 버그 판별

**스펙에 의한 정상 동작**. OTP 활성화 시 모든 사용자에게 OTP를 요구하는 것은 의도된 보안 정책. 관리자 퇴사 후 해제 수단이 제한적인 것은 보안상 의도된 설계[^4].

> 💡 **판단 근거**: `flex_auth.customer_credential_t_otp_setting`에서 customer_id=222855의 `required=1` 확인[^7]
> → Notion 가이드에 "보안상 이유로 operation API 없음" 명시[^2]
> → 의도된 보안 정책으로, 버그가 아닌 스펙에 의한 동작

## 다음에 같은 문의가 오면

1. OTP 설정 확인: `SELECT id, customer_id, required FROM flex_auth.customer_credential_t_otp_setting WHERE customer_id = {customer_id}`
2. `required = 1`이면 → 고객사 증적 확보 후 `UPDATE required = 0`
3. SSO가 설정되어 있다면 SSO 로그인으로 우회 가능
4. 해결 후 재활성화 시 신규 관리자 먼저 OTP 등록 → 전사 활성화 순서 안내

기능: 회사 OTP 2차인증 설정 관리
