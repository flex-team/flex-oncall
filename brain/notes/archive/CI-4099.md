# CI-4099: 휴일대체 취소가 되지 않는 점 확인 요청

> **상태**: Done — 2026-03-12 (KST)
> **verdict**: expected-behavior — 보상휴가 부여/근태마감 시 스케줄 편집 불가 → 휴일대체 아이콘 미노출은 의도된 스펙

## 증상

- **문제 정의**: 2/1(일) 주휴일 근무의 휴일대체를 2/20(금)으로 지정해두었으나, 2/1 근무 삭제 후에도 2/20은 휴일대체일로 유지. 2/1에 휴일대체 취소 버튼이 보이지 않아 2/20을 근무일로 되돌릴 수 없음. 보상휴가 부여 + 근태마감으로 잠긴 상태.

## 원인 분석

- 클라이언트 스펙: **스케줄 편집을 할 수 없으면 휴일대체 아이콘을 아예 미노출** (이지선 확인)
  - 조건: 근무 편집 권한 없음 / 입사전/퇴사후/휴직중 / 근태 마감됨 / **보상휴가가 부여됨**
- 해당 케이스에서 보상휴가가 부여되어 있어 근무가 잠긴 상태 → 수정 불가 → 아이콘 미노출
- 서버 API (`date-attributes`, `alternative-holidays/original-holiday-info`)에서는 휴일대체 값이 정상 반환됨
- 날짜 파란색 미표시 문의: 파란색은 휴일대체 표시가 아닌 "수정 중인 날짜" 표시임 (장우현 확인)

## 해결

- 고객사에 보상휴가 취소 → 대체일 취소 순서로 안내하여 이슈 해소
- 버그가 아닌 의도된 스펙임을 확인
- 보상휴가가 부여된 상태에서는 근무 편집이 잠기므로, 보상휴가 취소가 선행되어야 함
