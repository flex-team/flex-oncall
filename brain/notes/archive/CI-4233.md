## 원인 분석

**스펙이다.** 수동 가입(메리츠에 직접 요청)으로 처리된 구성원은 flex 제품 내에서 보험 탭이 표시되지 않는 것이 현재 설계.[^2]

### 보험 탭 노출 조건 (백엔드)

보험 탭 노출은 2가지 조건 모두 충족 필요[^5]:
1. `hasActiveFeature` = true (회사 레벨 feature flag, `INSURANCE`)
2. `profileActive` = true (진행 중인 보험 존재 + 해당 사용자가 피보험자로 등록)

### 중도 가입 케이스별 제품 내 확인 가능 여부

| 케이스 | 가입 가능 | 제품 내 확인 |
|--------|---------|-------------|
| 최초 청약 시 등록 구성원 | 가능 | **가능** — 제품에서 바로 진행 |
| 중도 가입 — 신규 입사 구성원 | 가능 | **가능** — 제품에서 바로 진행 |
| 중도 가입 — 기존 구성원(청약 당시 미가입)/복직자 | 가능 | **Operation API로 상태 변경 후** 가능 |
| 중도 가입 — 가족만 추가 (구성원은 이미 가입) | 가능 | **불가** — 메리츠에 직접 전달 |
| 수동 가입 (메리츠에 직접 요청) | 가능 | **불가** — 제품 반영 안 됨[^2] |

> 💡 **판단 근거**: 남지선(PM)이 Slack에서 "수동으로 가입하시면 제품내에서 확인 안됩니다", "청약당시 등록된 구성원아니면 제품에서 확인 안되요"로 확정.[^2]
> 안상연이 정리한 3가지 — (1) 최초 가입자만 제품 내 확인, (2) 수동 가입은 가능하지만 제품 내 확인 불가, (3) 원래부터 이러했음 — 에 남지선이 동의.[^6]
