> **verdict**: spec — F-4 임의가입 대상, 자격관리 취득일 미등록으로 EXCLUDED 처리는 정상 동작

  1. 2025년 6월까지 고용보험 정상 공제 → 7월부터 공제 제외

  3. 체류자격 수정 이력 또는 원인 파악 필요

### 원인

F-4(재외동포)는 고용보험법상 **임의가입** 대상[^6]이다. 시스템은 체류자격별로 고용보험 가입 의무를 3그룹으로 분류한다[^5]:

- **필수 제외**: A1, A2, A3, B1, B2, C1, C3, D1~D6, D10, F1, F3, G1, H1
- **필수 포함**: F2, F5, F6
- **임의/상호주의**: C4, D7~D9, E1~E10, **F4**, H2, UNKNOWN

임의/상호주의 그룹은 `employment_insurance_qualification_history`에 **취득일이 등록되어 있어야** 공제 대상이 된다. 취득일이 없으면 EXCLUDED 처리.

<details>
<summary>5 Whys 체인</summary>

1. 왜 7월부터 고용보험이 공제되지 않나? → remarks에 EXCLUDED가 설정됨[^3]
2. 왜 EXCLUDED가 설정되었나? → F4 체류자격은 임의가입 대상이고, 자격관리에 취득일이 없으면 EXCLUDED[^5]
3. 왜 6월에는 공제되었나? → 6월 시점 체류자격이 UNKNOWN이어서 외국인 로직이 적용되지 않았음[^1]
4. 왜 체류자격이 바뀌었나? → 2025-07-15에 UNKNOWN→F4, KR→CA로 수정됨[^2]
5. 왜 자격관리에 취득일이 없나? → 체류자격 변경 시 자격관리 등록이 동반되지 않음 (수동 등록 필요)

</details>

### 스펙 vs 버그

**스펙(정상 동작)**. F-4 재외동포의 고용보험은 임의가입이며, 자격관리에 취득일 등록이 필요하다. 이전에 UNKNOWN 상태에서 공제되었던 것 자체가 오히려 비정상 — 외국인 로직이 적용되지 않아 한국인처럼 처리된 것.

## 해결

- **즉시 대응**: 급여정산 > 사회보험 자격관리에서 해당 구성원의 **고용보험 자격 취득일을 등록**하면 이후 정산부터 공제 재적용
- **소급**: 7월 이후 미공제분에 대해 재정산 또는 수동 조정 필요 여부는 고객사와 협의

## 발견한 스펙/제약

- 외국인 고용보험 가입은 체류자격에 따라 필수/임의/제외 3그룹으로 분류되며, 임의가입 대상(F4 포함)은 사회보험 자격관리에 취득일이 등록되어야 공제 대상
- 체류자격이 UNKNOWN이면 외국인 로직이 적용되지 않아 한국인과 동일하게 처리됨

## 용어 발견

| 사용자 표현 | 시스템 용어 | 비고 |
|------------|------------|------|
| 외국인 고용보험 공제 제외 | EmploymentInsuranceRemark.EXCLUDED | remarks에 EXCLUDED 포함 시 공제 0원 |
| 체류자격 | UserPersonalResidenceQualification | core의 user_personal 테이블에 저장 |
| 사회보험 자격관리 | SocialInsuranceQualificationHistory | 4대보험 취득일/상실일 관리 |

## 다음에 같은 문의가 오면

1. **먼저 확인**: `work_income_settlement_payee`에서 해당 정산의 `residence_qualification` 값 확인
   - UNKNOWN이 아닌 외국인 체류자격이면 → 2로
   - UNKNOWN이면 → 체류자격 미등록 문제, core user_personal 확인
2. **자격관리 확인**: `employment_insurance_qualification_history`에서 취득일(qualified_date) 존재 여부 확인
   - 취득일 없음 → 사회보험 자격관리에서 고용보험 취득일 등록 안내
   - 취득일 있음 → `EmploymentInsuranceDeductionRecipient.remarks`에서 EXCLUDED 원인 추적
3. **조치**: 자격관리에 취득일 등록 후 정산 재계산하면 공제 재적용
