# CI-4395: 주법인 전자계약 서식 계열사 복제 요청

> **상태**: 해결 완료 — 2026-04-10
> **compact**: 원본 70줄 → 정제 70줄 (2026-04-10). 정제 대상 없음(단순 운영 대응). git log로 원본 추적 가능.

## 증상
- **문제 정의**: 주법인(엘케이홀딩스)의 전자계약 서식을 계열사(초록에프앤비)로 복제 요청
- **회사**: (주)초록에프앤비 (Customer ID[^5]: 230852)
- **요청자**: 김현욱 (CS팀, Intercom 인입)[^1]
- **대상자**: 초록에프앤비 관리자 (hyundal1705@naver.com)
- **영향 범위**: 1개 계열사
- **문제 시점**: 2026-04-10
- 문의 내용:
  1. 주법인(엘케이홀딩스, customer_id=208733)의 전자계약 서식[^6] 전체를 계열사(초록에프앤비, customer_id=230852)에 복사[^2]
  2. 계약서 제목 그대로 복사[^2]
  3. 복사 받은 회사(초록에프앤비)의 권한자가 계약서 편집 가능하도록 확인[^2]

## 원인 분석

### 조사 과정
> 💡 CI-4283과 동일한 주법인-계열사 구조(엘케이홀딩스 → 계열사)[^3]
> → 이전 건에서 확인된 `duplicateTemplates` Operation API를 동일하게 사용[^4]

### 원인
- 계열사 간 전자계약 서식 공유 기능이 제품에 없어 Operation API를 통한 수동 복제가 필요[^4]

## 해결
- raccoon prod에서 `duplicateTemplates` 오퍼레이션 실행[^7]
  - `originalCustomerId`: 208733 (엘케이홀딩스)
  - `targetCustomerIds`: [230852] (초록에프앤비)
  - `postfix`: "" (제목 그대로 복사)
- 검증: 두 회사의 템플릿 목록을 GET으로 조회하여 ID만 다르고 내용·개수가 동일함을 확인[^8]
  - `GET /api/operation/v2/digicon/customers/208733/templates`
  - `GET /api/operation/v2/digicon/customers/230852/templates`

## 다음에 같은 문의가 오면
1. **먼저 확인**: 주법인 customer_id와 복제 대상 계열사 customer_id 목록을 파악
2. **조치**: raccoon prod Swagger → digicon → `POST /api/operation/v2/digicon/duplicate-templates` 실행
   ```json
   {
     "originalCustomerId": {주법인 customer_id},
     "targetCustomerIds": [{대상 계열사 customer_id 목록}],
     "postfix": ""
   }
   ```
3. **검증**: `GET /api/operation/v2/digicon/customers/{customer_id}/templates` 로 원본·대상 템플릿 목록을 비교하여 복제 결과 확인
4. **주의**: postfix를 빈 문자열로 설정하면 원본 제목 그대로 복사. 구분이 필요하면 "_복사본" 등 접미사 지정
