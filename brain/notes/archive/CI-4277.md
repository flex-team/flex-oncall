# CI-4277: 캡스 연동 — 수동 전송/실시간 타각 미반영 (테이블 매핑 오류)

## 증상

- **문제 정의**: 캡스 연동 완료 후 수동 전송/실시간 타각이 flex에 반영되지 않음
- **회사**: (주)나유타 (Customer ID: 225712)
- **요청자**: 김한나 (CS팀)[^1]
- **대상자**: 해당 회사 캡스 연동 사용자 전체
- **영향 범위**: 전사 (캡스 연동 사용자 전원)
- **문제 시점**: 2026-04-01 문의 접수, 현재 진행 중
- 문의 내용:
  1. 캡스 연동을 가이드대로 완료했으나 수동 전송/실시간 타각이 반영 안 됨[^1]
  2. 오류 발생 없음, DB 설정도 정상으로 보임[^1]
  3. 사번 매핑, 실시간 근무 기록 ON 상태 확인됨[^2]

## 원인 분석

### 조사 과정

> 💡 **판단 근거**:
> → Metabase 연동 탭에서 연동 활성화/근무 기록 동기화 ON 확인[^3]
> → Grafana 캡스 RDB 모니터링에서 `caps_event` INSERT 정상 확인[^4]
> → Metabase에서 `v2_user_external_provider_event` 조회 → 데이터 없음[^5]
> → `v2_customer_external_provider` 설정 정상 (CAPS, active, odbc_connection_limit=2)[^6]
> → work-event-transmitter 로그 확인 → OpenSearch에 해당 고객 관련 발행/컨슘 로그 없음[^7]
> → CloudWatch INSERT 로그 분석: `employee_number` 컬럼이 INSERT에 **누락**되어 null로 저장됨[^8]
> → `CapsWorkEventTransmitterServiceImpl.makeFlexMessageModel()` 에서 `employeeNumber == null`이면 Kafka 발행 skip[^9]
> → 결론: 캡스 프로그램의 테이블 매핑 설정 오류로 `employee_number` 미전달

### 원인

- **캡스 프로그램의 테이블 컬럼 매핑 오류**[^8]
- `caps_event` INSERT 시 `employee_number` 컬럼이 빠져있어 null로 저장됨
- `employee_number`에 와야 할 값이 `employee_name`에 들어오고 있음[^10]
- work-event-transmitter는 `employeeNumber`가 null인 이벤트를 필터링하므로 Kafka 발행이 skip됨[^9]

### 데이터 흐름

```
캡스 PC → caps_event INSERT (employee_number 누락 ❌)
  → work-event-transmitter 1초마다 polling
  → filterNot { it.employeeNumber == null } 에서 skip
  → Kafka 발행 안 됨
  → tracking-consumer 컨슘 없음
  → v2_user_external_provider_event 데이터 없음
```

실제 INSERT 문 (CloudWatch 로그)[^8]:
```sql
insert into caps_event (
    event_date, event_time, employee_id, employee_name,
    group_id, event_type, auth_type, user_type,
    event_etc, event_logtime, event_logmode,
    event_picture, event_result, terminal_id
) values (
    '20260401', '122852', 0000000007, '2406D0016',
    00001, '1', '0', '0',
    '4', '20260401123006', '2', '0', '0', 0000000001
)
```

> ⚠️ `employee_number` 컬럼이 INSERT 목록에 아예 없음. `employee_name`에 `'2406D0016'`(사번으로 보이는 값)이 들어가고 있어 컬럼 매핑이 어긋난 상태.

## 해결

- 고객에게 캡스 근태처리옵션의 테이블 컬럼 매핑 재설정 안내 필요
- `employee_number` 컬럼이 올바른 값으로 매핑되도록 가이드 전달
- **코드 변경 불필요** — 고객 측 설정 수정으로 해결되는 건

## 다음에 같은 문의가 오면

1. **먼저 확인**: Grafana 캡스 RDB 모니터링[^11]에서 해당 회사의 INSERT 로그 확인 (`username`으로 필터링)
2. **원인 판별**:
   - INSERT에 `employee_number` 컬럼 누락 → 테이블 매핑 설정 오류
   - `e_date` 같은 캡스 원본 컬럼명이 보임 → 테이블 매핑 설정 오류 [CI-4202](./archive/CI-4202.md)
   - INSERT 정상인데 반영 안 됨 → consumer 로그 확인 (validation 실패, 잔존 위젯 등)
3. **조치**: 고객에게 올바른 테이블 매핑 설정 가이드 안내. `employee_number` 포함 여부 특히 확인
