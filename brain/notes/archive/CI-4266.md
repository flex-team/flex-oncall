# CI-4266: 결재 진행 중인 건 승인 라인의 휴직자 교체 요청(대결)

## 증상
- **문제 정의**: 휴직 예정자가 승인 라인에 포함된 미처리 승인건에 대해 강제 승인 요청
- **회사**: (주)디케이비엠시 (Customer ID: 207629)
- **요청자**: 양이슬 eishul.yang@dkbmc.com (CS팀 김현욱 대리 접수)[^1]
- **대상자**: 채희정 huijung.chae@dkbmc.com (userId: 791137) — 2026-04-01부터 휴직[^2]
- **영향 범위**: 채희정이 PENDING 승인자로 포함된 TIME_OFF 미처리 건 29건[^9]
- **문제 시점**: 2026-03-31
- 문의 내용:
  1. 휴직 처리 예정인 채희정이 결재 진행 중인 승인 라인에 있음[^1]
  2. 채희정이 들어가있는 휴가 건 모두 강제 승인 요청[^4]

## 원인 분석

### 조사 과정

> 💡 승인자 교체 시스템(`replacement-targets`)이 휴직자를 지원하는지 확인[^6]
> → `replacement-targets/migrate` API는 `resignationDateRange`(퇴직일 범위)가 필수 — 휴직자 미지원[^6]
> → 권민철(승인 도메인 담당): "휴직자는 고객사 동의 후 강제 승인 처리했던 것 같다"[^8]
> → `bulk-approve-for-user` API로 강제 승인 결정

### 원인
- 승인 시스템에 휴직자 처리 로직이 없음[^8]. 승인자 교체(`replacement-targets`)는 퇴사 이벤트 기반 퇴직자 전용[^6].
- 휴직자가 승인 라인에 남아있어도 시스템이 감지하지 못하므로, 운영 개입(강제 승인)이 필요한 구조.

## 해결

고객 동의 확인 후[^4] `bulk-approve-for-user` Operation API로 강제 승인 처리[^9]:

```
POST /proxy/approval/api/operation/v2/approval/process/customers/207629/users/791137/bulk-approve-for-user
Body: { "categories": ["TIME_OFF"] }
```

- succeededProcesses: **29건**, failedProcesses: **0건**[^9]
- 2차 호출로 잔여 건 없음 확인[^9]
- 김현욱이 고객사에 안내 완료[^10]

## 발견한 스펙/제약
- **승인 교체 시스템은 퇴직자 전용**: `replacement-targets`는 퇴사 이벤트 기반. 휴직자/휴가자는 대상이 아님[^6][^8].
- **`approval_process` 미처리 상태값은 `ONGOING`**: `REQUESTED`/`IN_PROGRESS`가 아님. 조회 시 주의[^5].

## 용어 발견
| 사용자 표현 | 시스템 용어 | 비고 |
|------------|------------|------|
| 대결 | 승인자 교체 / bulk-approve-for-user | 제품에 "대결" 기능은 없음. 운영으로 강제 승인 처리 |
| 결재 진행 중인 건 | approval_process.status = 'ONGOING' | 미처리 승인 프로세스 |

## 다음에 같은 문의가 오면

1. **먼저 확인**: 대상자가 퇴직자/삭제된 구성원/휴직자 중 어느 케이스인지 `view_user`에서 상태 확인
2. **원인 판별**:
   - 퇴직자(resigned) → 제품의 "퇴직자 승인자 교체" 기능 또는 `replacement-targets/migrate` API
   - 삭제된 구성원(deleted) → `bulk-approve-for-user` API (CI-4228 참고)
   - **휴직자** → `bulk-approve-for-user` API (이 케이스). 교체 시스템 미지원.
3. **조치**: 고객 동의 확인 후 `bulk-approve-for-user` 호출
   ```
   POST /proxy/approval/api/operation/v2/approval/process/customers/{customerId}/users/{userId}/bulk-approve-for-user
   Body: { "categories": ["TIME_OFF"] }
   ```
   - category: `TIME_OFF`(휴가), `WORK_RECORD`(근무) 등 대상에 맞게 선택
4. **향후 방지**: 고객에게 승인 정책에서 휴직자를 다른 승인자로 변경하도록 안내
