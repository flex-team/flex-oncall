# CI-4385: 승인완료이나 진행중 상태인 문서 확인 요청

## 증상

- **문제 정의**: 기안서 2건이 승인 완료되었으나 "진행중 - 승인 필요" 상태로 표시됨
- **회사**: 주식회사 워시스왓 (Customer ID: 52783)
- **요청자**: benedictus@washswat.com (김선우)[^1]
- **대상자**: actorUserId 661307[^7]
- **영향 범위**: 2건 확인 + 동일 오류 여파 추가 건수 선보정 진행 중[^8]
- **문제 시점**: 2026-04-07 (타임아웃 오류 발생 시점)[^5]
- 문의 내용:
  1. 문서번호 2026-2673, 2026-2675 두 건이 승인 완료되었으나 "진행중 - 승인 필요" 상태로 보임[^1]
  2. 고객사에서 기안서 승인이 완료된 것으로 확인했으나 화면에 반영되지 않음[^2]

## 원인 분석

### 조사 과정

> 💡 임형태가 access log 대시보드에서 benedictus@washswat.com의 2026-04-07 로그를 확인[^5]
> → `/action/v3/approval-document/approval-documents/{documentKey}/act` 엔드포인트에서 `FlexRemoteUnknownStateException: java.io.InterruptedIOException: timeout` 발생[^5]
> → 승인 동작(act) 실행 시 타임아웃으로 상태 전이 실패
> → `workflow_task` 테이블에서 문서 상태 확인 + `action_history_info` 활동 로그 확인[^6]
> → `sync-with-approval` Operation API로 보정 처리[^7]

### 원인

- 2026-04-07 승인 문서 act API 호출 시 타임아웃 오류(`FlexRemoteUnknownStateException: InterruptedIOException: timeout`) 발생[^5]
- 승인 프로세스 자체는 완료되었으나, 후속 상태 동기화(workflow_task 상태 전이)가 실패하여 "진행중 - 승인 필요"로 잔존[^4]
- 이 건은 단독 이슈가 아닌 2026-04-07 발생한 광범위한 타임아웃 오류의 여파[^3]

## 영향 범위

- 확인된 건수: 2건 (문서번호 2026-2673, 2026-2675)
- 동일 오류 여파로 추가 건이 있을 수 있으며, 임형태가 선보정 처리 진행 중[^8]
- 금전적/법적 영향: 해당 없음 (승인 상태 표시 문제)

## 해결

- 문서 2026-2673 (target_uid: `389d0a83f197442c8de0a6cf8af841df`): `sync-with-approval` Operation API로 보정 완료[^7]
- 문서 2026-2675 (target_uid: `934e7c71612c4846ad9cf87da399785c`): `sync-with-approval` Operation API로 보정 완료[^7]
- 나머지 오류 건도 확인하여 선보정 처리 예정[^8]
- 고객 확인 후 티켓 종료 예정[^9]

## 다음에 같은 문의가 오면

1. **먼저 확인**: `workflow_task` 테이블에서 해당 문서번호의 상태 확인 (`SELECT * FROM flex.workflow_task WHERE code = '{문서번호}' AND customer_id = {customer_id}`)
2. **원인 판별**: `workflow_task.status`가 ONGOING이고 실제 승인이 완료된 경우 → 타임아웃/이벤트 동기화 실패
3. **조치**: `sync-with-approval` Operation API 호출로 보정 — `POST /proxy/impact/api/operation/v3/impact/customers/{customerId}/approval-documents/{targetUid}/approval-process/sync?actorUserId={actorUserId}&dryRun=false&dataOnly=false`
