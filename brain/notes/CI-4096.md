# CI-4096: 급여정산 수정 시 정산이름 입력 오류

> **상태**: Todo — 2026-03-11 (KST)
> **verdict**: investigating — FE 정산이름 검증 로직 이슈 (특수문자 관련 의심)

## 증상

- **문제 정의**: 급여정산 완료 후 정산수정하기에서 정산이름을 정확히 입력(복사/붙여넣기 포함)해도 수정이 진행되지 않음. 삭제는 가능하고 수정만 안 됨. 특정 회사뿐 아니라 다른 회사에서도 동일 증상. `[ ]` 특수문자를 제거하면 수정이 되는 것을 확인했으나, 특수문자가 없는 경우에도 발생한 적 있음.
- **회사**: 미특정 (내부 제보, 여러 회사에서 발생)
- **요청자**: 최종진 (내부)
- **라벨**: Payroll Squad, bug
- **담당자**: 미배정 → FE 이슈로 판단, 이지우에게 하위 이슈 할당
- **Linear**: [CI-4096](https://linear.app/flexteam/issue/CI-4096/급여정산-완료-후-수정하려고-정산수정하기-할때n정산이름을-아무리-맞게-넣어도-수정이-안됩니다)

## 원인 분석

- 이지우가 FE 이슈로 판단, 하위 이슈 [FE-2086](https://linear.app/flexteam/issue/FE-2086) 생성
- 최종진 제보: `[ ]` 특수문자를 제거하면 수정 가능 → 특수문자 인식 문제 의심
- 단, 이전에 특수문자 없이도 동일 증상 발생한 적 있음
- 이지우가 자신의 PC에서 재현 불가, 로직도 이상해 보이지 않음
- 서버에 저장된 정산 이름 값을 확인해야 할 것으로 판단, 최종진에게 재현 시 제보 요청

## 해결

미해결. FE 정산이름 검증 로직 조사 중.
