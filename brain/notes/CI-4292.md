### 유급휴일시간 비례계산 문제

기본급 탭에서 두 사람 모두 유급휴일시간이 **6시간 24분**으로 동일하게 표시됨[^7]. 이재인은 56시간, 김가은은 96시간 근무했는데 유급휴일시간이 같다는 것은 **실근무시간이 아닌 비례계산**이 적용되었음을 시사한다.

## 원인 분석

### 조사 과정

> 💡 TT 근무기록(v2_user_work_record_event)에서 3/1, 3/2 출근 확인[^4a]
> → payroll DB(work_schedule_statutory_leisure_item)에서 03/01~07 무급(ALL_HOLIDAY + ANY_WORK_ABSENCE) 확인[^4]
> → v2_customer_holiday에 공휴일 0건 — 커스텀 공휴일은 없음[^8]
> → 그러나 **2026-03-01(삼일절)은 일요일** → 대체공휴일법에 의해 **3/2(월)가 삼일절 대체공휴일**
> → TT의 `ValidWeeklyPaidHolidayCalculator`에서 법정 공휴일 정책으로 3/1, 3/2 모두 `customerHoliday=true` 처리[^9]
> → 이재인의 소정근로일(일, 월) 2일이 전부 유급공휴일 → `ALL_HOLIDAY=true` → `ANY_WORK_ABSENCE=true` 연쇄[^9]

### 증상 1 원인: 주휴수당 03/01~07 무급 — 삼일절 + 대체공휴일

`ValidWeeklyPaidHolidayCalculator.kt`의 ALL_HOLIDAY 판정 로직[^9]:

```kotlin
// 소정근로일(WORKING_DAY)이 모두 paidHoliday이면 ALL_HOLIDAY
days.filter { it.category === WORKING_DAY }
    .let { it.isNotEmpty() && it.all { it.paidHoliday } }

// paidHoliday = category == WEEKLY_PAID_HOLIDAY || customerHoliday
```

이재인의 03/01~03/07 주차:
- 3/1(일) = 삼일절 → `customerHoliday=true` → `paidHoliday=true`
- 3/2(월) = 대체공휴일 → `customerHoliday=true` → `paidHoliday=true`
- 소정근로일 2일 전부 `paidHoliday` → **`ALL_HOLIDAY=true`**

ALL_HOLIDAY가 ANY_WORK_ABSENCE를 연쇄 유발[^9]:

```
ALL_HOLIDAY=true
→ ANY_WORK_ABSENCE 필터에서 .filterNot { paidHoliday }로 3/1, 3/2 제외
→ 남은 소정근로일 0일 → isEmpty()=true
→ ANY_WORK_ABSENCE=true → 해당 주 무급 처리
```

**실제로 3/1, 3/2 모두 출근했지만**, 소정근로일이 전부 법정 유급공휴일이라 주휴수당 무급 처리. 코드 로직상 의도된 동작이나, "공휴일에 근무해도 주휴수당 무급"이 올바른 스펙인지는 법적 해석 영역.

### 증상 1 보충: 03/22~03/28 유급은 정상

3/10에 등록된 근무 플랜(id=2834683)에서 **3/22가 비근무일로 변경**되었다[^10]. 최신 플랜 기준 03/22~03/28 소정근로일은 3/23(월)뿐이고, 3/23에 근무기록 있으므로 유급이 정상. CS의 "결근 주가 만근으로"라는 인식은 플랜 변경 이전 기준으로의 착각.

### 증상 2 원인: 유급휴일시간 384분 동일 — SHIFT 근무유형 고정값

payroll의 `ExtractPaidHolidayMinutesMethod`에서 SHIFT 근무유형은 `BaseAgreedExtractPaidHolidayMinutesMethod`를 사용[^11]:

```kotlin
paidHolidays.sumOf { workRule.baseAgreedDayWorkingMinutes }
```

`baseAgreedDayWorkingMinutes`는 근무유형 레벨 설정값(=384분)이며, 개인의 `agreedDailyWorkingMinutes`(이재인 192분)와 무관[^11]. 같은 근무유형이면 누구나 384분이 나온다.

| | 기대값 (개인 기준) | 실제값 (근무유형 기준) |
|--|-------------------|---------------------|
| 이재인 (일 192분) | 192 x 유급휴일일수 | **384** x 유급휴일일수 |
| 김가은 (일 384분) | 384 x 유급휴일일수 | **384** x 유급휴일일수 |

초과근무수당의 `holiday_working_minutes`는 실근무기록 기반이라 이재인=1200, 김가은=360으로 차이 나는 것이 정상[^12].
