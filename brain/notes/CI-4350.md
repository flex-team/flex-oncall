# CI-4350: 구글 캘린더 휴가 취소 후 일정 미회수

> **상태**: 즉시 대응 완료, 근본 해결 미완 — 2026-04-07

## 증상

- **문제 정의**: 워케이션 휴가 등록 후 취소 승인 완료됐으나 구글 캘린더에서 해당 일정이 회수되지 않음
- **회사**: 아보엠디코리아 (Customer ID: 113613)[^1]
- **요청자**: sj@avomd.io[^1]
- **대상자**: yhju@avomd.io[^1]
- **영향 범위**: 1명 (yhju@avomd.io)
- **문제 시점**: 2026-04-07, 워케이션 휴가 취소 승인 처리 직후[^1]
- 문의 내용:
  1. 4월 7일 "워케이션" 휴가를 등록했다가 취소 승인 완료[^1]
  2. 취소 승인 후에도 구글 캘린더에 해당 일정이 남아있음[^1]
  3. Metabase 확인 시 미연동 일정은 없는 것으로 보임[^1]

## 현재까지 파악된 내용

- 메타베이스 기준으로는 미연동 일정이 없는 것으로 확인됨[^1]
  - 이미 동기화된 상태로 기록되어 있거나, 취소 이벤트 처리가 별도 흐름으로 진행될 가능성 있음
- 구글 캘린더에 일정이 잔존하는 상태 — 회수(삭제) 이벤트가 구글 쪽에 전달됐는지 불명확[^1]

### L1 원인 확정

취소 승인 완료 후 `flex_calendar.calendar_event` 테이블의 이벤트 2건이 `deleted_at=NULL`, `status=CONFIRMED`를 유지하여 구글 캘린더 삭제 이벤트가 발행되지 않음.

**스펙/버그 판별**: 버그 — 취소 승인 완료 시 캘린더 이벤트가 정상적으로 삭제되지 않음

### L2 핵심 증거

- `flex_calendar.calendar_event` 이벤트 2건:
  - `id=01knjm0e0b3hm9659b53hyh210`: 4/7 09:30~14:30, status=CONFIRMED, deleted_at=NULL (google sync_id=01knjm0nkcsenvq3hvx0t848ct)
  - `id=01knjm0e0b3hm9659b53hyh213`: 4/7 14:30~18:30, status=CONFIRMED, deleted_at=NULL (google sync_id=01knjm0nkqe4endcpxq86q7jb5)
- `flex_calendar.google_calendar_event_delete_sync`에 해당 event_id 없음 → 구글에 삭제 요청 미전달 (사실)
- `flex_calendar.google_calendar_event_sync`에 sync row 존재 → Metabase "연동됨" 표시 유지 (사실)

## 해결

### 즉시 대응 (완료 — 2026-04-07)

`POST /proxy/calendar/api/operation/v2/calendar/customers/113613/events/cleansing` 호출로 구글 캘린더에서 2건의 이벤트를 직접 삭제 처리함.

```json
{
  "oauthTokenId": 1,
  "calendarConnectionId": 50246,
  "calendarConnectionHistoryId": null,
  "googleEventIds": [
    "019d6540380b1c6893152b28e3e88820",
    "019d6540380b1c6893152b28e3e88823"
  ]
}
```

- HTTP 200 응답 확인 → 구글 캘린더에서 일정 삭제 완료
- `calendarConnectionId=50246`: flex_calendar.v2_external_calendar_connection의 AVO Time Off 그룹 캘린더 연결 (user_id=495969)
- `oauthTokenId`는 API 내부에서 실제로 사용되지 않음 (findGroupBy에서 customerId+userId로 토큰 조회)

### 근본 해결

취소 승인 플로우에서 `calendar_event` 업데이트 로직 누락 여부 코드 확인 필요 → `ops-fix-issue` 실행 권장
