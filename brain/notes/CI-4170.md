# CI-4170: 채용 주소 변경 신청 확인 요청

## 증상
- **문제 정의**: 채용사이트 subdomain[^5] 변경 신청이 "검토중" 상태로 방치됨
- **회사**: BreezeBio Korea (Customer ID: 50684)[^1]
- **요청자**: 이주화 (CS팀)[^1]
- **대상자**: hlee@breezebio.com (고객사 담당자)[^1]
- **영향 범위**: 1건 (해당 회사의 채용사이트 subdomain 변경)
- **문제 시점**: 2026-03-10 신청, 2026-03-19까지 검토중 상태 유지[^1]
- 문의 내용:
  1. 채용사이트 subdomain 주소 변경을 3/10에 신청했으나 계속 "검토중" 상태[^1]
  2. 검토 완료 시점 문의[^1]

## 원인 분석

### 조사 과정
> 💡 채용사이트 subdomain 변경 요청은 자동 처리가 아니라, 온콜 담당자가 수동으로 도메인명을 확인한 후 operation API[^6]로 승인하는 프로세스[^2]
> → 신청 후 온콜 담당자에게 알림이 전달되지 않았거나 누락된 것으로 추정
> → 김성훈님이 확인 후 즉시 승인 처리 완료[^3]

### 원인
- subdomain 변경 요청 시 `#alarm-recruiting-operation` 슬랙 채널로 알림이 전달되는 구조[^10]. 해당 채널을 온콜 담당자가 모니터링하지 않아 9일간 방치됨

## 해결
- 김성훈님이 `flex_recruiting.site_subdomain_change` 테이블에서 데이터 확인 후, raccoon operation API로 승인 처리 완료[^3][^10]
- API 경로: `/customers/{customerId}/subdomains/{siteSubdomainChangeId}/change/approve` (또는 `/reject`)[^10]

## 다음에 같은 문의가 오면

1. **먼저 확인**: `#alarm-recruiting-operation` 채널에서 해당 회사의 subdomain 변경 요청 알림 확인[^10]
2. **데이터 확인**: `flex_recruiting.site_subdomain_change` 테이블에서 요청 건의 상태와 도메인명 확인[^10]
3. **조치**: 도메인명이 적절하면 raccoon operation API로 승인 처리[^10]
   - 승인: `POST /customers/{customerId}/subdomains/{siteSubdomainChangeId}/change/approve`
   - 반려: `POST /customers/{customerId}/subdomains/{siteSubdomainChangeId}/change/reject`
