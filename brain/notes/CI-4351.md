# CI-4351: 근무 스케줄 삭제 로그 요청

> **상태**: 진행 중 — 2026-04-07

## 증상
- **문제 정의**: 시몬스 고객사가 '갤러리F' 교대근무 스케줄 삭제 이력(삭제 시점 + 삭제자 이메일)을 요청
- **회사**: 시몬스 (Customer ID: 77952)
- **요청자**: lee0ju@simmons.co.kr[^1]
- **대상자**: lee0ju@simmons.co.kr[^1]
- **영향 범위**: 1개 회사(시몬스) — 특정 스케줄 1건의 삭제 이력 조회, 데이터 변경 없음
- **문제 시점**: 2026-04-04 이후[^1]
- 문의 내용:
  1. 교대 근무 설정 > 스케줄 관리에서 '갤러리F' 스케줄이 삭제되었으며, 삭제 발생 시점과 삭제를 수행한 구성원의 이메일을 확인해 달라는 요청[^1]

## 현재까지 파악된 내용
- 교대근무 스케줄 삭제는 raccoon audit operation API를 통해 확인 가능하다[^2]
- OpenSearch에는 감사로그 인덱스가 없으므로 로그 클러스터에서 직접 조회 불가[^2]
- 제품 내 감사로그 다운로드는 7일 기간 제한이 있으므로, 4월 4일 이후 이벤트는 API로 조회 가능하다[^2]

## 원인 분석

### 조사 방향 (미확정)

쿡북 `감사로그 다운로드` 플로우(F: 계정/구성원) 히트[^3] — raccoon audit API 호출로 삭제 이벤트 특정 예정

**가설 목록**

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | raccoon audit API DELETE 이벤트 조회로 갤러리F 스케줄 삭제자 파악 가능 | `POST /action/operation/v2/audit/77952/download` — eventCategory=DELETE, from=2026-04-04 | ⏳ API 호출 대기 중 |

### API 호출 준비

```
POST https://flex-raccoon.grapeisfruit.com/action/operation/v2/audit/77952/download?userId={userId}

Body (확인 필요):
{
  "eventCategory": "DELETE",
  "personalInformationType": "근태_근무정보",  ← 확인 필요
  "from": "2026-04-04T00:00:00+09:00",
  "to": "2026-04-07T23:59:59+09:00"
}
```

**미확정 항목**:
- `userId`: 이메일 수신자 userId (결정 필요)[^4]
- `personalInformationType`: shift schedule에 해당하는 값 (Swagger 인증 필요로 미확인)
