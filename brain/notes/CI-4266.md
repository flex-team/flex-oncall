# CI-4266: 결재 진행 중인 건 승인 라인의 휴직자 교체 요청(대결)

> **상태**: 진행 중 — 2026-03-31 (고객 동의 확인 완료, 2026-04-01 강제 승인 실행 예정)

## 증상
- **문제 정의**: 휴직 예정자가 승인 라인에 포함된 미처리 승인건 174건에 대해 승인자 교체(대결) 요청
- **회사**: (주)디케이비엠시 (Customer ID: 207629)
- **요청자**: 양이슬 eishul.yang@dkbmc.com (CS팀 김현욱 대리 접수)[^1]
- **대상자**: 채희정 huijung.chae@dkbmc.com (userId: 791137) — 현재 휴가 중, 2026-04-01부터 휴직 예정[^2]
- **영향 범위**: 채희정이 PENDING 승인자로 포함된 미처리 건 **174건** (대부분 TIME_OFF)[^5]
- **문제 시점**: 2026-03-31
- 문의 내용:
  1. 휴직 처리 예정인 채희정이 현재 결재 진행 중인 승인 라인에 있음[^1]
  2. 승인자를 양이슬(userId: 791138)로 교체 요청[^3]
  3. 이후 고객 확인 결과: 채희정이 들어가있는 **휴가 건 모두** 강제 승인 요청[^4]

## 현재까지 파악된 내용

### DB 조사 결과

초기 조사에서 `approval_process.status`를 `REQUESTED`/`IN_PROGRESS`로 검색하여 0건이 나왔으나, 실제 상태값은 **`ONGOING`**이었음[^5].

> 💡 **판단 근거**: approval_process 테이블의 미처리 상태값이 `ONGOING`임을 확인[^5]
> → `resolve_target_type = 'USER'`, `resolve_target_value = '791137'`인 PENDING actor 포함 건 재조회
> → **174건** 확인. 가장 오래된 건은 2025-06-11부터 쌓여있음[^5]

채희정(791137)의 사용자 상태:
- `view_user.deleted_date = NULL` — 삭제된 구성원 아님[^5]
- 현재 휴가 중, 내일(4/1)부터 휴직 예정[^2]

### 승인자 교체 API 조사

승인자 교체 시스템(`replacement-targets`)은 **퇴직자 전용**으로 설계되어 있음[^6]:

| API | 용도 | 적용 가능 여부 |
|-----|------|--------------|
| `replacement-targets/migrate` | 퇴직자 감지 → 교체 대상 등록 | **불가** — `resignationDateRange` 필수, 휴직자 미지원[^6] |
| `replace-actors` (User-Facing) | 교체 대상에 등록된 사용자의 승인자 교체 | **불가** — migrate 선행 필요[^6] |
| `bulk-approve-for-user` | 특정 사용자의 미처리 건 강제 승인 | **가능** — 고객 동의 필요[^7] |
| `act-approval-process` | 문서별 개별 강제 승인 | 가능하나 174건에 부적합 |

### 대응 방향 결정

권민철(승인 도메인 담당)이 "휴직자는 고객사 동의 후 강제 승인 처리했던 것 같다"고 확인[^8]. 고객(양이슬) 동의 확인 완료[^4].

## 발견한 스펙/제약
- **승인 시스템에 휴직자 처리 로직 없음**: 승인 요청 시점에 휴직 관련 처리를 하지 않음[^8]. 휴직자가 승인 라인에 남아있어도 시스템은 감지하지 못함.
- **교체 시스템은 퇴직자 전용**: `replacement-targets`는 퇴사 이벤트 기반으로 동작하며, 휴직자는 대상이 아님[^6].
- **`approval_process` 상태값**: 미처리 상태가 `REQUESTED`/`IN_PROGRESS`가 아닌 **`ONGOING`**임. 쿡북/가이드 수정 필요.

## 용어 발견
| 사용자 표현 | 시스템 용어 | 비고 |
|------------|------------|------|
| 대결 | 승인자 교체 / bulk-approve-for-user | 제품에 "대결" 기능은 없음. 운영으로 강제 승인 처리 |
| 결재 진행 중인 건 | approval_process.status = 'ONGOING' | 미처리 승인 프로세스 |
