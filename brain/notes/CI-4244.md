# CI-4244: 워크플로우 문서 강제 승인 요청

## 증상
- **문제 정의**: 승인 라인에 경영지원팀(조직)이 설정된 워크플로우 문서에 대해, 기존 팀원 전원 퇴사로 승인 진행 불가 — 강제 승인 요청
- **회사**: 세잎프닥 (Customer ID[^1]: 161808)
- **요청자**: wldus7312@safedoc.kr (CS 인터콤 문의)
- **대상자**: junhs@safedoc.io (현재 경영지원팀 구성원, 입사 전 문서 승인 불가)
- **영향 범위**: 워크플로우 문서 6건 — 2026-67, 2026-66, 2026-46, 2026-43, 2026-23, 2026-10
- **문제 시점**: 2026-03-27

## 원인

- `approval_line_actor` 의 `resolve_target_type` = `DEPT`, `resolve_target_value` = 499874 (경영지원팀)
- `approval_line_actor_resolved_user` 에 매핑된 user가 문서 생성 시점의 경영지원팀 소속 구성원 → **전원 퇴사**
- 조직 승인라인은 UI의 "퇴직자 승인자 교체" 기능이 차단됨[^2] → 운영 오퍼레이션 필요
- CI-4228과 동일 패턴이나, 개인이 아닌 **조직(DEPT)**이 승인라인이라는 차이점 존재

### 처리 절차

**`bulk-approve-for-user` API 사용**
- 초기에는 문서별 개별 처리(`act-approval-process` + `produce-approval-process-event`)를 계획했으나,
  metabase에서 각 퇴사자별 미처리 건이 모두 승인 처리 대상임을 확인하여 `bulk-approve-for-user` 로 전환[^4]

```
POST /api/operation/v2/approval/process/customers/161808/users/{userId}/bulk-approve-for-user
```

- 911597, 941623: 모든 미처리 건이 문의 범위 내 → 바로 처리
- 930246: 문의 외 추가 건(2026-59) 존재 → 이주화님 확인 후 처리[^3]

## 승인 테이블 ERD

```mermaid
erDiagram
    approval_process {
        CHAR_26 id PK
        BIGINT customer_id
        BIGINT requester_id
        VARCHAR status "ONGOING | APPROVED | REJECTED | CANCELED"
        VARCHAR target_category "APPROVAL_DOCUMENT | WORK_RECORD | TIME_OFF | ..."
        VARCHAR target_uid "승인 대상 문서 식별자 (task_key)"
        SMALLINT generation
        SMALLINT target_version
        DATETIME matched_at
        DATETIME requested_at
        DATETIME terminated_at
    }

    approval_line {
        CHAR_26 id PK
        CHAR_26 approval_process_id FK
        SMALLINT step "승인 단계 번호"
        VARCHAR status "PENDING | APPROVED | AUTO_APPROVED | REJECTED"
        VARCHAR action_type "NONE | SCHEDULED"
        VARCHAR action_trigger
    }

    approval_line_actor {
        CHAR_26 id PK
        CHAR_26 approval_line_id FK
        BIGINT acted_user_id "실제 처리한 사용자 (nullable)"
        DATETIME acted_at
        VARCHAR status "PENDING | APPROVED | REJECTED"
        VARCHAR resolve_target_type "USER | DEPT | DEPT_WITH_CHILDREN | ..."
        VARCHAR resolve_target_value "리졸브 대상 ID (e.g. 부서ID 499874)"
        DATETIME resolved_at
    }

    approval_line_actor_resolved_user {
        CHAR_26 id PK
        CHAR_26 approval_line_actor_id FK
        BIGINT user_id "리졸브된 실제 승인자 user_id"
    }

    workflow_task {
        BIGINT id PK
        VARCHAR task_key "문서 고유 식별자"
        BIGINT customer_id
        VARCHAR source_feature
        VARCHAR task_type
        VARCHAR status "IN_PROGRESS | DONE | DECLINED | CANCELED"
    }

    approval_process ||--o{ approval_line : "1:N (step별 승인라인)"
    approval_line ||--o{ approval_line_actor : "1:N (승인자/조직)"
    approval_line_actor ||--o{ approval_line_actor_resolved_user : "1:N (조직→개인 리졸브)"
    workflow_task ||--o| approval_process : "task_key = target_uid"
```

### 이 케이스의 데이터 흐름

    ALA["approval_line_actor<br/>resolve_target_type: DEPT<br/>resolve_target_value: 499874<br/>(경영지원팀)"]
