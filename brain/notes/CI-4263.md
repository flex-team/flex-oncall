### 코드 분석[^8]

실제 3/25 로그에서 확인된 세콤 STOP 처리 흐름[^12]:

```
16:34:03 세콤 퇴근 타각 수신 (eventTime=1774424043000)
16:34:12 UserWorkClockStopByDraftEventServiceImpl — dry-run 시작
16:34:12 ServerTriggered — 추천휴게 dry-run (targetWorkStopTimeStamp=16:34:03)
16:34:12 WorkScheduleRequiredMemoValidatorLogic — 메모 필수 승인 실패 (승인키 5개)
16:34:12 ServerTriggered — 연장근무입력설정 dry-run (targetWorkStopTimeStamp=16:34:03 — 시간 미조정)
16:34:12 결과: FAIL_DRY_RUN → snapshot 생성 → STOP_FAILURE_GO_CONFIRM 알림
```

> 💡 **핵심**: `adjustOverWorkInputRule`이 실행되었으나 **targetWorkStopTimeStamp이 변경되지 않았다** (여전히 16:34:03). 메모 필수 검증 실패가 overwork 조정보다 우선하여 전체가 FAIL 처리된 것으로 보인다. 또는 overwork 조정 자체가 동작하지 않았을 수 있다 — 코드 레벨의 추가 확인 필요.

연장근무 최소시간 미달 시 `adjustOverWorkInputRuleWithDryRunResult()`[^8]에서 소정근무 종료 시점으로 퇴근 시각을 자동 조정한다. **그러나 이 조정이 실행되기 전에 `WorkScheduleRequiredMemoValidatorLogic`의 메모 필수 검증이 먼저 실패하여 전체 dry-run이 FAIL 처리된다.**

> 💡 **김현욱님, 안희종님 판단**: 세콤 타각이면 정책에 맞는 시각으로 내려주는 게 기본 처리[^7]. 이 판단은 연장근무 조정 로직에 대해서는 맞지만, **메모 필수 승인 정책이 먼저 개입**하여 자동 조정 경로 자체를 차단하고 있다.

### 원인

> 💡 **판단 근거**: 세콤 연동 로그에서 `WorkScheduleRequiredMemoValidatorLogic` 실패 확인[^10]
> → 3/25, 3/30 모두 세콤 퇴근 이벤트는 정상 수신
> → ServerTriggered dry-run에서 **메모 필수 승인 정책** 검증 실패
> → 세콤은 외부 장비라 메모를 첨부할 수 없음 → 자동 퇴근 불가
> → draft event로 저장 + 사용자에게 `STOP_FAILURE_GO_CONFIRM` 알림 발송
> → 사용자가 앱에서 confirm 시도 시 연장근무 최소시간 제한까지 겹침

**근본 원인: 해당 고객사(223909)의 승인 정책에 메모 필수(`requiredMemo: true`)가 설정되어 있으며, 이것이 세콤 ServerTriggered 퇴근 경로에서도 적용된다.** 세콤은 외부 타각 장비라 메모를 첨부할 수 없으므로, 자동 퇴근이 항상 실패하고 사용자에게 수동 확인을 요구하게 된다.

**2차 원인: 사용자가 앱에서 수동 confirm 시도 시**, 소정근무 초과분(17분)이 연장근무 최소 입력 단위(30분) 미달 → FLOOR 정책으로 0분 내림 → 연장근무 등록 불가 → 메모 필수/최소시간 제한 UI 노출.

## 발견한 스펙/제약
- `WorkScheduleRequiredMemoValidatorLogic`의 메모 필수 검증은 세콤 ServerTriggered 경로에서도 적용된다[^10]. 세콤은 외부 장비라 메모를 첨부할 수 없으므로, 메모 필수 승인 정책이 설정된 고객사에서는 세콤 자동 퇴근이 항상 실패한다.
- 연장근무 최소시간 자동 조정(`adjustOverWorkInputRuleWithDryRunResult()`)은 메모 필수 검증보다 후순위로 실행된다. 메모 필수에서 먼저 실패하면 자동 조정 로직에 도달하지 못한다[^8].
- 세콤 STOP 실패 시 `STOP_FAILURE_GO_CONFIRM` 알림이 발송되어 사용자에게 수동 확인을 요구한다. 이 확인 화면에서 연장근무 최소시간 제한이 추가로 노출될 수 있다.
