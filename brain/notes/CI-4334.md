## 원인 분석

**현재까지 원인 미확정.** 3/5 이전 영수증이 팝업에서 제외되는 조건이 두 가지 경로로 압축됨.

### 가설 목록

| # | 가설 | 확인 방법 | 상태 |
|---|------|----------|------|
| 1 | 프론트 `transactedAtFromInclusive=3/5` 날짜 범위 필터 | access log 요청 body 확인 | ❌ 소거 |
| 2 | 3/5 이전 영수증이 다른 지출결의서에서 APPROVED → `evidenceState=CONFIRMED` → 팝업에서 제외 | DB에서 해당 영수증의 연결 지출결의서 상태 확인 | 🔍 일부 해당, 스펙 여부 미판별 |
| 3 | NULL 영수증(아무 지출결의서 미포함)의 Bullseye 인덱스 누락 또는 UNSUBMITTED 색인 | Bullseye 인덱스 직접 확인 또는 operation publish 후 현상 재확인 | 🔍 추가 확인 필요 |

### 조사 과정

> 💡 **가설 1 소거 경로**: access log 조회[^6] → 4/1 요청 4건에 `filter.transactedAtFromInclusive` 없음 + 응답에 3/4 영수증 정상 포함 → 날짜 범위 필터 소거

> 💡 **가설 2·3 근거**: DB 조회[^7] → 지출결의서 2026-175에 연결된 185건 전부 `IN_PROGRESS`(비정상) + 미포함 영수증들이 APPROVED 지출결의서에 포함됨 → `evidenceState=CONFIRMED`로 색인 → `approval/search` 필터(`SUBMITTED`만 통과)에서 제외
