# CI-4345 전자계약 변경 내역 업데이트 시 임금계약 종료일 미반영

## 증상

- **문제 정의**: 전자계약 "변경 내역 업데이트" 실행 후 구성원 프로필의 임금계약 종료일이 반영되지 않음
- **회사**: 법무법인 미션[^1] (Customer ID: 104433[^2])
- **요청자**: djkim@lawmission.net
- **대상자**: jykwon@lawmission.net (userId: 782105[^4])
- **영향 범위**: 특정 구성원 1명 (동일 조건 구성원 추가 확인 필요)
- **문제 시점**: 2026-04-07 오전 (변경 내역 업데이트 실행: set_id=147782, created 2026-04-07 08:35 KST)
- 문의 내용:
  1. 계약서에 임금계약 종료일 필드가 있고, 변경 내역 업데이트 시 해당 값이 입력되어 있음을 확인[^3]
  2. 다른 필드 값이 미입력된 항목이 있어 해당 필드를 채운 후 업데이트 진행[^3]
  3. 업데이트 후에도 구성원 프로필 → 임금 계약 정보에 임금계약 종료일이 반영되지 않음[^3]

### 핵심 발견: salaryContractEndDate 전달 경로 부재 [추론: 아래 근거 1~3 복합]

임금계약 종료일이 전자계약 → BulkChange unit까지 전달되지 않을 가능성이 높다.

**근거 1 — DigiconCoreData에 해당 필드 없음** (코드 확인):

`DigiconCoreData` (`flex-digicon-backend/protocol/src/main/kotlin/team/flex/digicon/DigiconCoreData.kt`):
```kotlin
data class DigiconCoreData(
    override val extraPays: List<DigiconCoreDataExtraPay> = emptyList(),
    override val comprehensivePayRule: DigiconCoreDataComprehensivePayRule? = null,
    override val userSignature: DigiconCoreDataSignature? = null,
    override val customerSignature: DigiconCoreDataSignature? = null,
    override val allowances: List<DigiconCoreDataAllowance> = emptyList(),
    override val regularPayPerHour: BigDecimal? = null,
    override val autoCalculation: Boolean? = null,
    // salaryContractEndDate 필드 없음
)
```

**근거 2 — Digicon에서 임금계약 종료일은 placeholderValues에 별도 저장** (코드 확인):

- `DigiconCandidateUnit.placeholderValues`의 `USER_SALARY_CONTRACT_END_DATE` 타입으로 관리
- `DigiconPlaceholderValueServiceImpl`이 전자계약 생성 시 `currentUserSalaryContract.endDate`로 자동 채움
- `DigiconCandidateUnitResponse`에는 `coreData` + `placeholderValues` 포함

**근거 3 — BulkChangeUnitData.salaryContractEndDate 기본값 null** (코드 확인):

`BulkChangeUnitData.kt:142`:
```kotlin
override val salaryContractEndDate: LocalDate? = null  // nullable, 기본값 null
```

**연결 고리**: Digicon의 `placeholderValues.USER_SALARY_CONTRACT_END_DATE` → BulkChange unit의 `salaryContractEndDate` 매핑이 존재하지 않거나 FE에서 누락될 가능성
[추론: FE 코드 미확인; 백엔드 코드상 DigiconCoreData에 해당 필드 없으므로 BackendData 경로로는 전달 불가]

### BulkChange 제출 주체 (코드 확인)

**FE가 core-backend에 직접 제출한다.** `flex-digicon-backend`는 BulkChange 상태를 `BulkChangeSetLookUpInternalService.getByReferenceIdsAndType()`로 **조회만** 하고, BulkChange 생성/제출 로직이 없다.

따라서 매핑 누락 위치는 **FE**: `DigiconCandidateUnitResponse.placeholderValues[USER_SALARY_CONTRACT_END_DATE]` → `BulkChangeUnitData.salaryContractEndDate` 변환 코드가 FE에 없거나 누락된 것.

### 과거 날짜 무시 로직 (코드 확인)

**`revertAfter` 플래그**가 이 동작을 제어한다.

- `UserSalaryContractServiceImpl.determineEndExpectedDate()` (`core/service-business/.../UserSalaryContractServiceImpl.kt:294`):
  - `revertAfter = true` → `endDateExpected = MAX_END_DATE` (과거 날짜 필터링 없음)
  - `revertAfter = false` → `beginDate`보다 뒤의 기존 계약 기준으로 `endDateExpected` 결정
- DIGITAL_CONTRACT BulkChange의 기본값: `revertAfter = bulkChangeUnitDto.revertAfter ?: true`

**결론**: DIGITAL_CONTRACT 처리 시 `revertAfter = true`(기본값)이므로 과거 날짜 필터링이 **적용되지 않는다.** `salaryContractEndDate`가 null이면 그대로 null로 저장되며, 차단 로직이 없다.

`validate` API(`UserSalaryContractValidatorImpl.checkWithSalaryContracts`)는 `beginDate` 충돌을 검증하지만, `endDate = null`은 유효한 값으로 통과한다.

현재 확인된 내용 기준으로 구성원의 임금계약 종료일이 BulkChange 처리 과정에서 누락된 것으로 추정되나, 확정 전 추가 확인 필요:

### 중기 (버그 판정 시)

- **버그 의심 지점**: 전자계약 "변경 내역 업데이트" 시 Digicon placeholder 값(`USER_SALARY_CONTRACT_END_DATE`)이 BulkChange unit의 `salaryContractEndDate`로 매핑되지 않는 구조 문제
- **수정 방향**: FE 또는 Digicon 백엔드에서 placeholder 값을 BulkChange unit DTO에 포함하도록 수정 필요

  - Option B: FE에서 `placeholderValues.USER_SALARY_CONTRACT_END_DATE` 값을 `BulkChangeUnitData.salaryContractEndDate`에 직접 매핑
- **영향 범위**: 전자계약 "변경 내역 업데이트" 기능을 사용하는 모든 회사
