# CI-4248: 전자계약 일괄 다운로드 링크 생성 안되는 문제

> **상태**: 조사 완료, 재시도 대기 — 2026-03-30

## 증상
- **문제 정의**: admin-shell(내부 운영 관리 도구)에서 전자계약 일괄 다운로드 링크 생성을 요청했으나 링크가 생성되지 않음
- **회사**: (주)큐리오시스 (Customer ID: 195074)
- **요청자**: CS팀 유지원 → 이한결(전자계약 담당) → 온콜 토스[^1]
- **대상자**: 허대성 (hur@curiosis.com)[^2]
- **영향 범위**: 해당 고객사 1건 (3/27 파일 서비스 장애 시간대)
- **문제 시점**: 2026-03-27 오전 10:05경, 오후 2:50경 (2회 시도)[^3]
- 문의 내용:
  1. admin-shell > 전자계약 > 대량 다운로드 링크 받기에서 Customer ID + User ID 입력 후 요청

### 근본 원인 추적 (5 Whys)

1. 왜 다운로드 링크가 안 나오나? → 파일 병합이 실패했고, 실패 콜백도 전달되지 않음
2. 왜 파일 병합이 실패했나? → S3 임시 파일 다운로드 시 파일이 이미 삭제됨
3. 왜 삭제되었나? → 임시 파일 TTL이 **600초(10분)**[^9]인데 병합 서비스가 수 시간 후에 작업 시작
4. 왜 병합 서비스가 지연되었나? → 3/27 파일 서비스 밀림 장애(CI-4236) 여파로 merge 큐 적체[^10]
5. 왜 실패 알림이 안 갔나? → 병합 실패 시 callback이 digicon 서비스로 전달되지 않음 (DIGICON MEREGED 로그 0건)[^8]

### 스펙 vs 버그

- **표면 원인**: CI-4236 파일 서비스 장애로 인한 일시적 큐 지연 → 재시도로 해결 가능 [추론: 장애 해소 후 큐 정상화 전제]
- **구조적 문제 (버그에 가까움)**: 임시 파일 TTL(600초)이 merge 큐 처리 시간보다 짧아서, 큐가 밀리면 항상 실패하는 구조. 또한 병합 실패 시 콜백이 전달되지 않아 사용자에게 실패 알림도 가지 않음

## 발견한 스펙/제약

- 전자계약 대량 다운로드 임시 파일 TTL(Time To Live, 만료 시간): **600초(10분)**, 파일 접근 제어 TTL: **6000초(~100분)**[^9]
- 대량 다운로드는 비동기 처리 — API는 즉시 응답하고 CoroutineScope.launch로 백그라운드 실행[^11]
- 파일 병합은 500건 단위로 chunk하여 요청[^12]
- 병합 완료/실패 시 Slack 알림으로 다운로드 링크 전달[^13]

## 다음에 같은 문의가 오면

1. **먼저 확인**: access log에서 `POST /api/operation/v2/digicon/bulk-download-pdf` 호출이 있는지 확인. 없으면 admin-shell 조작 문제.
2. **app log 확인**: traceId로 `[DIGICON UPLOAD] upload done` 로그가 있는지 확인 → 있으면 PDF 업로드는 성공.
3. **병합 결과 확인**: fileMergeId로 `[File Merge]` 로그 검색 → ERROR가 있으면 병합 실패.
4. **원인 판별**:
   - "S3 파일 다운로드 예외" → 임시 파일 만료 (merge 큐 지연). **재시도**로 해결.
   - merge callback 자체가 없음 → 파일 서비스 장애 확인 (CI-4236 유사).
5. **조치**: 파일 서비스 정상이면 admin-shell에서 **재시도** 요청.

## 해결안 / 조사 방향

### 즉시 대응 (표면 원인 해결)
- CI-4236 해소 확인 후 admin-shell에서 **재시도** → 정상 동작 예상

### 근본 해결 (구조적 문제)
1. 임시 파일 TTL 증가 (600초 → 3600초 이상) — merge 큐 지연에 대한 여유 확보
2. 병합 실패 시 callback이 digicon 서비스로 정상 전달되도록 파일 서비스 측 확인 필요
3. 사용자에게 실패 알림이 가도록 보장 (현재는 성공/실패 모두 알림 없는 상태)
