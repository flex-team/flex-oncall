# QNA-1936: 리뷰 마이그레이션 오류 문의

## 증상
- **문제 정의**: 리뷰 엑셀 마이그레이션 시 "Failed requirement." 오류만 표시되고 구체적 에러 내용 없음
- **회사**: customerId=3013[^1]
- **요청자**: 권남 (CS팀)[^1]
- **영향 범위**: 해당 고객사 리뷰 엑셀 마이그레이션 작업
- **문제 시점**: 2026-03-19 10:03 KST[^1]
- **사용 도구**: flex admin shell — raccoon 리뷰 마이그레이션 UI[^1]
- **요청 파라미터**: customerId=3013, reviewSetIdHash=`rM0AdQl8lR`, 파일=`2025년 팀원평가.xlsx`[^1]
- 문의 내용:
  1. 리뷰 엑셀 마이그레이션 작업 중 "Failed requirement." 에러 발생[^1]
  2. 에러메시지에 구체적인 내용이 표시되지 않아 원인 파악 불가[^1]

## 원인 분석

### 조사 과정

> 💡 **판단 근거**:
> → 권남이 `flex-raccoon.dev.grapeisfruit.com` (dev)에서 마이그레이션 시도 → 403 Forbidden[^2]
> → 같은 dev raccoon에서 재시도 → 400 "Failed requirement.", 응답 instance 경로에 **`-9223372036854775807`** 확인[^3]
> → `-9223372036854775807`은 HashedId 디코딩 실패 시 반환되는 `INVALID_NUMBER`[^5]
> → prod에서 생성된 해시를 dev에서 디코딩하면 Hashids salt[^6] 불일치로 디코딩 실패
> → 김주원이 dev/prod 환경 불일치를 지적[^4]
> → 권남이 `flex-raccoon.grapeisfruit.com` (prod)에서 재시도 → "올바르지않은 선택값입니다!" (구체적 에러 정상 출력)[^7]
> → 결론: "Failed requirement."는 환경 불일치로 인한 증상이었으며, prod에서는 정상 동작

### 원인
**사용자 운영 오류** — prod 해시를 dev raccoon에서 사용하여 Hashids salt 불일치로 디코딩 실패. 제품 버그 아님.[^8]

| # | 현상 | 원인 |
|---|------|------|
| 1 | 403 Forbidden | raccoon 세션 만료[^2] |
| 2 | 400 "Failed requirement." | prod 해시를 dev raccoon에서 디코딩 → **`INVALID_NUMBER`** → `require(reviewSetId > 0L)` 실패[^3] |
| 3 | "올바르지않은 선택값입니다!" | prod raccoon에서 재시도 시 엑셀 데이터 유효성 에러 — 정상 동작[^7] |

## 해결
- prod raccoon(`flex-raccoon.grapeisfruit.com`)에서 재시도하여 정상 에러 메시지 확인[^7]
- "Failed requirement."가 아닌 구체적 에러가 출력되어 제품 버그 아님으로 종결[^8]

## 다음에 같은 문의가 오면

1. **환경 확인**: raccoon prod(`flex-raccoon.grapeisfruit.com`)를 사용하고 있는지 확인. dev raccoon에 prod 해시를 쓰면 "Failed requirement." 발생
2. **"Failed requirement."**: Hashids salt 환경 불일치 가능성 우선 확인
3. **구체적 에러 메시지가 나오면**: 엑셀 데이터 문제이므로 메시지 내용대로 수정 안내
