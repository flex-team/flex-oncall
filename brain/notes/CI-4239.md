# CI-4239: 테스트 근무기록 완전 삭제 가능 여부 확인

> **상태**: 진행 중 — 2026-03-27

## 증상
- **문제 정의**: 첫 결제 후 테스트 중인 고객사가 4월 1일 실제 근무 시작 전에 테스트 근무기록을 완전 삭제할 수 있는지 확인 요청
- **회사**: 도안회계법인 (Customer ID: 222963)
- **요청자**: CS 담당 김선미(seonmi)[^1]
- **대상자**: 도안회계법인 전체 구성원 (테스트 데이터)
- **영향 범위**: 전사 (테스트 기간 전체 근무기록)
- **문제 시점**: 2026-03-27 접수, 4월 1일 실 근무 시작 예정
- 문의 내용:
  1. 테스트 과정에서 생성된 근무기록을 완전 삭제할 수 있는지[^1]

## 현재까지 파악된 내용

### 1차 확인: 완전 삭제 불가
- 김주원이 김영준(Enhance[^5])에게 데이터 무결성 관점에서 가능 여부 확인[^2]
- 김영준이 Notion 문서 링크와 함께 확인 응답[^3]
- 결론: **테스트 근무기록 완전 삭제는 불가능**[^4]

> 💡 **판단 근거**: 김주원이 "데이터 무결성 관점에서 정책상 불가능할 것 같긴 함"이라고 예상한 후[^2], 김영준(Enhance[^5])의 확인을 거쳐[^3] CS 담당에게 "안된다고 합니다"로 최종 전달[^4]

### 2차 논의: 대안 — 벌크 업로드 빈 값 덮어씌우기
- 김영준이 대안 제시: **벌크 업로드로 빈 날짜를 덮어씌우면 지워진 것처럼 동작**할 수 있음[^6]
  - 단, "완전 삭제"와는 다르므로 고객 요구에 맞는지 불확실[^6]
- 김선미(CS)가 요구 수준 확인: "근무 그래프 블록이랑 리포트에서 시간만 안 나오면 됨"[^7]
- 김영준 재확인: 빈 값으로 덮어씌우면 동작할 것[^8]
- 김선미: "빈 값이 업로드가 된다면 도전합니다"[^9]

### VOC 요청
- 김영준: 이 요청은 근태만이 아니라 테스트에 사용한 다른 데이터도 포함될 것이므로 VOC로 모아달라고 요청[^10]
- 김선미: VOC로 올려두겠다고 확인[^11]

## 발견한 스펙/제약
- 근무기록은 데이터 무결성 정책상 **완전 삭제(hard delete) 불가**[^4]
- 벌크 업로드로 빈 값 덮어씌우기는 UI상 삭제처럼 보이게 하는 workaround — DB에는 레코드가 남음[^6]
