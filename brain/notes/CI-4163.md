# CI-4163: 원텍 이메일 일괄 변경 (wtlaser.com → wontech.com)

> **compact**: 신규 생성 69줄 (2026-03-25). 정제 대상 없음.

## 증상
- **문제 정의**: 고객사 도메인 변경으로 전 직원 이메일 주소 일괄 변경 요청
- **회사**: 원텍 (Customer ID: 224689)
- **요청자**: 오시은 (@sieun, CS팀)[^1]
- **대상자**: 재직자 256명 (wtlaser.com 도메인 사용자)[^2]
- **영향 범위**: 전사
- **문제 시점**: 2026-03-19 접수, 2026-03-25 작업 실행
- 문의 내용:
  1. 고객사 도메인 변경(wtlaser.com → wontech.com)에 따른 이메일 일괄 변경[^1]
  2. 어드민쉘에서 이메일 변경 카테고리를 찾을 수 없어 별도 요청[^3]

## 원인 분석

### 조사 과정
> 💡 운영 작업 건으로 별도 원인 분석 불필요.
> → CI-4200에서 동일 유형(일괄 이메일 변경) 대응 절차 확인[^4]
> → Operation API bulk email change 엔드포인트 확인[^5]
> → 엑셀 Sheet1 기준 256건 변경 대상 확정[^6]

### 원인
- 운영 작업 요청 건. 고객사 도메인 변경에 따른 이메일 일괄 변경이 필요했으며, admin-shell에 해당 기능이 prod 미배포 상태[^3]

## 해결
- Operation API로 bulk email change 실행[^5]
- 엑셀 256건 중 1건(이형섭, `hs.lee`)은 이미 wontech.com으로 변경 완료 상태 → 제외[^7]
- 최종 255건 변경 성공[^7]

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| @wtlaser.com | 256 | **1** (엑셀 미포함 별도 계정) |
| @wontech.com | 24 | **279** |

- API: `PATCH /action/v2/operation/core/customers/224689/emails/change/bulk`[^5]
- 비밀번호는 기존과 동일 (이메일만 변경)[^8]

## 다음에 같은 문의가 오면

1. **먼저 확인**: 구성원 이메일 일괄 변경 요청인지 확인. 단건이면 단건 API 사용
2. **원인 판별**: Operation API(`PATCH /action/v2/operation/core/customers/{customerId}/emails/change/bulk`)로 처리 가능[^5]
3. **조치**:
   - 기존/변경 이메일 목록 엑셀 수령 (oldEmail, newEmail)
   - DB에서 oldEmail 존재 여부 사전 검증 — 이미 변경된 건, 없는 건 제외
   - bulk API 호출 후 DB 조회로 결과 검증
   - 소요 기간: 영업일 1일
