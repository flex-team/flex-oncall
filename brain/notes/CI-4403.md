### 근본 원인

`EvaluationRevieweeFactorGradeCalculator.kt` line 234-237:

```kotlin
val stepWeightsForFactorGrade = if (needWeightGroupToCalculateFactorGrade) {
    weightGroup?.stepWeightForFactorGrade
        ?.filter { it.evaluationFactorType == factorType }
        ?.filter { it.weight > BigDecimal.ZERO }
        ?: return null  // null만 체크, 빈 리스트는 통과
} else { ... }
```

line 271-282:
```kotlin
val sumOfWeight = stepWeightsForFactorGrade.sumOf { it.weight }  // 빈 리스트 → ZERO
// ...
val convertedScore = sumOfWeightedStepFactorConvertedScore.divide(sumOfWeight, SCORE_SCALE, RoundingMode.FLOOR)
// sumOfWeight가 ZERO → ArithmeticException: / by zero
```

**문제**: `?: return null`은 `weightGroup`이 null일 때만 동작. 필터링 후 빈 리스트(`emptyList()`)는 null이 아니므로 그대로 통과 → `sumOfWeight = 0` → division by zero.

### 발생 조건

1. `factor_grade_calculations`에 `WEIGHTED` 계산 방식 설정됨
2. 평가 단계(step)가 2개 이상 (`usableSteps.size > 1`)
3. 가중치 그룹에서 해당 factor의 유효 가중치(>0)가 없음 (미설정 또는 전부 0)

### DB 데이터 확인

- draft_evaluation `01knv21p6gd5j50q6x4568efd1`:
  - `grades_to_calculate`: `["FACTOR_GRADE"]`
  - `factor_grade_calculations`: `[{"calculation": "WEIGHTED", "evaluationFactorType": "CUSTOM"}]`
- customer가 동일 평가를 10회 이상 생성 시도 (05:27 ~ 06:42). 초기에는 `LAST_TOP_DOWN_GRADE`였다가 이후 `WEIGHTED`로 변경 시점부터 오류 발생.

- 금전/법적 영향: 없음 (평가 생성 단계에서 차단, 데이터 변경 없음)
- 동일 패턴: `UserFormCalculator.kt`에도 유사한 divide 패턴 존재 (잠재적 동일 버그)

### 미해결 — 가중치 미저장 원인 (도메인 팀 확인 필요)
