> ⚠️ 델리[^13]가 없는 고객사이며, 첨부된 스크린샷만 전달받은 상태[^3]

## 현재까지 파악된 내용

### 오류 API 특정
> 💡 **판단 근거**: 이성환 님이 traceId로 Tempo[^14]를 확인하여 오류 API를 특정[^9]
> → `DELETE /api/v3/evaluation/{evaluationId}/forms/{formId}/draft/blocks/{blockId}` API에서 오류 발생
> → 고객사가 달라도 동일 API에서 FE 에러 발생 확인[^10]
> → 스크린샷에서 evaluation ID 식별: `01kkxf721djq9vdkhrncz4cy3h`[^11]

### 원인 분석 과정
1. 이성환 님이 traceId 기반으로 Tempo에서 API 호출 추적[^9]
2. 오류 직후(약 20초 후) `/action/v3/evaluation/draft/01kkxf721djq9vdkhrncz4cy3h/stage-progress/complete` API 호출 확인[^15]
3. 김보라 님이 해당 evaluation을 origin으로 참조하는 update mode[^16]의 DraftEvaluation[^17] 존재 확인[^12]
4. 평가 "세팅 과정"이라 생성 중으로 예상했으나, 실제로는 **설정 수정(update mode)** 상태[^12]
5. access log가 남지 않아 정확한 `formId`/`blockId` 특정 불가[^18]

### 로그 확인 시도
- 센트리(Sentry)에 클라이언트 에러 로그가 찍히지 않음[^4]
- 서버 500 오류 로그도 찾지 못한 상태[^5]
- **access log가 해당 API에서 남지 않는 문제 확인** — 핫픽스로 로그 추가 필요[^18]

### 관련 Slack 스레드
이성환 님이 공유한 두 개의 Slack 스레드에서 동일 API 에러 확인:
1. `#CREP89MUG` 채널 스레드 — 리뷰 에러 관련[^6]
2. `#C09BFPSNTPY` 채널 스레드[^7]

> ⚠️ 이지선 님 확인 결과, 위 스레드의 이메일이 문의자와 다르다[^8]. 단, 이성환 님이 고객사가 달라도 동일 API에서 에러 발생을 확인[^10]하여, **같은 버그가 여러 고객사에서 발생**하는 것으로 추정.

### 대응 현황
1. 김보라 님이 `DraftFormNotFoundException`[^19] 발생 원인 조사[^12]
2. 김영준(Enhance)이 PR [#5148](https://github.com/flex-team/flex-review-backend/pull/5148) 생성 — DRAFT_FORM 한국어 메시지 키 불일치 수정[^20]
3. 고객에게 안내: 시스템 로그 불충분 → 임시 배포로 로그 추가 → 재현 요청 → 원인파악 후 조치 예정[^21]
4. **2026-03-18 14:28**: 배포 완료, 고객 재확인 요청[^22]
