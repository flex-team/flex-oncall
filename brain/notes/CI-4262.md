# CI-4262: 구글캘린더 일정 수동 연동 요청

## 증상
- **문제 정의**: 그룹 구글캘린더 재연동 후, 재연동 이전에 등록한 일정이 구글캘린더에 미반영
- **회사**: 사단법인 국제앰네스티 한국지부 (Customer ID: 27844)
- **요청자**: 유지원 (CS팀)[^1]
- **대상자**: heesumyung@amnesty.or.kr[^2]
- **영향 범위**: 1명 (해당 캘린더 연동 계정)
- **문제 시점**: 2026-02-12 ~ 2026-03-31 기간의 일정
- 문의 내용:
  1. 그룹 구글캘린더 연동이 원활하지 않아 재연동을 수행함[^2]
  2. 재연동 이전에 등록한 일정이 구글캘린더에 동기화되지 않음[^2]
  3. 2026-02-12 ~ 2026-03-31 기간 일정의 수동 연동 요청[^2]

## 원인 분석

### 조사 과정
> Metabase 대시보드 244에서 heesumyung@amnesty.or.kr의 캘린더 연동 상태 확인[^3]
> → 그룹캘린더가 2026-03-31 10:44에 재연동된 것 확인[^3]
> → 재연동 이전 일정 365건이 미연동 상태[^3]
> → COOKBOOK F1 플로우에 따라 Operation API로 재동기화 수행[^4]

### 원인
- 그룹 구글캘린더 재연동 시 기존 연동이 해제되면서, 재연동 이전에 생성된 flex 캘린더 이벤트가 구글캘린더에 동기화되지 않은 상태로 남음[^3]

## 해결
- Metabase 쿼리 7387로 미연동 eventIds 365건 추출[^5]
- raccoon Operation API `sync-created-member-schedule-group-calendar-by-ids` 호출[^6]
  - batch 20건 + 2초 delay, 총 19회 호출
  - 전체 200 OK 성공
- Slack 스레드 및 Linear 코멘트에 결과 보고 완료[^7]

## 다음에 같은 문의가 오면

1. **먼저 확인**: Metabase 대시보드 `/dashboard/244?email={사용자이메일}`에서 미연동 건 확인
2. **eventIds 추출**: Metabase 쿼리 7387로 미연동 eventIds 추출
3. **조치**: raccoon Operation API로 재동기화
   - 엔드포인트: `/proxy/calendar/api/operation/v2/calendar/customers/{customerId}/google-calendar-events/sync-created-member-schedule-group-calendar-by-ids`
   - 300건 이상 시 Google Calendar API rate limit 주의 → batch 20건 + 2초 delay 권장
   - 50건 이상 시 raccoon audit log URL 길이 초과 가능 → batch 5~20건 권장
