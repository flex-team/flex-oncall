# CI-4342 승인 정책 승인권자 일괄 변경 요청

## 증상

- **문제 정의**: SK케미칼(주)의 모든 승인 정책 승인권자를 일괄 변경 요청
- **회사**: SK케미칼(주) (Customer ID: 55747)
- **요청자**: dabin.s@skdiscovery.com
- **영향 범위**: 회사 전체 승인 정책
- **문제 시점**: 2026-04-06 접수
- 문의 내용:
  1. 기존 승인권자 `dabin.s@skdiscovery.com` → `jeongin.k@skdiscovery.com` 으로 일괄 변경 요청

## 원인 분석

데이터 조작 요청. 코드 버그 없음.

승인권자는 `approval_step_actor.resolve_target_value` 에 user_id(Long)로 저장된다.[^1]
과거 동일 패턴으로 처리한 이력이 있으며 (`ajkim → dabin` 변경, 2025-06-16), 같은 방식으로 처리한다.[^2]

### 가설 목록

| # | 가설 | 상태 |
|---|------|------|
| 1 | approval_step_actor에서 resolve_target_value(user_id)를 UPDATE하면 일괄 변경 가능 | ✅ 확정 |

## 조사 결과

| 항목 | 값 |
|------|-----|
| dabin (기존 승인권자) | user_id = **808508** |
| jeongin (변경 대상) | user_id = **897921** |
| 변경 대상 건수 | **729건** (미삭제 정책 기준) |
| DB | `flex_approval` |
| 미삭제 조건 | `deleted_at = '9090-09-09 09:09:09'` |

> 이전 처리(2025-06-16)에서는 47건이었으나 현재 729건으로 증가.

## 해결

### Step 1: 변경 대상 확인 (완료)

```sql
SELECT COUNT(*) AS cnt
FROM flex_approval.approval_step_actor asa
JOIN flex_approval.approval_step astp ON asa.approval_step_id = astp.id
JOIN flex_approval.approval_policy ap ON astp.approval_policy_id = ap.id
WHERE ap.customer_id = 55747
  AND asa.resolve_target_type = 'USER'
  AND asa.resolve_target_value = '808508'
  AND ap.deleted_at = '9090-09-09 09:09:09'
  AND astp.deleted_at = '9090-09-09 09:09:09';
-- 결과: 729건
```

### Step 2: 일괄 UPDATE (담당자 직접 실행)

```sql
UPDATE flex_approval.approval_step_actor asa
JOIN flex_approval.approval_step astp ON asa.approval_step_id = astp.id
JOIN flex_approval.approval_policy ap ON astp.approval_policy_id = ap.id
SET asa.resolve_target_value = '897921'
WHERE ap.customer_id = 55747
  AND asa.resolve_target_type = 'USER'
  AND asa.resolve_target_value = '808508'
  AND ap.deleted_at = '9090-09-09 09:09:09'
  AND astp.deleted_at = '9090-09-09 09:09:09';
```

### Step 3: 변경 확인

```sql
SELECT COUNT(*) AS cnt
FROM flex_approval.approval_step_actor asa
JOIN flex_approval.approval_step astp ON asa.approval_step_id = astp.id
JOIN flex_approval.approval_policy ap ON astp.approval_policy_id = ap.id
WHERE ap.customer_id = 55747
  AND asa.resolve_target_type = 'USER'
  AND asa.resolve_target_value = '808508'
  AND ap.deleted_at = '9090-09-09 09:09:09'
  AND astp.deleted_at = '9090-09-09 09:09:09';
-- 0건이어야 정상
```

## 비고

- `approval_step_actor` 에는 `resolve_target_type = 'USER'` 외에 `RELATIVE_DEPT_HEAD` 등 다른 타입도 존재. 이번 요청은 직접 지정(USER)된 승인권자만 변경하면 됨.
- 진행 중인 문서(`approval_line_actor`)의 승인권자는 이번 변경 대상에 포함되지 않음. 필요 시 별도 처리 필요.
