### 원인
- `goal-cron`의 `MigrationScheduler`가 예약된 뉴성과관리 전환을 자동 실행[^5]
- `ReviewMigrationServiceImpl.deleteAllInProgressReviewAndCalibrationSessions()`가 진행 중(`IN_PROGRESS`, `PREPARE_PROGRESS`, `EXTEND_PROGRESS`) 리뷰셋을 soft delete[^3]
- 3월 동안 217181 고객의 마이그레이션은 **1회만 실행**됨 (2026-03-25 00:00 KST)[^5]

### 스펙 vs 버그 판별
**스펙** — 마이그레이션 코드에 의도적으로 진행 중 리뷰를 삭제하는 로직이 존재[^3]. 고객이 뉴성과관리 전환 예약 시 이 동작을 인지하지 못한 것이 문제.

## 해결안 / 조사 방향

### 1. 구 리뷰셋 복구 (soft delete 복원)
- `review_set` 테이블의 `deleted = 0, deletedAt = NULL`로 복구 가능 (CI-4195 패턴[^7])
- **단, 뉴성과관리로 전환 완료된 상태에서 구 리뷰를 복구하는 것이 적절한지 김보라님 확인 필요**
- 복구 시 DB 쿼리:
  ```sql
  -- 삭제된 리뷰셋 확인
  SELECT id, title, progressStatus, deletedAt
  FROM flex_review.review_set
  WHERE customerId = 217181 AND deleted = 1
  ORDER BY deletedAt DESC;
  ```

### 2. 미리보기 목표 복구
- `ObjectiveMigrationService`가 disconnected objectives를 삭제[^5] — goal 도메인 담당자 확인 필요
- `flex-goal-backend`의 `ObjectiveMigrationService` 로직 조사 필요

### 3. 고객 안내
- 마이그레이션은 예약된 스케줄에 의한 자동 처리이며, 진행 중 리뷰 삭제는 의도된 동작
- 복구 가능 여부와 범위를 확정한 후 안내

## 발견한 스펙/제약
- 뉴성과관리 전환 시 `MigrationScheduler`가 매일 00:00 KST에 예약된 고객을 자동 마이그레이션[^5]
- 마이그레이션 시 진행 중(`IN_PROGRESS`, `PREPARE_PROGRESS`, `EXTEND_PROGRESS`) 리뷰셋 + 캘리브레이션 세션을 soft delete[^3]
- `evaluation` 테이블(뉴 성과관리)의 데이터는 마이그레이션에 영향받지 않음[^6]
- `review_set`의 soft delete는 `deleted` 플래그 + `deletedAt` 타임스탬프 방식[^4]
